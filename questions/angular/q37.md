**Ivy** es el motor de renderizado y compilación de Angular, introducido como preview en Angular 8 y convertido en el motor **por defecto desde Angular 9**. Reemplazó al antiguo motor View Engine, trayendo mejoras significativas en rendimiento, tamaño de bundle y experiencia de desarrollo.

## ¿Qué hace Ivy?

Ivy es responsable de:
1. **Compilar** componentes, directivas y pipes de Angular a código JavaScript optimizado.
2. **Renderizar** la interfaz de usuario en el navegador.
3. **Gestionar** el sistema de detección de cambios.

## Principio de localidad

Una de las innovaciones clave de Ivy es el **principio de localidad**: cada componente se compila de forma **independiente**, sin necesidad de conocer toda la aplicación.

```
View Engine (antiguo):
  Componente A → necesita info de Módulo → necesita info de toda la app

Ivy:
  Componente A → se compila solo con su propia información ✅
```

Esto permite compilación incremental: solo se recompila lo que cambia.

## Mejoras de Ivy sobre View Engine

### 1. Bundles más pequeños (Tree-shaking mejorado)

Ivy genera instrucciones específicas por componente en lugar de un intérprete genérico. Las partes de Angular que no se usan se eliminan del bundle final.

```
Hello World App:
  View Engine: ~130 KB (gzip)
  Ivy:          ~25 KB (gzip) ← ~80% más pequeño
```

### 2. Compilación más rápida

La compilación incremental de Ivy es significativamente más rápida en proyectos grandes ya que no recompila módulos enteros.

### 3. Mejor debugging

Ivy introduce mejoras en las herramientas de debugging:

```typescript
// En la consola del navegador, puedes hacer:
const componentInstance = ng.getComponent(document.querySelector('app-root'));
const directive = ng.getDirectives(element)[0];

// También disponible en Angular DevTools (extensión del navegador)
```

### 4. Detección de errores en plantillas mejorada

Ivy realiza una verificación de tipos más estricta en las plantillas, detectando errores en tiempo de compilación que antes solo se veían en runtime.

```typescript
// Con Ivy y strictTemplates: true en tsconfig.json
@Component({
  template: `<p>{{ usuario.edadTypo }}</p>` // ← Error en build, no en runtime
})
export class PerfilComponent {
  usuario: Usuario = { id: 1, nombre: 'Ana', edad: 30 };
}
```

### 5. Soporte de lazy loading a nivel de componente

Ivy permite cargar componentes de forma perezosa sin necesidad de módulos:

```typescript
// Lazy loading de componente standalone (Angular 14+, posible gracias a Ivy)
const routes: Routes = [
  {
    path: 'dashboard',
    loadComponent: () =>
      import('./dashboard/dashboard.component').then(m => m.DashboardComponent)
  }
];
```

### 6. Mejoras en `TestBed`

Con Ivy, `TestBed.configureTestingModule` solo recompila los componentes que han cambiado, haciendo los tests más rápidos.

## Activar el modo estricto de Ivy

En `tsconfig.json`:
```json
{
  "angularCompilerOptions": {
    "strictTemplates": true,
    "strictInjectionParameters": true,
    "strictInputAccessModifiers": true
  }
}
```

## Ivy y el futuro: Signals y Zoneless

Ivy sentó las bases para las innovaciones de Angular moderno:
- **Angular Signals** (v16+): un nuevo sistema de reactividad que funciona con el renderizado de Ivy.
- **Zoneless Angular** (v18+): posible gracias a que Ivy puede hacer renderizado fino sin Zone.js.
- **Standalone Components** (v14+): simplificados gracias al principio de localidad de Ivy.

**Consejo de entrevista**: Si te preguntan sobre Ivy, los puntos más importantes son: bundles más pequeños por tree-shaking mejorado, compilación incremental más rápida, detección de errores de plantilla en tiempo de compilación, y que es la base tecnológica sobre la que se construyen las características modernas de Angular (Signals, Standalone Components, Zoneless). Desde Angular 9, todos los proyectos usan Ivy por defecto.
