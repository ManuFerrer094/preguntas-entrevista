---
title: ¿Para qué sirve la directiva ngSwitch en Angular?
difficulty: easy
tags: [angular, ngSwitch, directives, template]
---

La directiva **`ngSwitch`** permite mostrar **uno de varios elementos posibles** según el valor de una expresión, de forma similar a la sentencia `switch/case` de JavaScript. Es ideal cuando se tienen múltiples opciones mutuamente excluyentes.

## Uso básico

`ngSwitch` es una directiva de **atributo** (sin `*`), pero trabaja junto con las directivas estructurales `*ngSwitchCase` y `*ngSwitchDefault`:

```typescript
@Component({
  standalone: true,
  imports: [CommonModule],
  template: `
    <div [ngSwitch]="estadoPedido">
      <p *ngSwitchCase="'pendiente'">⏳ Tu pedido está pendiente de confirmación.</p>
      <p *ngSwitchCase="'procesando'">🔄 Tu pedido está siendo procesado.</p>
      <p *ngSwitchCase="'enviado'">🚚 Tu pedido está en camino.</p>
      <p *ngSwitchCase="'entregado'">✅ Tu pedido ha sido entregado.</p>
      <p *ngSwitchDefault>❓ Estado desconocido.</p>
    </div>
  `
})
export class EstadoPedidoComponent {
  estadoPedido = 'procesando';
}
```

## Con componentes

`ngSwitch` también funciona con componentes:

```html
<ng-container [ngSwitch]="tipoUsuario">
  <app-panel-admin *ngSwitchCase="'admin'"></app-panel-admin>
  <app-panel-editor *ngSwitchCase="'editor'"></app-panel-editor>
  <app-panel-lector *ngSwitchCase="'lector'"></app-panel-lector>
  <app-acceso-denegado *ngSwitchDefault></app-acceso-denegado>
</ng-container>
```

## Múltiples casos para el mismo contenido

Para que varios valores muestren el mismo contenido, se pueden usar múltiples `*ngSwitchCase`:

```html
<div [ngSwitch]="dia">
  <p *ngSwitchCase="'sabado'">¡Es fin de semana!</p>
  <p *ngSwitchCase="'domingo'">¡Es fin de semana!</p>
  <p *ngSwitchDefault>Es un día laborable.</p>
</div>
```

## `ngSwitch` vs cadena de `*ngIf`

```html
<!-- ❌ Con *ngIf encadenados: verboso y menos semántico -->
<p *ngIf="color === 'rojo'">Rojo</p>
<p *ngIf="color === 'verde'">Verde</p>
<p *ngIf="color === 'azul'">Azul</p>

<!-- ✅ Con ngSwitch: más claro y eficiente -->
<ng-container [ngSwitch]="color">
  <p *ngSwitchCase="'rojo'">Rojo</p>
  <p *ngSwitchCase="'verde'">Verde</p>
  <p *ngSwitchCase="'azul'">Azul</p>
</ng-container>
```

## Nueva sintaxis @switch (Angular 17+)

Angular 17 introduce el control de flujo nativo `@switch`, que es más limpio y no requiere importar `CommonModule`:

```html
@switch (planSuscripcion) {
  @case ('free') {
    <div class="plan free">
      <h3>Plan Gratuito</h3>
      <p>5 GB de almacenamiento</p>
    </div>
  }
  @case ('pro') {
    <div class="plan pro">
      <h3>Plan Pro</h3>
      <p>100 GB de almacenamiento</p>
    </div>
  }
  @case ('enterprise') {
    <div class="plan enterprise">
      <h3>Plan Enterprise</h3>
      <p>Almacenamiento ilimitado</p>
    </div>
  }
  @default {
    <p>Plan no reconocido.</p>
  }
}
```

## Diferencias clave entre ngSwitch y @switch

| Característica         | `ngSwitch` (clásico)           | `@switch` (Angular 17+)      |
|------------------------|--------------------------------|------------------------------|
| Importación necesaria  | `CommonModule` o `NgSwitch`    | Ninguna                      |
| Sintaxis               | Directiva de atributo + `*`    | Sintaxis nativa del template |
| Type narrowing         | Limitado                       | Mejorado                     |
| Soporte en el IDE      | Básico                         | Mejorado con LSP             |
