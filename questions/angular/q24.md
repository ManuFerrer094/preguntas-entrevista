---
title: ¿Cuáles son los tipos de directivas en Angular?
difficulty: medium

---

Angular clasifica las directivas en **tres categorías** según su propósito y comportamiento sobre el DOM.

## 1. Directivas de componente

Son la categoría más numerosa. Técnicamente, **todo componente es una directiva** con una plantilla HTML asociada. Se definen con `@Component` y tienen su propio selector, template y estilos.

```typescript
@Component({
  selector: 'app-producto',
  standalone: true,
  template: `<div class="producto">{{ nombre }}</div>`
})
export class ProductoComponent {
  @Input() nombre!: string;
}
```

## 2. Directivas estructurales

Modifican la **estructura del DOM**: añaden, eliminan o reemplazan elementos. Se identifican por el prefijo `*` (azúcar sintáctico para `<ng-template>`).

### *ngIf / @if (Angular 17+)
```html
<!-- Sintaxis clásica -->
<div *ngIf="usuario; else sinUsuario">{{ usuario.nombre }}</div>
<ng-template #sinUsuario><p>No hay usuario</p></ng-template>

<!-- Nueva sintaxis Angular 17+ -->
@if (usuario) {
  <div>{{ usuario.nombre }}</div>
} @else {
  <p>No hay usuario</p>
}
```

### *ngFor / @for (Angular 17+)
```html
<!-- Sintaxis clásica -->
<ul>
  <li *ngFor="let item of lista; trackBy: trackById">{{ item.nombre }}</li>
</ul>

<!-- Nueva sintaxis Angular 17+ -->
@for (item of lista; track item.id) {
  <li>{{ item.nombre }}</li>
}
```

### *ngSwitch / @switch (Angular 17+)
```html
<!-- Sintaxis clásica -->
<ng-container [ngSwitch]="rol">
  <span *ngSwitchCase="'admin'">Administrador</span>
  <span *ngSwitchCase="'editor'">Editor</span>
  <span *ngSwitchDefault>Lector</span>
</ng-container>

<!-- Nueva sintaxis Angular 17+ -->
@switch (rol) {
  @case ('admin') { <span>Administrador</span> }
  @case ('editor') { <span>Editor</span> }
  @default { <span>Lector</span> }
}
```

### Directiva estructural personalizada
```typescript
@Directive({
  selector: '[appSiRol]',
  standalone: true
})
export class SiRolDirective {
  constructor(
    private templateRef: TemplateRef<any>,
    private viewContainer: ViewContainerRef,
    private authService: AuthService
  ) {}

  @Input() set appSiRol(rol: string) {
    if (this.authService.tieneRol(rol)) {
      this.viewContainer.createEmbeddedView(this.templateRef);
    } else {
      this.viewContainer.clear();
    }
  }
}
```
```html
<button *appSiRol="'admin'">Borrar usuario</button>
```

## 3. Directivas de atributo

Modifican la **apariencia o comportamiento** de un elemento, componente u otra directiva existente, sin alterar la estructura del DOM.

### Integradas en Angular
```html
<!-- NgClass -->
<div [ngClass]="{ 'destacado': esPrincipal, 'error': tieneError }">
  Contenido
</div>

<!-- NgStyle -->
<p [ngStyle]="{ color: colorTexto, fontSize: '16px' }">Texto</p>

<!-- NgModel (requiere FormsModule) -->
<input [(ngModel)]="nombre" placeholder="Tu nombre">
```

### Directiva de atributo personalizada
```typescript
@Directive({
  selector: '[appTooltip]',
  standalone: true
})
export class TooltipDirective {
  @Input('appTooltip') texto = '';
  private tooltipEl!: HTMLElement;

  constructor(private el: ElementRef, private renderer: Renderer2) {}

  @HostListener('mouseenter')
  mostrar(): void {
    this.tooltipEl = this.renderer.createElement('span');
    this.renderer.appendChild(
      this.tooltipEl,
      this.renderer.createText(this.texto)
    );
    this.renderer.addClass(this.tooltipEl, 'tooltip');
    this.renderer.appendChild(this.el.nativeElement, this.tooltipEl);
  }

  @HostListener('mouseleave')
  ocultar(): void {
    this.renderer.removeChild(this.el.nativeElement, this.tooltipEl);
  }
}
```
```html
<button appTooltip="Haz clic para guardar">Guardar</button>
```

## Resumen

| Tipo         | Decorador    | Prefijo | Efecto sobre el DOM         |
|--------------|--------------|---------|-----------------------------|
| Componente   | `@Component` | —       | Crea un nodo con template   |
| Estructural  | `@Directive` | `*`     | Añade o elimina elementos   |
| De atributo  | `@Directive` | —       | Modifica aspecto/comportamiento |

**Consejo de entrevista**: La diferencia clave entre directivas estructurales y de atributo es que las primeras trabajan con `TemplateRef` y `ViewContainerRef` para manipular el DOM, mientras que las segundas trabajan directamente con el `ElementRef` del elemento huésped. En Angular 17+, se recomienda usar la nueva sintaxis de control de flujo (`@if`, `@for`, `@switch`) en lugar de las directivas estructurales clásicas.
