La **internacionalización (i18n)** es el proceso de diseñar una aplicación para que pueda adaptarse a diferentes idiomas y regiones. Angular ofrece soporte nativo para i18n mediante el paquete `@angular/localize`, así como soluciones de terceros como `ngx-translate`.

## i18n nativo de Angular

### 1. Instalar el soporte i18n

```bash
ng add @angular/localize
```

### 2. Marcar textos para traducción

En la plantilla, usa el atributo `i18n` para marcar textos traducibles:

```html
<!-- Texto simple -->
<h1 i18n="Título de la página principal">Bienvenido a nuestra tienda</h1>

<!-- Con descripción y significado -->
<p i18n="@@MENSAJE_BIENVENIDA">Hola, ¡bienvenido!</p>

<!-- Texto con plural -->
<span i18n="{cantidad, plural, =0 {Sin artículos} =1 {1 artículo} other {{{cantidad}} artículos}}">
</span>

<!-- En atributos -->
<input i18n-placeholder="@@PLACEHOLDER_BUSCAR" placeholder="Buscar...">
<img i18n-alt="@@ALT_LOGO" alt="Logo de la empresa" src="logo.png">
```

### 3. Extraer los textos a traducir

```bash
ng extract-i18n --output-path src/locale
```

Esto genera `messages.xlf` (formato XLIFF por defecto):

```xml
<!-- messages.xlf -->
<xliff version="1.2">
  <file>
    <body>
      <trans-unit id="MENSAJE_BIENVENIDA">
        <source>Hola, ¡bienvenido!</source>
        <note>Mensaje de bienvenida en la pantalla principal</note>
      </trans-unit>
      <trans-unit id="PLACEHOLDER_BUSCAR">
        <source>Buscar...</source>
      </trans-unit>
    </body>
  </file>
</xliff>
```

### 4. Crear archivos de traducción

```xml
<!-- messages.en.xlf — Traducción al inglés -->
<xliff version="1.2">
  <file target-language="en">
    <body>
      <trans-unit id="MENSAJE_BIENVENIDA">
        <source>Hola, ¡bienvenido!</source>
        <target>Hello, welcome!</target>
      </trans-unit>
      <trans-unit id="PLACEHOLDER_BUSCAR">
        <source>Buscar...</source>
        <target>Search...</target>
      </trans-unit>
    </body>
  </file>
</xliff>
```

### 5. Configurar los locales en angular.json

```json
{
  "projects": {
    "mi-app": {
      "i18n": {
        "sourceLocale": "es",
        "locales": {
          "en": {
            "translation": "src/locale/messages.en.xlf"
          },
          "fr": {
            "translation": "src/locale/messages.fr.xlf"
          }
        }
      },
      "architect": {
        "build": {
          "configurations": {
            "en": { "localize": ["en"] },
            "fr": { "localize": ["fr"] },
            "production": { "localize": true }
          }
        }
      }
    }
  }
}
```

### 6. Construir y servir por idioma

```bash
# Construir para todos los locales
ng build --localize

# Servir un idioma específico en desarrollo
ng serve --configuration=en
```

## $localize en código TypeScript

```typescript
import '@angular/localize/init';

@Injectable({ providedIn: 'root' })
export class MensajesService {
  getMensajeBienvenida(nombre: string): string {
    return $localize`Hola, ${nombre}:nombre:! Bienvenido a la plataforma.`;
  }

  getErrorCampoRequerido(): string {
    return $localize`:@@ERROR_REQUERIDO:Este campo es obligatorio.`;
  }
}
```

## Pipes de i18n integrados en Angular

Angular incluye pipes que usan los datos de localización automáticamente:

```html
<!-- Fechas -->
<p>{{ fecha | date:'fullDate' }}</p>
<!-- es: "lunes, 15 de enero de 2024" -->
<!-- en: "Monday, January 15, 2024" -->

<!-- Números y moneda -->
<p>{{ precio | currency }}</p>        <!-- Usa la moneda del locale -->
<p>{{ numero | number:'1.2-3' }}</p>  <!-- Separadores según locale -->

<!-- Porcentajes -->
<p>{{ 0.85 | percent }}</p>            <!-- "85%" en todos los locales -->
```

## ngx-translate: alternativa dinámica

El enfoque nativo de Angular genera un build por idioma. Si necesitas **cambio dinámico de idioma en tiempo de ejecución**, `ngx-translate` es la solución más popular:

```bash
npm install @ngx-translate/core @ngx-translate/http-loader
```

```typescript
// app.config.ts
import { TranslateModule, TranslateLoader } from '@ngx-translate/core';
import { TranslateHttpLoader } from '@ngx-translate/http-loader';

export function createTranslateLoader(http: HttpClient) {
  return new TranslateHttpLoader(http, './assets/i18n/', '.json');
}
```

```json
// assets/i18n/es.json
{
  "BIENVENIDA": "Bienvenido, {{ nombre }}!",
  "NAVEGACION": {
    "INICIO": "Inicio",
    "PRODUCTOS": "Productos"
  }
}
```

```html
<!-- Uso en plantilla -->
<h1>{{ 'BIENVENIDA' | translate:{ nombre: usuario.nombre } }}</h1>
<nav>
  <a>{{ 'NAVEGACION.INICIO' | translate }}</a>
</nav>
```

```typescript
// Cambiar idioma dinámicamente
constructor(private translate: TranslateService) {
  translate.setDefaultLang('es');
  translate.use('es'); // Cambiar a español
}

cambiarIdioma(idioma: string): void {
  this.translate.use(idioma); // 'es', 'en', 'fr'...
}
```

## Comparativa: i18n nativo vs ngx-translate

| Característica          | i18n nativo              | ngx-translate                |
|-------------------------|--------------------------|------------------------------|
| Cambio de idioma        | Requiere recarga/rebuild | ✅ Dinámico en runtime       |
| Rendimiento             | ✅ Óptimo (AOT)          | Menor (runtime)              |
| Formato                 | XLIFF, XMB, JSON         | JSON                         |
| Configuración           | Más compleja             | Más simple                   |
| Ideal para              | Sitios con URLs por lang | Apps SPA con switch dinámico |

**Consejo de entrevista**: La elección entre i18n nativo y ngx-translate depende del requisito de cambio dinámico de idioma. El i18n nativo es más performante y está recomendado por el equipo de Angular cuando no se necesita cambio dinámico, ya que todo se resuelve en tiempo de build. Si se necesita cambio dinámico, ngx-translate es el estándar de la industria.
