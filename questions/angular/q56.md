---
title: ¿Qué es la interpolación en Angular?
difficulty: easy
tags: [angular, interpolation, template, data-binding]
---

La **interpolación** es la forma más básica de data binding en Angular. Usa la sintaxis de dobles llaves `{{ expresión }}` para evaluar una expresión TypeScript y mostrar su resultado como texto en la plantilla HTML.

## Sintaxis básica

```typescript
@Component({
  standalone: true,
  template: `
    <h1>{{ titulo }}</h1>
    <p>Bienvenido, {{ nombreUsuario }}!</p>
    <p>Versión: {{ version }}</p>
  `
})
export class AppComponent {
  titulo = 'Mi Aplicación Angular';
  nombreUsuario = 'Ana García';
  version = '17.0.0';
}
```

## Expresiones válidas en la interpolación

La interpolación puede evaluar cualquier **expresión** de TypeScript que produzca un valor convertible a string:

```html
<!-- Operaciones aritméticas -->
<p>Suma: {{ 5 + 3 }}</p>               <!-- 8 -->
<p>Precio con IVA: {{ precio * 1.21 }}</p>

<!-- Concatenación de strings -->
<p>{{ 'Hola, ' + nombreUsuario + '!' }}</p>

<!-- Llamadas a métodos del componente -->
<p>{{ obtenerSaludo() }}</p>
<p>{{ fecha | date:'dd/MM/yyyy' }}</p>  <!-- Con pipes -->

<!-- Expresiones condicionales (ternario) -->
<p>{{ esMayorDeEdad ? 'Adulto' : 'Menor' }}</p>

<!-- Operador de navegación segura -->
<p>{{ usuario?.direccion?.ciudad }}</p>

<!-- Acceso a propiedades de objetos y arrays -->
<p>{{ usuario.nombre }}</p>
<p>{{ frutas[0] }}</p>
<p>{{ configuracion['idioma'] }}</p>
```

## Limitaciones: qué NO se puede usar

```html
<!-- ❌ Asignaciones -->
{{ a = 10 }}

<!-- ❌ Operadores de incremento/decremento -->
{{ i++ }}
{{ ++contador }}

<!-- ❌ Operador new -->
{{ new Date() }}  <!-- ❌ Usar un método en el componente -->
{{ fecha }}       <!-- ✅ Calculado en el componente -->

<!-- ❌ Encadenamiento de expresiones con ; -->
{{ a; b }}

<!-- ❌ Operadores bitwise -->
{{ a | b }}  <!-- ⚠️ En plantillas, | es para pipes, no OR bitwise -->
```

## Interpolación vs Property Binding

Ambos pueden mostrar datos, pero hay diferencias importantes:

```html
<!-- Interpolación: convierte a STRING -->
<p>{{ titulo }}</p>
<img src="{{ imagenUrl }}">          <!-- ⚠️ Puede mostrar brevemente la URL literal -->

<!-- Property binding: enlaza el valor directamente -->
<img [src]="imagenUrl">             <!-- ✅ Recomendado para atributos -->
<input [disabled]="estaDeshabilitado"> <!-- ✅ Para booleanos, usar siempre binding -->

<!-- Diferencia crítica con HTML -->
<p title="{{ descripcion }}">...</p>     <!-- Atributo string -->
<p [title]="descripcion">...</p>         <!-- Property binding (igual resultado)  -->
<p [innerHTML]="htmlContent">...</p>     <!-- ← SOLO con property binding, no {{ }} -->
```

## Seguridad: XSS y la interpolación

Angular **escapa automáticamente** el contenido insertado mediante interpolación:

```typescript
contenidoMalicioso = '<script>alert("XSS")</script><b>Texto</b>';
```

```html
<!-- Angular muestra el texto literal, no ejecuta el script -->
<p>{{ contenidoMalicioso }}</p>
<!-- Renderiza: <script>alert("XSS")</script><b>Texto</b> (como texto plano) -->

<!-- Para HTML, usar [innerHTML] (Angular lo sanitiza también) -->
<p [innerHTML]="contenidoMalicioso"></p>
<!-- Renderiza: <b>Texto</b> (elimina el script) -->
```

## Interpolación con pipes

Los pipes se pueden combinar directamente en la interpolación:

```html
<p>{{ precio | currency:'EUR':'symbol':'1.2-2' }}</p>
<p>{{ fecha | date:'EEEE, d MMMM yyyy' | titlecase }}</p>
<p>{{ descripcion | truncar:100 | uppercase }}</p>
<p>{{ temperatura | number:'1.1-1' }}°C</p>
```

## Cambiar el delimitador de interpolación

En raras ocasiones (conflicto con otros sistemas de templates), puedes cambiar el delimitador:

```typescript
@Component({
  interpolation: ['[[', ']]'],  // Cambiar de {{ }} a [[ ]]
  template: `<p>[[ titulo ]]</p>`
})
export class ComponentePersonalizado {}
```
