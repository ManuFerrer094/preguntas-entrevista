title: ¿Qué es Angular Forms y la diferencia entre Template-driven y Reactive Forms?

Angular ofrece dos enfoques para trabajar con formularios: **Template-driven Forms** y **Reactive Forms**. Ambos gestionan la validación, el estado del formulario y el binding de datos, pero difieren en dónde reside la lógica y cómo se definen.

## Template-driven Forms

La lógica del formulario reside principalmente en el template HTML, usando directivas de Angular como `ngModel`. Son más sencillos para formularios simples.

```typescript
// Requiere importar FormsModule
@Component({
  imports: [FormsModule],
  template: `
    <form #myForm="ngForm" (ngSubmit)="onSubmit(myForm)">
      <input
        name="email"
        [(ngModel)]="user.email"
        required
        email
        #emailField="ngModel"
      />
      <span *ngIf="emailField.invalid && emailField.touched">
        Email inválido
      </span>

      <input
        name="password"
        type="password"
        [(ngModel)]="user.password"
        required
        minlength="8"
      />

      <button type="submit" [disabled]="myForm.invalid">Enviar</button>
    </form>
  `
})
export class LoginComponent {
  user = { email: '', password: '' };

  onSubmit(form: NgForm) {
    if (form.valid) {
      console.log(form.value);
    }
  }
}
```

## Reactive Forms

La lógica del formulario reside en el componente TypeScript. Ofrecen mayor control, testabilidad y son preferidos en aplicaciones profesionales.

```typescript
// Requiere importar ReactiveFormsModule
@Component({
  imports: [ReactiveFormsModule],
  template: `
    <form [formGroup]="loginForm" (ngSubmit)="onSubmit()">
      <input formControlName="email" />
      <span *ngIf="email.invalid && email.touched">
        {{ email.hasError('required') ? 'Campo requerido' : 'Email inválido' }}
      </span>

      <input type="password" formControlName="password" />

      <button type="submit" [disabled]="loginForm.invalid">Entrar</button>
    </form>
  `
})
export class LoginComponent {
  private fb = inject(FormBuilder);

  loginForm = this.fb.group({
    email: ['', [Validators.required, Validators.email]],
    password: ['', [Validators.required, Validators.minLength(8)]]
  });

  get email() { return this.loginForm.get('email')!; }

  onSubmit() {
    if (this.loginForm.valid) {
      console.log(this.loginForm.value);
    }
  }
}
```

**Validación personalizada con Reactive Forms:**

```typescript
function noSpacesValidator(control: AbstractControl): ValidationErrors | null {
  if (control.value && control.value.includes(' ')) {
    return { noSpaces: true };
  }
  return null;
}

// Validador asíncrono (por ejemplo, verificar si el email ya existe)
function uniqueEmailValidator(userService: UserService): AsyncValidatorFn {
  return (control: AbstractControl): Observable<ValidationErrors | null> => {
    return userService.checkEmailExists(control.value).pipe(
      map(exists => exists ? { emailTaken: true } : null),
      debounceTime(300)
    );
  };
}
```

## Comparación

| Aspecto | Template-driven | Reactive |
|---|---|---|
| Definición | En el HTML | En TypeScript |
| Testing | Más difícil | Fácil (sin DOM) |
| Validación dinámica | Compleja | Sencilla |
| Formularios complejos | Difícil de mantener | Escalable |
| Curva de aprendizaje | Menor | Mayor |

**Consejo de entrevista**: En proyectos profesionales se prefieren los Reactive Forms por su mayor control, testabilidad y capacidad para manejar formularios dinámicos. Los Template-driven forms son aceptables para prototipos o formularios muy simples.
