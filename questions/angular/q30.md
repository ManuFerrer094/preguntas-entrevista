---
title: ¿Para qué sirve la directiva *ngIf en Angular?
difficulty: easy
tags: [angular, ngIf, directives, template]
---

La directiva **`*ngIf`** es una directiva estructural que incluye o excluye un elemento del DOM según una condición booleana. Cuando la condición es `false`, el elemento **se elimina completamente del DOM** (no solo se oculta con CSS).

## Uso básico

```typescript
@Component({
  standalone: true,
  imports: [CommonModule],
  template: `
    <p *ngIf="isLoggedIn">¡Bienvenido de vuelta!</p>
    <p *ngIf="!isLoggedIn">Por favor, inicia sesión.</p>
  `
})
export class SaludoComponent {
  isLoggedIn = false;
}
```

## Con bloque `else`

Se usa junto con `<ng-template>` para definir el contenido alternativo:

```html
<div *ngIf="cargando; else contenido">
  <app-spinner></app-spinner>
</div>

<ng-template #contenido>
  <div class="datos">{{ datos | json }}</div>
</ng-template>
```

## Con bloques `then` y `else`

```html
<ng-container *ngIf="estado === 'ok'; then exito; else error"></ng-container>

<ng-template #exito>
  <p class="success">¡Operación completada!</p>
</ng-template>

<ng-template #error>
  <p class="error">Algo salió mal.</p>
</ng-template>
```

## Como alias (as)

Una característica muy útil para trabajar con Observables y el async pipe:

```html
<div *ngIf="usuario$ | async as usuario">
  <h2>{{ usuario.nombre }}</h2>
  <p>{{ usuario.email }}</p>
</div>
```

Esto evita múltiples suscripciones y la comprobación de nulos en cada binding.

## `*ngIf` vs `[hidden]`

| Característica        | `*ngIf`                       | `[hidden]`                    |
|-----------------------|-------------------------------|-------------------------------|
| Elemento en el DOM    | ❌ Se elimina del DOM         | ✅ Permanece en el DOM        |
| Ciclo de vida         | Se ejecuta al añadir/quitar   | Sin cambios en ciclo de vida  |
| Rendimiento           | ✅ Mejor (menos nodos DOM)    | ⚠️ El elemento siempre existe |
| Caso de uso           | Cuando el elemento no siempre se necesita | Cuando el toggle es frecuente |

```html
<!-- *ngIf: elimina el componente del DOM (destruye su estado) -->
<app-mapa *ngIf="mostrarMapa"></app-mapa>

<!-- [hidden]: oculta visualmente, pero el componente sigue vivo -->
<app-mapa [hidden]="!mostrarMapa"></app-mapa>
```

## Nueva sintaxis @if / @else (Angular 17+)

Angular 17 introduce una sintaxis de control de flujo nativa que reemplaza a `*ngIf`:

```html
@if (usuario) {
  <div class="perfil">
    <h2>{{ usuario.nombre }}</h2>
    <p>{{ usuario.email }}</p>
  </div>
} @else if (cargando) {
  <app-skeleton></app-skeleton>
} @else {
  <p>No se encontró el usuario.</p>
}
```

Ventajas del nuevo `@if` sobre `*ngIf`:
- **Mejor narrowing de tipos**: TypeScript reconoce que dentro del bloque el valor no es nulo.
- No requiere importar `CommonModule` ni `NgIf`.
- Sintaxis más legible, similar a JavaScript nativo.
- El bloque `@else if` no requiere un `<ng-template>` adicional.

```bash
# Migrar automáticamente de *ngIf a @if
ng generate @angular/core:control-flow
```
