---
title: ¿Qué son los Resolvers en Angular?
difficulty: medium
tags: [angular, resolvers, routing, data-fetching]
---

Los Resolvers en Angular son clases o funciones que pre-cargan datos antes de que una ruta se active. Su principal ventaja es que garantizan que el componente de destino recibe los datos necesarios desde el primer momento, evitando estados de carga intermedios y mejorando la experiencia de usuario.

Sin un resolver, el flujo sería: navegar → activar ruta → mostrar componente con estado vacío → hacer petición HTTP → mostrar datos. Con un resolver: navegar → cargar datos → activar ruta → mostrar componente con datos listos.

```typescript
// resolver funcional (Angular 14+)
import { inject } from '@angular/core';
import { ResolveFn, ActivatedRouteSnapshot } from '@angular/router';
import { ProductService } from './product.service';
import { Product } from './product.model';

export const productResolver: ResolveFn<Product> = (route: ActivatedRouteSnapshot) => {
  const productService = inject(ProductService);
  const id = route.paramMap.get('id')!;
  return productService.getProduct(id);
};
```

Configuración en el módulo de rutas:

```typescript
const routes: Routes = [
  {
    path: 'products/:id',
    component: ProductDetailComponent,
    resolve: {
      product: productResolver
    }
  }
];
```

Acceder a los datos resueltos en el componente a través de `ActivatedRoute`:

```typescript
@Component({...})
export class ProductDetailComponent implements OnInit {
  private route = inject(ActivatedRoute);
  product!: Product;

  ngOnInit() {
    // Los datos están disponibles inmediatamente
    this.product = this.route.snapshot.data['product'];

    // O de forma reactiva
    this.route.data.subscribe(data => {
      this.product = data['product'];
    });
  }
}
```

Una consideración importante es el manejo de errores. Si el resolver falla (por ejemplo, el recurso no existe), la navegación se cancela. Se puede mejorar usando `catchError` para redirigir a una página 404:

```typescript
export const productResolver: ResolveFn<Product | null> = (route) => {
  const productService = inject(ProductService);
  const router = inject(Router);
  const id = route.paramMap.get('id')!;

  return productService.getProduct(id).pipe(
    catchError(() => {
      router.navigate(['/not-found']);
      return of(null);
    })
  );
};
```

**Consejo de entrevista**: Los resolvers son ideales para rutas que dependen de datos críticos. Sin embargo, hay un trade-off: el usuario espera durante la carga antes de ver la nueva página. Para cargas largas puede ser mejor mostrar un skeleton en el componente directamente.
