---
title: ¿Qué es la proyección de contenido (Content Projection) en Angular?
difficulty: medium
tags: [angular, content-projection, ng-content, slots]
---

La **proyección de contenido** (Content Projection) es un patrón de Angular que permite a un componente padre insertar contenido HTML arbitrario dentro de un componente hijo, similar al concepto de **slots** en Web Components.

Esto se logra con la etiqueta especial `<ng-content>` en la plantilla del componente hijo.

## Proyección simple (slot único)

El caso más común: el componente acepta cualquier contenido entre sus etiquetas.

```typescript
// card.component.ts
@Component({
  selector: 'app-card',
  standalone: true,
  template: `
    <div class="card">
      <div class="card-body">
        <ng-content></ng-content>  <!-- El contenido del padre se proyecta aquí -->
      </div>
    </div>
  `,
  styles: [`
    .card { border: 1px solid #ddd; border-radius: 8px; padding: 16px; }
  `]
})
export class CardComponent {}
```

```html
<!-- Uso en el componente padre -->
<app-card>
  <h2>Título del artículo</h2>
  <p>Descripción del contenido proyectado.</p>
  <button>Leer más</button>
</app-card>
```

## Proyección múltiple (multi-slot) con `select`

Se pueden definir **múltiples slots** usando el atributo `select` con selectores CSS:

```typescript
@Component({
  selector: 'app-modal',
  standalone: true,
  template: `
    <div class="modal-overlay">
      <div class="modal">
        <div class="modal-header">
          <ng-content select="[modal-header]"></ng-content>
          <!-- Solo proyecta elementos con el atributo modal-header -->
        </div>
        <div class="modal-body">
          <ng-content select="[modal-body]"></ng-content>
        </div>
        <div class="modal-footer">
          <ng-content select="[modal-footer]"></ng-content>
        </div>
        <!-- Contenido sin selector específico va aquí -->
        <ng-content></ng-content>
      </div>
    </div>
  `
})
export class ModalComponent {}
```

```html
<!-- Uso con slots nombrados -->
<app-modal>
  <h3 modal-header>¿Confirmar eliminación?</h3>

  <p modal-body>
    Esta acción no se puede deshacer. ¿Deseas continuar?
  </p>

  <div modal-footer>
    <button (click)="cancelar()">Cancelar</button>
    <button (click)="confirmar()" class="danger">Eliminar</button>
  </div>
</app-modal>
```

## Selección por componente

```html
<!-- También puedes seleccionar por tipo de componente -->
<ng-content select="app-avatar"></ng-content>
<ng-content select=".titulo-principal"></ng-content>
<ng-content select="input[type=text]"></ng-content>
```

## Proyección condicional con `*ngIf` y `NgTemplateOutlet`

Para proyección más dinámica, se puede combinar con `TemplateRef`:

```typescript
@Component({
  selector: 'app-expansion-panel',
  standalone: true,
  imports: [CommonModule],
  template: `
    <div class="panel">
      <div class="panel-header" (click)="abierto = !abierto">
        <ng-content select="[panel-titulo]"></ng-content>
        <span>{{ abierto ? '▲' : '▼' }}</span>
      </div>
      @if (abierto) {
        <div class="panel-content">
          <ng-content></ng-content>
        </div>
      }
    </div>
  `
})
export class ExpansionPanelComponent {
  abierto = false;
}
```

## Acceder al contenido proyectado con @ContentChild

```typescript
@Component({
  selector: 'app-formulario-campo',
  standalone: true,
  template: `
    <label>{{ etiqueta }}</label>
    <ng-content select="input, select, textarea"></ng-content>
    @if (control?.invalid && control?.touched) {
      <span class="error">Campo inválido</span>
    }
  `
})
export class FormularioCampoComponent implements AfterContentInit {
  @Input() etiqueta = '';
  @ContentChild(NgControl) control!: NgControl;

  ngAfterContentInit(): void {
    console.log('Input proyectado:', this.control);
  }
}
```

## Ejemplo completo: componente Tarjeta reutilizable

```html
<!-- Uso del componente -->
<app-card>
  <img src="foto.jpg" alt="Producto" card-imagen>
  <h3 card-titulo>Producto Premium</h3>
  <p card-descripcion>La mejor calidad del mercado.</p>
  <button card-accion>Comprar ahora</button>
</app-card>
```

**Consejo de entrevista**: La proyección de contenido es clave para crear componentes **reutilizables y flexibles** (como los de una librería de UI). La diferencia importante con `@Input()` es que la proyección permite pasar **HTML/componentes completos**, mientras que `@Input()` pasa datos primitivos u objetos. `@ContentChild` / `@ContentChildren` se usan en `ngAfterContentInit()` para acceder programáticamente a los elementos proyectados.
