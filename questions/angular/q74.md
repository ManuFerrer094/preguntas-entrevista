El patrón **Smart/Dumb Components** (también llamado Container/Presentational o Stateful/Stateless) es una arquitectura de componentes que separa la lógica de negocio de la lógica de presentación, mejorando la reutilización, la testeabilidad y el mantenimiento.

## Concepto

- **Smart Components (Contenedor)**: conocen el estado de la aplicación, inyectan servicios y coordinan la lógica. Pasan datos a los hijos mediante `@Input()`.
- **Dumb Components (Presentacional)**: solo reciben datos mediante `@Input()` y emiten eventos mediante `@Output()`. Son puramente visuales y reutilizables.

## Ejemplo práctico: Lista de tareas

### Componente Dumb — Solo muestra, no sabe nada del estado global

```typescript
// tarea-item.component.ts
@Component({
  selector: 'app-tarea-item',
  standalone: true,
  imports: [CommonModule],
  template: `
    <li [class.completada]="tarea.completada">
      <span>{{ tarea.titulo }}</span>
      <button (click)="toggleClick()">
        {{ tarea.completada ? 'Deshacer' : 'Completar' }}
      </button>
      <button (click)="eliminarClick()">Eliminar</button>
    </li>
  `
})
export class TareaItemComponent {
  @Input({ required: true }) tarea!: Tarea;

  @Output() toggle = new EventEmitter<number>();
  @Output() eliminar = new EventEmitter<number>();

  toggleClick() { this.toggle.emit(this.tarea.id); }
  eliminarClick() { this.eliminar.emit(this.tarea.id); }
}
```

### Componente Dumb — Lista de tareas

```typescript
// tarea-lista.component.ts
@Component({
  selector: 'app-tarea-lista',
  standalone: true,
  imports: [TareaItemComponent],
  template: `
    @if (tareas.length === 0) {
      <p>No hay tareas pendientes.</p>
    } @else {
      <ul>
        @for (tarea of tareas; track tarea.id) {
          <app-tarea-item
            [tarea]="tarea"
            (toggle)="toggle.emit($event)"
            (eliminar)="eliminar.emit($event)"
          />
        }
      </ul>
    }
  `
})
export class TareaListaComponent {
  @Input({ required: true }) tareas: Tarea[] = [];
  @Output() toggle = new EventEmitter<number>();
  @Output() eliminar = new EventEmitter<number>();
}
```

### Componente Smart — Gestiona el estado y llama al servicio

```typescript
// tareas-container.component.ts
@Component({
  selector: 'app-tareas-container',
  standalone: true,
  imports: [TareaListaComponent, TareaFormComponent],
  template: `
    <h2>Mis Tareas ({{ pendientes() }})</h2>
    <app-tarea-form (agregar)="agregarTarea($event)" />
    <app-tarea-lista
      [tareas]="tareas()"
      (toggle)="toggleTarea($event)"
      (eliminar)="eliminarTarea($event)"
    />
  `
})
export class TareasContainerComponent {
  private tareaService = inject(TareaService);

  tareas = this.tareaService.tareas;
  pendientes = computed(() => this.tareas().filter(t => !t.completada).length);

  agregarTarea(titulo: string) {
    this.tareaService.agregar(titulo);
  }

  toggleTarea(id: number) {
    this.tareaService.toggleCompletada(id);
  }

  eliminarTarea(id: number) {
    this.tareaService.eliminar(id);
  }
}
```

## Beneficios del patrón

```
Smart Component (sabe de servicios, estado, NgRx)
    └── Dumb Component A (solo @Input/@Output)
    └── Dumb Component B (solo @Input/@Output)
            └── Dumb Component C (reutilizable en otro Smart)
```

| Característica      | Smart Component       | Dumb Component       |
|---------------------|-----------------------|----------------------|
| Inyecta servicios   | ✅ Sí                | ❌ No                |
| Conoce estado global| ✅ Sí                | ❌ No                |
| Reutilizable        | ❌ Acoplado al dominio| ✅ Muy reutilizable  |
| Testeable           | Necesita mocks        | ✅ Sin dependencias  |
| Change Detection    | Default               | ✅ OnPush ideal      |

## Optimización con ChangeDetectionStrategy.OnPush

```typescript
// Los Dumb Components son ideales para OnPush
@Component({
  selector: 'app-tarea-item',
  standalone: true,
  changeDetection: ChangeDetectionStrategy.OnPush,  // ✅ Máximo rendimiento
  template: `...`
})
export class TareaItemComponent {
  @Input({ required: true }) tarea!: Tarea;
  // Solo re-renderiza si @Input cambia por referencia
}
```

**Consejo de entrevista**: Este patrón es fundamental en Angular de producción. Los Dumb Components con `ChangeDetectionStrategy.OnPush` reducen drásticamente los ciclos de detección de cambios. Combinado con Signals y NgRx, los Smart Components se vuelven delgados y los Dumb Components son completamente reutilizables y fáciles de testear con inputs aislados.
