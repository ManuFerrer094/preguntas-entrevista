---
title: ¿Qué son los Guards en Angular?
difficulty: medium
tags: [angular, guards, routing, security]
---

Los Guards en Angular son interfaces que permiten controlar el acceso a las rutas de la aplicación. Actúan como guardianes que se ejecutan antes de activar o desactivar una ruta, permitiendo implementar lógica de autorización, autenticación y confirmación de cambios.

Angular ofrece varios tipos de guards:

- **CanActivate**: Determina si una ruta puede ser activada. Ideal para verificar autenticación.
- **CanActivateChild**: Similar a CanActivate pero para rutas hijas.
- **CanDeactivate**: Controla si el usuario puede abandonar una ruta (útil para formularios con cambios sin guardar).
- **CanLoad**: Evita que un módulo lazy-loaded se cargue si el usuario no tiene permisos.
- **Resolve**: Precarga datos antes de activar la ruta (técnicamente un resolver, no un guard).

A partir de Angular 14, los guards pueden implementarse como funciones simples en lugar de clases, lo que los hace más ligeros:

```typescript
// Guard funcional (Angular 14+)
import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';
import { AuthService } from './auth.service';

export const authGuard: CanActivateFn = (route, state) => {
  const authService = inject(AuthService);
  const router = inject(Router);

  if (authService.isLoggedIn()) {
    return true;
  }

  // Redirigir al login con la URL de retorno
  router.navigate(['/login'], { queryParams: { returnUrl: state.url } });
  return false;
};
```

Para aplicarlo en las rutas:

```typescript
const routes: Routes = [
  {
    path: 'dashboard',
    component: DashboardComponent,
    canActivate: [authGuard]
  },
  {
    path: 'admin',
    component: AdminComponent,
    canActivate: [authGuard, roleGuard] // múltiples guards
  }
];
```

Un guard `CanDeactivate` típico para proteger formularios con cambios sin guardar:

```typescript
export const unsavedChangesGuard: CanDeactivateFn<FormComponent> = (component) => {
  if (component.hasUnsavedChanges()) {
    return confirm('¿Tienes cambios sin guardar. ¿Deseas salir de todas formas?');
  }
  return true;
};
```

**Consejo de entrevista**: Los guards pueden devolver `boolean`, `UrlTree` (para redirecciones), o `Observable<boolean | UrlTree>` para operaciones asíncronas como llamadas a la API. Mencionar que desde Angular 15 se recomienda el enfoque funcional sobre las clases es un plus.
