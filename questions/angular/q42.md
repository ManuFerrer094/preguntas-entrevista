**TestBed** es la principal utilidad de testing de Angular. Crea un módulo de prueba configurable que simula un entorno Angular real, permitiendo instanciar componentes, inyectar servicios, y testear el comportamiento de los elementos de la aplicación de forma aislada.

## Configuración básica

```typescript
import { TestBed, ComponentFixture } from '@angular/core/testing';
import { SaludoComponent } from './saludo.component';

describe('SaludoComponent', () => {
  let componente: SaludoComponent;
  let fixture: ComponentFixture<SaludoComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      imports: [SaludoComponent]  // Para standalone components
      // declarations: [SaludoComponent]  // Para componentes con NgModule
    }).compileComponents();

    fixture = TestBed.createComponent(SaludoComponent);
    componente = fixture.componentInstance;
    fixture.detectChanges(); // Ejecuta ngOnInit y actualiza la vista
  });

  it('debería crearse correctamente', () => {
    expect(componente).toBeTruthy();
  });
});
```

## ComponentFixture

`ComponentFixture` es el contenedor del componente instanciado. Proporciona acceso al componente, al DOM y a las utilidades de detección de cambios:

```typescript
const fixture = TestBed.createComponent(MiComponente);

fixture.componentInstance  // La instancia del componente TypeScript
fixture.nativeElement      // El elemento DOM raíz del componente
fixture.debugElement       // Wrapper de Angular con utilidades de testing
fixture.detectChanges()    // Dispara la detección de cambios manualmente
fixture.whenStable()       // Promise que se resuelve cuando no hay tareas pendientes
```

## Ejemplo completo: componente con @Input y lógica

```typescript
// contador.component.ts
@Component({
  selector: 'app-contador',
  standalone: true,
  template: `
    <p data-testid="valor">{{ valor }}</p>
    <button (click)="incrementar()">+</button>
    <button (click)="decrementar()">-</button>
  `
})
export class ContadorComponent {
  @Input() valorInicial = 0;
  valor = 0;

  ngOnInit(): void {
    this.valor = this.valorInicial;
  }

  incrementar(): void { this.valor++; }
  decrementar(): void { this.valor--; }
}
```

```typescript
// contador.component.spec.ts
describe('ContadorComponent', () => {
  let fixture: ComponentFixture<ContadorComponent>;
  let componente: ContadorComponent;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      imports: [ContadorComponent]
    }).compileComponents();

    fixture = TestBed.createComponent(ContadorComponent);
    componente = fixture.componentInstance;
  });

  it('debería mostrar el valor inicial', () => {
    componente.valorInicial = 5;
    fixture.detectChanges(); // Ejecuta ngOnInit

    const parrafo = fixture.nativeElement.querySelector('[data-testid="valor"]');
    expect(parrafo.textContent).toBe('5');
  });

  it('debería incrementar el valor al hacer click en "+"', () => {
    componente.valorInicial = 0;
    fixture.detectChanges();

    const boton = fixture.nativeElement.querySelector('button');
    boton.click();
    fixture.detectChanges();

    expect(componente.valor).toBe(1);
  });
});
```

## Inyectar servicios y usar mocks

```typescript
describe('UsuarioComponent', () => {
  let mockUsuarioService: jasmine.SpyObj<UsuarioService>;

  beforeEach(async () => {
    mockUsuarioService = jasmine.createSpyObj('UsuarioService', ['getUsuario']);
    mockUsuarioService.getUsuario.and.returnValue(
      of({ id: 1, nombre: 'Ana' })
    );

    await TestBed.configureTestingModule({
      imports: [UsuarioComponent],
      providers: [
        { provide: UsuarioService, useValue: mockUsuarioService }
      ]
    }).compileComponents();
  });

  it('debería cargar el usuario al iniciar', () => {
    const fixture = TestBed.createComponent(UsuarioComponent);
    fixture.detectChanges();

    expect(mockUsuarioService.getUsuario).toHaveBeenCalled();
    expect(fixture.nativeElement.querySelector('h2').textContent).toContain('Ana');
  });
});
```

## Acceder al servicio inyectado

```typescript
// Obtener la instancia real del servicio desde TestBed
const servicio = TestBed.inject(UsuarioService);
const router = TestBed.inject(Router);
```

## Comandos para ejecutar tests

```bash
# Ejecutar todos los tests
ng test

# Ejecutar sin modo watch (para CI)
ng test --watch=false

# Con cobertura de código
ng test --code-coverage

# Ejecutar solo un archivo específico
ng test --include='**/contador.component.spec.ts'
```

**Consejo de entrevista**: Una buena práctica en tests de componentes Angular es siempre llamar a `fixture.detectChanges()` después de cambiar propiedades del componente, ya que Angular no actualiza la vista automáticamente en el entorno de testing. Otra práctica recomendada es usar `data-testid` en lugar de selectores CSS o texto para localizar elementos, lo que hace los tests más resilientes a cambios de estilo.
