---
title: ¿Qué es el async pipe y cuándo usarlo?
difficulty: medium
tags: [angular, async-pipe, rxjs, observable, template]
---

El **`async` pipe** es un pipe integrado de Angular que se **suscribe automáticamente** a un `Observable` o `Promise` en la plantilla y muestra el último valor emitido. Lo más importante: se **desuscribe automáticamente** cuando el componente se destruye, previniendo memory leaks.

## Uso básico

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe, CommonModule],
  template: `
    <!-- La suscripción es automática, sin ngOnInit ni ngOnDestroy -->
    <p>{{ tiempo$ | async }}</p>

    @if (usuario$ | async; as usuario) {
      <h2>{{ usuario.nombre }}</h2>
    }
  `
})
export class RelojComponent {
  // Observable que emite cada segundo
  tiempo$ = interval(1000).pipe(
    map(() => new Date().toLocaleTimeString('es-ES'))
  );

  // Observable HTTP (se gestiona automáticamente)
  usuario$ = this.usuarioService.getUsuario(1);

  constructor(private usuarioService: UsuarioService) {}
}
```

## Comparación: manual vs async pipe

```typescript
// ❌ Suscripción manual: requiere código de limpieza
@Component({
  template: `<p>{{ usuario?.nombre }}</p>`
})
export class UsuarioManualComponent implements OnInit, OnDestroy {
  usuario: Usuario | null = null;
  private subscription!: Subscription;

  ngOnInit(): void {
    this.subscription = this.usuarioService.getUsuario(1)
      .subscribe(u => this.usuario = u);
  }

  ngOnDestroy(): void {
    this.subscription.unsubscribe(); // Si olvidamos esto → memory leak
  }
}
```

```typescript
// ✅ Con async pipe: sin gestión de suscripciones
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    @if (usuario$ | async; as usuario) {
      <p>{{ usuario.nombre }}</p>
    }
  `
})
export class UsuarioAsyncComponent {
  usuario$ = this.usuarioService.getUsuario(1);
  // Angular gestiona la suscripción automáticamente ✅

  constructor(private usuarioService: UsuarioService) {}
}
```

## Múltiples suscripciones al mismo Observable

```html
<!-- ❌ Sin alias: se crean DOS suscripciones (dos peticiones HTTP) -->
<div>
  <p>{{ (usuario$ | async)?.nombre }}</p>
  <p>{{ (usuario$ | async)?.email }}</p>
</div>

<!-- ✅ Con alias 'as': una sola suscripción -->
@if (usuario$ | async; as usuario) {
  <div>
    <p>{{ usuario.nombre }}</p>
    <p>{{ usuario.email }}</p>
  </div>
}

<!-- Alternativa con ng-container (sin condicional) -->
<ng-container *ngIf="usuario$ | async as usuario">
  <p>{{ usuario.nombre }}</p>
</ng-container>
```

## Combinación con operadores RxJS

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe, CommonModule],
  template: `
    @if (datosVista$ | async; as datos) {
      <app-lista [items]="datos.productos"></app-lista>
      <p>Total: {{ datos.total }}</p>
    }
  `
})
export class DashboardComponent {
  datosVista$ = combineLatest([
    this.productoService.getProductos(),
    this.categoriaService.getCategorias()
  ]).pipe(
    map(([productos, categorias]) => ({
      productos,
      categorias,
      total: productos.reduce((sum, p) => sum + p.precio, 0)
    }))
  );

  constructor(
    private productoService: ProductoService,
    private categoriaService: CategoriaService
  ) {}
}
```

## async pipe con estado de carga y error

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe, CommonModule],
  template: `
    @if (estado$ | async; as estado) {
      @if (estado.cargando) {
        <app-spinner></app-spinner>
      } @else if (estado.error) {
        <app-error [mensaje]="estado.error"></app-error>
      } @else {
        <app-lista [items]="estado.datos!"></app-lista>
      }
    }
  `
})
export class ListaConEstadoComponent {
  estado$ = this.servicio.getDatos().pipe(
    map(datos => ({ datos, cargando: false, error: null })),
    startWith({ datos: null, cargando: true, error: null }),
    catchError(err => of({ datos: null, cargando: false, error: err.message }))
  );
}
```

## Cuándo usar async pipe vs suscripción manual

| Situación                                | Recomendación                    |
|------------------------------------------|----------------------------------|
| Mostrar datos en plantilla               | ✅ `async` pipe siempre          |
| Lógica compleja con múltiples efectos    | Suscripción manual con `takeUntil`|
| Necesitas el valor en el componente TS   | `toSignal()` (Angular 16+)       |
| Con `ChangeDetectionStrategy.OnPush`     | ✅ `async` pipe (notifica automáticamente) |

**Consejo de entrevista**: El async pipe es la forma más segura y recomendada de manejar Observables en plantillas Angular. Su mayor ventaja es que **nunca hay memory leaks** porque Angular se encarga de la desuscripción automáticamente. Una alternativa moderna es `toSignal()` de Angular 16+, que convierte un Observable en una Signal para usar en el componente TypeScript y en la plantilla sin necesidad del async pipe.
