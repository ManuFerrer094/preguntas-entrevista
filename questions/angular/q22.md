Los **componentes** son los bloques de construcción fundamentales de cualquier aplicación Angular. Cada componente controla una parte de la interfaz de usuario (una vista) y encapsula su lógica, plantilla HTML y estilos CSS.

## Anatomía de un componente

Un componente Angular se define con el decorador `@Component` que incluye:

- **`selector`**: el nombre de la etiqueta HTML con la que se usará el componente.
- **`template` / `templateUrl`**: el HTML de la vista.
- **`styles` / `styleUrls`**: los estilos CSS encapsulados.
- **`standalone`**: si es `true`, no pertenece a ningún NgModule (recomendado desde Angular 17).

```typescript
import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';

@Component({
  selector: 'app-user-card',
  standalone: true,
  imports: [CommonModule],
  templateUrl: './user-card.component.html',
  styleUrl: './user-card.component.scss'
})
export class UserCardComponent {
  title = 'Bienvenido';
  userName = 'Ana García';

  greet(): string {
    return `Hola, ${this.userName}!`;
  }
}
```

```html
<!-- user-card.component.html -->
<div class="card">
  <h2>{{ title }}</h2>
  <p>{{ greet() }}</p>
</div>
```

## Crear un componente con Angular CLI

La forma más rápida y recomendada de crear componentes es usando el CLI:

```bash
# Componente standalone (Angular 17+ por defecto)
ng generate component user-card
# o su forma corta:
ng g c user-card

# Con opciones específicas
ng g c features/user/user-card --standalone --skip-tests
```

El CLI genera automáticamente 4 archivos:
1. `user-card.component.ts` — la clase TypeScript
2. `user-card.component.html` — la plantilla
3. `user-card.component.scss` — los estilos
4. `user-card.component.spec.ts` — los tests unitarios

## Árbol de componentes

Las aplicaciones Angular se organizan como un **árbol de componentes**, donde `AppComponent` es la raíz:

```
AppComponent
├── NavbarComponent
├── MainLayoutComponent
│   ├── UserListComponent
│   │   └── UserCardComponent (×N)
│   └── SidebarComponent
└── FooterComponent
```

## Encapsulación de estilos

Por defecto, Angular usa **ViewEncapsulation.Emulated**, que emula el Shadow DOM añadiendo atributos únicos a los elementos para que los estilos no se filtren al exterior. Opciones:

- `ViewEncapsulation.Emulated` (por defecto): estilos encapsulados.
- `ViewEncapsulation.None`: estilos globales.
- `ViewEncapsulation.ShadowDom`: Shadow DOM nativo del navegador.
