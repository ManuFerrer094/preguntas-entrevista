Los **Feature Modules** (módulos de funcionalidad) son NgModules que organizan el código según las **funcionalidades o dominios** de la aplicación, en lugar de agrupar todo en un solo módulo raíz. Mejoran la mantenibilidad, la separación de responsabilidades y habilitan el **lazy loading**.

## Tipos de módulos en Angular

Angular documenta cinco tipos de feature modules, cada uno con un propósito específico:

| Tipo             | Propósito                                                     |
|------------------|---------------------------------------------------------------|
| **Domain**       | Agrupa componentes de una funcionalidad de negocio             |
| **Routing**      | Configuración de rutas de una funcionalidad                   |
| **Service**      | Servicios singleton (sin componentes exportados)              |
| **Widget**       | Componentes/pipes/directivas reutilizables                    |
| **Entry**        | Componentes de entrada (punto de arranque de lazy loading)    |

## Crear un Feature Module

```bash
# Generar módulo de funcionalidad con routing
ng generate module features/productos --routing
```

Estructura resultante:
```
src/
└── app/
    └── features/
        └── productos/
            ├── productos.module.ts
            ├── productos-routing.module.ts
            ├── productos-list/
            │   └── productos-list.component.ts
            └── productos-detail/
                └── productos-detail.component.ts
```

```typescript
// productos.module.ts
@NgModule({
  declarations: [
    ProductosListComponent,
    ProductosDetailComponent,
    ProductoCardComponent
  ],
  imports: [
    CommonModule,
    ReactiveFormsModule,
    ProductosRoutingModule,
    SharedModule
  ]
})
export class ProductosModule {}
```

```typescript
// productos-routing.module.ts
const routes: Routes = [
  { path: '', component: ProductosListComponent },
  { path: ':id', component: ProductosDetailComponent }
];

@NgModule({
  imports: [RouterModule.forChild(routes)],
  exports: [RouterModule]
})
export class ProductosRoutingModule {}
```

## Lazy Loading con Feature Modules

El beneficio principal es cargar el módulo solo cuando el usuario navega a esa ruta:

```typescript
// app-routing.module.ts
const routes: Routes = [
  { path: '', redirectTo: 'inicio', pathMatch: 'full' },
  {
    path: 'productos',
    loadChildren: () =>
      import('./features/productos/productos.module')
        .then(m => m.ProductosModule)
    // El bundle de productos solo se descarga cuando se navega a /productos
  },
  {
    path: 'usuarios',
    loadChildren: () =>
      import('./features/usuarios/usuarios.module')
        .then(m => m.UsuariosModule)
  }
];
```

## Lazy Loading con Standalone Components (Angular 17+)

Con la arquitectura moderna, se puede hacer lazy loading sin NgModules:

```typescript
// app.routes.ts
export const routes: Routes = [
  {
    path: 'productos',
    loadComponent: () =>
      import('./features/productos/productos-list.component')
        .then(m => m.ProductosListComponent)
  },
  {
    path: 'usuarios',
    loadChildren: () =>
      import('./features/usuarios/usuarios.routes')
        .then(m => m.USUARIOS_ROUTES)
  }
];
```

```typescript
// features/usuarios/usuarios.routes.ts
export const USUARIOS_ROUTES: Routes = [
  { path: '', component: UsuariosListComponent },
  { path: ':id', component: UsuarioDetailComponent }
];
```

## Arquitectura recomendada

```
src/app/
├── core/             # CoreModule: servicios singleton, guards, interceptors
│   ├── guards/
│   ├── interceptors/
│   └── services/
├── shared/           # SharedModule: componentes, pipes, directivas reutilizables
│   ├── components/
│   ├── directives/
│   └── pipes/
├── features/         # Feature Modules: una carpeta por dominio
│   ├── productos/
│   ├── usuarios/
│   └── dashboard/
└── app.component.ts
```

## Verificar el lazy loading

```bash
ng build --configuration=production
# Busca en el output archivos como: chunk-XXXXXXXX.js
# Estos son los bundles generados por lazy loading
```

**Consejo de entrevista**: El lazy loading es una de las optimizaciones de rendimiento más importantes en aplicaciones Angular grandes. La pregunta clave es: ¿qué hacer cuando múltiples feature modules necesitan los mismos servicios? La respuesta es un **CoreModule** (con `forRoot()`) o, en la arquitectura moderna, `providedIn: 'root'`. Nunca proveas servicios singleton en un SharedModule, ya que se crearía una instancia por módulo que lo importe.
