---
title: ¿Qué es el operador de navegación segura (?.) en Angular?
difficulty: easy
tags: [angular, template, safe-navigation, null-safety]
---

El **operador de navegación segura** (`?.`), también llamado **optional chaining** en JavaScript moderno, es una sintaxis que permite acceder a propiedades de objetos que podrían ser `null` o `undefined` sin lanzar un error. Angular lo soporta directamente en sus plantillas.

## El problema sin `?.`

```typescript
@Component({
  template: `
    <!-- ❌ Error si 'usuario' es null: Cannot read property 'nombre' of null -->
    <p>{{ usuario.nombre }}</p>
    <p>{{ usuario.direccion.ciudad }}</p>
  `
})
export class PerfilComponent {
  usuario: Usuario | null = null; // Puede ser null al inicio (antes de cargar datos)
}
```

## Solución con `?.`

```typescript
@Component({
  template: `
    <!-- ✅ Si 'usuario' es null, devuelve undefined (no lanza error) -->
    <p>{{ usuario?.nombre }}</p>

    <!-- ✅ Encadenamiento profundo -->
    <p>{{ usuario?.direccion?.ciudad }}</p>
    <p>{{ usuario?.contacto?.telefono?.numero }}</p>

    <!-- ✅ También funciona con métodos -->
    <p>{{ usuario?.obtenerNombreCompleto() }}</p>

    <!-- ✅ Con arrays -->
    <p>{{ pedidos?.[0]?.estado }}</p>
  `
})
export class PerfilComponent {
  usuario: Usuario | null = null;
  pedidos: Pedido[] | null = null;
}
```

## Combinación con el operador de fusión nula `??`

En las plantillas de Angular también puedes combinar `?.` con `||` para valores por defecto (aunque `??` no está soportado directamente en todas las versiones de template):

```html
<!-- Usar || para valor por defecto -->
<p>{{ usuario?.nombre || 'Usuario anónimo' }}</p>
<p>{{ usuario?.edad || 'Edad no especificada' }}</p>

<!-- También se puede usar con ngIf para mayor claridad -->
<p *ngIf="usuario; else anonimo">{{ usuario.nombre }}</p>
<ng-template #anonimo>Usuario anónimo</ng-template>
```

## Casos de uso comunes

### Datos cargados de forma asíncrona

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    @if (usuario$ | async; as usuario) {
      <!-- Dentro del @if, usuario nunca es null -->
      <p>{{ usuario.nombre }}</p>
    }

    <!-- O con async pipe directamente (sin @if) -->
    <p>{{ (usuario$ | async)?.nombre }}</p>
    <p>{{ (perfil$ | async)?.direccion?.calle }}</p>
  `
})
export class UsuarioComponent {
  usuario$ = this.userService.getUsuario(1);
  perfil$ = this.userService.getPerfil(1);

  constructor(private userService: UserService) {}
}
```

### Acceso condicional a índices de array

```html
<!-- ✅ Acceso seguro al primer elemento -->
<p>{{ listaProductos?.[0]?.nombre }}</p>

<!-- ✅ Verificar si existe antes de iterar -->
@if (listaProductos?.length) {
  @for (producto of listaProductos!; track producto.id) {
    <li>{{ producto.nombre }}</li>
  }
}
```

## Diferencia con `[hidden]` y verificación manual

```html
<!-- Sin ?. (más verboso pero explícito) -->
<p *ngIf="usuario">{{ usuario!.nombre }}</p>

<!-- Con ?. (más conciso) -->
<p>{{ usuario?.nombre }}</p>

<!-- ✅ Mejor práctica con @if (Angular 17+) - mejor type narrowing -->
@if (usuario) {
  <p>{{ usuario.nombre }}</p>  <!-- TypeScript sabe que usuario no es null aquí -->
}
```

## En TypeScript moderno (clases del componente)

El operador `?.` también se usa en el código TypeScript del componente, no solo en las plantillas:

```typescript
export class PedidoComponent {
  pedido: Pedido | undefined;

  get ciudadEntrega(): string {
    return this.pedido?.direccionEntrega?.ciudad ?? 'No especificada';
  }

  procesarPedido(): void {
    // ✅ Solo llama al método si existe
    this.pedido?.confirmar();
  }
}
```
