---
title: ¿Cuál es la diferencia entre Promise y Observable?

**Promise** y **Observable** son dos mecanismos para trabajar con operaciones asíncronas en JavaScript/TypeScript, pero tienen diferencias fundamentales. Angular utiliza Observables de **RxJS** de forma extensiva (HttpClient, Router, Forms, etc.).

## Promise

Una Promise representa un **único valor futuro**. Una vez que se resuelve o rechaza, no puede volver a emitir.

- **Eager**: se ejecuta inmediatamente al crearse.
- **Un solo valor**: emite exactamente un resultado (o un error).
- **No cancelable**: no hay forma nativa de cancelar una Promise en curso.
- **No compuesta**: la API de composición es limitada comparada con RxJS.

```typescript
// Promise: eager, un solo valor
const promesa = new Promise<Usuario>((resolve, reject) => {
  // Se ejecuta INMEDIATAMENTE
  fetch('/api/usuario/1')
    .then(res => res.json())
    .then(data => resolve(data))
    .catch(err => reject(err));
});

promesa
  .then(usuario => console.log('Usuario:', usuario))
  .catch(err => console.error('Error:', err));
```

## Observable

Un Observable de RxJS puede emitir **múltiples valores a lo largo del tiempo**. Es la base del enfoque reactivo.

- **Lazy**: no se ejecuta hasta que alguien se suscribe.
- **Múltiples valores**: puede emitir cero, uno o infinitos valores.
- **Cancelable**: la suscripción se puede cancelar con `unsubscribe()`.
- **Operadores**: una librería enorme de operadores para transformar, combinar y filtrar flujos.

```typescript
import { Observable, interval } from 'rxjs';
import { map, take, filter } from 'rxjs/operators';

// Observable: lazy, múltiples valores
const observable$ = new Observable<number>(subscriber => {
  // NO se ejecuta hasta que alguien se suscriba
  let contador = 0;
  const id = setInterval(() => {
    subscriber.next(contador++);
    if (contador > 5) subscriber.complete();
  }, 1000);

  // Función de limpieza al cancelar
  return () => clearInterval(id);
});

// Solo se empieza a ejecutar aquí:
const suscripcion = observable$.subscribe({
  next: valor => console.log('Valor:', valor),
  error: err => console.error('Error:', err),
  complete: () => console.log('Completado')
});

// Cancelar después de 3 segundos
setTimeout(() => suscripcion.unsubscribe(), 3000);
```

## Uso en Angular (HttpClient)

Angular devuelve Observables desde `HttpClient`, pero puedes convertirlos a Promise si lo necesitas:

```typescript
@Injectable({ providedIn: 'root' })
export class UsuarioService {
  constructor(private http: HttpClient) {}

  // ✅ Observable (recomendado): lazy, cancelable, composable
  getUsuario$(id: number): Observable<Usuario> {
    return this.http.get<Usuario>(`/api/usuarios/${id}`).pipe(
      retry(2),
      catchError(this.manejarError)
    );
  }

  // También posible: convertir a Promise
  async getUsuario(id: number): Promise<Usuario> {
    return firstValueFrom(this.http.get<Usuario>(`/api/usuarios/${id}`));
  }
}
```

## Tabla comparativa

| Característica        | Promise                    | Observable (RxJS)                    |
|-----------------------|----------------------------|--------------------------------------|
| Número de valores     | Uno                        | Cero, uno o múltiples                |
| Ejecución             | Eager (inmediata)          | Lazy (al suscribirse)                |
| Cancelable            | ❌ No                      | ✅ Sí (`unsubscribe()`)              |
| Operadores            | `.then()`, `.catch()`      | `map`, `filter`, `switchMap`, etc.   |
| Manejo de errores     | `.catch()`                 | `catchError()`, `retry()`            |
| Composición           | Limitada                   | Muy potente (combinación de flujos)  |
| Streams infinitos     | ❌ No                      | ✅ Sí (WebSockets, eventos, timers)  |
| Integración Angular   | Parcial                    | ✅ Nativa (HttpClient, Forms, Router)|

## Operadores RxJS equivalentes a métodos de Promise

```typescript
// Promise.all → forkJoin
forkJoin([obs1$, obs2$, obs3$]).subscribe(([r1, r2, r3]) => { /* ... */ });

// Promise.race → race
race(obs1$, obs2$).subscribe(primero => { /* ... */ });

// .then() → map + switchMap
observable$.pipe(
  switchMap(id => this.http.get<Usuario>(`/api/usuarios/${id}`)),
  map(usuario => usuario.nombre)
).subscribe(nombre => console.log(nombre));
```

**Consejo de entrevista**: En Angular, es mejor quedarse con Observables y no convertirlos a Promises, ya que permiten cancelar peticiones HTTP al destruir el componente (con `takeUntil` o `AsyncPipe`), aplicar operadores de búsqueda como `debounceTime` y `switchMap`, y gestionar peticiones en paralelo con `forkJoin`. Las Promises son útiles en contextos donde el ecosistema espera una Promise (async/await, interop con librerías externas).
