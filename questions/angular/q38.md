**Zone.js** es una librería que **parchea** (monkey-patches) las APIs asíncronas del navegador para interceptar cuándo comienzan y terminan las operaciones asíncronas, permitiendo a Angular saber cuándo debe ejecutar la detección de cambios.

## ¿Por qué lo necesita Angular?

Angular necesita saber cuándo actualizar la vista. Sin Zone.js, Angular no sabría cuándo terminó una operación asíncrona (setTimeout, fetch, evento del DOM) y, por tanto, cuándo re-renderizar.

```
Sin Zone.js:
  1. Usuario hace click → handler ejecutado → estado cambia
  2. Angular no sabe que el estado cambió → vista no se actualiza ❌

Con Zone.js:
  1. Zone.js intercepta el evento click
  2. Handler ejecutado → estado cambia
  3. Zone.js notifica a Angular → Angular ejecuta change detection → vista actualizada ✅
```

## APIs que Zone.js parchea

Zone.js intercepta automáticamente:
- **Timers**: `setTimeout`, `setInterval`, `clearTimeout`
- **Promesas**: `Promise.then`, `Promise.catch`
- **Eventos del DOM**: `addEventListener`, `removeEventListener`
- **XHR / Fetch**: peticiones HTTP
- **Microtareas**: `queueMicrotask`, `MutationObserver`

## NgZone: la interfaz de Angular con Zone.js

Angular expone el servicio `NgZone` para controlar manualmente cuándo las operaciones están dentro o fuera de la zona de Angular.

### `ngZone.runOutsideAngular()` — Mejora de rendimiento

Ejecuta código **fuera** de la zona de Angular, evitando que dispare change detection innecesariamente:

```typescript
@Component({
  selector: 'app-animacion',
  template: `<canvas #miCanvas></canvas>`
})
export class AnimacionComponent implements OnInit, OnDestroy {
  @ViewChild('miCanvas') canvas!: ElementRef<HTMLCanvasElement>;
  private animFrameId!: number;

  constructor(private ngZone: NgZone) {}

  ngOnInit(): void {
    // ✅ El loop de animación no dispara change detection en cada frame
    this.ngZone.runOutsideAngular(() => {
      this.iniciarLoop();
    });
  }

  private iniciarLoop(): void {
    const render = () => {
      // Lógica de renderizado (60fps, no necesita change detection)
      this.dibujarFrame();
      this.animFrameId = requestAnimationFrame(render);
    };
    this.animFrameId = requestAnimationFrame(render);
  }

  ngOnDestroy(): void {
    cancelAnimationFrame(this.animFrameId);
  }
}
```

### `ngZone.run()` — Volver a la zona

Cuando necesitas actualizar la vista desde código fuera de la zona:

```typescript
constructor(private ngZone: NgZone) {
  // Callback de librería de terceros (fuera de la zona Angular)
  algunaLibreriaExterna.onDatoRecibido((dato: string) => {
    // ❌ Sin ngZone.run(): la vista no se actualiza
    // this.mensaje = dato;

    // ✅ Con ngZone.run(): Angular detecta el cambio y actualiza la vista
    this.ngZone.run(() => {
      this.mensaje = dato;
    });
  });
}
```

## Problemas de rendimiento con Zone.js

Zone.js dispara change detection con cualquier evento asíncrono, lo que puede ser excesivo en aplicaciones grandes:

```typescript
// Cada mousemove dispara change detection en toda la app ❌
@HostListener('mousemove', ['$event'])
onMouseMove(e: MouseEvent): void {
  this.posicion = { x: e.clientX, y: e.clientY };
}

// ✅ Mejor: ejecutar fuera de la zona y actualizar manualmente
constructor(private ngZone: NgZone, private cd: ChangeDetectorRef) {
  this.ngZone.runOutsideAngular(() => {
    document.addEventListener('mousemove', (e: MouseEvent) => {
      this.posicion = { x: e.clientX, y: e.clientY };
      // Solo marca el componente, no ejecuta change detection global
      this.cd.markForCheck();
    });
  });
}
```

## Angular Zoneless (Angular 18+)

Desde Angular 17 (experimental) y Angular 18 (más estable), es posible usar Angular **sin Zone.js**, basándose exclusivamente en **Signals** para la reactividad:

```typescript
// main.ts — Angular Zoneless
bootstrapApplication(AppComponent, {
  providers: [
    provideExperimentalZonelessChangeDetection() // Angular 18+
  ]
});
```

```typescript
// Componente zoneless con Signals
@Component({
  selector: 'app-contador',
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `<p>{{ contador() }}</p> <button (click)="incrementar()">+1</button>`
})
export class ContadorComponent {
  contador = signal(0);

  incrementar(): void {
    this.contador.update(v => v + 1); // Angular actualiza la vista automáticamente
  }
}
```

## Tabla resumen

| Característica       | Con Zone.js                    | Sin Zone.js (Zoneless)         |
|----------------------|--------------------------------|--------------------------------|
| Change detection     | Automática (todas las zonas)   | Manual con Signals/markForCheck|
| Bundle size          | +Zone.js (~100KB minificado)   | Sin Zone.js                    |
| Rendimiento          | Puede ser excesivo             | Más eficiente y predecible     |
| Adopción             | Default hasta Angular 17       | Experimental/estable desde v18 |

**Consejo de entrevista**: Zone.js es "magia" que Angular usa para detectar automáticamente las actualizaciones del estado. Su principal desventaja es que puede disparar change detection innecesariamente. La solución tradicional es `runOutsideAngular()` + `OnPush`. La solución moderna es Angular Zoneless con Signals, que da al desarrollador control total sobre cuándo se actualiza la vista, con mejor rendimiento y sin dependencia de Zone.js.
