---

Los **pipes** en Angular transforman datos en las plantillas de forma declarativa. Angular incluye pipes integrados (`DatePipe`, `CurrencyPipe`, `UpperCasePipe`, etc.), pero también es posible crear pipes personalizados para transformaciones específicas del negocio.

## Estructura de un pipe personalizado

Un pipe debe:
1. Implementar la interfaz `PipeTransform` con el método `transform()`.
2. Decorarse con `@Pipe` indicando su nombre.

```typescript
import { Pipe, PipeTransform } from '@angular/core';

@Pipe({
  name: 'truncar',
  standalone: true  // Recomendado desde Angular 17
})
export class TruncarPipe implements PipeTransform {
  transform(valor: string, limite: number = 100, sufijo: string = '...'): string {
    if (!valor) return '';
    if (valor.length <= limite) return valor;
    return valor.substring(0, limite).trim() + sufijo;
  }
}
```

Uso en la plantilla:

```html
<!-- Sin argumentos: usa los valores por defecto (100 caracteres, '...') -->
<p>{{ descripcion | truncar }}</p>

<!-- Con argumentos personalizados -->
<p>{{ descripcion | truncar:50 }}</p>
<p>{{ descripcion | truncar:30:'→' }}</p>
```

## Ejemplo: pipe de capitalización de palabras

```typescript
@Pipe({
  name: 'capitalizarPalabras',
  standalone: true
})
export class CapitalizarPalabrasPipe implements PipeTransform {
  transform(valor: string): string {
    if (!valor) return '';
    return valor
      .toLowerCase()
      .split(' ')
      .map(palabra => palabra.charAt(0).toUpperCase() + palabra.slice(1))
      .join(' ');
  }
}
```

```html
<h1>{{ 'hola mundo desde angular' | capitalizarPalabras }}</h1>
<!-- Resultado: "Hola Mundo Desde Angular" -->
```

## Ejemplo: pipe de formato de teléfono

```typescript
@Pipe({
  name: 'telefono',
  standalone: true
})
export class TelefonoPipe implements PipeTransform {
  transform(numero: string | number): string {
    const limpio = String(numero).replace(/\D/g, '');
    if (limpio.length === 9) {
      // Formato español: 612 345 678
      return limpio.replace(/(\d{3})(\d{3})(\d{3})/, '$1 $2 $3');
    }
    return String(numero);
  }
}
```

```html
<p>{{ 612345678 | telefono }}</p>
<!-- Resultado: "612 345 678" -->
```

## Generar con Angular CLI

```bash
ng generate pipe pipes/truncar
# o forma corta
ng g p pipes/truncar
```

## Registrar el pipe

**En un componente standalone** (recomendado):
```typescript
@Component({
  standalone: true,
  imports: [TruncarPipe, CapitalizarPalabrasPipe],
  template: `<p>{{ texto | truncar:50 | capitalizarPalabras }}</p>`
})
export class ArticuloComponent { texto = 'contenido del artículo...'; }
```

**En un NgModule** (arquitectura clásica):
```typescript
@NgModule({
  declarations: [TruncarPipe],
  exports: [TruncarPipe]
})
export class SharedModule {}
```

## Encadenamiento de pipes

Los pipes se pueden encadenar con `|`:

```html
{{ fecha | date:'dd/MM/yyyy' | uppercase }}
{{ nombre | capitalizarPalabras | truncar:20 }}
```

**Consejo de entrevista**: Por defecto, los pipes son **puros** (`pure: true`), lo que significa que solo se recalculan cuando cambia la referencia del valor de entrada. Si necesitas que el pipe se ejecute en cada ciclo de detección de cambios (por ejemplo, para filtrar un array que se muta), debes usar `pure: false`, pero esto tiene un coste de rendimiento. La mayoría de las veces, lo más correcto es **no mutar** el array y trabajar con nuevas referencias.
