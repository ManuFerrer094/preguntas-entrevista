**`NgZone`** es un servicio de Angular que encapsula la funcionalidad de Zone.js, permitiendo al desarrollador controlar explícitamente qué código corre dentro y fuera de la zona de Angular, optimizando así el rendimiento de la detección de cambios.

## El problema que NgZone resuelve

Angular usa Zone.js para detectar cuándo ejecutar change detection. Pero algunas operaciones (animaciones, WebSockets, intervalos frecuentes) pueden disparar change detection innecesariamente, degradando el rendimiento.

```typescript
@Component({
  selector: 'app-reloj',
  template: `<p>{{ hora }}</p>`
})
export class RelojComponent implements OnInit {
  hora = '';

  ngOnInit(): void {
    // ❌ setInterval dispara change detection 60 veces por segundo
    // en TODA la aplicación, no solo en este componente
    setInterval(() => {
      this.hora = new Date().toLocaleTimeString();
    }, 1000);
  }
}
```

## NgZone.runOutsideAngular()

Ejecuta código fuera de la zona de Angular: los eventos no disparan change detection automáticamente.

```typescript
@Component({
  selector: 'app-reloj-optimizado',
  template: `<p>{{ hora }}</p>`,
  changeDetection: ChangeDetectionStrategy.OnPush
})
export class RelojOptimizadoComponent implements OnInit, OnDestroy {
  hora = '';
  private intervalId!: ReturnType<typeof setInterval>;

  constructor(
    private ngZone: NgZone,
    private cdr: ChangeDetectorRef
  ) {}

  ngOnInit(): void {
    // ✅ El intervalo corre fuera de la zona de Angular
    this.ngZone.runOutsideAngular(() => {
      this.intervalId = setInterval(() => {
        this.hora = new Date().toLocaleTimeString('es-ES');
        // Notificar a Angular que este componente necesita actualizarse
        this.cdr.markForCheck();
      }, 1000);
    });
  }

  ngOnDestroy(): void {
    clearInterval(this.intervalId);
  }
}
```

## NgZone.run()

Vuelve a ejecutar código dentro de la zona de Angular desde código externo:

```typescript
@Component({ /* ... */ })
export class WebSocketComponent implements OnInit {
  mensajes: string[] = [];

  constructor(private ngZone: NgZone) {}

  ngOnInit(): void {
    const socket = new WebSocket('wss://api.ejemplo.com/ws');

    this.ngZone.runOutsideAngular(() => {
      socket.addEventListener('message', (event) => {
        // ❌ Sin ngZone.run(): la vista no se actualiza
        // this.mensajes.push(event.data);

        // ✅ Con ngZone.run(): Angular detecta el cambio
        this.ngZone.run(() => {
          this.mensajes.push(event.data);
        });
      });
    });
  }
}
```

## Caso de uso: animaciones de alto rendimiento

```typescript
@Component({
  selector: 'app-particulas',
  template: `<canvas #canvas width="800" height="600"></canvas>`,
  changeDetection: ChangeDetectionStrategy.OnPush
})
export class ParticulasComponent implements OnInit, OnDestroy {
  @ViewChild('canvas') canvasRef!: ElementRef<HTMLCanvasElement>;
  private animFrameId!: number;
  private ctx!: CanvasRenderingContext2D;

  constructor(private ngZone: NgZone) {}

  ngAfterViewInit(): void {
    this.ctx = this.canvasRef.nativeElement.getContext('2d')!;

    // ✅ Todo el loop de animación fuera de la zona Angular
    this.ngZone.runOutsideAngular(() => {
      this.iniciarAnimacion();
    });
  }

  private iniciarAnimacion(): void {
    const frame = () => {
      this.dibujar(); // Solo opera con el canvas, sin tocar el estado Angular
      this.animFrameId = requestAnimationFrame(frame);
    };
    this.animFrameId = requestAnimationFrame(frame);
  }

  private dibujar(): void {
    this.ctx.clearRect(0, 0, 800, 600);
    // ... lógica de dibujo a 60fps
  }

  ngOnDestroy(): void {
    cancelAnimationFrame(this.animFrameId);
  }
}
```

## NgZone + ChangeDetectorRef: el dúo perfecto con OnPush

```typescript
@Component({
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `<p>Posición: {{ posX }}, {{ posY }}</p>`
})
export class SeguimientoRatonComponent implements OnInit, OnDestroy {
  posX = 0;
  posY = 0;
  private listener!: EventListener;

  constructor(
    private ngZone: NgZone,
    private cdr: ChangeDetectorRef
  ) {}

  ngOnInit(): void {
    this.ngZone.runOutsideAngular(() => {
      this.listener = (e: Event) => {
        const mouseEvent = e as MouseEvent;
        this.posX = mouseEvent.clientX;
        this.posY = mouseEvent.clientY;
        // Solo actualiza la vista cada 16ms (≈60fps)
        this.cdr.markForCheck();
      };
      document.addEventListener('mousemove', this.listener);
    });
  }

  ngOnDestroy(): void {
    document.removeEventListener('mousemove', this.listener);
  }
}
```

## Hacia el futuro: Angular Zoneless

En Angular 18+, se puede eliminar Zone.js completamente usando Signals:

```typescript
// ✅ Con Signals, NgZone ya no es necesario
@Component({
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `<p>{{ hora() }}</p>`
})
export class RelojSignalsComponent implements OnInit, OnDestroy {
  hora = signal('');
  private intervalId!: ReturnType<typeof setInterval>;

  ngOnInit(): void {
    // Sin Zone.js, Angular actualiza solo los signals que cambian
    this.intervalId = setInterval(() => {
      this.hora.set(new Date().toLocaleTimeString('es-ES'));
    }, 1000);
  }

  ngOnDestroy(): void {
    clearInterval(this.intervalId);
  }
}
```

**Consejo de entrevista**: `NgZone.runOutsideAngular()` es la optimización de rendimiento más efectiva para código que actualiza el DOM frecuentemente (animaciones, WebSockets, eventos de ratón). Siempre combínalo con `ChangeDetectionStrategy.OnPush` y `ChangeDetectorRef.markForCheck()` para máximo rendimiento. En Angular 18+ con el modo Zoneless, este patrón ya no es necesario, ya que los Signals gestionan la reactividad de forma granular y eficiente.
