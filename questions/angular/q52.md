Angular Router proporciona varias formas de pasar información entre rutas: **parámetros de ruta** (parte del path) y **parámetros de consulta** (query params), cada uno con sus casos de uso específicos.

## Parámetros de ruta (Route Params)

Son parte del **path de la URL** y se definen con `:nombre` en la configuración de rutas:

```typescript
// app.routes.ts
export const routes: Routes = [
  { path: 'usuarios', component: UsuariosListComponent },
  { path: 'usuarios/:id', component: UsuarioDetailComponent },
  { path: 'productos/:categoria/:id', component: ProductoDetailComponent }
];
```

### Leer parámetros de ruta

```typescript
// usuario-detail.component.ts
@Component({
  standalone: true,
  imports: [CommonModule, AsyncPipe],
  template: `
    @if (usuario$ | async; as usuario) {
      <h1>{{ usuario.nombre }}</h1>
    }
  `
})
export class UsuarioDetailComponent implements OnInit {
  usuario$!: Observable<Usuario>;

  constructor(
    private route: ActivatedRoute,
    private usuarioService: UsuarioService
  ) {}

  ngOnInit(): void {
    // Opción 1: Observable (recomendado) — reacciona a cambios de parámetro
    this.usuario$ = this.route.paramMap.pipe(
      map(params => params.get('id')!),
      switchMap(id => this.usuarioService.getUsuario(Number(id)))
    );

    // Opción 2: Snapshot (simple, no reacciona a cambios)
    const id = this.route.snapshot.paramMap.get('id');
    console.log('ID del usuario:', id);
  }
}
```

### Input binding con parámetros (Angular 16+)

Con `withComponentInputBinding()`, los parámetros se inyectan directamente como `@Input()`:

```typescript
// main.ts
bootstrapApplication(AppComponent, {
  providers: [
    provideRouter(routes, withComponentInputBinding())
  ]
});
```

```typescript
// usuario-detail.component.ts — más limpio con binding de inputs
@Component({ /* ... */ })
export class UsuarioDetailComponent implements OnInit {
  @Input() id!: string; // ← Angular inyecta el parámetro de ruta automáticamente

  ngOnInit(): void {
    this.usuarioService.getUsuario(Number(this.id)).subscribe(/* ... */);
  }
}
```

## Parámetros de consulta (Query Params)

Son los parámetros opcionales después del `?` en la URL: `/usuarios?pagina=2&orden=nombre`

```typescript
// Navegar con query params
@Component({ template: `<button (click)="buscar()">Buscar</button>` })
export class BusquedaComponent {
  constructor(private router: Router) {}

  buscar(): void {
    this.router.navigate(['/resultados'], {
      queryParams: { termino: 'angular', pagina: 1, orden: 'relevancia' }
    });
    // URL resultante: /resultados?termino=angular&pagina=1&orden=relevancia
  }
}
```

```typescript
// Leer query params
@Component({ /* ... */ })
export class ResultadosComponent implements OnInit {
  constructor(private route: ActivatedRoute) {}

  ngOnInit(): void {
    // Observable (reacciona a cambios)
    this.route.queryParamMap.pipe(
      map(params => ({
        termino: params.get('termino') ?? '',
        pagina: Number(params.get('pagina') ?? '1'),
        orden: params.get('orden') ?? 'relevancia'
      }))
    ).subscribe(({ termino, pagina, orden }) => {
      this.buscarResultados(termino, pagina, orden);
    });

    // Snapshot (valor actual único)
    const termino = this.route.snapshot.queryParamMap.get('termino');
  }
}
```

## Navegar con parámetros programáticamente

```typescript
// Navegación con parámetros de ruta
this.router.navigate(['/usuarios', usuario.id]);
// URL: /usuarios/42

// Navegación con query params (preservando los actuales)
this.router.navigate(['/usuarios'], {
  queryParams: { pagina: 2 },
  queryParamsHandling: 'merge' // 'merge' | 'preserve' | ''
});

// Navegación relativa
this.router.navigate(['../editar'], { relativeTo: this.route });
```

## RouterLink con parámetros en la plantilla

```html
<!-- Parámetros de ruta -->
<a [routerLink]="['/usuarios', usuario.id]">Ver perfil</a>
<a [routerLink]="['/productos', categoria, producto.id]">Ver producto</a>

<!-- Query params -->
<a [routerLink]="['/busqueda']"
   [queryParams]="{ termino: 'angular', pagina: 1 }">
  Buscar Angular
</a>

<!-- Activar clase cuando la ruta coincide -->
<a routerLink="/usuarios" routerLinkActive="activo">Usuarios</a>
<a [routerLink]="['/usuarios', 1]" routerLinkActive="seleccionado">Ana</a>
```

## Parámetros de ruta vs Query params

| Característica    | Parámetros de ruta (`:id`)     | Query params (`?key=value`)       |
|-------------------|--------------------------------|-----------------------------------|
| Obligatorio       | Generalmente sí                | Generalmente no (opcionales)      |
| En la URL         | `/usuarios/42`                 | `/usuarios?pagina=2`              |
| Uso típico        | Identificar un recurso         | Filtros, paginación, búsquedas    |
| SEO               | Mejor (URLs limpias)           | Aceptable                         |

**Consejo de entrevista**: Usa **parámetros de ruta** para identificar recursos (IDs) y **query params** para opciones opcionales como filtros, paginación y ordenación. En Angular 16+, `withComponentInputBinding()` es la forma más elegante de leer parámetros, ya que los convierte en `@Input()` del componente, simplificando el código y mejorando la testabilidad.
