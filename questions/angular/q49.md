`<ng-template>` y `<ng-container>` son dos elementos especiales de Angular que no generan elementos DOM reales, pero tienen propósitos muy diferentes.

## `<ng-template>`

`<ng-template>` define un **fragmento de plantilla** que Angular no renderiza automáticamente. Solo se renderiza cuando se le indica explícitamente.

### Uso con `*ngIf` (else/then)

```html
<!-- Caso de uso más común: bloque else -->
<div *ngIf="cargando; else contenido">
  <mat-spinner></mat-spinner>
</div>

<ng-template #contenido>
  <app-lista-usuarios></app-lista-usuarios>
</ng-template>

<!-- Con then y else -->
<ng-container *ngIf="usuario; then perfilTemplate; else loginTemplate">
</ng-container>

<ng-template #perfilTemplate>
  <app-perfil [usuario]="usuario"></app-perfil>
</ng-template>

<ng-template #loginTemplate>
  <app-login></app-login>
</ng-template>
```

### Uso con `NgTemplateOutlet`

`NgTemplateOutlet` permite renderizar un `ng-template` en cualquier lugar y pasarle contexto:

```typescript
@Component({
  standalone: true,
  imports: [CommonModule],
  template: `
    <!-- Definir template reutilizable con contexto -->
    <ng-template #tarjetaUsuario let-usuario let-indice="indice">
      <div class="card">
        <span>{{ indice + 1 }}.</span>
        <strong>{{ usuario.nombre }}</strong>
        <em>{{ usuario.email }}</em>
      </div>
    </ng-template>

    <!-- Renderizar el template con contexto -->
    @for (u of usuarios; track u.id; let i = $index) {
      <ng-container
        [ngTemplateOutlet]="tarjetaUsuario"
        [ngTemplateOutletContext]="{ $implicit: u, indice: i }">
      </ng-container>
    }
  `
})
export class ListaComponent {
  usuarios = [
    { id: 1, nombre: 'Ana', email: 'ana@ejemplo.com' },
    { id: 2, nombre: 'Luis', email: 'luis@ejemplo.com' }
  ];
}
```

### Pasar templates como @Input()

Un patrón poderoso para componentes reutilizables:

```typescript
@Component({
  selector: 'app-lista-generica',
  standalone: true,
  imports: [CommonModule],
  template: `
    @for (item of items; track $index) {
      <ng-container
        [ngTemplateOutlet]="itemTemplate"
        [ngTemplateOutletContext]="{ $implicit: item }">
      </ng-container>
    }
  `
})
export class ListaGenericaComponent<T> {
  @Input() items: T[] = [];
  @Input() itemTemplate!: TemplateRef<{ $implicit: T }>;
}
```

```html
<!-- Uso: el padre define el template de cada fila -->
<app-lista-generica [items]="productos" [itemTemplate]="productoTpl">
</app-lista-generica>

<ng-template #productoTpl let-producto>
  <div class="producto-row">
    <img [src]="producto.imagen" [alt]="producto.nombre">
    <span>{{ producto.nombre }}</span>
  </div>
</ng-template>
```

## `<ng-container>`

`<ng-container>` es un **elemento de agrupación lógica** que no añade ningún elemento al DOM. Es útil cuando necesitas aplicar una directiva o agrupar elementos sin introducir un elemento HTML envolvente.

### Evitar elementos DOM innecesarios

```html
<!-- ❌ Sin ng-container: el <div> se añade al DOM aunque sea innecesario -->
<div *ngIf="mostrar">
  <p>Párrafo 1</p>
  <p>Párrafo 2</p>
</div>

<!-- ✅ Con ng-container: no añade elemento al DOM -->
<ng-container *ngIf="mostrar">
  <p>Párrafo 1</p>
  <p>Párrafo 2</p>
</ng-container>
```

### Múltiples directivas en el mismo elemento

En HTML, no puedes tener dos directivas estructurales en el mismo elemento:

```html
<!-- ❌ Error: no se pueden combinar *ngFor y *ngIf en el mismo elemento -->
<li *ngFor="let item of items" *ngIf="item.activo">{{ item.nombre }}</li>

<!-- ✅ Con ng-container para el *ngIf externo -->
<ng-container *ngFor="let item of items">
  <li *ngIf="item.activo">{{ item.nombre }}</li>
</ng-container>
```

### En tablas (preservar estructura HTML válida)

```html
<!-- Evita añadir <div> dentro de <table>, que rompe el HTML -->
<table>
  <tbody>
    <ng-container *ngFor="let grupo of grupos">
      <tr class="grupo-header">
        <td colspan="3">{{ grupo.nombre }}</td>
      </tr>
      <tr *ngFor="let item of grupo.items">
        <td>{{ item.nombre }}</td>
        <td>{{ item.precio }}</td>
      </tr>
    </ng-container>
  </tbody>
</table>
```

## Resumen comparativo

| Elemento         | Genera DOM | Renderizado        | Uso principal                                  |
|------------------|------------|--------------------|------------------------------------------------|
| `<ng-template>`  | No         | Solo cuando se pide| Bloques reutilizables, plantillas condicionales|
| `<ng-container>` | No         | Siempre            | Agrupación lógica sin afectar el DOM           |

**Consejo de entrevista**: La confusión más común es entre los dos. Recuerda: `<ng-template>` es como un "borrador" que Angular guarda pero no muestra hasta que se lo pides; `<ng-container>` es un "grupo invisible" que siempre está en el árbol lógico pero no añade elementos al DOM real. Ambos son fundamentales para escribir plantillas Angular limpias y semánticamente correctas.
