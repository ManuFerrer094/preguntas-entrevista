Los **validadores** en Angular son funciones que comprueban si el valor de un `FormControl` cumple ciertos criterios. Si la validación falla, el control tiene errores y el formulario es inválido.

## Validadores integrados (Validators)

Angular proporciona un conjunto de validadores en la clase `Validators`:

```typescript
import { Validators, FormControl } from '@angular/forms';

const nombre = new FormControl('', [
  Validators.required,                    // No puede estar vacío
  Validators.minLength(3),               // Mínimo 3 caracteres
  Validators.maxLength(50),              // Máximo 50 caracteres
  Validators.pattern(/^[a-zA-ZÀ-ú\s]+$/) // Solo letras y espacios
]);

const email = new FormControl('', [
  Validators.required,
  Validators.email                        // Formato de email válido
]);

const edad = new FormControl(null, [
  Validators.required,
  Validators.min(18),                    // Valor mínimo 18
  Validators.max(120)                    // Valor máximo 120
]);
```

## Mostrar mensajes de error en la plantilla

```typescript
@Component({
  standalone: true,
  imports: [ReactiveFormsModule, CommonModule],
  template: `
    <form [formGroup]="form" (ngSubmit)="onSubmit()">
      <!-- Campo nombre -->
      <div class="campo" [class.invalido]="nombre.invalid && nombre.touched">
        <label for="nombre">Nombre *</label>
        <input id="nombre" formControlName="nombre" type="text">

        @if (nombre.invalid && nombre.touched) {
          <div class="errores">
            @if (nombre.hasError('required')) {
              <span>El nombre es obligatorio.</span>
            }
            @if (nombre.hasError('minlength')) {
              <span>
                Mínimo {{ nombre.errors?.['minlength'].requiredLength }} caracteres.
                Tienes {{ nombre.errors?.['minlength'].actualLength }}.
              </span>
            }
            @if (nombre.hasError('pattern')) {
              <span>Solo se permiten letras y espacios.</span>
            }
          </div>
        }
      </div>

      <button type="submit" [disabled]="form.invalid">Guardar</button>
    </form>
  `
})
export class PerfilFormComponent {
  private fb = inject(NonNullableFormBuilder);

  form = this.fb.group({
    nombre: ['', [Validators.required, Validators.minLength(3), Validators.maxLength(50)]]
  });

  get nombre() { return this.form.controls.nombre; }
}
```

## Validadores personalizados síncronos

```typescript
import { AbstractControl, ValidationErrors, ValidatorFn } from '@angular/forms';

// Validador de contraseña segura
export function contrasenaSeguraValidator(): ValidatorFn {
  return (control: AbstractControl): ValidationErrors | null => {
    const valor = control.value as string;
    if (!valor) return null; // No validar si está vacío (required se encarga)

    const tieneMayuscula = /[A-Z]/.test(valor);
    const tieneMinuscula = /[a-z]/.test(valor);
    const tieneNumero = /\d/.test(valor);
    const tieneEspecial = /[!@#$%^&*(),.?":{}|<>]/.test(valor);

    const esValida = tieneMayuscula && tieneMinuscula && tieneNumero && tieneEspecial;

    return esValida ? null : {
      contrasenaDebil: {
        tieneMayuscula,
        tieneMinuscula,
        tieneNumero,
        tieneEspecial
      }
    };
  };
}

// Uso
const password = new FormControl('', [
  Validators.required,
  Validators.minLength(8),
  contrasenaSeguraValidator()
]);
```

## Validadores de grupo (cross-field)

Para validar que dos campos coincidan (como confirmar contraseña):

```typescript
export function confirmarContrasenaValidator(
  controlNombre: string,
  confirmarNombre: string
): ValidatorFn {
  return (group: AbstractControl): ValidationErrors | null => {
    const control = group.get(controlNombre);
    const confirmar = group.get(confirmarNombre);

    if (!control || !confirmar) return null;

    if (control.value !== confirmar.value) {
      confirmar.setErrors({ noCoincide: true });
      return { noCoincide: true };
    }

    // Limpiar el error si ahora coinciden
    if (confirmar.hasError('noCoincide')) {
      confirmar.setErrors(null);
    }

    return null;
  };
}

// Uso en el formulario
form = this.fb.group({
  password: ['', [Validators.required, Validators.minLength(8)]],
  confirmarPassword: ['', Validators.required]
}, {
  validators: confirmarContrasenaValidator('password', 'confirmarPassword')
});
```

## Validadores asíncronos

Para verificar disponibilidad en el servidor (email único, usuario existente):

```typescript
export function emailUnicoValidator(usuarioService: UsuarioService): AsyncValidatorFn {
  return (control: AbstractControl): Observable<ValidationErrors | null> => {
    if (!control.value) return of(null);

    return timer(300).pipe( // Debounce de 300ms
      switchMap(() => usuarioService.verificarEmailDisponible(control.value)),
      map(disponible => disponible ? null : { emailOcupado: true }),
      catchError(() => of(null)) // No bloquear si hay error de red
    );
  };
}

// Uso: los async validators van en el TERCER argumento
email = new FormControl('', [Validators.required, Validators.email],
  [emailUnicoValidator(this.usuarioService)]
);
```

```html
<!-- Mostrar estado de validación asíncrona -->
@if (email.pending) {
  <span class="verificando">Verificando disponibilidad...</span>
}
@if (email.hasError('emailOcupado') && !email.pending) {
  <span class="error">Este email ya está registrado.</span>
}
```

**Consejo de entrevista**: Tres aspectos clave sobre validadores: 1) Los validadores deben retornar `null` si son válidos y un objeto de errores si son inválidos. 2) Los validadores asíncronos solo se ejecutan cuando los síncronos pasan, para evitar peticiones innecesarias al servidor. 3) Para validaciones cross-field, aplica el validador al `FormGroup`, no al `FormControl` individual.
