Optimizar el rendimiento en Angular implica estrategias a múltiples niveles: detección de cambios, bundle size, rendering y gestión de memoria. Conocer estas técnicas es fundamental para construir aplicaciones Angular rápidas y escalables.

## 1. OnPush Change Detection

Por defecto, Angular verifica todos los componentes en cada ciclo de detección de cambios. Con `ChangeDetectionStrategy.OnPush`, un componente solo se actualiza cuando sus inputs cambian por referencia, o cuando se emite un evento interno.

```typescript
@Component({
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `<div>{{ data.name }}</div>`
})
export class OptimizedComponent {
  @Input() data!: UserData;
  // Este componente solo se re-renderiza si la referencia de 'data' cambia
}
```

## 2. Signals para reactividad granular

Los Signals de Angular permiten actualizaciones quirúrgicas: solo se re-renderiza la parte del template que depende del signal que cambió.

```typescript
@Component({
  template: `
    <p>{{ counter() }}</p>
    <p>{{ expensiveComputation() }}</p>
  `
})
export class SignalsComponent {
  counter = signal(0);
  expensiveComputation = computed(() => heavyCalc(this.counter()));
}
```

## 3. Lazy Loading y Code Splitting

Dividir la aplicación en chunks que se cargan bajo demanda:

```typescript
const routes: Routes = [
  { path: 'feature', loadComponent: () => import('./feature.component').then(m => m.FeatureComponent) }
];
```

## 4. TrackBy en ngFor

Sin `trackBy`, Angular re-renderiza toda la lista cuando cambia. Con `trackBy`, solo re-renderiza los elementos que cambiaron.

```typescript
@Component({
  template: `
    <div *ngFor="let item of items; trackBy: trackById">{{ item.name }}</div>
  `
})
export class ListComponent {
  trackById(index: number, item: Item): number {
    return item.id;
  }
}
```

## 5. Virtual Scrolling para listas largas

```typescript
// Con @angular/cdk/scrolling para listas de miles de elementos
<cdk-virtual-scroll-viewport itemSize="60" style="height: 500px">
  <div *cdkVirtualFor="let item of items">{{ item }}</div>
</cdk-virtual-scroll-viewport>
```

## 7. Pipes puras para transformaciones

Las pipes puras solo se recalculan cuando cambia el input, a diferencia de los métodos llamados en el template:

```typescript
@Pipe({ name: 'filter', pure: true })
export class FilterPipe implements PipeTransform {
  transform(items: Item[], search: string): Item[] {
    return items.filter(i => i.name.includes(search));
  }
}
```

## 9. Deferrable Views (Angular 17+)

```typescript
@Component({
  template: `
    <!-- Carga el componente pesado solo cuando entra en el viewport -->
    @defer (on viewport) {
      <app-heavy-chart />
    } @placeholder {
      <div>Cargando gráfico...</div>
    }
  `
})
export class DashboardComponent {}
```
