---
title: ¿Qué es un servicio en Angular?
difficulty: easy

Un **servicio** en Angular es una clase TypeScript con el decorador `@Injectable` que encapsula lógica de negocio, acceso a datos o funcionalidad reutilizable que puede compartirse entre múltiples componentes.

Los servicios siguen el **principio de responsabilidad única**: mientras los componentes gestionan la interfaz de usuario, los servicios gestionan los datos y la lógica.

## Creación de un servicio

```bash
# Generar con Angular CLI
ng generate service services/usuario
# o forma corta
ng g s services/usuario
```

Esto genera `usuario.service.ts`:

```typescript
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

export interface Usuario {
  id: number;
  nombre: string;
  email: string;
}

@Injectable({
  providedIn: 'root'  // Singleton a nivel de aplicación
})
export class UsuarioService {
  private readonly apiUrl = 'https://api.ejemplo.com/usuarios';

  constructor(private http: HttpClient) {}

  getUsuarios(): Observable<Usuario[]> {
    return this.http.get<Usuario[]>(this.apiUrl);
  }

  getUsuario(id: number): Observable<Usuario> {
    return this.http.get<Usuario>(`${this.apiUrl}/${id}`);
  }

  crearUsuario(usuario: Omit<Usuario, 'id'>): Observable<Usuario> {
    return this.http.post<Usuario>(this.apiUrl, usuario);
  }

  actualizarUsuario(id: number, usuario: Partial<Usuario>): Observable<Usuario> {
    return this.http.put<Usuario>(`${this.apiUrl}/${id}`, usuario);
  }

  eliminarUsuario(id: number): Observable<void> {
    return this.http.delete<void>(`${this.apiUrl}/${id}`);
  }
}
```

## Usando el servicio en un componente

```typescript
@Component({
  selector: 'app-lista-usuarios',
  standalone: true,
  imports: [CommonModule, AsyncPipe],
  template: `
    @if (usuarios$ | async; as usuarios) {
      <ul>
        @for (usuario of usuarios; track usuario.id) {
          <li>{{ usuario.nombre }} - {{ usuario.email }}</li>
        }
      </ul>
    }
  `
})
export class ListaUsuariosComponent implements OnInit {
  usuarios$!: Observable<Usuario[]>;

  // Angular inyecta el servicio automáticamente
  constructor(private usuarioService: UsuarioService) {}

  ngOnInit(): void {
    this.usuarios$ = this.usuarioService.getUsuarios();
  }
}
```

## `providedIn: 'root'` — Singleton global

La opción `providedIn: 'root'` es la forma más recomendada. Significa que:

- Angular crea **una sola instancia** del servicio para toda la aplicación.
- Es **tree-shakeable**: si no se usa, no se incluye en el bundle de producción.
- No necesitas registrarlo en ningún módulo manualmente.

## Servicio con estado compartido

Los servicios son perfectos para compartir estado entre componentes no relacionados:

```typescript
@Injectable({ providedIn: 'root' })
export class CarritoService {
  private items = signal<ProductoCarrito[]>([]);

  readonly items$ = this.items.asReadonly();
  readonly total = computed(() =>
    this.items().reduce((sum, item) => sum + item.precio * item.cantidad, 0)
  );

  agregarItem(producto: Producto): void {
    this.items.update(items => [...items, { ...producto, cantidad: 1 }]);
  }

  vaciarCarrito(): void {
    this.items.set([]);
  }
}
```

## Jerarquía de proveedores

| Nivel          | Configuración                     | Instancias     |
|----------------|-----------------------------------|----------------|
| Aplicación     | `providedIn: 'root'`              | Una (singleton)|
| Módulo         | En `providers` del `@NgModule`    | Una por módulo |
| Componente     | En `providers` del `@Component`   | Una por componente|
