---
title: ¿Qué son los Interceptors HTTP en Angular?
difficulty: medium
tags: [angular, interceptors, http, middleware]
---

Los Interceptors HTTP en Angular son servicios que interceptan todas las solicitudes y respuestas HTTP de la aplicación. Funcionan como middleware en la cadena de peticiones, permitiendo modificar requests salientes y responses entrantes de forma centralizada sin necesidad de duplicar lógica en cada servicio.

Los casos de uso más comunes son: añadir tokens de autenticación, manejar errores globalmente, mostrar indicadores de carga, transformar datos, implementar caché y hacer logging de peticiones.

**Interceptor funcional (Angular 15+, forma moderna):**

```typescript
import { HttpInterceptorFn, HttpRequest, HttpHandlerFn } from '@angular/common/http';
import { inject } from '@angular/core';
import { AuthService } from './auth.service';

export const authInterceptor: HttpInterceptorFn = (req: HttpRequest<unknown>, next: HttpHandlerFn) => {
  const authService = inject(AuthService);
  const token = authService.getToken();

  if (token) {
    const authReq = req.clone({
      headers: req.headers.set('Authorization', `Bearer ${token}`)
    });
    return next(authReq);
  }

  return next(req);
};
```

**Interceptor para manejo global de errores:**

```typescript
export const errorInterceptor: HttpInterceptorFn = (req, next) => {
  const router = inject(Router);
  const notificationService = inject(NotificationService);

  return next(req).pipe(
    catchError((error: HttpErrorResponse) => {
      if (error.status === 401) {
        router.navigate(['/login']);
      } else if (error.status === 403) {
        notificationService.error('No tienes permisos para esta acción');
      } else if (error.status === 500) {
        notificationService.error('Error interno del servidor');
      }
      return throwError(() => error);
    })
  );
};
```

**Interceptor de loading:**

```typescript
export const loadingInterceptor: HttpInterceptorFn = (req, next) => {
  const loadingService = inject(LoadingService);

  loadingService.show();
  return next(req).pipe(
    finalize(() => loadingService.hide())
  );
};
```

**Registro de interceptors en la configuración de la aplicación:**

```typescript
// app.config.ts
export const appConfig: ApplicationConfig = {
  providers: [
    provideHttpClient(
      withInterceptors([authInterceptor, errorInterceptor, loadingInterceptor])
    )
  ]
};
```

Los interceptors se ejecutan en el orden en que se registran para las solicitudes salientes, y en orden inverso para las respuestas entrantes, formando una cadena. Esto es importante al diseñar su orden: el interceptor de autenticación debe añadir el token antes de que la petición salga, mientras que el de errores debe procesar la respuesta antes de que llegue al componente.

**Consejo de entrevista**: Es importante mencionar que las requests HTTP son inmutables en Angular. Para modificarlas se debe usar `req.clone()` con los cambios deseados, nunca mutar el objeto directamente.
