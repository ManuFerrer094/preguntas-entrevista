---
title: ¿Qué son toSignal() y toObservable() en Angular?
difficulty: medium
tags: [angular, signals, rxjs, interop]
---

`toSignal()` y `toObservable()` son funciones de interoperabilidad que permiten convertir entre el mundo reactivo de **RxJS Observables** y el nuevo sistema de **Signals** de Angular, facilitando la migración incremental y el uso mixto de ambos enfoques.

## toSignal() — Convierte un Observable en Signal

```typescript
import { Component, inject } from '@angular/core';
import { toSignal } from '@angular/core/rxjs-interop';
import { HttpClient } from '@angular/common/http';

@Component({
  standalone: true,
  template: `
    @if (usuarios()) {
      @for (u of usuarios(); track u.id) {
        <li>{{ u.nombre }}</li>
      }
    } @else {
      <p>Cargando...</p>
    }
  `
})
export class ListaUsuariosComponent {
  private http = inject(HttpClient);

  // El Observable se convierte automáticamente en Signal
  usuarios = toSignal(
    this.http.get<Usuario[]>('/api/usuarios'),
    { initialValue: null }   // Valor hasta que llegue la primera emisión
  );
}
```

### Opciones de toSignal()

```typescript
// initialValue: valor inicial antes de la primera emisión
const items = toSignal(items$, { initialValue: [] });

// requireSync: lanza error si el Observable no emite síncronamente
const valorSync = toSignal(of(42), { requireSync: true });

// injector: usar fuera del contexto de inyección
const signal = toSignal(observable$, { injector: this.injector });

// manualCleanup: no desuscribir automáticamente al destruir el contexto
const signal = toSignal(observable$, { manualCleanup: true });
```

### Manejo de errores con toSignal()

```typescript
import { toSignal } from '@angular/core/rxjs-interop';
import { catchError, of } from 'rxjs';

@Component({ standalone: true, template: `{{ datos() | json }}` })
export class DatosComponent {
  private datos$ = this.http.get('/api/datos').pipe(
    catchError(err => of({ error: err.message }))
  );

  datos = toSignal(this.datos$, { initialValue: null });
}
```

## toObservable() — Convierte un Signal en Observable

```typescript
import { Component, signal, computed } from '@angular/core';
import { toObservable } from '@angular/core/rxjs-interop';
import { debounceTime, switchMap } from 'rxjs/operators';

@Component({
  standalone: true,
  template: `
    <input (input)="busqueda.set($event.target.value)" />
    @for (r of resultados(); track r.id) {
      <div>{{ r.nombre }}</div>
    }
  `
})
export class BuscadorComponent {
  busqueda = signal('');

  // Convierte el signal en Observable para aplicar operadores RxJS
  resultados = toSignal(
    toObservable(this.busqueda).pipe(
      debounceTime(300),
      switchMap(query => this.http.get<Resultado[]>(`/api/buscar?q=${query}`))
    ),
    { initialValue: [] }
  );

  constructor(private http: HttpClient) {}
}
```

## Patrón combinado: búsqueda reactiva

```typescript
@Component({ standalone: true })
export class BusquedaAvanzadaComponent {
  // Signals para parámetros
  filtro = signal('');
  pagina = signal(1);

  // Computed signal que agrupa los parámetros
  private params = computed(() => ({
    q: this.filtro(),
    page: this.pagina()
  }));

  // Convierte params Signal → Observable → aplica debounce → vuelve a Signal
  resultados = toSignal(
    toObservable(this.params).pipe(
      debounceTime(300),
      switchMap(({ q, page }) =>
        this.http.get<Resultado[]>(`/api/buscar`, { params: { q, page } })
      )
    ),
    { initialValue: [] }
  );
}
```

## Comparativa de enfoques

| Caso de uso                    | Usar                        |
|--------------------------------|-----------------------------|
| HTTP request en template       | `toSignal(http.get(...))`   |
| Debounce sobre un Signal       | `toObservable()` + `debounceTime` |
| Migrar código existente        | `toSignal()` progresivamente |
| Estado local simple            | `signal()` directamente     |
| Streams complejos (merge, zip) | Observable + `toSignal()`   |

**Consejo de entrevista**: `toSignal()` y `toObservable()` son el puente entre RxJS y Signals. La ventaja de `toSignal()` es que el componente no necesita `async pipe` ni suscripciones manuales: Angular gestiona la suscripción y desuscripción automáticamente dentro del contexto de inyección. Esto simplifica enormemente el código de los componentes.
