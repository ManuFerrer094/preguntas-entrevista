`@Input()` y `@Output()` son los decoradores fundamentales para la **comunicación entre componentes padre e hijo** en Angular. Definen la API pública de un componente.

## @Input() — Datos del padre al hijo

`@Input()` marca una propiedad del componente hijo para que el componente padre pueda establecer su valor mediante property binding.

```typescript
// hijo.component.ts
@Component({
  selector: 'app-tarjeta-usuario',
  standalone: true,
  template: `
    <div class="card">
      <h3>{{ nombre }}</h3>
      <p>Edad: {{ edad }}</p>
      @if (activo) {
        <span class="badge activo">Activo</span>
      }
    </div>
  `
})
export class TarjetaUsuarioComponent {
  @Input() nombre!: string;          // Requerido (el ! indica que se asignará)
  @Input() edad: number = 0;         // Con valor por defecto
  @Input() activo: boolean = false;  // Booleano con valor por defecto
}
```

```html
<!-- padre.component.html -->
<app-tarjeta-usuario
  [nombre]="usuario.nombre"
  [edad]="usuario.edad"
  [activo]="usuario.estaActivo">
</app-tarjeta-usuario>
```

### @Input() con alias

```typescript
@Input('colorFondo') color!: string; // Nombre externo vs interno diferente
```
```html
<app-boton [colorFondo]="'#ff0000'"></app-boton>
```

### @Input() con transformación (Angular 16+)

```typescript
import { booleanAttribute, numberAttribute } from '@angular/core';

@Input({ transform: booleanAttribute }) disabled = false;
@Input({ transform: numberAttribute }) maxLength = 100;
```

## @Output() — Eventos del hijo al padre

`@Output()` marca una propiedad de tipo `EventEmitter` que permite al hijo emitir eventos hacia el padre.

```typescript
// buscador.component.ts
import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-buscador',
  standalone: true,
  imports: [FormsModule],
  template: `
    <div class="buscador">
      <input [(ngModel)]="termino" placeholder="Buscar...">
      <button (click)="buscar()">🔍</button>
      <button (click)="limpiar()">✕</button>
    </div>
  `
})
export class BuscadorComponent {
  termino = '';

  @Output() busquedaRealizada = new EventEmitter<string>();
  @Output() busquedaLimpiada = new EventEmitter<void>();

  buscar(): void {
    if (this.termino.trim()) {
      this.busquedaRealizada.emit(this.termino.trim());
    }
  }

  limpiar(): void {
    this.termino = '';
    this.busquedaLimpiada.emit();
  }
}
```

```html
<!-- padre.component.html -->
<app-buscador
  (busquedaRealizada)="onBusqueda($event)"
  (busquedaLimpiada)="onLimpiar()">
</app-buscador>
```

```typescript
// padre.component.ts
onBusqueda(termino: string): void {
  this.resultados = this.productos.filter(p =>
    p.nombre.toLowerCase().includes(termino.toLowerCase())
  );
}

onLimpiar(): void {
  this.resultados = this.productos;
}
```

## Señales: la alternativa moderna (Angular 17.1+)

Angular 17.1 introduce `input()` y `output()` como alternativas basadas en signals:

```typescript
import { Component, input, output, computed } from '@angular/core';

@Component({
  selector: 'app-contador',
  standalone: true,
  template: `
    <p>{{ valorDoble() }}</p>
    <button (click)="incrementar()">+1</button>
  `
})
export class ContadorComponent {
  // input() devuelve una Signal — más reactivo y type-safe
  valorInicial = input<number>(0);
  // input requerido sin valor por defecto:
  titulo = input.required<string>();

  // output() reemplaza EventEmitter
  valorCambiado = output<number>();

  private contador = signal(0);
  valorDoble = computed(() => this.contador() * 2);

  ngOnInit() {
    this.contador.set(this.valorInicial());
  }

  incrementar(): void {
    this.contador.update(v => v + 1);
    this.valorCambiado.emit(this.contador());
  }
}
```

## Tabla comparativa: decoradores vs signals API

| Característica          | `@Input()` / `@Output()`  | `input()` / `output()` (signals) |
|-------------------------|--------------------------|-----------------------------------|
| Reactivo                | A través de `ngOnChanges` | ✅ Signal nativa                  |
| Valor por defecto       | En la propiedad           | En `input(initialValue)`          |
| Requerido               | Sin soporte nativo        | `input.required<T>()`             |
| Type safety             | Bueno                     | ✅ Excelente                      |
| Interop con OnPush      | Requiere cuidado          | ✅ Nativo                         |
