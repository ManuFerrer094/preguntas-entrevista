---
title: ¿Cómo implementar autenticación con Guards funcionales en Angular?
difficulty: medium
tags: [angular, guards, authentication, routing, security]
---

Desde Angular 15+, los guards pueden implementarse como **funciones puras** (en lugar de clases con interfaces), lo que simplifica enormemente el código manteniendo toda la potencia de la inyección de dependencias.

## Guard funcional básico (canActivate)

```typescript
// auth.guard.ts
import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';
import { AuthService } from './auth.service';

export const authGuard: CanActivateFn = (route, state) => {
  const authService = inject(AuthService);
  const router = inject(Router);

  if (authService.estaAutenticado()) {
    return true;
  }

  // Redirigir al login guardando la URL de destino
  return router.createUrlTree(['/login'], {
    queryParams: { returnUrl: state.url }
  });
};

// Uso en las rutas
const routes: Routes = [
  {
    path: 'dashboard',
    component: DashboardComponent,
    canActivate: [authGuard]  // ✅ Array de funciones
  }
];
```

## Guard con roles (canActivate con parámetros)

```typescript
// roles.guard.ts
import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';
import { AuthService } from './auth.service';

// Factory function que devuelve un guard con el rol requerido
export function rolesGuard(rolesPermitidos: string[]): CanActivateFn {
  return (route, state) => {
    const authService = inject(AuthService);
    const router = inject(Router);

    const usuario = authService.usuarioActual();

    if (!usuario) {
      return router.createUrlTree(['/login']);
    }

    const tienePermiso = rolesPermitidos.some(rol =>
      usuario.roles.includes(rol)
    );

    if (tienePermiso) {
      return true;
    }

    // Redirigir a página de acceso denegado
    return router.createUrlTree(['/acceso-denegado']);
  };
}

// Uso:
const routes: Routes = [
  {
    path: 'admin',
    loadChildren: () => import('./admin/admin.routes'),
    canActivate: [authGuard, rolesGuard(['admin', 'superadmin'])]
  },
  {
    path: 'reportes',
    component: ReportesComponent,
    canActivate: [rolesGuard(['admin', 'analista'])]
  }
];
```

## canDeactivate — Guard para evitar salida sin guardar

```typescript
// unsaved-changes.guard.ts
import { CanDeactivateFn } from '@angular/router';

// Interfaz para componentes que pueden tener cambios sin guardar
export interface PuedeDesactivarse {
  tieneCambiosSinGuardar(): boolean;
}

export const unsavedChangesGuard: CanDeactivateFn<PuedeDesactivarse> = (
  component,
  currentRoute,
  currentState,
  nextState
) => {
  if (component.tieneCambiosSinGuardar()) {
    return confirm('¿Tienes cambios sin guardar. ¿Quieres salir de todas formas?');
  }
  return true;
};

// Componente que implementa la interfaz
@Component({ standalone: true })
export class EditarPerfilComponent implements PuedeDesactivarse {
  formulario = new FormGroup({ nombre: new FormControl('') });

  tieneCambiosSinGuardar(): boolean {
    return this.formulario.dirty;
  }
}

// Ruta
{ path: 'perfil/editar', component: EditarPerfilComponent, canDeactivate: [unsavedChangesGuard] }
```

## canMatch — Guard para rutas condicionales

```typescript
// feature-flag.guard.ts
import { CanMatchFn } from '@angular/router';
import { inject } from '@angular/core';
import { FeatureFlagService } from './feature-flag.service';

export function featureFlagGuard(feature: string): CanMatchFn {
  return () => {
    const featureService = inject(FeatureFlagService);
    return featureService.isEnabled(feature);
  };
}

// Muestra diferente componente según feature flag
const routes: Routes = [
  {
    path: 'checkout',
    component: CheckoutNuevoComponent,
    canMatch: [featureFlagGuard('nuevo-checkout')]
  },
  {
    path: 'checkout',
    component: CheckoutLegacyComponent
    // Sin guard: fallback si el feature flag está desactivado
  }
];
```

## Guard asíncrono (verifica con el backend)

```typescript
export const tokenValidGuard: CanActivateFn = async (route, state) => {
  const authService = inject(AuthService);
  const router = inject(Router);

  try {
    const esValido = await authService.verificarTokenEnServidor();
    if (esValido) return true;

    authService.cerrarSesion();
    return router.createUrlTree(['/login']);
  } catch {
    return router.createUrlTree(['/error']);
  }
};
```

## Diferencia entre tipos de guards

| Guard           | Cuándo se ejecuta                        | Uso típico                          |
|-----------------|------------------------------------------|-------------------------------------|
| `canActivate`   | Antes de activar la ruta                 | Autenticación, autorización         |
| `canActivateChild` | Antes de activar rutas hijas          | Proteger secciones completas        |
| `canDeactivate` | Antes de salir de la ruta actual         | Cambios sin guardar                 |
| `canMatch`      | Al hacer match de la ruta                | Feature flags, A/B testing          |
| `canLoad`       | Antes de cargar módulo lazy *(deprecated)* | Reemplazado por `canMatch`        |
| `resolve`       | Antes de activar, resuelve datos         | Pre-cargar datos para la ruta       |

**Consejo de entrevista**: Los guards funcionales (Angular 15+) son la forma moderna de implementar guards. Son más simples que las clases y permiten usar `inject()` directamente. La función factory pattern (`rolesGuard(['admin'])`) es el patrón recomendado para guards parametrizables. Evita `canLoad` (deprecated) y usa `canMatch` en su lugar.
