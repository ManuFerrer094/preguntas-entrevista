El **Angular CLI** (Command Line Interface) es la herramienta oficial de línea de comandos para crear, desarrollar, testear y desplegar aplicaciones Angular. Automatiza las tareas repetitivas y asegura que los proyectos sigan las convenciones y mejores prácticas del equipo de Angular.

## Instalación

```bash
npm install -g @angular/cli
ng version  # Verificar la instalación
```

## Comandos principales

### `ng new` — Crear un proyecto nuevo

```bash
ng new mi-aplicacion
ng new mi-app --routing --style=scss --standalone
ng new mi-app --skip-tests --inline-template --inline-style
```

### `ng serve` — Servidor de desarrollo

```bash
ng serve                    # http://localhost:4200
ng serve --port 3000        # Puerto personalizado
ng serve --open             # Abre el navegador automáticamente
ng serve --host 0.0.0.0     # Accesible en la red local
ng serve --configuration=staging  # Usar configuración específica
```

### `ng generate` — Generar código (scaffolding)

Es el comando más usado en el día a día. Genera código siguiendo las convenciones de Angular:

```bash
# Componentes
ng generate component features/usuarios/lista-usuarios
ng g c features/login --standalone --skip-tests

# Servicios
ng generate service services/autenticacion
ng g s core/services/api

# Directivas y Pipes
ng generate directive shared/directives/resaltar
ng generate pipe shared/pipes/truncar

# Guards
ng generate guard guards/auth --implements CanActivate
ng g guard guards/role --functional  # Guard funcional (Angular 15+)

# Interfaces y clases
ng generate interface models/usuario
ng generate class models/producto
ng generate enum models/estado-pedido

# Módulos
ng generate module features/productos --routing
ng g m features/admin --route admin --module app.module

# Resolvers e Interceptors
ng generate resolver resolvers/usuario
ng generate interceptor interceptors/auth
```

### `ng build` — Compilar la aplicación

```bash
ng build                                # Build de desarrollo
ng build --configuration=production     # Build de producción (optimizado)
ng build --stats-json                   # Genera stats para análisis de bundle
ng build --source-map                   # Incluye source maps
```

### `ng test` — Ejecutar tests unitarios

```bash
ng test                                 # Modo watch (desarrollo)
ng test --watch=false                   # Una sola ejecución (CI/CD)
ng test --code-coverage                 # Reporte de cobertura
ng test --include='**/*.spec.ts'        # Incluir archivos específicos
```

### `ng e2e` — Tests end-to-end

```bash
ng e2e                  # Requiere configurar Cypress, Playwright o Protractor
```

### `ng lint` — Análisis estático de código

```bash
ng lint                                 # Analizar todo el proyecto
ng lint --fix                           # Corregir automáticamente lo que se pueda
```

### `ng update` — Actualizar Angular

```bash
ng update                               # Ver qué puede actualizarse
ng update @angular/core @angular/cli    # Actualizar Angular
ng update --all                         # Actualizar todo (con cuidado)
```

### `ng add` — Añadir funcionalidades

```bash
ng add @angular/material                # Añadir Angular Material
ng add @angular/pwa                     # Añadir soporte PWA
ng add @angular/ssr                     # Añadir Server-Side Rendering
ng add @ngrx/store                      # Añadir NgRx
```

### `ng extract-i18n` — Extraer textos para traducción

```bash
ng extract-i18n --output-path src/locale --format json
```

## Schematics personalizados

El CLI usa **Schematics** como motor de generación de código. Puedes crear tus propios schematics para generar código que siga las convenciones de tu empresa:

```bash
# Usar schematics de terceros
ng generate @ngrx/schematics:store usuarios --root --module app.module.ts

# Ver todos los schematics disponibles
ng generate --list-schematics
```

## Configuración del CLI (angular.json)

```json
{
  "projects": {
    "mi-app": {
      "schematics": {
        "@schematics/angular:component": {
          "style": "scss",
          "standalone": true,
          "changeDetection": "OnPush"  // Por defecto OnPush en todos los componentes
        }
      }
    }
  }
}
```
