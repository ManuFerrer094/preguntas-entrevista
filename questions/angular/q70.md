---
title: ¿Qué es el nuevo sistema de control de flujo en Angular 17?
difficulty: medium

---

Angular 17 introdujo una nueva sintaxis nativa de **control de flujo** en las plantillas que reemplaza a las directivas estructurales clásicas (`*ngIf`, `*ngFor`, `ngSwitch`). Esta sintaxis es más similar a JavaScript, no requiere importar módulos, tiene mejor soporte de TypeScript y mejora el rendimiento.

## Comparativa: Sintaxis clásica vs nueva

### Condicional: `@if / @else if / @else`

```html
<!-- ❌ Antes (Angular < 17): requiere importar NgIf o CommonModule -->
<div *ngIf="usuario; else noAutenticado">
  <p>Bienvenido, {{ usuario.nombre }}!</p>
</div>
<ng-template #noAutenticado>
  <p>Por favor, inicia sesión.</p>
</ng-template>

<!-- ✅ Ahora (Angular 17+): sintaxis nativa, sin imports adicionales -->
@if (usuario) {
  <p>Bienvenido, {{ usuario.nombre }}!</p>
} @else {
  <p>Por favor, inicia sesión.</p>
}
```

La nueva sintaxis también soporta `@else if`:

```html
@if (estado === 'cargando') {
  <app-spinner></app-spinner>
} @else if (estado === 'error') {
  <app-mensaje-error [mensaje]="mensajeError"></app-mensaje-error>
} @else if (datos.length === 0) {
  <p class="vacio">No hay datos disponibles.</p>
} @else {
  <app-tabla [datos]="datos"></app-tabla>
}
```

### Bucle: `@for / @empty`

```html
<!-- ❌ Antes -->
<ul>
  <li *ngFor="let producto of productos; trackBy: trackById; let i = index">
    {{ i + 1 }}. {{ producto.nombre }}
  </li>
</ul>
<p *ngIf="productos.length === 0">Sin productos.</p>

<!-- ✅ Ahora: track es obligatorio, @empty integrado -->
<ul>
  @for (producto of productos; track producto.id; let i = $index) {
    <li>{{ i + 1 }}. {{ producto.nombre }}</li>
  } @empty {
    <li>Sin productos disponibles.</li>
  }
</ul>
```

Variables locales disponibles en `@for`:

```html
@for (item of items; track item.id;
      let idx = $index;
      let primero = $first;
      let ultimo = $last;
      let par = $even;
      let impar = $odd;
      let total = $count) {
  <div [class.primero]="primero" [class.ultimo]="ultimo">
    {{ idx + 1 }}/{{ total }}: {{ item.nombre }}
  </div>
}
```

### Switch: `@switch / @case / @default`

```html
<!-- ❌ Antes -->
<ng-container [ngSwitch]="planSuscripcion">
  <div *ngSwitchCase="'free'">Plan Gratuito</div>
  <div *ngSwitchCase="'pro'">Plan Pro</div>
  <div *ngSwitchDefault>Plan Desconocido</div>
</ng-container>

<!-- ✅ Ahora -->
@switch (planSuscripcion) {
  @case ('free') {
    <div class="plan-free">Plan Gratuito — 5GB</div>
  }
  @case ('pro') {
    <div class="plan-pro">Plan Pro — 100GB</div>
  }
  @case ('enterprise') {
    <div class="plan-enterprise">Plan Enterprise — Ilimitado</div>
  }
  @default {
    <div>Plan no reconocido</div>
  }
}
```

## Ventajas del nuevo sistema

### 1. Sin imports necesarios

```typescript
// ❌ Antes: había que importar CommonModule, NgIf, NgFor...
@Component({
  standalone: true,
  imports: [CommonModule], // o NgIf, NgFor individualmente
  template: `<div *ngIf="show">...</div>`
})

// ✅ Ahora: no se necesita ningún import para el control de flujo
@Component({
  standalone: true,
  imports: [], // El @if, @for, @switch son nativos de Angular
  template: `@if (show) { <div>...</div> }`
})
```

### 2. Mejor type narrowing (TypeScript)

```html
<!-- Con @if, TypeScript sabe que dentro del bloque usuario no es null -->
@if (usuario) {
  <p>{{ usuario.nombre }}</p>          <!-- TypeScript: usuario es Usuario (no null) -->
  <p>{{ usuario.direccion.ciudad }}</p> <!-- Sin ?. necesario -->
}

<!-- Con *ngIf clásico, TypeScript no siempre narrow correctamente -->
<p *ngIf="usuario">{{ usuario!.nombre }}</p> <!-- Necesita assertion ! -->
```

### 3. `track` obligatorio en `@for`

```html
<!-- ❌ @for SIN track: error de compilación -->
@for (item of items) { ... }  <!-- Error: track is required -->

<!-- ✅ @for CON track: obligatorio, fomenta buenas prácticas -->
@for (item of items; track item.id) { ... }
```

### 4. Bloque `@empty` integrado

```html
<!-- No necesitas un *ngIf adicional para el caso vacío -->
@for (tarea of tareas; track tarea.id) {
  <app-tarea [tarea]="tarea"></app-tarea>
} @empty {
  <p class="sin-tareas">¡No tienes tareas pendientes! 🎉</p>
}
```

## Migración automática

```bash
# Migrar automáticamente de *ngIf/*ngFor/ngSwitch a la nueva sintaxis
ng generate @angular/core:control-flow

# Para un archivo específico
ng generate @angular/core:control-flow --path src/app/mi-componente.component.html
```

## @defer — Carga diferida (también Angular 17)

Junto con el nuevo control de flujo, Angular 17 introduce `@defer` para carga lazy de componentes en la plantilla:

```html
@defer (on viewport) {
  <!-- Se carga cuando el elemento es visible en pantalla -->
  <app-grafico-pesado></app-grafico-pesado>
} @placeholder {
  <div class="skeleton"></div>
} @loading {
  <app-spinner></app-spinner>
} @error {
  <p>No se pudo cargar el componente.</p>
}
```

**Consejo de entrevista**: El nuevo sistema de control de flujo de Angular 17 no solo mejora la legibilidad sino también el rendimiento: el compilador puede optimizar mejor las plantillas con la sintaxis nativa. Hay cuatro puntos clave: 1) no requiere imports, 2) mejor type narrowing, 3) `track` obligatorio en `@for`, 4) `@empty` integrado. La migración es automática con `ng generate @angular/core:control-flow`. En nuevos proyectos Angular 17+, debe usarse siempre la nueva sintaxis.
