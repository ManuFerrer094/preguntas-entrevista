Un **Subject** en RxJS es una clase especial que actúa simultáneamente como **Observable** (se puede suscribir) y como **Observer** (se pueden emitir valores manualmente con `next()`, `error()` y `complete()`). También implementa **multicast**, es decir, comparte la misma ejecución entre todos sus suscriptores.

## Subject básico

```typescript
import { Subject } from 'rxjs';

const sujeto$ = new Subject<string>();

// Suscriptor 1
sujeto$.subscribe(valor => console.log('Suscriptor 1:', valor));

sujeto$.next('Mensaje A'); // Suscriptor 1: Mensaje A

// Suscriptor 2 (se suscribe después)
sujeto$.subscribe(valor => console.log('Suscriptor 2:', valor));

sujeto$.next('Mensaje B'); // Suscriptor 1: Mensaje B / Suscriptor 2: Mensaje B

// El Suscriptor 2 no recibió 'Mensaje A' (ya había pasado)
```

### Uso típico: comunicación entre componentes

```typescript
@Injectable({ providedIn: 'root' })
export class NotificacionService {
  private notificacion$ = new Subject<string>();

  // Solo exponemos el Observable (evita emisiones externas)
  readonly notificaciones$ = this.notificacion$.asObservable();

  enviar(mensaje: string): void {
    this.notificacion$.next(mensaje);
  }
}
```

## BehaviorSubject

Almacena el **último valor emitido** y lo entrega inmediatamente a cualquier nuevo suscriptor. Requiere un valor inicial.

```typescript
import { BehaviorSubject } from 'rxjs';

const estado$ = new BehaviorSubject<boolean>(false); // Valor inicial

console.log(estado$.value); // false → acceso sincrónico al valor actual

estado$.next(true);

// Nuevo suscriptor recibe inmediatamente el último valor (true)
estado$.subscribe(val => console.log('Estado:', val)); // Estado: true
```

### Caso de uso: gestión de estado de usuario

```typescript
@Injectable({ providedIn: 'root' })
export class AuthService {
  private usuarioActual$ = new BehaviorSubject<Usuario | null>(null);

  readonly usuario$ = this.usuarioActual$.asObservable();
  get estaAutenticado(): boolean {
    return this.usuarioActual$.value !== null;
  }

  login(usuario: Usuario): void {
    this.usuarioActual$.next(usuario);
  }

  logout(): void {
    this.usuarioActual$.next(null);
  }
}
```

## ReplaySubject

Almacena y **repite los últimos N valores** a cada nuevo suscriptor.

```typescript
import { ReplaySubject } from 'rxjs';

// Almacena los últimos 3 valores
const historial$ = new ReplaySubject<number>(3);

historial$.next(1);
historial$.next(2);
historial$.next(3);
historial$.next(4);

// Nuevo suscriptor recibe 2, 3, 4 (los últimos 3)
historial$.subscribe(val => console.log('Historial:', val));
// Historial: 2
// Historial: 3
// Historial: 4
```

### Caso de uso: caché de resultados

```typescript
@Injectable({ providedIn: 'root' })
export class LogService {
  private logs$ = new ReplaySubject<string>(50); // Guarda últimos 50 logs

  log(mensaje: string): void {
    this.logs$.next(`[${new Date().toISOString()}] ${mensaje}`);
  }

  getLogs(): Observable<string> {
    return this.logs$.asObservable();
  }
}
```

## AsyncSubject

Solo emite el **último valor** y únicamente cuando el Subject se completa con `complete()`.

```typescript
import { AsyncSubject } from 'rxjs';

const async$ = new AsyncSubject<number>();

async$.subscribe(val => console.log('Async:', val));

async$.next(1); // No emite aún
async$.next(2); // No emite aún
async$.next(3); // No emite aún

async$.complete(); // Ahora emite: Async: 3 (solo el último)
```

## Resumen comparativo

| Subject          | Valor inicial | Nuevos suscriptores reciben         | Caso de uso                    |
|------------------|---------------|--------------------------------------|--------------------------------|
| `Subject`        | No            | Solo valores futuros                 | Eventos, notificaciones        |
| `BehaviorSubject`| Sí            | El último valor emitido              | Estado global, usuario actual  |
| `ReplaySubject`  | No (buffer)   | Los últimos N valores                | Historial, caché               |
| `AsyncSubject`   | No            | Solo el último valor al completar    | Operaciones que terminan       |

**Consejo de entrevista**: `BehaviorSubject` es el más utilizado en aplicaciones Angular para gestión de estado simple (alternativa ligera a NgRx). Siempre expón el Subject como `asObservable()` para encapsular la emisión y evitar que componentes externos puedan hacer `next()` directamente sobre el Subject. Desde Angular 16+, los **Signals** son una alternativa más ergonómica para el estado síncrono dentro de componentes.
