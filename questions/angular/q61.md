Los **componentes dinámicos** son componentes que se crean y destruyen en **tiempo de ejecución**, en respuesta a acciones del usuario o lógica de la aplicación, sin estar definidos estáticamente en la plantilla.

## `ViewContainerRef.createComponent()` — El método principal

`ViewContainerRef` representa un contenedor en el DOM donde se pueden insertar vistas. Se accede a él mediante `@ViewChild` o inyección directa.

```typescript
// Componentes que se cargarán dinámicamente
@Component({
  selector: 'app-alerta-exito',
  standalone: true,
  template: `<div class="alerta exito">✅ {{ mensaje }}</div>`
})
export class AlertaExitoComponent {
  @Input() mensaje = 'Operación exitosa';
}

@Component({
  selector: 'app-alerta-error',
  standalone: true,
  template: `<div class="alerta error">❌ {{ mensaje }}</div>`
})
export class AlertaErrorComponent {
  @Input() mensaje = 'Ha ocurrido un error';
}
```

```typescript
// componente-alertas.component.ts
@Component({
  selector: 'app-alertas',
  standalone: true,
  template: `
    <div class="contenedor-alertas">
      <!-- Ancla donde se insertarán los componentes dinámicos -->
      <ng-container #ancla></ng-container>
    </div>
    <button (click)="mostrarExito()">Mostrar Éxito</button>
    <button (click)="mostrarError()">Mostrar Error</button>
    <button (click)="limpiar()">Limpiar</button>
  `
})
export class ComponenteAlertasComponent {
  @ViewChild('ancla', { read: ViewContainerRef })
  ancla!: ViewContainerRef;

  mostrarExito(): void {
    this.ancla.clear();
    const ref = this.ancla.createComponent(AlertaExitoComponent);
    ref.setInput('mensaje', '¡Guardado correctamente!');
    // Auto-eliminar después de 3 segundos
    setTimeout(() => ref.destroy(), 3000);
  }

  mostrarError(): void {
    this.ancla.clear();
    const ref = this.ancla.createComponent(AlertaErrorComponent);
    ref.setInput('mensaje', 'No se pudo guardar. Intenta de nuevo.');
  }

  limpiar(): void {
    this.ancla.clear();
  }
}
```

## Pasar datos al componente dinámico

```typescript
const ref = vcr.createComponent(MiComponenteDinamico);

// Opción 1: setInput() (Angular 14+ — recomendado)
ref.setInput('titulo', 'Hola Mundo');
ref.setInput('datos', { id: 1, nombre: 'Ana' });

// Opción 2: Acceder a la instancia directamente
ref.instance.titulo = 'Hola Mundo';
ref.instance.onCerrar.subscribe(() => ref.destroy());

// Forzar detección de cambios después de modificar la instancia
ref.changeDetectorRef.detectChanges();
```

## NgComponentOutlet — La forma declarativa

`NgComponentOutlet` es una directiva que simplifica la creación de componentes dinámicos directamente en la plantilla:

```typescript
@Component({
  standalone: true,
  imports: [NgComponentOutlet, CommonModule],
  template: `
    <!-- Forma básica -->
    <ng-container [ngComponentOutlet]="componenteActual"></ng-container>

    <!-- Con inputs (Angular 16+) -->
    <ng-container
      [ngComponentOutlet]="componenteActual"
      [ngComponentOutletInputs]="{ titulo: 'Hola', activo: true }">
    </ng-container>
  `
})
export class TabsComponent {
  pestañas = [
    { etiqueta: 'Perfil', componente: PerfilComponent },
    { etiqueta: 'Ajustes', componente: AjustesComponent },
    { etiqueta: 'Seguridad', componente: SeguridadComponent }
  ];

  pestañaActiva = this.pestañas[0];

  get componenteActual() {
    return this.pestañaActiva.componente;
  }
}
```

## Caso de uso: sistema de modales dinámicos

```typescript
// modal.service.ts
@Injectable({ providedIn: 'root' })
export class ModalService {
  private vcr!: ViewContainerRef;

  registrarContenedor(vcr: ViewContainerRef): void {
    this.vcr = vcr;
  }

  abrir<T>(
    componente: Type<T>,
    datos?: Partial<T>
  ): ComponentRef<T> {
    this.vcr.clear();

    // Crear overlay
    const overlayRef = this.vcr.createComponent(ModalOverlayComponent);

    // Crear el componente de contenido dentro del overlay
    const contenidoRef = overlayRef.instance.viewContainerRef
      .createComponent(componente);

    // Pasar datos al componente
    if (datos) {
      Object.entries(datos).forEach(([key, value]) => {
        contenidoRef.setInput(key, value);
      });
    }

    return contenidoRef;
  }

  cerrar(): void {
    this.vcr.clear();
  }
}
```

## Lazy loading de componentes dinámicos

Combinado con dynamic import para cargar el componente solo cuando se necesita:

```typescript
async cargarEditorPesado(): Promise<void> {
  // El bundle del editor solo se carga cuando se necesita
  const { EditorAvanazadoComponent } = await import(
    './editor-avanzado/editor-avanzado.component'
  );

  const ref = this.ancla.createComponent(EditorAvanazadoComponent);
  ref.setInput('documento', this.documentoActual);
}
```

## Ciclo de vida de componentes dinámicos

```typescript
const ref = vcr.createComponent(MiComponente);

// Los lifecycle hooks se ejecutan normalmente
// ngOnInit, ngAfterViewInit, etc.

// Destruir manualmente (llama a ngOnDestroy)
ref.destroy();

// O limpiar todo el contenedor
vcr.clear(); // Destruye todos los componentes creados
```

**Consejo de entrevista**: Los componentes dinámicos son la base de implementaciones avanzadas como sistemas de modales, tooltips, notificaciones y sistemas de plugins. El método moderno es `createComponent()` con `setInput()` (Angular 14+), que es más type-safe que acceder a `instance` directamente. `NgComponentOutlet` es preferible cuando puedes expresarlo declarativamente en la plantilla. Para el lazy loading, combina siempre los componentes dinámicos con dynamic import para evitar incluir código pesado en el bundle principal.
