---
title: ¿Qué es el data binding en Angular y cuáles son sus tipos?
difficulty: easy
tags: [angular, data-binding, template]
---

El **data binding** es el mecanismo de Angular para sincronizar automáticamente los datos entre la clase del componente (TypeScript) y su plantilla (HTML). Elimina la necesidad de manipular el DOM manualmente.

Angular ofrece **cuatro tipos** de data binding:

## 1. Interpolación `{{ }}`

Flujo: **Componente → Template** (unidireccional)

Evalúa una expresión TypeScript y la convierte a texto para mostrarla en el HTML.

```typescript
@Component({
  template: `
    <h1>{{ titulo }}</h1>
    <p>Precio: {{ precio | currency:'EUR' }}</p>
    <p>Resultado: {{ 2 + 2 }}</p>
    <p>Saludo: {{ obtenerSaludo() }}</p>
  `
})
export class EjemploComponent {
  titulo = 'Mi Tienda';
  precio = 29.99;

  obtenerSaludo(): string {
    return '¡Hola, mundo!';
  }
}
```

## 2. Property Binding `[propiedad]`

Flujo: **Componente → Template** (unidireccional)

Vincula una propiedad del DOM o un `@Input()` de un componente hijo con un valor de la clase. Usa corchetes `[]`.

```html
<!-- Propiedad del DOM -->
<img [src]="imagenUrl" [alt]="descripcion">
<button [disabled]="estaDeshabilitado">Enviar</button>
<input [value]="nombreUsuario">

<!-- Input de un componente hijo -->
<app-tarjeta-usuario [usuario]="usuarioActual" [mostrarAvatar]="true">
</app-tarjeta-usuario>
```

```typescript
export class EjemploComponent {
  imagenUrl = 'https://ejemplo.com/foto.jpg';
  descripcion = 'Foto de perfil';
  estaDeshabilitado = false;
  usuarioActual: Usuario = { id: 1, nombre: 'Ana' };
}
```

## 3. Event Binding `(evento)`

Flujo: **Template → Componente** (unidireccional)

Escucha eventos del DOM y ejecuta un método de la clase. Usa paréntesis `()`.

```html
<button (click)="guardar()">Guardar</button>
<input (input)="onInput($event)" (keyup.enter)="buscar()">
<form (ngSubmit)="onSubmit($event)">
  <!-- ... -->
</form>
<div (mouseover)="resaltar()" (mouseout)="quitarResalte()">
  Pasa el ratón aquí
</div>
```

```typescript
export class EjemploComponent {
  guardar(): void {
    console.log('Guardando...');
  }

  onInput(event: Event): void {
    const valor = (event.target as HTMLInputElement).value;
    console.log('Valor:', valor);
  }
}
```

## 4. Two-Way Binding `[(ngModel)]`

Flujo: **Componente ↔ Template** (bidireccional)

Combina property binding y event binding. El valor se sincroniza en ambas direcciones. Requiere `FormsModule`.

```typescript
import { FormsModule } from '@angular/forms';

@Component({
  standalone: true,
  imports: [FormsModule],
  template: `
    <input [(ngModel)]="nombre" placeholder="Tu nombre">
    <p>Hola, {{ nombre }}!</p>

    <!-- Equivalente manual (sin ngModel): -->
    <input [value]="nombre" (input)="nombre = $any($event.target).value">
  `
})
export class FormularioComponent {
  nombre = '';
}
```

> En formularios reactivos (`ReactiveFormsModule`), no se usa `[(ngModel)]`. Se prefiere `formControlName` o `[formControl]`.

## Resumen visual

```
INTERPOLACIÓN:   Componente ──────→ Template    {{ valor }}
PROPERTY:        Componente ──────→ Template    [prop]="valor"
EVENT:           Template   ──────→ Componente  (evento)="metodo()"
TWO-WAY:         Componente ←────→ Template    [(ngModel)]="valor"
```

## Diferencia entre `{{ valor }}` y `[textContent]="valor"`

Ambos muestran texto, pero:
- `{{ valor }}` es interpolación y **sanitiza** el contenido automáticamente (seguro contra XSS).
- `[innerHTML]="valor"` renderiza HTML pero Angular lo sanitiza también. Para HTML sin sanitizar (peligroso), se necesita `DomSanitizer`.
