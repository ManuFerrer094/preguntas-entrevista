---
title: ¿Qué es effect() en Angular Signals y cómo funciona?
difficulty: medium
tags: [angular, signals, effect, reactivity]
---

`effect()` es una función de Angular Signals que ejecuta un **efecto secundario** cada vez que uno o más signals de los que depende cambian su valor. Es la equivalencia de `computed()` pero para operaciones con efectos secundarios (logging, sincronización con DOM externo, llamadas a APIs, etc.).

## Uso básico

```typescript
import { Component, signal, effect, inject } from '@angular/core';

@Component({
  standalone: true,
  template: `
    <button (click)="contador.update(n => n + 1)">+1</button>
    <span>{{ contador() }}</span>
  `
})
export class ContadorComponent {
  contador = signal(0);

  constructor() {
    // Se ejecuta cada vez que `contador` cambia
    effect(() => {
      console.log('Contador cambió a:', this.contador());
    });
  }
}
```

> **Importante**: `effect()` debe llamarse dentro del contexto de inyección (constructor, factory, o `runInInjectionContext`).

## Dependencias automáticas

```typescript
@Component({ standalone: true })
export class ThemeComponent {
  tema = signal<'claro' | 'oscuro'>('claro');
  idioma = signal('es');
  usuario = signal<string | null>(null);

  constructor() {
    effect(() => {
      // Effect rastrea automáticamente los signals que se leen aquí
      const t = this.tema();
      const lang = this.idioma();
      document.documentElement.setAttribute('data-theme', t);
      document.documentElement.setAttribute('lang', lang);
      // `usuario` NO se lee → no es una dependencia de este effect
    });
  }
}
```

## Limpieza con onCleanup

```typescript
@Component({ standalone: true })
export class TimerComponent {
  activo = signal(false);

  constructor() {
    effect((onCleanup) => {
      if (!this.activo()) return;

      const id = setInterval(() => {
        console.log('tick');
      }, 1000);

      // Se llama antes de la próxima ejecución o cuando el efecto se destruye
      onCleanup(() => clearInterval(id));
    });
  }
}
```

## Evitar ciclos con allowSignalWrites

Por defecto, escribir en un signal dentro de un `effect()` lanza un error para prevenir ciclos infinitos:

```typescript
effect(() => {
  const val = this.a();
  // this.b.set(val); // ❌ Error: no se puede escribir signals dentro de effect por defecto
});

// Si necesitas escribir signals (úsalo con cuidado):
effect(() => {
  const val = this.a();
  this.b.set(val);
}, { allowSignalWrites: true }); // ✅ Permitido explícitamente
```

## Destruir un effect manualmente

```typescript
@Component({ standalone: true })
export class MiComponent implements OnDestroy {
  private effectRef: EffectRef;

  constructor() {
    // Guardamos la referencia
    this.effectRef = effect(() => {
      console.log('valor:', this.miSignal());
    });
  }

  ngOnDestroy() {
    // Destruir el effect antes de que el componente se destruya
    this.effectRef.destroy();
  }
}
```

## Usar effect() fuera del constructor

```typescript
import { inject, Injector, runInInjectionContext, effect } from '@angular/core';

@Component({ standalone: true })
export class MiComponent {
  private injector = inject(Injector);
  datos = signal<string[]>([]);

  iniciarEfecto() {
    // Crear un effect fuera del constructor
    runInInjectionContext(this.injector, () => {
      effect(() => {
        localStorage.setItem('datos', JSON.stringify(this.datos()));
      });
    });
  }
}
```

## Comparativa: effect() vs computed()

```typescript
// computed: transforma un valor, NO tiene efectos secundarios
const nombreCompleto = computed(() => `${nombre()} ${apellido()}`);

// effect: reacciona a cambios para hacer algo (efecto secundario)
effect(() => {
  analytics.track('nombre_cambiado', { nombre: nombre() });
});
```

| Característica        | `computed()`         | `effect()`                |
|-----------------------|----------------------|---------------------------|
| Retorna valor         | ✅ Sí               | ❌ No (void)              |
| Propósito             | Derivar estado       | Ejecutar efectos secundarios |
| Lazy                  | ✅ Sí (memoized)    | ❌ Se ejecuta siempre     |
| Escribir signals      | ❌ No permitido      | ✅ Con `allowSignalWrites` |
| Limpieza              | ❌ No necesario      | ✅ Con `onCleanup`        |

**Consejo de entrevista**: Usa `effect()` solo cuando necesitas sincronizar el estado de Angular con algo externo (DOM, localStorage, analytics, WebSockets). Para derivar valores, usa siempre `computed()`. Un anti-patrón común es usar `effect()` para actualizar otro signal —en ese caso, `computed()` es la solución correcta y más eficiente.
