---
title: ¿Cuál es la diferencia entre compilación JIT y AOT en Angular?
difficulty: medium
tags: [angular, jit, aot, compilation, performance]
---

Angular compila las plantillas HTML y los componentes TypeScript antes de que el navegador pueda ejecutarlos. Existen dos estrategias de compilación: **JIT** (Just-In-Time) y **AOT** (Ahead-Of-Time), que difieren en *cuándo* se realiza esa compilación.

## JIT (Just-In-Time) — Compilación en tiempo de ejecución

Con JIT, la compilación ocurre **en el navegador del usuario**, en tiempo de ejecución, justo antes de que la aplicación se ejecute.

**Características:**
- El compilador de Angular se incluye en el bundle enviado al navegador.
- Tiempo de inicio de la aplicación más lento.
- Los errores de plantilla se detectan en tiempo de ejecución.
- Bundle de mayor tamaño (incluye el compilador).
- Era el modo por defecto en desarrollo en versiones antiguas.

```bash
# Compilación JIT (development, modo antiguo)
ng serve --no-aot  # Ya no es el comportamiento por defecto
```

## AOT (Ahead-Of-Time) — Compilación en tiempo de build

Con AOT, la compilación ocurre **durante el proceso de build**, antes de que el código llegue al navegador.

**Características:**
- El navegador recibe HTML y JavaScript ya compilados y optimizados.
- Inicio de aplicación mucho más rápido (no hay que compilar en el cliente).
- El compilador de Angular **no se incluye** en el bundle → bundles más pequeños.
- Los errores de plantilla se detectan en tiempo de compilación (build).
- **Es el modo por defecto** en producción y desarrollo desde Angular 9+.

```bash
# AOT siempre activo en Angular 9+ (modo por defecto)
ng serve                           # Desarrollo con AOT
ng build                           # Producción con AOT + optimizaciones
ng build --configuration=production # Producción completa
```

## Ventajas de AOT sobre JIT

### 1. Detección temprana de errores

```html
<!-- Este error se detecta en BUILD con AOT, no en runtime -->
<p>{{ usuario.nombreMalEscrito }}</p>
<!-- Error: Property 'nombreMalEscrito' does not exist on type 'Usuario' -->
```

### 2. Mejor seguridad

AOT compila las plantillas antes del despliegue, lo que reduce el riesgo de ataques de inyección ya que no se evalúan expresiones dinámicas en el cliente.

### 3. Bundle más pequeño

```
Con JIT:  bundle.js → incluye Angular Compiler (~hundreds KB)
Con AOT:  bundle.js → sin compilador, solo el runtime de Angular
```

### 4. Inicio más rápido

```
JIT:  Descarga → Parsea JS → Compila plantillas → Renderiza
AOT:  Descarga → Parsea JS → Renderiza (sin paso de compilación)
```

## Tabla comparativa

| Característica          | JIT                          | AOT                           |
|-------------------------|------------------------------|-------------------------------|
| Momento de compilación  | En el navegador (runtime)    | Durante el build              |
| Detección de errores    | En tiempo de ejecución       | En tiempo de compilación      |
| Tamaño del bundle       | Mayor (incluye compilador)   | Menor (sin compilador)        |
| Velocidad de inicio     | Más lento                    | Más rápido                    |
| Tiempo de build         | Más rápido                   | Más lento                     |
| Modo por defecto        | Antiguas versiones           | Angular 9+ (siempre)          |
| Seguridad               | Menor                        | Mayor                         |

## Ivy y AOT

Desde Angular 9, el motor **Ivy** trabaja exclusivamente con AOT. Ivy mejoró significativamente:
- Compilación más rápida con el modo **incremental**.
- **Tree-shaking** más efectivo: solo incluye el código Angular que realmente se usa.
- Mejor diagnóstico de errores en plantillas.

**Consejo de entrevista**: En Angular 9+, AOT está activo tanto en desarrollo como en producción. La diferencia práctica entre `ng serve` y `ng build --production` no es JIT vs AOT, sino las **optimizaciones adicionales** de producción: minificación, tree-shaking agresivo, source maps desactivados y presupuesto de bundle. Saber que AOT detecta errores de plantilla en tiempo de compilación es un punto clave para destacar en entrevistas.
