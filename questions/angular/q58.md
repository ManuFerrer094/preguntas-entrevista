**Angular Elements** es un paquete (`@angular/elements`) que permite convertir componentes Angular en **Web Components estándar** (Custom Elements), que pueden usarse en cualquier entorno web, con o sin Angular.

## ¿Qué son los Custom Elements?

Los **Custom Elements** son parte del estándar de Web Components del W3C. Permiten definir nuevas etiquetas HTML con comportamiento personalizado:

```html
<!-- Un Custom Element se puede usar como cualquier etiqueta HTML -->
<mi-widget-angular data-titulo="Hola" data-color="#ff0000"></mi-widget-angular>
```

## Instalación y configuración

```bash
npm install @angular/elements
```

## Convertir un componente Angular en un Custom Element

```typescript
// widget.component.ts
@Component({
  selector: 'app-widget',
  standalone: true,
  template: `
    <div [style.background-color]="color" class="widget">
      <h2>{{ titulo }}</h2>
      <p>{{ contenido }}</p>
      <button (click)="emitirEvento()">Acción</button>
    </div>
  `
})
export class WidgetComponent {
  @Input() titulo = 'Widget Angular';
  @Input() color = '#e3f2fd';
  @Input() contenido = '';

  @Output() accionEjecutada = new EventEmitter<string>();

  emitirEvento(): void {
    this.accionEjecutada.emit('Acción ejecutada en: ' + new Date().toISOString());
  }
}
```

```typescript
// main.ts — Registrar el Custom Element
import { createApplication } from '@angular/platform-browser';
import { createCustomElement } from '@angular/elements';
import { WidgetComponent } from './widget.component';

(async () => {
  const app = await createApplication({
    providers: [/* proveedores necesarios */]
  });

  // Convertir el componente Angular en un Custom Element
  const WidgetElement = createCustomElement(WidgetComponent, {
    injector: app.injector
  });

  // Registrar el Custom Element con el nombre de la etiqueta
  customElements.define('mi-widget-angular', WidgetElement);
})();
```

## Usar el Custom Element en HTML puro (sin Angular)

```html
<!DOCTYPE html>
<html>
<head>
  <script src="mi-widget-angular.js"></script> <!-- Bundle generado por Angular -->
</head>
<body>
  <!-- Se usa como cualquier etiqueta HTML estándar -->
  <mi-widget-angular
    titulo="Panel de Control"
    color="#fff3e0"
    contenido="Datos importantes aquí">
  </mi-widget-angular>

  <script>
    // Escuchar eventos del Custom Element
    const widget = document.querySelector('mi-widget-angular');
    widget.addEventListener('accionEjecutada', (event) => {
      console.log('Evento recibido:', event.detail);
    });

    // Cambiar propiedades dinámicamente
    widget.titulo = 'Nuevo Título';
    widget.color = '#e8f5e9';
  </script>
</body>
</html>
```

## Usar el Custom Element en React o Vue

```jsx
// En React (como cualquier elemento HTML nativo)
function App() {
  const widgetRef = useRef(null);

  useEffect(() => {
    const widget = widgetRef.current;
    widget.addEventListener('accionEjecutada', handleAccion);
    return () => widget.removeEventListener('accionEjecutada', handleAccion);
  }, []);

  return (
    <mi-widget-angular
      ref={widgetRef}
      titulo="Widget en React"
      contenido="Funciona en React!">
    </mi-widget-angular>
  );
}
```

## Construir el Custom Element como bundle independiente

```bash
# Construir como un único archivo JS (bundle)
ng build --output-hashing=none --single-bundle
```

```typescript
// angular.json - configuración para bundle único
"scripts": [
  "node scripts/build-elements.js"
]
```

```javascript
// scripts/build-elements.js
const { execSync } = require('child_process');
const fs = require('fs');

// Construir
execSync('ng build --configuration=production --output-hashing=none');

// Concatenar en un solo archivo
const distPath = 'dist/mi-elemento/browser';
const archivos = ['runtime.js', 'polyfills.js', 'main.js'];
const contenido = archivos.map(f => fs.readFileSync(`${distPath}/${f}`)).join('\n');
fs.writeFileSync('dist/mi-widget-angular.js', contenido);
```

## Casos de uso de Angular Elements

1. **Micro-frontends**: diferentes equipos con diferentes tecnologías.
2. **Widgets embebibles**: reutilizar componentes Angular en sitios web legacy.
3. **Aplicaciones legacy**: migración gradual de AngularJS a Angular.
4. **CMS y editores**: integrar componentes Angular en WordPress, Drupal, etc.
5. **Design Systems**: publicar componentes que funcionen en cualquier framework.

## Consideraciones y limitaciones

- El bundle puede ser grande si no se optimiza (incluye el runtime de Angular).
- Los `@Input()` reciben valores como strings desde los atributos HTML; usa `transform` para conversión.
- El ciclo de vida de Angular se mapea a los callbacks de Custom Elements.
- Para múltiples elementos, comparten el mismo bundle de Angular.

**Consejo de entrevista**: Angular Elements es la respuesta de Angular al problema de la interoperabilidad entre frameworks. Su caso de uso más común es crear **micro-frontends** o widgets que se integren en aplicaciones que no son Angular. El coste principal es el tamaño del bundle (Angular runtime + librería de elementos), que se puede mitigar usando `@angular/elements` con Zone.js desactivado y Ivy para máximo tree-shaking.
