---
title: ¿Cuál es la diferencia entre pipes puros e impuros en Angular?

---

En Angular, todos los pipes tienen una propiedad `pure` que controla cuándo se ejecuta la función `transform()`. Esta propiedad tiene un impacto directo en el **rendimiento de la aplicación**.

## Pipes puros (pure: true)

Son los **pipes por defecto**. Angular solo ejecuta el método `transform()` cuando detecta un **cambio de referencia** en el valor de entrada (o en sus argumentos).

- Para **tipos primitivos** (string, number, boolean): se ejecuta cuando cambia el valor.
- Para **objetos y arrays**: se ejecuta cuando cambia la referencia (no las mutaciones internas).

```typescript
@Pipe({
  name: 'formatearMoneda',
  standalone: true,
  pure: true // ← Por defecto, no hace falta indicarlo explícitamente
})
export class FormatearMonedaPipe implements PipeTransform {
  transform(valor: number, moneda: string = 'EUR'): string {
    return new Intl.NumberFormat('es-ES', {
      style: 'currency',
      currency: moneda
    }).format(valor);
  }
}
```

```html
<!-- Solo se recalcula cuando cambia 'precio' o 'moneda' -->
<p>{{ precio | formatearMoneda:'USD' }}</p>
```

### Problema con pipes puros y arrays/objetos mutados

```typescript
@Component({
  template: `
    <!-- ❌ No se actualiza: items es el mismo array (mutado, no reemplazado) -->
    <li *ngFor="let item of items | filtrarActivos">{{ item.nombre }}</li>
    <button (click)="agregarItem()">Agregar</button>
  `
})
export class ListaComponent {
  items: Item[] = [];

  agregarItem(): void {
    this.items.push({ nombre: 'Nuevo', activo: true }); // Mutación ❌
    // this.items = [...this.items, { nombre: 'Nuevo', activo: true }]; // Nueva referencia ✅
  }
}
```

## Pipes impuros (pure: false)

Se ejecutan en **cada ciclo de detección de cambios**, independientemente de si el valor ha cambiado. Esto los hace más reactivos pero también más costosos para el rendimiento.

```typescript
@Pipe({
  name: 'filtrarActivos',
  standalone: true,
  pure: false // ← Ejecuta transform() en cada change detection
})
export class FiltrarActivosPipe implements PipeTransform {
  transform(items: Item[]): Item[] {
    if (!items) return [];
    return items.filter(item => item.activo);
  }
}
```

### Pipes impuros integrados en Angular

Angular incluye algunos pipes impuros por necesidad:
- **`AsyncPipe`**: escucha valores de Observables/Promises (impuro por naturaleza).
- **`KeyValuePipe`**: necesita detectar cambios en objetos.
- **`SlicePipe`**: puede necesitar detectar cambios en arrays mutados.

## Tabla comparativa

| Característica             | Pipe puro (`pure: true`)          | Pipe impuro (`pure: false`)         |
|----------------------------|-----------------------------------|-------------------------------------|
| Cuándo se ejecuta          | Solo cuando cambia la referencia  | En cada ciclo de change detection   |
| Rendimiento                | ✅ Alto                           | ⚠️ Puede ser costoso               |
| Detecta mutaciones internas| ❌ No                             | ✅ Sí                               |
| Uso recomendado            | La mayoría de los casos           | Solo cuando es imprescindible       |
| Ejemplos integrados        | `DatePipe`, `CurrencyPipe`        | `AsyncPipe`, `KeyValuePipe`         |

## Recomendación: preferir inmutabilidad

La mejor práctica es **evitar pipes impuros** y trabajar con inmutabilidad:

```typescript
// ✅ En lugar de mutar el array (que requiere pipe impuro):
this.items.push(nuevoItem); // ❌ Mutación

// Crear una nueva referencia (funciona con pipe puro):
this.items = [...this.items, nuevoItem]; // ✅ Nueva referencia
this.items = this.items.filter(i => i.id !== idAEliminar); // ✅ Nuevo array
```

Esto es compatible con la estrategia `OnPush` y los Signals de Angular, que también se basan en la detección de cambios de referencia.

**Consejo de entrevista**: Si en una entrevista te preguntan por qué un pipe no se actualiza cuando agregas elementos a un array, la respuesta es que los pipes puros no detectan mutaciones internas. La solución correcta es crear una nueva referencia del array en lugar de mutarlo. Si alguien propone `pure: false` como solución, hay que mencionar el impacto en el rendimiento y que es preferible cambiar la estrategia de actualización del array.
