---
title: ¿Qué es la estrategia de precarga (preloading) de rutas en Angular?
difficulty: medium

---

La estrategia de **precarga** (*preloading*) en Angular permite cargar módulos de forma diferida (lazy) en segundo plano después de que la aplicación inicial ha cargado, mejorando la experiencia de usuario al reducir el tiempo de navegación entre rutas.

## Sin precarga (comportamiento por defecto)

```typescript
// Sin preloading: los módulos se cargan solo cuando el usuario navega a la ruta
const routes: Routes = [
  { path: 'dashboard', loadComponent: () => import('./dashboard/dashboard.component') },
  { path: 'usuarios', loadComponent: () => import('./usuarios/usuarios.component') },
];
```
El módulo solo se descarga cuando el usuario hace clic en el enlace → hay latencia.

## PreloadAllModules — Precarga todo en segundo plano

```typescript
import { RouterModule, PreloadAllModules } from '@angular/router';

@NgModule({
  imports: [
    RouterModule.forRoot(routes, {
      preloadingStrategy: PreloadAllModules  // ✅ Precarga todos los módulos lazy
    })
  ]
})
export class AppModule {}
```

Con standalone bootstrap:

```typescript
// main.ts
bootstrapApplication(AppComponent, {
  providers: [
    provideRouter(routes, withPreloading(PreloadAllModules))
  ]
});
```

## Estrategia de precarga personalizada

```typescript
import { Injectable } from '@angular/core';
import { PreloadingStrategy, Route } from '@angular/router';
import { Observable, of } from 'rxjs';

@Injectable({ providedIn: 'root' })
export class SelectivePreloadingStrategy implements PreloadingStrategy {
  preload(route: Route, load: () => Observable<any>): Observable<any> {
    // Solo precargar rutas marcadas con data.preload: true
    if (route.data?.['preload']) {
      console.log(`Precargando: ${route.path}`);
      return load();
    }
    return of(null);
  }
}

// Configuración de rutas con marcado selectivo
const routes: Routes = [
  {
    path: 'dashboard',
    loadComponent: () => import('./dashboard/dashboard.component'),
    data: { preload: true }  // ✅ Se precarga en segundo plano
  },
  {
    path: 'admin',
    loadChildren: () => import('./admin/admin.routes'),
    data: { preload: false }  // ❌ No se precarga (solo admins la necesitan)
  },
  {
    path: 'perfil',
    loadComponent: () => import('./perfil/perfil.component'),
    data: { preload: true }  // ✅ Se precarga
  }
];

// Usar la estrategia personalizada
bootstrapApplication(AppComponent, {
  providers: [
    provideRouter(routes, withPreloading(SelectivePreloadingStrategy))
  ]
});
```

## Precarga basada en conexión de red

```typescript
@Injectable({ providedIn: 'root' })
export class NetworkAwarePreloadingStrategy implements PreloadingStrategy {
  preload(route: Route, load: () => Observable<any>): Observable<any> {
    // No precargar en conexiones lentas (2G, slow-2g)
    const connection = (navigator as any).connection;
    if (connection) {
      const isSlowConnection = ['slow-2g', '2g'].includes(connection.effectiveType);
      const isSavingData = connection.saveData;

      if (isSlowConnection || isSavingData) {
        return of(null);  // No precargar
      }
    }

    return route.data?.['preload'] ? load() : of(null);
  }
}
```

## QuicklinkStrategy — Precarga rutas visibles en pantalla

```typescript
// Instalar: npm install ngx-quicklink
import { QuicklinkStrategy, QuicklinkModule } from 'ngx-quicklink';

bootstrapApplication(AppComponent, {
  providers: [
    provideRouter(routes, withPreloading(QuicklinkStrategy))
  ]
});
// Precarga automáticamente las rutas cuyos <a routerLink> están visibles en el viewport
```

## Comparativa de estrategias

| Estrategia                  | Cuándo precargar              | Ideal para                     |
|-----------------------------|-------------------------------|--------------------------------|
| `NoPreloading` (defecto)    | Nunca (solo bajo demanda)     | Apps muy grandes               |
| `PreloadAllModules`         | Todo en segundo plano         | Apps medianas                  |
| Personalizada con `data`    | Solo rutas marcadas           | Control granular               |
| `QuicklinkStrategy`         | Links visibles en pantalla    | Mejor UX automática            |
| Basada en red               | Solo en buenas conexiones     | PWA y usuarios móviles         |

**Consejo de entrevista**: La estrategia de precarga perfecta para la mayoría de apps es la personalizada con `data.preload`. `PreloadAllModules` puede desperdiciar ancho de banda en rutas que el usuario nunca visita. La combinación ideal es: lazy loading para la carga inicial + precarga selectiva para las rutas más visitadas. Esto se puede combinar con análisis de uso (Google Analytics) para precargar inteligentemente las rutas más frecuentes.
