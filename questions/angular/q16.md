---
title: ¿Qué son ViewChild y ContentChild en Angular?
difficulty: medium
tags: [angular, viewchild, contentchild, template-reference]
---

`ViewChild` y `ContentChild` son decoradores (o funciones de señal en Angular 17+) que permiten a un componente obtener referencias a elementos dentro de su template o a componentes/directivas hijos. Son fundamentales para interactuar programáticamente con la vista.

**ViewChild** da acceso a elementos del propio template del componente (su View):

```typescript
@Component({
  template: `
    <input #myInput type="text" />
    <canvas #myCanvas></canvas>
    <app-child #childComp></app-child>
  `
})
export class ParentComponent implements AfterViewInit {
  @ViewChild('myInput') inputRef!: ElementRef<HTMLInputElement>;
  @ViewChild('myCanvas') canvasRef!: ElementRef<HTMLCanvasElement>;
  @ViewChild('childComp') childComponent!: ChildComponent;

  ngAfterViewInit() {
    // Solo disponible a partir de AfterViewInit
    this.inputRef.nativeElement.focus();
    this.childComponent.someMethod();
  }
}
```

**ViewChildren** para múltiples elementos:

```typescript
@ViewChildren('item') items!: QueryList<ElementRef>;

ngAfterViewInit() {
  this.items.forEach(item => console.log(item.nativeElement));
}
```

**ContentChild** da acceso al contenido proyectado mediante `<ng-content>` (Content Projection):

```typescript
// card.component.ts
@Component({
  selector: 'app-card',
  template: `
    <div class="card">
      <ng-content select="[cardHeader]"></ng-content>
      <ng-content></ng-content>
    </div>
  `
})
export class CardComponent implements AfterContentInit {
  @ContentChild('cardTitle') titleRef!: ElementRef;

  ngAfterContentInit() {
    // El contenido proyectado está disponible aquí
    console.log(this.titleRef.nativeElement.textContent);
  }
}

// Uso del componente
// <app-card>
//   <h2 #cardTitle cardHeader>Mi Título</h2>
//   <p>Contenido del card</p>
// </app-card>
```

**Signal-based queries (Angular 17+):**

```typescript
@Component({...})
export class ModernComponent {
  // Nueva API basada en signals
  myInput = viewChild<ElementRef>('myInput');
  items = viewChildren<ItemComponent>(ItemComponent);
  projectedContent = contentChild<ElementRef>('title');

  ngAfterViewInit() {
    // El valor se obtiene llamando la signal como función
    this.myInput()?.nativeElement.focus();
  }
}
```

La diferencia clave entre `ViewChild` y `ContentChild`:
- **ViewChild**: accede al template del propio componente.
- **ContentChild**: accede al contenido que le pasa el componente padre mediante content projection (`<ng-content>`).

**Consejo de entrevista**: Es importante aclarar cuándo están disponibles los valores: `ViewChild` solo está disponible en `ngAfterViewInit`, y `ContentChild` en `ngAfterContentInit`. Acceder a ellos en `ngOnInit` devolverá `undefined`. Con la nueva API de signals, los valores son reactivos y siempre están actualizados.
