RxJS es la librería de programación reactiva que Angular utiliza extensivamente. Sus operadores permiten transformar, filtrar y combinar flujos de datos de forma declarativa. Conocer los operadores más importantes es fundamental para trabajar eficientemente con Angular.

## Operadores de transformación

### `map` — Transformar cada valor

```typescript
this.http.get<{ data: Usuario[] }>('/api/usuarios').pipe(
  map(respuesta => respuesta.data),           // Extrae el array
  map(usuarios => usuarios.filter(u => u.activo)) // Filtra activos
).subscribe(usuarios => this.usuarios = usuarios);
```

### `switchMap` — Cancelar y reemplazar (el más usado)

Cancela el Observable anterior al llegar un nuevo valor. Ideal para búsquedas y navegación:

```typescript
// Búsqueda en tiempo real: cancela petición anterior si el usuario sigue escribiendo
this.searchControl.valueChanges.pipe(
  debounceTime(300),
  distinctUntilChanged(),
  switchMap(termino => this.productoService.buscar(termino))
).subscribe(resultados => this.resultados = resultados);

// Navegación: cancela petición del producto anterior
this.route.paramMap.pipe(
  switchMap(params => this.productoService.getProducto(params.get('id')!))
).subscribe(producto => this.producto = producto);
```

### `mergeMap` (flatMap) — En paralelo, sin cancelar

Permite múltiples Observables internos en paralelo. Útil para operaciones independientes:

```typescript
// Eliminar múltiples elementos en paralelo
from(idsAEliminar).pipe(
  mergeMap(id => this.productoService.eliminar(id))
).subscribe();
```

### `concatMap` — Secuencial (espera al anterior)

Procesa los Observables uno a uno, en orden:

```typescript
// Subir archivos en orden secuencial
from(archivos).pipe(
  concatMap(archivo => this.uploadService.subir(archivo))
).subscribe(resultado => this.progresos.push(resultado));
```

### `exhaustMap` — Ignorar mientras está en curso

Ignora nuevos valores mientras el Observable interno está activo. Ideal para prevenir dobles envíos:

```typescript
// Evitar doble clic en botón de envío
this.submitBtn.clicks.pipe(
  exhaustMap(() => this.formService.guardar(this.form.value))
).subscribe();
```

## Comparativa de operadores de aplanamiento

| Operador      | Comportamiento                              | Caso de uso                    |
|---------------|---------------------------------------------|--------------------------------|
| `switchMap`   | Cancela el anterior                         | Búsquedas, rutas               |
| `mergeMap`    | Paralelo, sin cancelar                      | Operaciones independientes     |
| `concatMap`   | Secuencial, espera al anterior              | Orden importante               |
| `exhaustMap`  | Ignora nuevos hasta que el actual termine   | Prevenir dobles envíos         |

## Operadores de filtrado

```typescript
// filter — filtrar valores
observable$.pipe(
  filter(valor => valor > 0),
  filter(usuario => usuario.activo)
)

// take — tomar solo N valores y completar
observable$.pipe(take(1))     // Solo el primer valor

// takeUntil — completar cuando otro Observable emite
this.datosService.getDatos().pipe(
  takeUntil(this.destroy$)   // Cancelar al destruir el componente
).subscribe(datos => this.datos = datos);

// debounceTime — esperar N ms sin actividad
searchInput.valueChanges.pipe(
  debounceTime(400)          // Espera 400ms después del último teclazo
)

// distinctUntilChanged — ignorar valores repetidos consecutivos
observable$.pipe(
  distinctUntilChanged()     // 1,1,2,2,3 → 1,2,3
)
```

## Operadores de combinación

```typescript
import { forkJoin, combineLatest, zip } from 'rxjs';

// forkJoin — esperar que todos completen (equivalente a Promise.all)
forkJoin({
  usuario: this.userService.getUsuario(id),
  permisos: this.authService.getPermisos(id),
  preferencias: this.prefService.getPreferencias(id)
}).subscribe(({ usuario, permisos, preferencias }) => {
  // Todos los datos disponibles a la vez
});

// combineLatest — emite cuando cualquiera de los fuentes cambia
combineLatest([
  this.filtroService.filtroActivo$,
  this.paginaService.paginaActual$,
  this.ordenService.ordenActual$
]).pipe(
  switchMap(([filtro, pagina, orden]) =>
    this.productoService.getProductos({ filtro, pagina, orden })
  )
).subscribe(productos => this.productos = productos);
```

## Operadores de manejo de errores

```typescript
// catchError — manejar errores sin romper el stream
this.http.get('/api/datos').pipe(
  catchError(error => {
    console.error('Error:', error);
    return of([]); // Valor por defecto en caso de error
  })
)

// retry — reintentar en caso de error
this.http.get('/api/datos').pipe(
  retry(3)          // Reintenta hasta 3 veces
)

// retryWhen / retry con delay (RxJS 7+)
this.http.get('/api/datos').pipe(
  retry({ count: 3, delay: 1000 }) // 3 reintentos con 1s de espera
)
```

## Patrón completo: búsqueda con todo

```typescript
@Component({ /* ... */ })
export class BusquedaComponent implements OnInit, OnDestroy {
  private destroy$ = new Subject<void>();
  resultados: Producto[] = [];
  cargando = false;
  error: string | null = null;

  searchControl = new FormControl('');

  ngOnInit(): void {
    this.searchControl.valueChanges.pipe(
      debounceTime(400),              // Espera 400ms
      distinctUntilChanged(),         // Ignora si no cambió
      filter(termino => (termino?.length ?? 0) >= 2), // Mínimo 2 caracteres
      tap(() => { this.cargando = true; this.error = null; }),
      switchMap(termino =>            // Cancela petición anterior
        this.productoService.buscar(termino!).pipe(
          catchError(err => {
            this.error = 'Error al buscar';
            return of([]);
          })
        )
      ),
      tap(() => this.cargando = false),
      takeUntil(this.destroy$)        // Limpiar al destruir
    ).subscribe(resultados => this.resultados = resultados);
  }

  ngOnDestroy(): void {
    this.destroy$.next();
    this.destroy$.complete();
  }
}
```

**Consejo de entrevista**: El operador más importante que debes dominar es `switchMap`, especialmente la diferencia con `mergeMap`. La regla práctica: si quieres cancelar la petición anterior (búsquedas, navegación) → `switchMap`; si quieres ejecutarlas en paralelo → `mergeMap`; si el orden importa → `concatMap`; si quieres ignorar mientras está en curso → `exhaustMap`. El patrón `debounceTime + distinctUntilChanged + switchMap` para búsquedas es uno de los más preguntados en entrevistas Angular avanzadas.
