Un **Service Worker** es un script JavaScript que el navegador ejecuta en segundo plano, separado de la página web, actuando como proxy entre la aplicación y la red. Permite interceptar peticiones de red, gestionar caché y habilitar funcionalidades offline.

Una **PWA (Progressive Web App)** es una aplicación web que usa Service Workers (entre otras tecnologías) para ofrecer una experiencia similar a una app nativa: instalable, offline-first y con notificaciones push.

## Agregar soporte PWA a un proyecto Angular

```bash
# ng add configura automáticamente todo lo necesario
ng add @angular/pwa
```

Este comando realiza los siguientes cambios automáticamente:
1. Instala `@angular/service-worker`.
2. Añade `ServiceWorkerModule` a `app.module.ts` o configura el provider.
3. Crea `ngsw-config.json` (configuración del Service Worker).
4. Crea `manifest.webmanifest` (metadatos de la PWA).
5. Añade los iconos y meta tags necesarios en `index.html`.

## Configuración del Service Worker (ngsw-config.json)

```json
{
  "$schema": "./node_modules/@angular/service-worker/config/schema.json",
  "index": "/index.html",
  "assetGroups": [
    {
      "name": "app",
      "installMode": "prefetch",
      "resources": {
        "files": ["/favicon.ico", "/index.html", "/manifest.webmanifest", "/*.css", "/*.js"]
      }
    },
    {
      "name": "assets",
      "installMode": "lazy",
      "updateMode": "prefetch",
      "resources": {
        "files": ["/assets/**", "/*.(svg|cur|jpg|jpeg|png|apng|webp|avif|gif|otf|ttf|woff|woff2)"]
      }
    }
  ],
  "dataGroups": [
    {
      "name": "api-freshness",
      "urls": ["/api/**"],
      "cacheConfig": {
        "strategy": "freshness",
        "maxSize": 100,
        "maxAge": "3d",
        "timeout": "10s"
      }
    },
    {
      "name": "api-performance",
      "urls": ["/api/static/**"],
      "cacheConfig": {
        "strategy": "performance",
        "maxSize": 50,
        "maxAge": "7d"
      }
    }
  ]
}
```

## Estrategias de caché

Angular Service Worker ofrece dos estrategias principales para datos (`dataGroups`):

- **`freshness`** (Network First): intenta la red primero; si falla, usa la caché. Ideal para datos que cambian frecuentemente.
- **`performance`** (Cache First): usa la caché primero; si no existe, va a la red. Ideal para recursos estáticos o que cambian poco.

## Configuración en el bootstrap (Angular 17+)

```typescript
// main.ts
import { bootstrapApplication } from '@angular/platform-browser';
import { provideServiceWorker } from '@angular/service-worker';
import { isDevMode } from '@angular/core';

bootstrapApplication(AppComponent, {
  providers: [
    provideServiceWorker('ngsw-worker.js', {
      enabled: !isDevMode(),           // Solo en producción
      registrationStrategy: 'registerWhenStable:30000'
    })
  ]
});
```

## Detectar actualizaciones de la PWA

```typescript
@Injectable({ providedIn: 'root' })
export class ActualizacionPwaService {
  constructor(private swUpdate: SwUpdate) {
    // Verificar si hay actualizaciones disponibles
    swUpdate.versionUpdates.subscribe(event => {
      if (event.type === 'VERSION_READY') {
        if (confirm('Nueva versión disponible. ¿Actualizar ahora?')) {
          document.location.reload();
        }
      }
    });
  }

  async comprobarActualizaciones(): Promise<void> {
    if (this.swUpdate.isEnabled) {
      await this.swUpdate.checkForUpdate();
    }
  }
}
```

## Probar la PWA localmente

```bash
# Construir en modo producción (el SW no funciona en dev)
ng build

# Servir con un servidor HTTP estático
npx http-server -p 8080 -c-1 dist/mi-app/browser
```

**Consejo de entrevista**: Recuerda que los Service Workers **solo funcionan en producción** (HTTPS o localhost). En `ng serve` (desarrollo), el Service Worker está deshabilitado por defecto. Para probar offline, debes hacer `ng build` y servir los archivos con un servidor estático. También es importante mencionar `SwPush` para notificaciones push y el control de actualizaciones con `SwUpdate`.
