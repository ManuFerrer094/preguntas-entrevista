Los **decoradores** son una característica de TypeScript (y propuesta para JavaScript estándar) que permiten adjuntar **metadatos** a clases, métodos, propiedades y parámetros. Angular los usa extensivamente para declarar componentes, servicios, pipes y para configurar el comportamiento de la inyección de dependencias.

## ¿Cómo funcionan los decoradores?

Un decorador es una función que recibe como argumento el elemento decorado y puede modificar su comportamiento o adjuntar metadatos:

```typescript
// Decorador simplificado (conceptual)
function Component(metadata: ComponentMetadata) {
  return function(target: Function) {
    // Angular registra el componente con sus metadatos
    Reflect.defineMetadata('annotations', [metadata], target);
  };
}
```

## Decoradores de clase

### `@Component`

El más importante. Convierte una clase TypeScript en un componente Angular:

```typescript
@Component({
  selector: 'app-hero',
  standalone: true,
  imports: [CommonModule],
  templateUrl: './hero.component.html',
  styleUrl: './hero.component.scss',
  changeDetection: ChangeDetectionStrategy.OnPush,
  encapsulation: ViewEncapsulation.Emulated,
  providers: [HeroService]  // Proveedor a nivel de componente
})
export class HeroComponent implements OnInit {
  // ...
}
```

### `@Directive`

Convierte una clase en una directiva:

```typescript
@Directive({
  selector: '[appResaltar]',
  standalone: true,
  host: { '(mouseenter)': 'onEnter()', '(mouseleave)': 'onLeave()' }
})
export class ResaltarDirective { /* ... */ }
```

### `@Pipe`

Convierte una clase en un pipe:

```typescript
@Pipe({
  name: 'truncar',
  standalone: true,
  pure: true
})
export class TruncarPipe implements PipeTransform {
  transform(valor: string, limite: number = 100): string { /* ... */ }
}
```

### `@Injectable`

Marca una clase como inyectable por el sistema de DI de Angular:

```typescript
@Injectable({
  providedIn: 'root'       // Singleton global y tree-shakeable
  // providedIn: 'platform'  // Singleton a nivel de plataforma
  // providedIn: 'any'       // Nueva instancia por módulo cargado lazy
})
export class UsuarioService { /* ... */ }
```

### `@NgModule`

Define un módulo Angular (arquitectura clásica):

```typescript
@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule, RouterModule],
  providers: [{ provide: TOKEN, useValue: valor }],
  bootstrap: [AppComponent],
  exports: [AppComponent]
})
export class AppModule {}
```

## Decoradores de propiedad

### `@Input()` y `@Output()`

```typescript
export class TarjetaComponent {
  @Input() titulo!: string;
  @Input({ required: true }) id!: number;        // Angular 16+
  @Input({ transform: booleanAttribute }) activo = false;

  @Output() cerrar = new EventEmitter<void>();
  @Output('guardado') onGuardado = new EventEmitter<Entidad>(); // Con alias
}
```

### `@ViewChild()` y `@ViewChildren()`

```typescript
export class FormComponent implements AfterViewInit {
  // Primer elemento que coincide
  @ViewChild('miInput') input!: ElementRef<HTMLInputElement>;
  @ViewChild(SpinnerComponent) spinner!: SpinnerComponent;
  @ViewChild('ancla', { read: ViewContainerRef }) ancla!: ViewContainerRef;

  // Todos los elementos que coinciden (QueryList)
  @ViewChildren(ItemComponent) items!: QueryList<ItemComponent>;
  @ViewChildren('fila') filas!: QueryList<ElementRef>;

  ngAfterViewInit(): void {
    this.input.nativeElement.focus();
    this.items.changes.subscribe(lista => console.log('Items:', lista.length));
  }
}
```

### `@ContentChild()` y `@ContentChildren()`

```typescript
export class ContenedorComponent implements AfterContentInit {
  @ContentChild(CabeceraComponent) cabecera!: CabeceraComponent;
  @ContentChildren(TabComponent) tabs!: QueryList<TabComponent>;

  ngAfterContentInit(): void {
    console.log('Tabs proyectados:', this.tabs.length);
  }
}
```

### `@HostBinding()` y `@HostListener()`

```typescript
export class BotonDirective {
  @HostBinding('class.activo') estaActivo = false;
  @HostBinding('attr.disabled') deshabilitado: boolean | null = null;

  @HostListener('click', ['$event'])
  onClick(event: MouseEvent): void { /* ... */ }
}
```

## Decoradores de parámetro (en el constructor)

### `@Inject()`

Especifica manualmente el token a inyectar (necesario para tokens no-clase):

```typescript
const API_URL = new InjectionToken<string>('api.url');

export class ApiService {
  constructor(@Inject(API_URL) private apiUrl: string) {}
}
```

### `@Optional()`

Marca una dependencia como opcional (no lanza error si no existe):

```typescript
export class LogService {
  constructor(@Optional() private logger: ExternalLogger | null) {
    if (this.logger) {
      this.logger.log('LogService inicializado');
    }
  }
}
```

### `@Self()`, `@SkipSelf()`, `@Host()`

Controlan el alcance de la búsqueda en el árbol de inyectores:

```typescript
export class ChildComponent {
  constructor(
    @Self() private localService: MiServicio,      // Solo en este inyector
    @SkipSelf() private parentService: MiServicio, // Saltar el inyector local
    @Host() private hostService: MiServicio         // Solo el host y sus padres hasta el host
  ) {}
}
```

## Tabla resumen

| Decorador        | Tipo      | Propósito                                    |
|------------------|-----------|----------------------------------------------|
| `@Component`     | Clase     | Definir un componente                        |
| `@Directive`     | Clase     | Definir una directiva                        |
| `@Pipe`          | Clase     | Definir un pipe                              |
| `@Injectable`    | Clase     | Permitir la inyección de dependencias        |
| `@NgModule`      | Clase     | Definir un módulo                            |
| `@Input`         | Propiedad | Recibir datos del componente padre           |
| `@Output`        | Propiedad | Emitir eventos al componente padre           |
| `@ViewChild`     | Propiedad | Acceder a elementos de la vista              |
| `@ContentChild`  | Propiedad | Acceder a contenido proyectado               |
| `@HostBinding`   | Propiedad | Bindear al elemento huésped                  |
| `@HostListener`  | Método    | Escuchar eventos del elemento huésped        |
| `@Inject`        | Parámetro | Inyección manual por token                   |
| `@Optional`      | Parámetro | Dependencia opcional                         |

**Consejo de entrevista**: Los decoradores son metaprogramación: permiten añadir comportamiento sin modificar el código de la clase directamente. Angular los usa para construir un grafo de dependencias en tiempo de compilación (AOT), lo que permite el tree-shaking y la detección de errores de tipo en plantillas. La tendencia moderna es reemplazar algunos decoradores de propiedad (`@Input`, `@Output`, `@ViewChild`) con las nuevas APIs funcionales de signals (`input()`, `output()`, `viewChild()`).
