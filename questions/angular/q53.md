title: ¿Qué es FormBuilder en Angular y cómo simplifica los formularios reactivos?

**`FormBuilder`** es un servicio de Angular que proporciona métodos de fábrica para crear instancias de `FormControl`, `FormGroup` y `FormArray` con menos código repetitivo. Es una alternativa más concisa a instanciar estas clases directamente con `new`.

## Sin FormBuilder vs Con FormBuilder

```typescript
// ❌ Sin FormBuilder: verboso
import { FormGroup, FormControl, Validators } from '@angular/forms';

export class LoginComponent {
  form = new FormGroup({
    email: new FormControl('', [Validators.required, Validators.email]),
    password: new FormControl('', [Validators.required, Validators.minLength(8)]),
    recordarme: new FormControl(false)
  });
}
```

```typescript
// ✅ Con FormBuilder: más conciso y legible
import { FormBuilder, Validators } from '@angular/forms';

export class LoginComponent {
  constructor(private fb: FormBuilder) {}

  form = this.fb.group({
    email: ['', [Validators.required, Validators.email]],
    password: ['', [Validators.required, Validators.minLength(8)]],
    recordarme: [false]
  });
}
```

## FormBuilder con tipado fuerte (Angular 14+)

Desde Angular 14, los formularios reactivos son **tipados estrictamente** por defecto:

```typescript
// NonNullableFormBuilder (Angular 14+): todos los controles son NonNullable
import { NonNullableFormBuilder } from '@angular/forms';

@Component({
  standalone: true,
  imports: [ReactiveFormsModule]
})
export class RegistroComponent {
  // Inyección con inject() (Angular 14+)
  private fb = inject(NonNullableFormBuilder);

  form = this.fb.group({
    nombre: ['', Validators.required],        // string (nunca null)
    email: ['', [Validators.required, Validators.email]],
    edad: [18, [Validators.min(18), Validators.max(120)]],
    rol: ['usuario' as 'usuario' | 'admin']
  });
}
```

## Formulario de login completo

```typescript
@Component({
  selector: 'app-login',
  standalone: true,
  imports: [ReactiveFormsModule, CommonModule],
  template: `
    <form [formGroup]="loginForm" (ngSubmit)="onSubmit()">
      <!-- Email -->
      <div class="campo">
        <label>Email</label>
        <input type="email" formControlName="email">
        @if (email.invalid && email.touched) {
          @if (email.hasError('required')) {
            <span class="error">El email es requerido</span>
          }
          @if (email.hasError('email')) {
            <span class="error">Formato de email inválido</span>
          }
        }
      </div>

      <!-- Contraseña -->
      <div class="campo">
        <label>Contraseña</label>
        <input type="password" formControlName="password">
        @if (password.invalid && password.touched) {
          @if (password.hasError('minlength')) {
            <span class="error">Mínimo 8 caracteres</span>
          }
        }
      </div>

      <button type="submit" [disabled]="loginForm.invalid || cargando">
        {{ cargando ? 'Iniciando sesión...' : 'Iniciar sesión' }}
      </button>
    </form>
  `
})
export class LoginComponent {
  private fb = inject(NonNullableFormBuilder);
  private authService = inject(AuthService);
  private router = inject(Router);

  cargando = false;

  loginForm = this.fb.group({
    email: ['', [Validators.required, Validators.email]],
    password: ['', [Validators.required, Validators.minLength(8)]]
  });

  // Getters para acceso fácil en la plantilla y reducir repetición
  get email() { return this.loginForm.controls.email; }
  get password() { return this.loginForm.controls.password; }

  onSubmit(): void {
    if (this.loginForm.invalid) return;

    this.cargando = true;
    const { email, password } = this.loginForm.getRawValue();

    this.authService.login(email, password).subscribe({
      next: () => this.router.navigate(['/dashboard']),
      error: () => {
        this.loginForm.setErrors({ credencialesInvalidas: true });
        this.cargando = false;
      }
    });
  }
}
```

## FormBuilder con FormArray

```typescript
@Component({ standalone: true, imports: [ReactiveFormsModule, CommonModule] })
export class PedidoComponent {
  private fb = inject(FormBuilder);

  pedidoForm = this.fb.group({
    cliente: ['', Validators.required],
    productos: this.fb.array([
      this.crearProducto()
    ])
  });

  get productosArray() {
    return this.pedidoForm.get('productos') as FormArray;
  }

  crearProducto() {
    return this.fb.group({
      nombre: ['', Validators.required],
      cantidad: [1, [Validators.required, Validators.min(1)]],
      precio: [0, Validators.min(0)]
    });
  }

  agregarProducto(): void {
    this.productosArray.push(this.crearProducto());
  }

  eliminarProducto(index: number): void {
    this.productosArray.removeAt(index);
  }
}
```

**Consejo de entrevista**: Prefiere `NonNullableFormBuilder` sobre `FormBuilder` en proyectos con TypeScript estricto, ya que evita que los valores de los controles sean `null` al hacer `reset()` o al leer el valor. Con `fb.group()`, el array `['valor inicial', validadores]` es el patrón más común. El tipado fuerte de formularios en Angular 14+ es una mejora enorme para detectar errores en tiempo de compilación al acceder a `form.controls.campoInexistente`.
