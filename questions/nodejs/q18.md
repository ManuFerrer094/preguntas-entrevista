---
title: ¿Cómo manejar variables de entorno en Node.js?
difficulty: easy

---

Las variables de entorno separan la configuración del código, permitiendo un mismo código en múltiples entornos (desarrollo, staging, producción) con valores diferentes. En Node.js se accede a ellas mediante `process.env`.

## Acceso básico a variables de entorno

```javascript
// Variables de entorno disponibles en process.env
const port = process.env.PORT || 3000;
const nodeEnv = process.env.NODE_ENV || 'development';
const dbUrl = process.env.DATABASE_URL;

if (!dbUrl) {
  console.error('ERROR: DATABASE_URL no está definida');
  process.exit(1);
}

console.log(`Servidor en modo: ${nodeEnv}`);
console.log(`Escuchando en puerto: ${port}`);
```

## Fichero .env con la librería dotenv

```bash
# .env — NUNCA commitear a git
PORT=3000
NODE_ENV=development
DATABASE_URL=postgresql://user:password@localhost:5432/mydb
JWT_SECRET=mi-secreto-super-seguro-cambiar-en-produccion
REDIS_URL=redis://localhost:6379
API_KEY=abc123def456
```

```javascript
// Cargar .env al inicio de la aplicación (antes de cualquier otro import)
require('dotenv').config();
// o con ES Modules:
import 'dotenv/config';

// Ahora process.env tiene las variables del fichero .env
console.log(process.env.DATABASE_URL); // 'postgresql://...'
```

## Validación de variables de entorno con zod

```javascript
// config/env.js — validación al arrancar la app
const { z } = require('zod');

const envSchema = z.object({
  NODE_ENV: z.enum(['development', 'test', 'production']).default('development'),
  PORT: z.string().transform(Number).default('3000'),
  DATABASE_URL: z.string().url('DATABASE_URL debe ser una URL válida'),
  JWT_SECRET: z.string().min(32, 'JWT_SECRET debe tener al menos 32 caracteres'),
  JWT_EXPIRES_IN: z.string().default('7d'),
  REDIS_URL: z.string().url().optional(),
  LOG_LEVEL: z.enum(['error', 'warn', 'info', 'debug']).default('info'),
});

const result = envSchema.safeParse(process.env);

if (!result.success) {
  console.error('❌ Variables de entorno inválidas:');
  result.error.issues.forEach(issue => {
    console.error(`  - ${issue.path.join('.')}: ${issue.message}`);
  });
  process.exit(1);
}

module.exports = result.data;
```

```javascript
// Uso en otros módulos — importar la config validada, no process.env directamente
const config = require('./config/env');

const app = express();
app.listen(config.PORT, () => {
  console.log(`Servidor en puerto ${config.PORT} (${config.NODE_ENV})`);
});
```

## Múltiples ficheros .env por entorno

```
.env                  # Valores por defecto (puede commitarse)
.env.local            # Sobreescrituras locales (no commitear)
.env.development      # Entorno de desarrollo
.env.test             # Entorno de test
.env.production       # Variables de producción (no commitear)
```

```javascript
// Cargar el fichero correcto según el entorno
require('dotenv').config({
  path: `.env.${process.env.NODE_ENV || 'development'}`
});

// dotenv-flow gestiona esto automáticamente
require('dotenv-flow').config();
```

## Buenas prácticas de seguridad

```javascript
// ✅ Siempre añadir .env al .gitignore
// .gitignore:
// .env
// .env.local
// .env.*.local

// ✅ Proporcionar un .env.example sin valores reales
// DATABASE_URL=postgresql://user:password@host:5432/dbname
// JWT_SECRET=change-me-in-production-min-32-chars
// PORT=3000

// ❌ Nunca loggear process.env completo (expone secretos)
console.log(process.env); // ❌ Muestra todos los secretos

// ✅ Solo loggear variables no sensibles
console.log({ port: config.PORT, env: config.NODE_ENV }); // ✅

// ✅ Rotar secretos periódicamente
// ✅ Usar valores diferentes en cada entorno
// ✅ Usar servicios como Vault, AWS Secrets Manager en producción
```

**Consejo de entrevista**: Valida las variables de entorno al arrancar la aplicación y falla rápido si faltan. Nunca hardcodees secretos ni los loggees. En producción, usa gestores de secretos (AWS Secrets Manager, HashiCorp Vault) en lugar de ficheros .env. Proporcionar un `.env.example` en el repositorio facilita el onboarding del equipo.
