**Prisma** es un ORM moderno para Node.js y TypeScript que proporciona un cliente de base de datos type-safe, migraciones automáticas y una interfaz intuitiva. Soporta PostgreSQL, MySQL, SQLite, SQL Server y MongoDB.

## Instalación y configuración inicial

```bash
npm install prisma @prisma/client
npx prisma init  # Crea prisma/schema.prisma y .env
```

```prisma
// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// Modelos
model Usuario {
  id          Int       @id @default(autoincrement())
  email       String    @unique
  nombre      String
  password    String
  rol         Rol       @default(USUARIO)
  activo      Boolean   @default(true)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  // Relaciones
  pedidos     Pedido[]
  perfil      Perfil?

  @@index([email])
  @@map("usuarios")  // Nombre real de la tabla en la BD
}

model Perfil {
  id        Int     @id @default(autoincrement())
  bio       String?
  avatar    String?
  usuarioId Int     @unique
  usuario   Usuario @relation(fields: [usuarioId], references: [id])

  @@map("perfiles")
}

model Pedido {
  id        Int          @id @default(autoincrement())
  total     Decimal      @db.Decimal(10, 2)
  estado    EstadoPedido @default(PENDIENTE)
  items     Json
  createdAt DateTime     @default(now())

  usuarioId Int
  usuario   Usuario @relation(fields: [usuarioId], references: [id])

  @@map("pedidos")
}

enum Rol {
  ADMIN
  USUARIO
  MODERADOR
}

enum EstadoPedido {
  PENDIENTE
  PROCESANDO
  ENVIADO
  ENTREGADO
  CANCELADO
}
```

## Migraciones

```bash
# Crear y aplicar migración
npx prisma migrate dev --name "crear-tabla-usuarios"

# En producción (aplica migraciones pendientes)
npx prisma migrate deploy

# Ver estado de migraciones
npx prisma migrate status

# Generar el cliente Prisma (después de cambios en el schema)
npx prisma generate

# Abrir Prisma Studio (GUI para inspeccionar datos)
npx prisma studio
```

## Operaciones CRUD con Prisma Client

```typescript
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient({
  log: process.env.NODE_ENV === 'development' ? ['query', 'info', 'warn', 'error'] : ['error'],
});

// Singleton para evitar múltiples instancias en desarrollo
export { prisma };

// CREATE
const nuevoUsuario = await prisma.usuario.create({
  data: {
    email: 'ana@ejemplo.com',
    nombre: 'Ana García',
    password: await hashPassword('contraseña123'),
    // Crear relación anidada (nested create)
    perfil: {
      create: { bio: 'Desarrolladora web', avatar: '/avatars/ana.jpg' }
    }
  },
  include: { perfil: true }  // Incluir relación en el resultado
});

// READ - findUnique
const usuario = await prisma.usuario.findUnique({
  where: { id: 1 },
  include: {
    perfil: true,
    pedidos: {
      where: { estado: 'PENDIENTE' },
      orderBy: { createdAt: 'desc' },
      take: 5
    }
  }
});

// READ - findMany con filtros
const usuariosActivos = await prisma.usuario.findMany({
  where: {
    activo: true,
    rol: { in: ['ADMIN', 'MODERADOR'] },
    createdAt: {
      gte: new Date('2024-01-01')
    }
  },
  select: {
    id: true,
    nombre: true,
    email: true,
    // password: false → No se incluye (select explícito)
    _count: { select: { pedidos: true } }  // Contar pedidos
  },
  orderBy: { nombre: 'asc' },
  skip: 0,
  take: 20,  // Paginación
});

// UPDATE
const actualizado = await prisma.usuario.update({
  where: { id: 1 },
  data: {
    nombre: 'Ana López',
    updatedAt: new Date()
  }
});

// UPDATE muchos
await prisma.usuario.updateMany({
  where: { activo: false, createdAt: { lt: new Date('2023-01-01') } },
  data: { activo: false }
});

// DELETE
await prisma.usuario.delete({ where: { id: 1 } });

// UPSERT (crear si no existe, actualizar si existe)
const usuario = await prisma.usuario.upsert({
  where: { email: 'ana@ejemplo.com' },
  create: { email: 'ana@ejemplo.com', nombre: 'Ana', password: hash },
  update: { nombre: 'Ana García' },
});
```

## Transacciones

```typescript
// Transacción interactiva (puede fallar y hacer rollback automático)
const resultado = await prisma.$transaction(async (tx) => {
  const pedido = await tx.pedido.create({
    data: {
      usuarioId: 1,
      total: 99.99,
      estado: 'PENDIENTE',
      items: [{ nombre: 'Laptop', precio: 99.99, cantidad: 1 }]
    }
  });

  await tx.inventario.update({
    where: { productoId: 123 },
    data: { stock: { decrement: 1 } }  // Reducir stock
  });

  // Si esto falla, todo el bloque hace rollback
  if (pedido.total > 1000) {
    await tx.alerta.create({ data: { tipo: 'compra-alta', pedidoId: pedido.id } });
  }

  return pedido;
});
```

## Middleware de Prisma (logging, soft delete, etc.)

```typescript
// Soft delete middleware
prisma.$use(async (params, next) => {
  // Interceptar operaciones de delete
  if (params.action === 'delete' && params.model === 'Usuario') {
    params.action = 'update';
    params.args.data = { deletedAt: new Date() };
  }

  // Filtrar registros borrados en queries
  if (['findUnique', 'findFirst', 'findMany'].includes(params.action)) {
    if (!params.args.where?.deletedAt) {
      params.args.where = { ...params.args.where, deletedAt: null };
    }
  }

  return next(params);
});
```

**Consejo de entrevista**: Prisma destaca sobre otros ORMs (TypeORM, Sequelize) por el **tipo-safety completo**: el cliente Prisma se genera a partir del schema, por lo que los tipos de retorno son exactamente los campos que seleccionas. Si usas `select: { nombre: true }`, el tipo retornado solo tiene `{ nombre: string }`. Menciona también el problema N+1 y cómo evitarlo con `include` y el batching automático de Prisma. La alternativa más ligera es Drizzle ORM, que también tiene excelente soporte TypeScript.
