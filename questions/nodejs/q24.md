**GraphQL** es un lenguaje de consultas para APIs desarrollado por Meta. A diferencia de REST, el cliente especifica exactamente qué datos necesita, eliminando el over-fetching (recibir más datos de los necesarios) y el under-fetching (necesitar múltiples requests).

## Conceptos fundamentales

```graphql
# Schema: define la estructura de los datos y las operaciones disponibles
type Usuario {
  id: ID!           # ! = non-nullable (obligatorio)
  nombre: String!
  email: String!
  edad: Int
  pedidos: [Pedido!]!  # Array de Pedidos (puede estar vacío pero no ser null)
}

type Pedido {
  id: ID!
  total: Float!
  estado: EstadoPedido!
  usuario: Usuario!
}

enum EstadoPedido {
  PENDIENTE
  PROCESANDO
  ENVIADO
  ENTREGADO
}

# Query: operaciones de lectura
type Query {
  usuario(id: ID!): Usuario
  usuarios: [Usuario!]!
  pedido(id: ID!): Pedido
}

# Mutation: operaciones de escritura
type Mutation {
  crearUsuario(nombre: String!, email: String!): Usuario!
  actualizarUsuario(id: ID!, nombre: String): Usuario
  eliminarUsuario(id: ID!): Boolean!
}

# Subscription: tiempo real con WebSockets
type Subscription {
  pedidoActualizado(usuarioId: ID!): Pedido!
}
```

## Implementación con Apollo Server y Express

```javascript
// npm install @apollo/server graphql express
import { ApolloServer } from '@apollo/server';
import { expressMiddleware } from '@apollo/server/express4';
import express from 'express';
import { json } from 'body-parser';

// Schema en formato SDL (Schema Definition Language)
const typeDefs = `#graphql
  type Libro {
    id: ID!
    titulo: String!
    autor: String!
    año: Int
    precio: Float
  }

  type Query {
    libros: [Libro!]!
    libro(id: ID!): Libro
  }

  type Mutation {
    crearLibro(titulo: String!, autor: String!, año: Int, precio: Float): Libro!
    eliminarLibro(id: ID!): Boolean!
  }
`;

// Datos de ejemplo (reemplazar con DB real)
let libros = [
  { id: '1', titulo: 'Don Quijote', autor: 'Cervantes', año: 1605, precio: 12.99 },
  { id: '2', titulo: 'Cien Años de Soledad', autor: 'García Márquez', año: 1967, precio: 14.99 }
];

// Resolvers: implementación de cada campo/operación
const resolvers = {
  Query: {
    libros: () => libros,
    libro: (_, { id }) => libros.find(l => l.id === id) ?? null
  },

  Mutation: {
    crearLibro: (_, args) => {
      const libro = { id: String(Date.now()), ...args };
      libros.push(libro);
      return libro;
    },

    eliminarLibro: (_, { id }) => {
      const indice = libros.findIndex(l => l.id === id);
      if (indice === -1) return false;
      libros.splice(indice, 1);
      return true;
    }
  }
};

// Configurar Apollo Server
const server = new ApolloServer({ typeDefs, resolvers });
await server.start();

const app = express();
app.use('/graphql', json(), expressMiddleware(server));

app.listen(4000, () => {
  console.log('🚀 Server en http://localhost:4000/graphql');
});
```

## Consultas desde el cliente

```graphql
# El cliente pide EXACTAMENTE los campos que necesita
query ObtenerLibros {
  libros {
    id
    titulo
    autor
    # No pedimos `año` ni `precio` → no se envían en la respuesta
  }
}

# Con variables
query ObtenerLibro($id: ID!) {
  libro(id: $id) {
    titulo
    autor
    precio
  }
}

# Mutation
mutation CrearLibro {
  crearLibro(titulo: "Clean Code", autor: "Robert Martin", precio: 29.99) {
    id
    titulo
  }
}
```

## Context y autenticación

```javascript
const server = new ApolloServer({
  typeDefs,
  resolvers,
});

app.use('/graphql', json(), expressMiddleware(server, {
  // Context: se ejecuta en cada request, disponible en todos los resolvers
  context: async ({ req }) => {
    const token = req.headers.authorization?.replace('Bearer ', '');

    let usuario = null;
    if (token) {
      try {
        usuario = await verificarToken(token);
      } catch {
        // Token inválido: usuario null → resolver puede decidir qué hacer
      }
    }

    return { usuario, db };  // Disponible en todos los resolvers
  }
}));

// Uso en resolvers
const resolvers = {
  Query: {
    miPerfil: (_, __, { usuario }) => {
      if (!usuario) throw new GraphQLError('No autenticado', {
        extensions: { code: 'UNAUTHENTICATED' }
      });
      return usuario;
    }
  }
};
```

## DataLoader — Solución al problema N+1

```javascript
import DataLoader from 'dataloader';

// Sin DataLoader: consulta N+1 (1 query por cada libro)
const resolvers = {
  Libro: {
    autor: async (libro) => {
      return await db.autores.findById(libro.autorId);  // N queries!
    }
  }
};

// Con DataLoader: batch de queries (1 sola query para todos)
const crearAutorLoader = () => new DataLoader(async (ids) => {
  const autores = await db.autores.findByIds(ids);  // Una sola query con IN (id1, id2, ...)
  return ids.map(id => autores.find(a => a.id === id));
});

// En el context:
context: async ({ req }) => ({
  autorLoader: crearAutorLoader()
})

// En el resolver:
Libro: {
  autor: async (libro, _, { autorLoader }) => {
    return autorLoader.load(libro.autorId);  // Batch automático
  }
}
```

**Consejo de entrevista**: GraphQL resuelve dos problemas de REST: over-fetching (recibir demasiados datos) y under-fetching (necesitar múltiples requests). El problema más común de GraphQL en producción es el **N+1 queries** —cuando resuelves una lista de objetos, cada uno hace una query adicional para sus relaciones. La solución es DataLoader, que agrupa las IDs y hace una sola query con `WHERE id IN (...)`. Menciona también que GraphQL no siempre es mejor que REST: para APIs públicas simples, REST puede ser más apropiado.
