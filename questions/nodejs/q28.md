**Zod** es una librería de validación y parsing de esquemas con primera clase en TypeScript. Permite definir esquemas de datos, validar entradas y obtener tipos TypeScript inferidos automáticamente a partir de los esquemas.

## Instalación

```bash
npm install zod
```

## Schemas básicos

```typescript
import { z } from 'zod';

// Tipos primitivos
const nombreSchema = z.string().min(2).max(100);
const edadSchema = z.number().int().min(0).max(150);
const emailSchema = z.string().email();
const urlSchema = z.string().url();
const activoSchema = z.boolean();

// Validación y parsing
const resultado = emailSchema.safeParse('no-es-email');
if (!resultado.success) {
  console.log(resultado.error.issues);
  // [{ code: 'invalid_string', validation: 'email', message: 'Invalid email' }]
}

const emailValido = emailSchema.parse('ana@ejemplo.com');  // 'ana@ejemplo.com'
// .parse() lanza ZodError si falla
// .safeParse() retorna { success: boolean, data/error }
```

## Schemas de objetos

```typescript
// Definir el schema
const UsuarioSchema = z.object({
  nombre: z.string().min(2, 'Mínimo 2 caracteres').max(100),
  email: z.string().email('Email no válido'),
  edad: z.number().int().min(0).max(150).optional(),
  rol: z.enum(['admin', 'usuario', 'moderador']).default('usuario'),
  password: z.string().min(8, 'Contraseña muy corta')
    .regex(/[A-Z]/, 'Debe tener al menos una mayúscula')
    .regex(/[0-9]/, 'Debe tener al menos un número'),
  fechaNacimiento: z.string().datetime().optional(),
  activo: z.boolean().default(true)
});

// Inferir el tipo TypeScript automáticamente
type Usuario = z.infer<typeof UsuarioSchema>;
/*
type Usuario = {
  nombre: string;
  email: string;
  edad?: number;
  rol: "admin" | "usuario" | "moderador";
  password: string;
  fechaNacimiento?: string;
  activo: boolean;
}
*/

// Crear DTO parcial para actualizaciones
const ActualizarUsuarioSchema = UsuarioSchema
  .omit({ password: true })          // Excluir campos
  .partial()                          // Todos opcionales
  .required({ email: true });         // Pero email obligatorio

type ActualizarUsuario = z.infer<typeof ActualizarUsuarioSchema>;
```

## Middleware de validación para Express

```typescript
// middleware/validar.middleware.ts
import { Request, Response, NextFunction } from 'express';
import { ZodSchema, ZodError } from 'zod';

export function validar(schema: ZodSchema) {
  return (req: Request, res: Response, next: NextFunction) => {
    const resultado = schema.safeParse(req.body);

    if (!resultado.success) {
      const errores = resultado.error.issues.map(issue => ({
        campo: issue.path.join('.'),
        mensaje: issue.message,
        codigo: issue.code
      }));

      return res.status(400).json({
        error: 'Datos de entrada inválidos',
        detalles: errores
      });
    }

    // Reemplazar req.body con los datos parseados y transformados
    req.body = resultado.data;
    next();
  };
}

// Uso en rutas
import express from 'express';
import { validar } from './middleware/validar.middleware';
import { UsuarioSchema, ActualizarUsuarioSchema } from './schemas/usuario.schema';

const router = express.Router();

router.post('/usuarios', validar(UsuarioSchema), async (req, res) => {
  const nuevoUsuario = req.body;  // Tipado y validado ✅
  const usuario = await usuarioService.create(nuevoUsuario);
  res.status(201).json(usuario);
});

router.put('/usuarios/:id', validar(ActualizarUsuarioSchema), async (req, res) => {
  const actualizado = await usuarioService.update(req.params.id, req.body);
  res.json(actualizado);
});
```

## Validación de variables de entorno

```typescript
// config/env.schema.ts
import { z } from 'zod';

const EnvSchema = z.object({
  NODE_ENV: z.enum(['development', 'production', 'test']).default('development'),
  PORT: z.coerce.number().int().min(1).max(65535).default(3000),
  DATABASE_URL: z.string().url(),
  JWT_SECRET: z.string().min(32, 'JWT_SECRET debe tener al menos 32 caracteres'),
  REDIS_URL: z.string().url().optional(),
  MAX_UPLOAD_SIZE_MB: z.coerce.number().positive().default(10),
  ALLOWED_ORIGINS: z.string().transform(s => s.split(',')).default('http://localhost:4200'),
});

// Validar al iniciar la aplicación
const env = EnvSchema.parse(process.env);

// Si falta NODE_ENV, DATABASE_URL o JWT_SECRET → lanza ZodError con detalles
export { env };

// Uso tipado:
import { env } from './config/env.schema';
console.log(env.PORT);    // number (no string) gracias a z.coerce
```

## Schemas avanzados

```typescript
// Refine: validaciones personalizadas
const PedidoSchema = z.object({
  fechaInicio: z.string().datetime(),
  fechaFin: z.string().datetime(),
  importe: z.number().positive(),
  descuento: z.number().min(0).max(100).default(0),
}).refine(
  data => new Date(data.fechaFin) > new Date(data.fechaInicio),
  {
    message: 'La fecha de fin debe ser posterior a la de inicio',
    path: ['fechaFin']  // El error apunta a este campo
  }
).transform(data => ({
  ...data,
  importeConDescuento: data.importe * (1 - data.descuento / 100)
}));

// Union y discriminated union
const NotificacionSchema = z.discriminatedUnion('tipo', [
  z.object({
    tipo: z.literal('email'),
    destinatario: z.string().email(),
    asunto: z.string()
  }),
  z.object({
    tipo: z.literal('sms'),
    telefono: z.string().regex(/^[6-9]\d{8}$/)
  }),
  z.object({
    tipo: z.literal('push'),
    tokenDispositivo: z.string()
  })
]);

// Array con validación
const TagsSchema = z.array(z.string().min(1).max(50)).min(1).max(10);

// Record con validación de claves y valores
const ConfigSchema = z.record(
  z.string(),
  z.union([z.string(), z.number(), z.boolean()])
);
```

**Consejo de entrevista**: Zod se ha convertido en el estándar de validación en el ecosistema TypeScript, reemplazando a Joi y Yup en muchos proyectos. Su ventaja principal sobre otras librerías es que **los tipos TypeScript se infieren del schema** (`z.infer<typeof Schema>`), eliminando la duplicación entre el schema de validación y los tipos TypeScript. Menciona también `z.coerce` para convertir automáticamente strings de `process.env` a números/booleanos —esencial para validar variables de entorno.
