CORS (Cross-Origin Resource Sharing) es un mecanismo de seguridad del navegador que bloquea peticiones HTTP entre orígenes distintos (dominio, protocolo o puerto diferente). El servidor debe indicar explícitamente qué orígenes están permitidos mediante cabeceras HTTP.

## Por qué existe CORS

```
// Sin CORS, el navegador bloquea:
// Origen de la app:  http://mifrontend.com
// Petición a la API: http://miapirest.com/users  ← origen diferente

// El navegador lanza: "Access to fetch at 'http://miapirest.com/users'
// from origin 'http://mifrontend.com' has been blocked by CORS policy"
```

## Configuración básica con el paquete cors

```javascript
const express = require('express');
const cors = require('cors');

const app = express();

// ❌ Permitir TODOS los orígenes (solo para desarrollo)
app.use(cors());

// ✅ Configuración específica para producción
app.use(cors({
  origin: 'https://mifrontend.com',     // Un solo origen permitido
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization'],
  credentials: true,                     // Permitir cookies y credenciales
  maxAge: 86400                          // Cachear preflight 24 horas
}));
```

## Múltiples orígenes permitidos

```javascript
const allowedOrigins = [
  'https://mifrontend.com',
  'https://admin.mifrontend.com',
  process.env.NODE_ENV === 'development' ? 'http://localhost:4200' : null,
  process.env.NODE_ENV === 'development' ? 'http://localhost:3000' : null,
].filter(Boolean);

app.use(cors({
  origin: (origin, callback) => {
    // Permitir peticiones sin origen (Postman, curl, SSR)
    if (!origin) return callback(null, true);

    if (allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error(`Origen ${origin} no permitido por CORS`));
    }
  },
  credentials: true
}));
```

## Configuración manual sin paquete cors

```javascript
app.use((req, res, next) => {
  const origin = req.headers.origin;

  if (allowedOrigins.includes(origin)) {
    res.setHeader('Access-Control-Allow-Origin', origin);
    res.setHeader('Vary', 'Origin'); // Para caché correcta con múltiples orígenes
  }

  res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, PATCH, DELETE, OPTIONS');
  res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
  res.setHeader('Access-Control-Allow-Credentials', 'true');
  res.setHeader('Access-Control-Max-Age', '86400');

  // Responder inmediatamente a peticiones preflight
  if (req.method === 'OPTIONS') {
    return res.sendStatus(204);
  }

  next();
});
```

## CORS por ruta específica

```javascript
const publicCors = cors({ origin: '*' });
const privateCors = cors({
  origin: allowedOrigins,
  credentials: true
});

// API pública: cualquier origen
app.get('/api/public/health', publicCors, (req, res) => {
  res.json({ status: 'ok' });
});

// API privada: solo orígenes autorizados con credenciales
app.use('/api/private', privateCors);
app.get('/api/private/profile', privateCors, authMiddleware, (req, res) => {
  res.json(req.user);
});
```

## Qué son las peticiones Preflight

```javascript
// El navegador envía automáticamente una petición OPTIONS antes de:
// - Métodos no simples (PUT, DELETE, PATCH)
// - Cabeceras personalizadas (Authorization, X-Custom-Header)
// - Content-Type: application/json

// Ejemplo de petición preflight que envía el navegador:
// OPTIONS /api/users HTTP/1.1
// Origin: https://mifrontend.com
// Access-Control-Request-Method: POST
// Access-Control-Request-Headers: Content-Type, Authorization

// El servidor debe responder con las cabeceras apropiadas:
// Access-Control-Allow-Origin: https://mifrontend.com
// Access-Control-Allow-Methods: POST
// Access-Control-Allow-Headers: Content-Type, Authorization

// Con el paquete cors, el preflight se gestiona automáticamente
app.options('*', cors(corsOptions)); // Habilitar preflight para todas las rutas
```

**Consejo de entrevista**: CORS es una política del **navegador**, no del servidor. Las peticiones de servidor a servidor (microservicios, curl, Postman) no se ven afectadas. Activa `credentials: true` solo cuando necesites enviar cookies o cabeceras de autenticación. La cabecera `Vary: Origin` es importante cuando tienes múltiples orígenes permitidos para que los proxies cacheen correctamente.
