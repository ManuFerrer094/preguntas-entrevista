---
title: ¿Qué es el módulo child_process en Node.js?

El módulo `child_process` de Node.js permite crear procesos hijos para ejecutar comandos del sistema operativo, scripts de shell u otros programas desde una aplicación Node.js. Esto es útil para ejecutar tareas que no se pueden hacer en JavaScript puro: convertir imágenes con ImageMagick, ejecutar scripts de Python, comprimir archivos, etc.

El módulo ofrece cuatro métodos principales: `exec`, `execFile`, `spawn` y `fork`.

## exec — ejecutar comandos shell (salida en buffer)

```javascript
const { exec } = require('child_process');
const { promisify } = require('util');

const execAsync = promisify(exec);

// Con promisify para uso con async/await
async function getGitLog() {
  const { stdout, stderr } = await execAsync('git log --oneline -10');
  return stdout.trim().split('\n');
}

// Caso de uso: información del sistema
async function getSystemInfo() {
  const { stdout } = await execAsync('df -h && free -m');
  return stdout;
}
```

⚠️ **NUNCA interpolar variables de usuario en exec** — vulnerabilidad de inyección de comandos:

```javascript
// ❌ PELIGROSO: inyección de comandos (command injection)
exec(`ls ${userInput}`); // Si userInput = "; rm -rf /", ¡desastre!

// ✅ Seguro: usar execFile con argumentos separados
const { execFile } = require('child_process');
execFile('ls', [userInput], callback); // Los argumentos son escapados
```

## spawn — streams de salida (ideal para procesos largos)

```javascript
const { spawn } = require('child_process');

function convertVideo(inputPath, outputPath) {
  return new Promise((resolve, reject) => {
    const ffmpeg = spawn('ffmpeg', [
      '-i', inputPath,
      '-c:v', 'libx264',
      '-preset', 'medium',
      outputPath
    ]);

    let errorOutput = '';

    ffmpeg.stdout.on('data', (data) => {
      console.log(`stdout: ${data}`);
    });

    ffmpeg.stderr.on('data', (data) => {
      // ffmpeg escribe el progreso en stderr
      errorOutput += data.toString();
      const progress = parseProgress(data.toString());
      if (progress) emitProgressUpdate(progress);
    });

    ffmpeg.on('close', (code) => {
      if (code === 0) resolve(outputPath);
      else reject(new Error(`ffmpeg falló: ${errorOutput}`));
    });
  });
}
```

## fork — crear procesos Node.js hijos con comunicación IPC

```javascript
// parent.js
const { fork } = require('child_process');

const child = fork('./child.js');

// Enviar mensaje al hijo
child.send({ type: 'process', data: largeDataset });

// Recibir resultados
child.on('message', (result) => {
  console.log('Resultado:', result);
  child.kill(); // Terminar cuando se haya procesado
});

child.on('exit', (code) => {
  console.log(`Hijo terminó con código ${code}`);
});

// child.js
process.on('message', async (msg) => {
  if (msg.type === 'process') {
    const result = await processData(msg.data); // Operación CPU-intensiva
    process.send({ type: 'result', data: result });
  }
});
```

## Caso de uso: ejecutar script Python

```javascript
const { spawn } = require('child_process');

async function runPythonScript(scriptPath, args = []) {
  return new Promise((resolve, reject) => {
    const python = spawn('python3', [scriptPath, ...args]);

    let output = '';
    let errorOutput = '';

    python.stdout.on('data', (data) => { output += data.toString(); });
    python.stderr.on('data', (data) => { errorOutput += data.toString(); });

    python.on('close', (code) => {
      if (code === 0) {
        resolve(JSON.parse(output));
      } else {
        reject(new Error(`Script Python falló: ${errorOutput}`));
      }
    });
  });
}

// En una ruta de Express
app.post('/analyze', async (req, res, next) => {
  try {
    const result = await runPythonScript('./ml_model.py', [req.body.text]);
    res.json(result);
  } catch (error) {
    next(error);
  }
});
```

**Consejo de entrevista**: La distinción entre `exec` y `spawn` es importante: `exec` guarda toda la salida en buffer (malo para procesos que generan mucha salida), mientras que `spawn` usa streams (mejor para procesos largos). Para procesos Node.js hijos con comunicación bidireccional, `fork` es la mejor opción. Mencionar `Worker Threads` como alternativa moderna cuando se trata de código JavaScript CPU-intensivo.
