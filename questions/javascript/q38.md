Optimizar JavaScript no empieza por microtrucos, sino por medir y atacar cuellos de botella reales. Las mejoras útiles suelen venir de reducir trabajo innecesario, no de perseguir supersticiones.

## Estrategias frecuentes
- minimizar manipulaciones del DOM y agrupar cambios;
- evitar recalcular datos costosos si no han cambiado;
- elegir estructuras de datos adecuadas;
- reducir trabajo dentro de bucles críticos;
- evitar renders, reflows o repaints innecesarios;
- mover tareas pesadas fuera del hilo principal cuando procede.

## Ejemplo de mejora sencilla
```js
// Peor: toca el DOM en cada iteración
for (const item of items) {
  lista.innerHTML += `<li>${item}</li>`;
}

// Mejor: construye primero y actualiza una vez
const html = items.map((item) => `<li>${item}</li>`).join('');
lista.innerHTML = html;
```

## Qué hacer en serio
- perfila con herramientas reales;
- encuentra el punto lento;
- aplica un cambio medible;
- vuelve a medir.

## Para interiorizarlo
La optimización buena no nace de adivinar. Nace de observar dónde se gasta tiempo o memoria y atacar exactamente ese punto.
