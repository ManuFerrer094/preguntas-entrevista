title: ¿Qué es el lexical scoping y en qué se diferencia del dynamic scoping?

JavaScript usa lexical scoping: la visibilidad de una variable se determina por dónde está escrita la función en el código fuente, no por quién la llama en tiempo de ejecución.

## Ejemplo de código
```js
const nombre = 'global';

function exterior() {
  const nombre = 'local';

  function interior() {
    console.log(nombre);
  }

  return interior;
}

const fn = exterior();
fn(); // 'local'
```

## Qué demuestra este ejemplo
`interior` imprime `local` porque fue creada dentro de `exterior`. No importa desde dónde se invoque después: su entorno léxico ya quedó fijado.

## Y qué sería dynamic scoping
En un lenguaje con dynamic scoping, la variable se resolvería según la cadena de llamadas en runtime. JavaScript no funciona así.

## Por qué importa
Esto explica closures, módulos, callbacks y buena parte del comportamiento de las funciones. Si entiendes el scope léxico, muchas “sorpresas” dejan de ser sorpresas.

## Para interiorizarlo
Cuando dudes de qué variable ve una función, no mires primero quién la llama. Mira dónde fue definida.
