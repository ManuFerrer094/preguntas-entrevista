El event loop es el mecanismo con el que el entorno de ejecución coordina tareas asíncronas sin bloquear el hilo principal. JavaScript ejecuta una cosa cada vez en la call stack, pero el entorno puede preparar callbacks, timers, eventos o respuestas de red para procesarlos más tarde.

## Modelo mental útil
Piensa en estas piezas:
- call stack: lo que se está ejecutando ahora;
- Web APIs o APIs del entorno: timers, red, eventos, etc.;
- task queue y microtask queue: colas donde quedan pendientes callbacks;
- event loop: decide cuándo pasar trabajo pendiente a la stack.

## Ejemplo de código
```js
console.log('A');

setTimeout(() => console.log('B'), 0);
Promise.resolve().then(() => console.log('C'));

console.log('D');
```

La salida es:
```js
A
D
C
B
```

## Qué está pasando
- `A` y `D` salen primero porque son síncronos.
- La promesa resuelta agenda una microtarea.
- `setTimeout` agenda una tarea normal.
- El event loop vacía antes las microtareas y luego atiende la cola de tareas.

## Para interiorizarlo
No memorices solo "JavaScript es asíncrono". Entiende que el lenguaje ejecuta de forma secuencial, mientras el entorno le va entregando trabajo pendiente en el momento adecuado.
