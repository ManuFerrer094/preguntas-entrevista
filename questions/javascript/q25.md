---
title: ¿Cuál es la diferencia entre arrow functions y funciones normales en JavaScript?
difficulty: easy
tags: [javascript, arrow-functions, this, functions, es6]
---

Las **arrow functions** (funciones flecha, ES6) tienen diferencias fundamentales respecto a las funciones normales en cuanto a `this`, `arguments`, constructores y uso como métodos.

## Sintaxis

```javascript
// Función normal (declaración)
function sumar(a, b) {
  return a + b;
}

// Función normal (expresión)
const sumar = function(a, b) {
  return a + b;
};

// Arrow function
const sumar = (a, b) => a + b;

// Variantes de sintaxis de arrow function
const doble = x => x * 2;           // Un parámetro: sin paréntesis
const saludar = () => 'Hola!';      // Sin parámetros: paréntesis obligatorio
const sumar = (a, b) => a + b;      // Múltiples parámetros

// Cuerpo con múltiples líneas: necesita llaves y return
const calcular = (a, b) => {
  const suma = a + b;
  return suma * 2;
};

// Retornar objeto literal: envolver en paréntesis
const crearUsuario = (nombre, edad) => ({ nombre, edad });
// Sin paréntesis, las llaves se interpretarían como bloque de código
```

## La diferencia más importante: `this`

```javascript
// Función normal: `this` depende de CÓMO se llama la función
const persona = {
  nombre: 'Ana',

  saludarNormal: function() {
    console.log(`Hola, soy ${this.nombre}`);  // this = persona ✅
  },

  saludarArrow: () => {
    // Arrow function NO tiene su propio `this`
    // Hereda `this` del scope léxico donde fue definida (aquí, el módulo/global)
    console.log(`Hola, soy ${this.nombre}`);  // this = undefined (strict) o window ❌
  }
};

persona.saludarNormal();  // "Hola, soy Ana"
persona.saludarArrow();   // "Hola, soy undefined"

// El problema clásico: callbacks en métodos
const temporizador = {
  nombre: 'Timer',
  iniciar: function() {
    // ❌ Función normal: `this` cambia dentro del callback
    setTimeout(function() {
      console.log(this.nombre);  // undefined (this es window/undefined en strict)
    }, 1000);

    // ✅ Arrow function: hereda `this` del método `iniciar`
    setTimeout(() => {
      console.log(this.nombre);  // "Timer" ✅
    }, 1000);
  }
};
```

## Arrow functions en clases

```javascript
class Contador {
  constructor() {
    this.cuenta = 0;
  }

  // ❌ Método normal: `this` puede perderse al pasar como callback
  incrementarNormal() {
    this.cuenta++;
  }

  // ✅ Arrow function como propiedad: `this` siempre es la instancia
  incrementarArrow = () => {
    this.cuenta++;
  };
}

const contador = new Contador();

// Arrow function: mantiene el binding correcto
document.getElementById('btn').addEventListener('click', contador.incrementarArrow);  // ✅

// Función normal: pierde el binding
document.getElementById('btn').addEventListener('click', contador.incrementarNormal);  // ❌
document.getElementById('btn').addEventListener('click',
  contador.incrementarNormal.bind(contador)  // ✅ Con bind
);
```

## `arguments` — No disponible en arrow functions

```javascript
// Función normal: tiene objeto `arguments`
function sumarTodos() {
  let total = 0;
  for (let arg of arguments) {
    total += arg;
  }
  return total;
}
sumarTodos(1, 2, 3, 4);  // 10

// Arrow function: NO tiene `arguments`
const sumarTodosArrow = () => {
  // arguments → ReferenceError o hereda el de la función contenedora
};

// Solución moderna: usar rest params (también en funciones normales)
const sumarTodosRest = (...numeros) => numeros.reduce((acc, n) => acc + n, 0);
sumarTodosRest(1, 2, 3, 4);  // 10
```

## No son constructores

```javascript
// Función normal: puede usarse como constructor
function Persona(nombre) {
  this.nombre = nombre;
}
const ana = new Persona('Ana');  // ✅

// Arrow function: NO puede usarse como constructor
const PersonaArrow = (nombre) => { this.nombre = nombre; };
const carlos = new PersonaArrow('Carlos');  // ❌ TypeError: PersonaArrow is not a constructor
```

## No tienen `prototype`

```javascript
function funcionNormal() {}
console.log(funcionNormal.prototype);  // { constructor: f }

const funcionFlecha = () => {};
console.log(funcionFlecha.prototype);  // undefined
```

## Comparativa completa

| Característica          | Función normal              | Arrow function              |
|-------------------------|-----------------------------|-----------------------------|
| `this`                  | Dinámico (depende de cómo se llama) | Léxico (del scope externo) |
| `arguments`             | ✅ Disponible               | ❌ No disponible            |
| Constructor (`new`)     | ✅ Puede usarse            | ❌ No puede usarse          |
| `prototype`             | ✅ Tiene                   | ❌ No tiene                 |
| `super`                 | ✅ Disponible               | ❌ No disponible            |
| Método de objeto        | ✅ Recomendado              | ❌ Evitar                   |
| Callback                | ⚠️ Cuidado con `this`      | ✅ Recomendado              |
| Función generadora      | ✅ `function*`             | ❌ No puede ser generadora   |

**Consejo de entrevista**: La diferencia más importante es el comportamiento de `this`. Las arrow functions son ideales para callbacks (setTimeout, addEventListener, Array.map/filter) porque heredan el `this` del contexto donde se definen. Para métodos de objeto que necesitan acceder a `this`, usa funciones normales. El caso más común de bug en JavaScript es pasar un método de clase como callback sin hacer `bind` o usar arrow function.
