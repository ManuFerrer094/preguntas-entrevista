La delegación de eventos consiste en colocar un único listener en un elemento ancestro para gestionar eventos que ocurren en sus descendientes. Funciona porque muchos eventos hacen bubbling y suben por el árbol DOM.

## Ejemplo de código
```js
const lista = document.querySelector('#tareas');

lista.addEventListener('click', (event) => {
  const boton = event.target.closest('button[data-id]');
  if (!boton) return;

  console.log('Eliminar tarea', boton.dataset.id);
});
```

## Por qué es útil
- evita registrar cientos de listeners en elementos hijos;
- sigue funcionando aunque añadas elementos dinámicamente;
- reduce memoria y simplifica mantenimiento.

## Qué hay que entender bien
No se trata solo de “escuchar arriba”. Se trata de usar el bubbling a tu favor y filtrar el objetivo real del evento con `event.target` o `closest()`.

## Matices importantes
- No todos los eventos hacen bubbling del mismo modo.
- Si la estructura interna del DOM cambia mucho, conviene que tus selectores para delegación sean robustos.
- A veces necesitas `stopPropagation()` con cuidado, pero usarlo demasiado suele ocultar problemas de diseño.

## Para interiorizarlo
Cada vez que vayas a crear muchos listeners parecidos, pregúntate si en realidad deberías delegar el evento desde un ancestro común.
