Cuando te preguntan por la cadena de prototipos no basta con decir "JavaScript hereda". Lo que suele medir la entrevista es si entiendes el mecanismo real de resolución de propiedades y métodos, porque de ahí salen bugs de `this`, sombreado de propiedades, APIs compartidas y decisiones de memoria.

## Qué evalúa el entrevistador
- Si sabes que la búsqueda empieza en el propio objeto y solo sube al prototipo cuando la propiedad no existe localmente.
- Si puedes nombrar el final de la cadena: `Object.prototype`, y después `null`.
- Si distingues entre la propiedad `prototype` de una función constructora y el enlace interno `[[Prototype]]` de una instancia, que solemos ver como `__proto__` o con `Object.getPrototypeOf()`.

## Respuesta sólida
- Cada objeto tiene un enlace interno a otro objeto que actúa como su prototipo.
- Cuando haces `obj.prop`, JavaScript primero mira si `prop` existe en `obj`.
- Si no existe, busca en el prototipo de `obj`; si tampoco está, sigue subiendo por la cadena.
- El proceso termina cuando llega a `Object.prototype`. Si ahí tampoco aparece, el resultado es `undefined`.
- Esto permite compartir comportamiento sin copiar métodos en cada instancia.

## Ejemplo de código
Este ejemplo sirve para explicar el recorrido exacto de la búsqueda:

```js
function Cuenta(titular) {
  this.titular = titular;
}

Cuenta.prototype.getResumen = function () {
  return `Cuenta de ${this.titular}`;
};

const cuenta = new Cuenta('Ana');

console.log(cuenta.titular); // propia de la instancia
console.log(cuenta.getResumen()); // encontrada en Cuenta.prototype
console.log(cuenta.toString()); // heredada desde Object.prototype
```

Si `getResumen` no está en `cuenta`, JavaScript la localiza en `Cuenta.prototype`. Si después llamas a `toString`, sigue subiendo hasta `Object.prototype`.

## Matiz importante
- `__proto__` existe en muchos entornos, pero en entrevista es mejor hablar de `[[Prototype]]` o de `Object.getPrototypeOf()`, porque `__proto__` es una forma histórica de inspección y no el concepto central.
- Las `class` de ES2015 no eliminan este modelo. Solo ponen una sintaxis más familiar encima de la misma cadena de prototipos.

## Error común
Confundir `MiFuncion.prototype` con el prototipo de la propia función. `prototype` es el objeto que usarán las instancias creadas con `new MiFuncion()`. El prototipo de la función en sí se consulta aparte, porque las funciones también son objetos.

## Frase corta de entrevista
> La cadena de prototipos es el algoritmo de búsqueda de propiedades de JavaScript: primero el objeto, luego su prototipo, luego el prototipo del prototipo, hasta `Object.prototype`.
