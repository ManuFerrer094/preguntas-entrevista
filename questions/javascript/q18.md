`undefined` representa la ausencia de un valor asignado. Aparece en varias situaciones normales del lenguaje y no siempre significa que haya un error, aunque sí suele ser una señal de que conviene revisar el flujo.

## Ejemplo de código
```js
let nombre;

const usuario = {};

function noDevuelveNada() {}

console.log(nombre); // undefined
console.log(usuario.email); // undefined
console.log(noDevuelveNada()); // undefined
```

## Cuándo aparece
- una variable declarada aún no tiene valor;
- una propiedad no existe;
- una función no hace `return`;
- un parámetro no se pasó.

## Matices importantes
- No confundas `undefined` con `null`: `null` suele usarse cuando quieres expresar una ausencia intencional.
- Muchas APIs usan `undefined` para decir "no se proporcionó nada".

## Para interiorizarlo
Si ves `undefined`, pregúntate si la ausencia es esperada por diseño o si es el rastro de un dato que nunca llegó donde debía.
