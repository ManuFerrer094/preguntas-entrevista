`Symbol` es un tipo primitivo único e inmutable introducido en ES6. Cada símbolo creado es único, lo que lo hace ideal como clave de propiedad para evitar colisiones de nombres, especialmente en librerías y APIs.

## Creación y unicidad

```javascript
const sym1 = Symbol('descripción');
const sym2 = Symbol('descripción');

console.log(sym1 === sym2);   // false — siempre únicos
console.log(typeof sym1);     // 'symbol'
console.log(sym1.toString()); // 'Symbol(descripción)'
console.log(sym1.description);// 'descripción'

// No se puede usar como constructor
// new Symbol() → TypeError
```

## Symbols como claves de objeto

```javascript
const ID = Symbol('id');
const user = {
  name: 'Ana',
  [ID]: 12345   // Clave símbolo con computed property syntax
};

console.log(user[ID]);     // 12345
console.log(user.name);    // 'Ana'

// Las claves Symbol NO aparecen en iteraciones normales
console.log(Object.keys(user));            // ['name']
console.log(Object.getOwnPropertyNames(user)); // ['name']
console.log(JSON.stringify(user));         // {"name":"Ana"}

// Para acceder a las claves Symbol:
console.log(Object.getOwnPropertySymbols(user)); // [Symbol(id)]
console.log(Reflect.ownKeys(user)); // ['name', Symbol(id)]
```

## Symbol.for() — Registro global de símbolos

```javascript
// Symbol.for() busca o crea en el registro global
const a = Symbol.for('shared');
const b = Symbol.for('shared');

console.log(a === b); // true — mismo símbolo del registro global

// Útil para compartir símbolos entre módulos
const TOKEN = Symbol.for('app.auth.token');
```

## Well-known Symbols (protocolos internos)

JavaScript usa símbolos predefinidos para personalizar comportamientos del lenguaje:

```javascript
// Symbol.iterator — hace un objeto iterable
class Range {
  constructor(start, end) {
    this.start = start;
    this.end = end;
  }

  [Symbol.iterator]() {
    let current = this.start;
    const end = this.end;
    return {
      next() {
        if (current <= end) {
          return { value: current++, done: false };
        }
        return { value: undefined, done: true };
      }
    };
  }
}

const range = new Range(1, 5);
console.log([...range]); // [1, 2, 3, 4, 5]
for (const n of range) console.log(n); // 1, 2, 3, 4, 5

// Symbol.toPrimitive — controla la conversión de tipo
class Money {
  constructor(amount, currency) {
    this.amount = amount;
    this.currency = currency;
  }

  [Symbol.toPrimitive](hint) {
    if (hint === 'number') return this.amount;
    if (hint === 'string') return `${this.amount} ${this.currency}`;
    return this.amount; // 'default'
  }
}

const price = new Money(100, 'EUR');
console.log(`Precio: ${price}`); // 'Precio: 100 EUR' (string hint)
console.log(price * 2);          // 200 (number hint)
console.log(price + 50);         // 150 (default hint)

// Symbol.hasInstance — personaliza instanceof
class EvenNumber {
  static [Symbol.hasInstance](num) {
    return typeof num === 'number' && num % 2 === 0;
  }
}

console.log(4 instanceof EvenNumber);  // true
console.log(5 instanceof EvenNumber);  // false
```

## Caso de uso: evitar colisiones en librerías

```javascript
// Librería externa
const INTERNAL_STATE = Symbol('internalState');

class EventBus {
  constructor() {
    this[INTERNAL_STATE] = { listeners: new Map() };
  }

  on(event, callback) {
    const listeners = this[INTERNAL_STATE].listeners;
    if (!listeners.has(event)) listeners.set(event, []);
    listeners.get(event).push(callback);
  }

  emit(event, data) {
    const callbacks = this[INTERNAL_STATE].listeners.get(event) ?? [];
    callbacks.forEach(cb => cb(data));
  }
}

// El usuario no puede acceder accidentalmente a INTERNAL_STATE
// porque el Symbol no está expuesto
```

**Consejo de entrevista**: Los Symbol son especialmente útiles para dos cosas: crear identificadores únicos que no colisionen con otras propiedades, y personalizar el comportamiento del lenguaje mediante well-known symbols (`Symbol.iterator`, `Symbol.toPrimitive`, `Symbol.hasInstance`). Mencionar que los Symbol no se serializan con `JSON.stringify` también es un punto importante en entrevistas.
