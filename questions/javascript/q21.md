El scope define desde qué partes del programa puede verse una variable. Es una regla de visibilidad: no dice cuánto vive un valor en memoria, sino desde dónde se puede acceder a él.

## Idea clave
En JavaScript conviven varios ámbitos importantes:
- scope global: visible desde casi cualquier parte del programa;
- scope de función: visible dentro de la función donde se declara;
- scope de bloque: visible dentro de `{}` cuando usas `let` o `const`.

## Ejemplo de código
```js
const global = 'estoy fuera';

function ejemplo() {
  const local = 'solo dentro de la función';

  if (true) {
    const deBloque = 'solo dentro del if';
    console.log(global);   // ok
    console.log(local);    // ok
    console.log(deBloque); // ok
  }

  // console.log(deBloque); // ReferenceError
}

// console.log(local); // ReferenceError
```

## Qué conviene entender de verdad
- Una variable global es fácil de usar, pero también fácil de acoplar, pisar o mutar desde demasiados sitios.
- El scope local ayuda a encapsular lógica y reducir superficie mental.
- El scope de bloque evita fugas de variables temporales que antes eran comunes con `var`.

## Error frecuente
Confundir scope con tiempo de vida. Una variable puede salir del scope accesible pero seguir viva si una closure todavía la referencia.

## Para interiorizarlo
Cuando declares una variable, pregúntate: "¿cuál es el ámbito más pequeño donde realmente necesito que exista?". Casi siempre esa respuesta mejora el diseño.
