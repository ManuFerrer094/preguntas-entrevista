En JavaScript moderno hay siete tipos primitivos:
- `string`
- `number`
- `bigint`
- `boolean`
- `undefined`
- `symbol`
- `null`

Además está el tipo `object`, que agrupa arrays, funciones, fechas, mapas y casi cualquier estructura compleja.

## Idea clave
Los primitivos se manejan por valor; los objetos se manejan por referencia. Esa diferencia explica muchísimos bugs de comparación, copia y mutación.

## Ejemplo de código
```js
const texto = 'hola';
const numero = 42;
const enteroGrande = 42n;
const activo = true;
const vacio = null;
let sinValor;
const id = Symbol('id');
const usuario = { nombre: 'Ana' };

console.log(typeof texto); // 'string'
console.log(typeof numero); // 'number'
console.log(typeof enteroGrande); // 'bigint'
console.log(typeof activo); // 'boolean'
console.log(typeof sinValor); // 'undefined'
console.log(typeof id); // 'symbol'
console.log(typeof usuario); // 'object'
console.log(typeof vacio); // 'object' (quirk histórico)
```

## Matices importantes
- `null` representa una ausencia de valor intencional.
- `undefined` suele significar "todavía no hay valor" o "no existe".
- `typeof null === 'object'` es un comportamiento histórico del lenguaje, no algo lógico.
- Las funciones son objetos invocables, por eso `typeof fn === 'function'`.

## Para interiorizarlo
Antes de aprender frameworks, acostúmbrate a pensar: "¿estoy trabajando con un primitivo o con una referencia a objeto?". Esa pregunta ahorra muchos errores.
