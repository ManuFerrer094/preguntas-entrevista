---
title: ¿Qué es la coerción de tipos en JavaScript?
difficulty: medium
tags: [javascript, type-coercion, implicit-conversion, equality]
---

La coerción de tipos es la conversión automática (implícita) o manual (explícita) que realiza JavaScript cuando una operación involucra valores de tipos diferentes. Entenderla es clave para evitar bugs sutiles.

## Coerción implícita

JavaScript convierte tipos automáticamente en ciertos contextos:

```javascript
// Coerción a string (operador +)
console.log('5' + 3);        // '53' — 3 se convierte a string
console.log('5' + true);     // '5true'
console.log('' + null);      // 'null'

// Coerción a número (otros operadores aritméticos)
console.log('5' - 3);        // 2     — '5' se convierte a número
console.log('5' * '2');      // 10
console.log(true + true);    // 2     — true → 1
console.log(false + 1);      // 1     — false → 0
console.log(null + 1);       // 1     — null → 0
console.log(undefined + 1);  // NaN   — undefined → NaN

// Coerción a boolean (contextos condicionales)
if ('') console.log('vacío');       // No se ejecuta — '' es falsy
if ('hola') console.log('truthy'); // Se ejecuta
if (0) console.log('cero');        // No se ejecuta — 0 es falsy
if ([]) console.log('array');      // Se ejecuta — [] es truthy
```

## Valores falsy y truthy

```javascript
// Valores falsy (se convierten a false)
Boolean(false);      // false
Boolean(0);          // false
Boolean(-0);         // false
Boolean(0n);         // false (BigInt)
Boolean('');         // false
Boolean(null);       // false
Boolean(undefined);  // false
Boolean(NaN);        // false

// Todo lo demás es truthy
Boolean([]);         // true  ← array vacío es truthy
Boolean({});         // true  ← objeto vacío es truthy
Boolean('0');        // true  ← string '0' es truthy
Boolean(-1);         // true
```

## El algoritmo de comparación abstracta (==)

```javascript
// Reglas del operador ==:
// 1. Si mismo tipo → compara directamente (sin coerción)
// 2. null == undefined (solo entre ellos)
console.log(null == undefined);  // true
console.log(null == 0);          // false
console.log(null == false);      // false

// 3. Si un lado es número y el otro string → convierte string a número
console.log('5' == 5);           // true  ('5' → 5)
console.log('' == 0);            // true  ('' → 0)

// 4. Si un lado es boolean → convierte boolean a número
console.log(false == 0);         // true  (false → 0)
console.log(true == 1);          // true  (true → 1)
console.log(true == '1');        // true  (true → 1, '1' → 1)

// 5. Si un lado es objeto → llama a valueOf() o toString()
console.log([] == 0);            // true  ([].toString() → '' → 0)
console.log([] == '');           // true  ([].toString() → '')
console.log([1] == 1);           // true  ([1].toString() → '1' → 1)
```

## Coerción explícita (conversión manual)

```javascript
// A número
Number('42');        // 42
Number('');          // 0
Number('abc');       // NaN
Number(true);        // 1
Number(null);        // 0
Number(undefined);   // NaN
parseInt('42px');    // 42 — parsea hasta el primer carácter no numérico
parseFloat('3.14m'); // 3.14

// A string
String(42);          // '42'
String(null);        // 'null'
String(undefined);   // 'undefined'
(42).toString();     // '42'

// A boolean
Boolean(0);          // false
!!0;                 // false (doble negación: patrón idiomático)
!![];                // true
```

## Trampas comunes

```javascript
// ❌ Errores frecuentes
console.log(typeof NaN === 'number'); // true  — NaN ES de tipo number
console.log(NaN === NaN);             // false — NaN no es igual a sí mismo
console.log(Number.isNaN(NaN));       // true  — forma correcta de verificar

console.log(typeof null === 'object'); // true  — bug histórico de JS

// Suma de arrays y objetos
console.log([] + []);    // ''   — ambos → '', '' + '' = ''
console.log([] + {});    // '[object Object]'
console.log({} + []);    // '[object Object]' (o 0 dependiendo del contexto)
```

**Consejo de entrevista**: Usa siempre `===` (igualdad estricta) en lugar de `==` para evitar coerciones inesperadas. La única excepción justificada es `value == null`, que comprueba tanto `null` como `undefined` de forma concisa. Entender la tabla de verdad de `==` demuestra conocimiento profundo del lenguaje.
