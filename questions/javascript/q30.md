Aunque a veces se usan como si fueran lo mismo, `null` y `undefined` comunican ideas distintas.

## Idea clave
- `undefined` suele significar que no hay valor asignado o que algo no existe.
- `null` suele usarse cuando quieres expresar una ausencia intencional de valor.

## Ejemplo de código
```js
let pendiente;
const usuario = { avatar: null };

console.log(pendiente);      // undefined
console.log(usuario.avatar); // null
```

## Cuándo aparece cada uno
- `undefined`: variable declarada sin asignar, propiedad inexistente, parámetro omitido, función sin `return`.
- `null`: valor puesto explícitamente para indicar “vacío”, “sin dato” o “sin relación”.

## Matices importantes
- `null == undefined` es `true`, pero `null === undefined` es `false`.
- Elegir entre ambos afecta cómo modelas APIs, formularios y validaciones.

## Para interiorizarlo
Cuando diseñes datos, úsalo con intención: `undefined` para “no se proporcionó” y `null` para “sí se proporcionó, y su valor es ausencia”.
