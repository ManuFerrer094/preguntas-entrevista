---
title: ¿Cómo funciona el encadenamiento opcional (?.) y el operador de fusión nula (??) en JavaScript?
difficulty: easy
tags: [javascript, optional-chaining, nullish-coalescing, es2020, null, undefined]
---

El **encadenamiento opcional** (`?.`) y el **operador de fusión nula** (`??`) son características de ES2020 que simplifican el acceso seguro a propiedades anidadas y la gestión de valores nulos/indefinidos.

## Encadenamiento opcional (?.)

```javascript
// El problema sin ?.
const usuario = {
  nombre: 'Ana',
  direccion: {
    ciudad: 'Madrid',
    codigoPostal: '28001'
  }
};

// Sin ?.  necesitas verificaciones manuales (verbose):
const ciudad = usuario && usuario.direccion && usuario.direccion.ciudad;
const pais = usuario && usuario.direccion && usuario.direccion.pais;

// ✅ Con ?. (acceso seguro):
const ciudad2 = usuario?.direccion?.ciudad;           // 'Madrid'
const pais2 = usuario?.direccion?.pais;               // undefined (no lanza error)
const telefono = usuario?.contacto?.telefono;         // undefined (contacto no existe)

// Si cualquier parte es null/undefined, devuelve undefined (no lanza TypeError)
const usuarioNull = null;
console.log(usuarioNull?.nombre);          // undefined (no: TypeError: Cannot read properties of null)
```

## Acceso a arrays y llamadas a métodos

```javascript
const usuario = {
  nombre: 'Ana',
  roles: ['admin', 'editor'],
  obtenerEmail() { return 'ana@ejemplo.com'; }
};

// Acceso a arrays
console.log(usuario?.roles?.[0]);           // 'admin'
console.log(usuario?.permisos?.[0]);        // undefined

// Llamadas a métodos opcionales
console.log(usuario?.obtenerEmail());       // 'ana@ejemplo.com'
console.log(usuario?.obtenerTelefono?.());  // undefined (método no existe)

// También funciona con funciones
function ejecutarSiExiste(fn) {
  return fn?.();  // Solo llama si fn no es null/undefined
}
ejecutarSiExiste(null);          // undefined
ejecutarSiExiste(() => 'Hola');  // 'Hola'
```

## Casos de uso con respuestas de API

```javascript
// Sin necesidad de múltiples verificaciones:
async function obtenerCiudadUsuario(usuarioId) {
  const respuesta = await fetch(`/api/usuarios/${usuarioId}`);
  const datos = await respuesta.json();

  // Acceso seguro a datos profundamente anidados
  return datos?.perfil?.ubicacion?.ciudad ?? 'Ciudad desconocida';
}

// Listas de elementos
const respuesta = {
  data: {
    usuarios: [
      { id: 1, perfil: { nombre: 'Ana' } },
      { id: 2, perfil: null },
      { id: 3 }  // Sin perfil
    ]
  }
};

// Extraer nombres de forma segura
const nombres = respuesta.data?.usuarios?.map(u => u?.perfil?.nombre ?? 'Sin nombre');
// ['Ana', 'Sin nombre', 'Sin nombre']
```

## Operador de fusión nula (??)

```javascript
// ?? devuelve el lado derecho SOLO si el izquierdo es null o undefined
// (no si es 0, '' o false)

const nombre = null ?? 'Usuario anónimo';     // 'Usuario anónimo'
const edad = undefined ?? 18;                 // 18
const activo = false ?? true;                 // false (false NO es null/undefined)
const contador = 0 ?? 10;                     // 0 (0 NO es null/undefined)
const texto = '' ?? 'valor por defecto';      // '' (vacío NO es null/undefined)

// Comparativa con || (OR lógico)
const valorA = 0 || 'predeterminado';         // 'predeterminado' (0 es falsy)
const valorB = 0 ?? 'predeterminado';         // 0 (0 no es null/undefined)

const textoA = '' || 'predeterminado';        // 'predeterminado' ('' es falsy)
const textoB = '' ?? 'predeterminado';        // '' ('' no es null/undefined)

// ✅ Usar ?? cuando 0, '' y false son valores válidos
function configurar({ puerto, debug, prefijo }) {
  return {
    puerto: puerto ?? 3000,     // 0 como puerto sería inválido de todas formas
    debug: debug ?? false,      // false es un valor válido, ?? correcto
    prefijo: prefijo ?? '/api'  // '' podría ser válido → usar ??
  };
}

configurar({ puerto: 0, debug: false, prefijo: '' });
// { puerto: 0, debug: false, prefijo: '' }  ← Correcto con ??

// Con ||:
// { puerto: 3000, debug: false, prefijo: '/api' }  ← Incorrecto (0 y '' reemplazados)
```

## Combinación de ?. con ??

```javascript
// La combinación más poderosa para acceso seguro con valor por defecto

const perfil = {
  usuario: {
    configuracion: {
      tema: null,
      idioma: 'es',
      notificaciones: {
        email: false,
        push: undefined
      }
    }
  }
};

const tema = perfil?.usuario?.configuracion?.tema ?? 'claro';        // 'claro'
const idioma = perfil?.usuario?.configuracion?.idioma ?? 'en';       // 'es'
const email = perfil?.usuario?.configuracion?.notificaciones?.email ?? true;  // false (no undefined)
const push = perfil?.usuario?.configuracion?.notificaciones?.push ?? true;    // true

// Ejemplo real: configuración con múltiples niveles
function obtenerConfig(config) {
  return {
    baseUrl: config?.api?.baseUrl ?? 'http://localhost:3000',
    timeout: config?.api?.timeout ?? 5000,
    reintentos: config?.api?.reintentos ?? 3,
    debug: config?.dev?.debug ?? false,
    logNivel: config?.dev?.logNivel ?? 'error'
  };
}
```

## Asignación con operadores combinados (ES2021)

```javascript
// Operador ??= (asigna si es null/undefined)
let usuario = { nombre: null };
usuario.nombre ??= 'Anónimo';      // Se asigna porque nombre es null
usuario.nombre ??= 'Otro';          // No se asigna porque ya tiene valor

// Operador ||= (asigna si es falsy)
let texto = '';
texto ||= 'Predeterminado';         // Se asigna porque '' es falsy

// Operador &&= (asigna si es truthy)
let activo = true;
activo &&= obtenerEstadoActualizado();  // Solo actualiza si activo es truthy
```

**Consejo de entrevista**: La diferencia clave entre `??` y `||` es que `??` solo considera `null` y `undefined` como "vacíos", mientras que `||` trata cualquier valor falsy (`0`, `''`, `false`, `NaN`) como vacío. Usa `??` cuando `0`, cadena vacía o `false` son valores válidos en tu dominio. En React, el patrón `valor ?? 'predeterminado'` es más preciso que `valor || 'predeterminado'` para valores por defecto en props.
