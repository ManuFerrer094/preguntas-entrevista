Cuando ocurre un evento en el DOM, no se queda solo en el elemento objetivo. Puede recorrer el árbol en distintas fases: captura, objetivo y bubbling.

## Idea clave
- capturing: el evento baja desde los ancestros hacia el objetivo;
- target: el evento llega al elemento que lo disparó;
- bubbling: el evento sube desde el objetivo hacia los ancestros.

## Ejemplo de código
```js
padre.addEventListener('click', () => console.log('padre bubbling'));
padre.addEventListener('click', () => console.log('padre capturing'), true);
hijo.addEventListener('click', () => console.log('hijo'));
```

Al hacer click en `hijo`, verás primero la fase de captura del padre, luego el handler del hijo y después el bubbling del padre.

## Cómo frenar la propagación
```js
boton.addEventListener('click', (event) => {
  event.stopPropagation();
});
```

## Para qué importa
Entender estas fases es clave para delegación de eventos, composición de componentes y debugging de interacciones complejas.

## Para interiorizarlo
Cuando un evento dispare varios handlers y no entiendas por qué, deja de mirar solo el objetivo. Mira el recorrido completo del evento por el árbol.
