title: ¿Qué es el hoisting en JavaScript y cómo afecta a variables y funciones?

El hoisting es el comportamiento por el que declaraciones de variables y funciones quedan registradas antes de ejecutar el código de su ámbito. No significa que el código “suba físicamente”, sino que el motor prepara esas vinculaciones durante la creación del contexto.

## Ejemplo con `var`
```js
console.log(x); // undefined
var x = 10;
```

Eso ocurre porque `var` se registra al inicio del scope y se inicializa con `undefined`.

## Ejemplo con `let` y `const`
```js
// console.log(y); // ReferenceError
let y = 20;
```

Aquí también hay hoisting, pero la variable queda en temporal dead zone hasta su declaración.

## Ejemplo con funciones
```js
saludar();

function saludar() {
  console.log('hola');
}
```

Las declaraciones de función sí quedan disponibles antes de la línea donde aparecen.

## Para interiorizarlo
Lo importante del hoisting no es memorizar rarezas, sino entender que JavaScript prepara primero el entorno y ejecuta después. Eso explica muchas diferencias entre `var`, `let`, `const` y las distintas formas de declarar funciones.
