Los módulos permiten dividir el código en archivos con responsabilidades más claras y controlar qué partes expones al exterior. En JavaScript moderno, el sistema estándar es ES Modules.

## Ejemplo de exportación
```js
// math.js
export function sumar(a, b) {
  return a + b;
}

export const PI = 3.14159;
```

## Ejemplo de importación
```js
// app.js
import { sumar, PI } from './math.js';

console.log(sumar(2, 3));
console.log(PI);
```

## También existe export default
```js
export default function saludar(nombre) {
  return `Hola, ${nombre}`;
}
```

## Qué ganamos con módulos
- separación de responsabilidades;
- dependencias explícitas;
- mejor mantenibilidad y testabilidad;
- posibilidad de optimización por parte de herramientas de build.

## Para interiorizarlo
Un módulo no es solo un archivo: es una frontera. La pregunta importante es qué expones y qué dejas oculto dentro de esa frontera.
