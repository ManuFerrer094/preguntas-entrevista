`Symbol` es un tipo primitivo que representa identificadores únicos. Dos símbolos con la misma descripción siguen siendo distintos entre sí.

## Ejemplo de código
```js
const id = Symbol('id');
const otroId = Symbol('id');

console.log(id === otroId); // false

const usuario = {
  [id]: 123,
  nombre: 'John',
};

console.log(usuario[id]); // 123
```

## Para qué resulta útil
- evitar colisiones de nombres en objetos;
- crear claves que no choquen con propiedades normales;
- trabajar con well-known symbols del lenguaje, como `Symbol.iterator`.

## Matices importantes
- No es una herramienta de privacidad real; solo reduce visibilidad accidental.
- Las propiedades con símbolo no aparecen en recorridos sencillos como `Object.keys()`.

## Para interiorizarlo
Piensa en `Symbol` como una forma de crear llaves realmente únicas cuando una string corriente podría colisionar con otras propiedades del sistema o de terceros.
