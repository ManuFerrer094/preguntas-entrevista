---
title: ¿Qué diferencia hay entre shallow copy y deep copy en JavaScript?
difficulty: hard
tags: [objects, immutability, memory]
lastReviewed: 2026-04-06
---

Una shallow copy copia solo el primer nivel; los objetos anidados siguen compartiendo referencia. Una deep copy clona también las estructuras internas.

## Ejemplo de código
```js
const original = {
  nombre: 'Ana',
  direccion: { ciudad: 'Madrid' },
};

const superficial = { ...original };
superficial.direccion.ciudad = 'Sevilla';

console.log(original.direccion.ciudad); // 'Sevilla'
```

## Ejemplo de deep copy moderno
```js
const profundo = structuredClone(original);
profundo.direccion.ciudad = 'Valencia';

console.log(original.direccion.ciudad); // sigue como estaba
```

## Qué conviene saber
- El spread (`...`) y `Object.assign()` hacen copia superficial.
- `JSON.parse(JSON.stringify(obj))` puede servir en demos, pero pierde fechas, `Map`, `Set`, `undefined`, funciones y más.
- `structuredClone()` es la opción moderna cuando el entorno la soporta y el dato es clonable.

## Para interiorizarlo
Cuando copies un objeto, no pienses solo “he creado otro”. Pregunta: “¿qué partes siguen compartiendo referencia y cuáles no?”.
