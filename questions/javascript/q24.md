---
title: ¿Qué son Map y Set en JavaScript?
difficulty: easy

`Map` y `Set` son estructuras de datos introducidas en ES6. `Map` es una colección de pares clave-valor donde las **claves pueden ser de cualquier tipo** (no solo strings). `Set` es una colección de **valores únicos** sin duplicados.

## Map — Colección de pares clave-valor

```javascript
// Crear un Map
const usuarios = new Map();

// Operaciones básicas
usuarios.set(1, { nombre: 'Ana', email: 'ana@ejemplo.com' });
usuarios.set(2, { nombre: 'Carlos', email: 'carlos@ejemplo.com' });
usuarios.set('admin', { nombre: 'Admin', rol: 'superadmin' });

// Leer valores
console.log(usuarios.get(1));          // { nombre: 'Ana', ... }
console.log(usuarios.has(2));          // true
console.log(usuarios.size);            // 3

// Eliminar
usuarios.delete(2);

// Iterar
for (const [clave, valor] of usuarios) {
  console.log(`${clave}: ${valor.nombre}`);
}

// Métodos de iteración
usuarios.keys();    // Iterador de claves
usuarios.values();  // Iterador de valores
usuarios.entries(); // Iterador de [clave, valor]

// Inicializar con datos
const precios = new Map([
  ['manzana', 1.2],
  ['naranja', 0.8],
  ['plátano', 0.5]
]);

// Convertir a Array
const array = [...precios.entries()];  // [['manzana', 1.2], ...]
```

### Map vs Object: cuándo usar cada uno

```javascript
// ✅ Usar Map cuando:
// 1. Las claves no son strings/symbols (objetos, números, funciones como clave)
const cacheConObjeto = new Map();
const elemento = document.getElementById('btn');
cacheConObjeto.set(elemento, { clicks: 0 });  // DOM element como clave ✅

// 2. Necesitas el tamaño con .size (Object requiere Object.keys().length)
console.log(cacheConObjeto.size);  // O(1)

// 3. Necesitas iterar en orden de inserción de forma garantizada
// 4. Necesitas insertar/borrar frecuentemente (mejor rendimiento que Object)

// ✅ Usar Object cuando:
// 1. Las claves son siempre strings conocidos en tiempo de compilación
// 2. Necesitas serializar a JSON (Map no es serializable directamente)
const config = { host: 'localhost', puerto: 3000 };  // Object es más apropiado

// Serializar Map a JSON:
const json = JSON.stringify([...precios]);
const mapDeserializado = new Map(JSON.parse(json));
```

## Set — Colección de valores únicos

```javascript
// Crear un Set
const tecnologias = new Set(['JavaScript', 'TypeScript', 'Vue', 'React']);

// Añadir (los duplicados se ignoran)
tecnologias.add('Node.js');
tecnologias.add('JavaScript');  // Ya existe: ignorado
console.log(tecnologias.size);  // 5

// Verificar existencia
console.log(tecnologias.has('Vue'));       // true
console.log(tecnologias.has('Angular'));   // false

// Eliminar
tecnologias.delete('Vue');

// Iterar
for (const tech of tecnologias) {
  console.log(tech);
}

// Convertir a Array
const array = [...tecnologias];
const array2 = Array.from(tecnologias);
```

### Casos de uso de Set

```javascript
// 1. Eliminar duplicados de un array (el uso más frecuente)
const numeros = [1, 2, 3, 2, 1, 4, 3, 5];
const unicos = [...new Set(numeros)];  // [1, 2, 3, 4, 5]

const nombres = ['Ana', 'Carlos', 'Ana', 'María', 'Carlos'];
const nombresUnicos = [...new Set(nombres)];  // ['Ana', 'Carlos', 'María']

// 2. Operaciones de conjuntos
const setA = new Set([1, 2, 3, 4]);
const setB = new Set([3, 4, 5, 6]);

// Unión
const union = new Set([...setA, ...setB]);  // {1, 2, 3, 4, 5, 6}

// Intersección
const interseccion = new Set([...setA].filter(x => setB.has(x)));  // {3, 4}

// Diferencia (A - B)
const diferencia = new Set([...setA].filter(x => !setB.has(x)));  // {1, 2}

// 3. Verificar unicidad eficientemente (O(1) vs O(n) de Array.includes)
const vistos = new Set();

function procesarSinDuplicados(items) {
  return items.filter(item => {
    if (vistos.has(item.id)) return false;
    vistos.add(item.id);
    return true;
  });
}
```

## WeakMap y WeakSet — Versiones con referencias débiles

```javascript
// WeakMap: claves son objetos, se recolectan por GC si no hay otras referencias
const metadatos = new WeakMap();

function registrarTiempoCreacion(objeto) {
  metadatos.set(objeto, Date.now());
}

// Si el objeto se elimina, la entrada del WeakMap también se elimina
// → No causa memory leaks

// WeakSet: solo objetos, se recolectan por GC
const objetosProcesados = new WeakSet();

function procesarSiNoProcesado(objeto) {
  if (objetosProcesados.has(objeto)) return;
  objetosProcesados.add(objeto);
  // procesar...
}
```

**Consejo de entrevista**: El uso más común de `Set` en entrevistas es eliminar duplicados: `[...new Set(array)]`. Para `Map`, la ventaja clave sobre Object es que puede tener cualquier tipo como clave (incluyendo objetos DOM), tiene `.size` y el orden de inserción está garantizado. Menciona también que `Map` tiene mejor rendimiento para inserciones/eliminaciones frecuentes comparado con Object, ya que no sufre las optimizaciones del motor que pueden degradarse en patrones dinámicos.
