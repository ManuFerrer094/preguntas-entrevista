JavaScript reutiliza comportamiento principalmente mediante prototipos. Cuando un objeto no encuentra una propiedad en sí mismo, la busca en su prototipo, luego en el prototipo del prototipo, y así sucesivamente hasta agotar la cadena.

## Ejemplo de código
```js
function Usuario(nombre) {
  this.nombre = nombre;
}

Usuario.prototype.saludar = function () {
  return `Hola, soy ${this.nombre}`;
};

const ana = new Usuario('Ana');
console.log(ana.saludar());
```

## Idea clave
La herencia clásica suele modelarse como clases que heredan de otras clases. En JavaScript moderno puedes escribir `class`, pero por debajo sigue funcionando sobre prototipos. Es decir: la sintaxis cambió mucho; el modelo real del lenguaje, no tanto.

## Qué conviene entender
- `prototype` en una función constructora no es lo mismo que `[[Prototype]]` de una instancia.
- Los métodos en el prototipo se comparten entre instancias y ahorran memoria.
- La composición suele ser más flexible que construir jerarquías profundas.

## Para interiorizarlo
Si quieres aprender JavaScript a fondo, no te quedes en la sintaxis de `class`. Entender la cadena de prototipos te explica cómo funciona de verdad la delegación de comportamiento.
