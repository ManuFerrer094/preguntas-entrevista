Esta pregunta suele aparecer cuando quieren separar a alguien que usa la sintaxis de `class` de alguien que entiende el modelo real del lenguaje. En JavaScript no heredan clases en tiempo de compilación como en Java o C#: heredan objetos enlazados mediante prototipos.

## Qué evalúa el entrevistador
- Si sabes que en JavaScript la herencia real es prototipal, aunque escribas `class`.
- Si explicas que las `class` son azúcar sintáctico sobre funciones constructoras y prototipos.
- Si puedes hablar de flexibilidad y coste de memoria sin quedarte en una definición académica.

## Respuesta sólida
- En herencia clásica, una clase define una estructura cerrada y las instancias se crean a partir de esa clase.
- En herencia prototipal, un objeto puede delegar comportamiento en otro objeto a través de la cadena de prototipos.
- JavaScript permite ambas sintaxis de trabajo, pero el motor sigue resolviendo miembros mediante prototipos.
- Eso hace que el modelo sea más dinámico: puedes componer objetos, ajustar prototipos y compartir comportamiento sin depender de jerarquías rígidas.

## Cómo lo explicaría en entrevista
- Si quiero sonar preciso, diría que `class Cuenta extends ProductoFinanciero {}` termina generando enlaces prototipales entre `Cuenta.prototype` y `ProductoFinanciero.prototype`.
- La ventaja no es "que JavaScript no tenga clases", sino que su mecanismo nativo de reutilización sigue siendo la delegación entre objetos.
- En sistemas grandes eso ayuda a escoger entre jerarquía, composición o factorías según el problema, en vez de convertir todo en árboles de herencia.

## Ejemplo práctico
Con `class`:

```js
class Animal {
  hablar() {
    return 'sonido';
  }
}

class Perro extends Animal {}
```

Con prototipos explícitos:

```js
function Animal() {}
Animal.prototype.hablar = function () {
  return 'sonido';
};

function Perro() {}
Perro.prototype = Object.create(Animal.prototype);
Perro.prototype.constructor = Perro;
```

Los dos enfoques acaban apoyándose en la misma idea: si `hablar` no está en la instancia, se busca en el prototipo.

## Error común
Decir que JavaScript "no usa clases". Desde ES2015 sí tienes sintaxis de clases, pero internamente no cambió el mecanismo de herencia. La respuesta buena reconoce las dos cosas al mismo tiempo.

## Frase corta de entrevista
> En JavaScript las `class` mejoran la sintaxis, pero la herencia real sigue siendo prototipal: los objetos delegan comportamiento a través de la cadena de prototipos.
