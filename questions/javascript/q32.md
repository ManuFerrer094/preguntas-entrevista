Las **expresiones regulares** (RegExp) son patrones para buscar, validar y transformar texto. JavaScript tiene soporte nativo mediante el tipo `RegExp` y métodos de `String`.

## Sintaxis básica

```javascript
// Literal (más común)
const patron = /hola/;
const patronConFlags = /hola/gi;

// Constructor (para patrones dinámicos)
const busqueda = 'mundo';
const patronDinamico = new RegExp(busqueda, 'gi');
const patronEscapado = new RegExp(`\\b${busqueda}\\b`, 'gi');  // Cuidado con escapes
```

## Metacaracteres y cuantificadores fundamentales

```javascript
// Caracteres especiales
/./    // Cualquier carácter excepto \n
/\d/   // Dígito: [0-9]
/\D/   // No dígito: [^0-9]
/\w/   // Carácter de palabra: [a-zA-Z0-9_]
/\W/   // No carácter de palabra
/\s/   // Espacio en blanco: [ \t\n\r\f\v]
/\S/   // No espacio en blanco
/\b/   // Límite de palabra
/^/    // Inicio de cadena
/$/    // Fin de cadena

// Cuantificadores
/a+/   // Uno o más 'a'
/a*/   // Cero o más 'a'
/a?/   // Cero o una 'a' (opcional)
/a{3}/ // Exactamente 3 'a'
/a{2,4}/ // Entre 2 y 4 'a'
/a{2,}/  // 2 o más 'a'

// Grupos y alternativas
/(ab)+/   // Grupo: 'ab' repetido
/cat|dog/  // Alternativa: 'cat' o 'dog'
/[aeiou]/  // Clase de caracteres
/[^aeiou]/ // Negación de clase
/[a-z]/    // Rango
```

## Flags (modificadores)

```javascript
/patron/g   // global: busca todas las ocurrencias
/patron/i   // case insensitive
/patron/m   // multiline: ^ y $ aplican a cada línea
/patron/s   // dotAll: . también coincide con \n (ES2018)
/patron/u   // unicode: habilita soporte Unicode completo
/patron/v   // unicodeSets: soporte avanzado Unicode (ES2024)
/patron/d   // indices: incluye índices de los grupos
```

## Métodos de String con RegExp

```javascript
const texto = 'Mi email es ana@ejemplo.com y carlos@test.org. Llámame.';

// test(): boolean — ¿el patrón existe?
const emailRegex = /\w+@\w+\.\w+/;
console.log(emailRegex.test(texto));  // true

// match(): array de coincidencias (con /g, todas; sin /g, primera + grupos)
const emails = texto.match(/\w+@[\w.]+\.\w+/g);
console.log(emails);  // ['ana@ejemplo.com', 'carlos@test.org']

// matchAll(): iterador de todas las coincidencias con grupos (ES2020)
const patronEmail = /(\w+)@([\w.]+)\.(\w+)/g;
for (const coincidencia of texto.matchAll(patronEmail)) {
  console.log({
    completo: coincidencia[0],   // 'ana@ejemplo.com'
    usuario: coincidencia[1],    // 'ana'
    dominio: coincidencia[2],    // 'ejemplo'
    extension: coincidencia[3]   // 'com'
  });
}

// search(): índice de la primera coincidencia
console.log(texto.search(/email/i));   // 3

// replace(): reemplazar primera coincidencia (o todas con /g)
const textoMascarado = texto.replace(/\w+@[\w.]+\.\w+/g, '[OCULTO]');
// 'Mi email es [OCULTO] y [OCULTO]. Llámame.'

// replaceAll() con regex necesita flag /g

// split(): dividir usando regex
const palabras = 'uno  dos   tres'.split(/\s+/);
// ['uno', 'dos', 'tres'] — cualquier cantidad de espacios como separador
```

## Grupos de captura y named groups

```javascript
// Grupos de captura numerados
const fechaRegex = /(\d{4})-(\d{2})-(\d{2})/;
const match = '2024-03-15'.match(fechaRegex);
console.log(match[1]);  // '2024' (año)
console.log(match[2]);  // '03' (mes)
console.log(match[3]);  // '15' (día)

// Named groups (ES2018) — más legible
const fechaConNombres = /(?<año>\d{4})-(?<mes>\d{2})-(?<dia>\d{2})/;
const matchNombrado = '2024-03-15'.match(fechaConNombres);
console.log(matchNombrado.groups.año);  // '2024'
console.log(matchNombrado.groups.mes);  // '03'
console.log(matchNombrado.groups.dia);  // '15'

// Named groups en replace
const reformatear = '2024-03-15'.replace(
  /(?<año>\d{4})-(?<mes>\d{2})-(?<dia>\d{2})/,
  '$<dia>/$<mes>/$<año>'
);
console.log(reformatear);  // '15/03/2024'
```

## Validaciones comunes

```javascript
// Email (simplificado)
const EMAIL_REGEX = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

// URL
const URL_REGEX = /^https?:\/\/(www\.)?[-a-zA-Z0-9@:%._+~#=]{1,256}\.[a-zA-Z0-9()]{1,6}\b([-a-zA-Z0-9()@:%_+.~#?&//=]*)$/;

// Teléfono español
const TELEFONO_REGEX = /^[6-9]\d{8}$/;

// Código postal español
const CP_REGEX = /^[0-5]\d{4}$/;

// NIF/DNI español
const DNI_REGEX = /^\d{8}[A-Z]$/i;

// Contraseña: mínimo 8 caracteres, al menos 1 mayúscula, 1 minúscula, 1 número
const PASSWORD_REGEX = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$/;

// Verificar con lookahead
function validarContraseña(pass) {
  const tieneMinuscula = /(?=.*[a-z])/.test(pass);
  const tieneMayuscula = /(?=.*[A-Z])/.test(pass);
  const tieneNumero = /(?=.*\d)/.test(pass);
  const longitudSuficiente = pass.length >= 8;
  return { tieneMinuscula, tieneMayuscula, tieneNumero, longitudSuficiente };
}
```

**Consejo de entrevista**: En entrevistas, los casos más comunes de RegExp son: validar email/URL/teléfono, extraer datos de strings, transformar formatos de fecha, y sanitizar input. Menciona los named groups (`?<nombre>`) como la forma moderna y legible de capturar grupos. Para regex complejas en producción, usa herramientas como regex101.com para testear y documentar el patrón. Evita regex excesivamente complejas para validaciones HTML —usa parsers específicos.
