---
title: ¿Qué es el método Object.entries(), Object.keys() y Object.values() en JavaScript?
difficulty: easy
tags: [javascript, object, entries, keys, values, iteration]
---

`Object.keys()`, `Object.values()` y `Object.entries()` son métodos estáticos de ES2017 que permiten iterar sobre las propiedades enumerables de un objeto, retornando arrays de claves, valores o pares [clave, valor] respectivamente.

## Los tres métodos

```javascript
const producto = {
  id: 1,
  nombre: 'Laptop Pro',
  precio: 999,
  stock: 15,
  activo: true
};

// Object.keys() — Array de claves
const claves = Object.keys(producto);
// ['id', 'nombre', 'precio', 'stock', 'activo']

// Object.values() — Array de valores
const valores = Object.values(producto);
// [1, 'Laptop Pro', 999, 15, true]

// Object.entries() — Array de pares [clave, valor]
const entradas = Object.entries(producto);
// [['id', 1], ['nombre', 'Laptop Pro'], ['precio', 999], ['stock', 15], ['activo', true]]
```

## Iteración sobre objetos

```javascript
const precios = { manzana: 1.2, naranja: 0.8, plátano: 0.5, uva: 2.1 };

// Iterar con for...of + Object.entries()
for (const [fruta, precio] of Object.entries(precios)) {
  console.log(`${fruta}: ${precio}€`);
}

// Transformar con map (los objetos no tienen .map directo)
const preciosConIVA = Object.fromEntries(
  Object.entries(precios).map(([fruta, precio]) => [fruta, precio * 1.21])
);
// { manzana: 1.452, naranja: 0.968, plátano: 0.605, uva: 2.541 }

// Filtrar propiedades de un objeto
const caros = Object.fromEntries(
  Object.entries(precios).filter(([_, precio]) => precio > 1)
);
// { manzana: 1.2, uva: 2.1 }
```

## Object.fromEntries() — El inverso de entries()

```javascript
// Object.fromEntries() convierte un iterable de [clave, valor] en objeto
const array = [['a', 1], ['b', 2], ['c', 3]];
const objeto = Object.fromEntries(array);
// { a: 1, b: 2, c: 3 }

// Conversión Map → Object
const mapa = new Map([['nombre', 'Ana'], ['edad', 30]]);
const obj = Object.fromEntries(mapa);
// { nombre: 'Ana', edad: 30 }

// Patrón pipeline: transformar objetos como si fueran arrays
const config = { HOST: 'LOCALHOST', PORT: '3000', DEBUG: 'TRUE' };

// Convertir todas las claves a minúsculas
const configNormalizada = Object.fromEntries(
  Object.entries(config).map(([k, v]) => [k.toLowerCase(), v])
);
// { host: 'LOCALHOST', port: '3000', debug: 'TRUE' }
```

## Casos de uso prácticos

```javascript
// 1. Contar ocurrencias en un array
function contarOcurrencias(array) {
  return array.reduce((contador, elemento) => {
    contador[elemento] = (contador[elemento] ?? 0) + 1;
    return contador;
  }, {});
}

const frecuencia = contarOcurrencias(['a', 'b', 'a', 'c', 'b', 'a']);
// { a: 3, b: 2, c: 1 }

const porFrecuencia = Object.entries(frecuencia)
  .sort(([, a], [, b]) => b - a)  // Ordenar por frecuencia descendente
  .map(([letra, cuenta]) => `${letra}: ${cuenta}`);
// ['a: 3', 'b: 2', 'c: 1']

// 2. Invertir un objeto (claves ↔ valores)
const PAIS_CODIGO = { ES: 'España', MX: 'México', AR: 'Argentina' };
const CODIGO_PAIS = Object.fromEntries(
  Object.entries(PAIS_CODIGO).map(([k, v]) => [v, k])
);
// { España: 'ES', México: 'MX', Argentina: 'AR' }

// 3. Limpiar propiedades undefined/null de un objeto
function limpiarObjeto(obj) {
  return Object.fromEntries(
    Object.entries(obj).filter(([_, v]) => v != null)
  );
}

const conNull = { nombre: 'Ana', email: null, edad: undefined, activo: true };
const limpio = limpiarObjeto(conNull);
// { nombre: 'Ana', activo: true }

// 4. Agrupar array de objetos por propiedad (groupBy)
function agruparPor(array, clave) {
  return array.reduce((grupos, item) => {
    const grupo = item[clave];
    return {
      ...grupos,
      [grupo]: [...(grupos[grupo] ?? []), item]
    };
  }, {});
}

const empleados = [
  { nombre: 'Ana', dpto: 'Engineering' },
  { nombre: 'Carlos', dpto: 'Marketing' },
  { nombre: 'María', dpto: 'Engineering' },
  { nombre: 'Luis', dpto: 'Marketing' },
];

const porDepartamento = agruparPor(empleados, 'dpto');
// {
//   Engineering: [{ nombre: 'Ana', ... }, { nombre: 'María', ... }],
//   Marketing: [{ nombre: 'Carlos', ... }, { nombre: 'Luis', ... }]
// }
```

## Diferencias con for...in

```javascript
// Object.keys/values/entries solo retornan propiedades PROPIAS y ENUMERABLES
// for...in itera también las propiedades HEREDADAS (del prototipo)

function Animal(nombre) {
  this.nombre = nombre;
}
Animal.prototype.tipo = 'animal';

const perro = new Animal('Rex');
perro.raza = 'Labrador';

// for...in incluye propiedades heredadas:
for (const key in perro) {
  console.log(key);  // nombre, raza, tipo (¡incluye 'tipo' del prototipo!)
}

// Object.keys solo propias:
console.log(Object.keys(perro));  // ['nombre', 'raza'] — sin 'tipo'

// Equivalente seguro con for...in:
for (const key in perro) {
  if (Object.hasOwn(perro, key)) {  // hasOwn es más moderno que hasOwnProperty
    console.log(key);  // nombre, raza
  }
}
```

**Consejo de entrevista**: El patrón `Object.entries()` + `map/filter` + `Object.fromEntries()` es el equivalente al pipeline funcional de arrays (map/filter/reduce) para objetos. Es la forma más limpia de transformar objetos sin mutarlos. Menciona también `Object.hasOwn()` (ES2022) como alternativa moderna y más segura a `hasOwnProperty()`, que puede ser sobreescrito en objetos creados con `Object.create(null)`.
