tags: [prototypes, memory, performance]

Aquí lo que buscan no es una regla dogmática, sino si entiendes qué se comparte y qué se duplica al instanciar muchos objetos. En una app con cientos o miles de entidades en memoria, meter funciones dentro del constructor multiplica trabajo y consumo sin aportar valor.

## Qué evalúa el entrevistador
- Si sabes que las funciones creadas dentro del constructor generan una nueva referencia por instancia.
- Si entiendes que un método en el prototipo se comparte entre todas las instancias.
- Si puedes conectar esta decisión con rendimiento real y no solo con estilo de código.

## Respuesta sólida
- Los datos propios de cada objeto van en la instancia: por ejemplo, saldo, titular o estado.
- El comportamiento común debería vivir en el prototipo para que todas las instancias reutilicen la misma función.
- Eso reduce memoria, facilita comparar comportamiento y deja más claro qué forma parte de la API común.

## Ejemplo de código
La diferencia es más visible cuando comparas ambos enfoques:

```js
function CuentaMala(titular) {
  this.titular = titular;
  this.getResumen = function () {
    return `Cuenta de ${this.titular}`;
  };
}

function CuentaBuena(titular) {
  this.titular = titular;
}

CuentaBuena.prototype.getResumen = function () {
  return `Cuenta de ${this.titular}`;
};
```

En `CuentaMala`, cada `new CuentaMala()` crea otra función `getResumen`. En `CuentaBuena`, todas las instancias comparten una sola implementación.

## Cuándo sí tendría sentido crear funciones por instancia
- Cuando necesitas cerrar sobre variables privadas del constructor y ese cierre forma parte del diseño.
- Cuando el comportamiento realmente depende de datos encapsulados que no quieres exponer en la instancia.

Pero incluso ahí conviene explicitar el coste, porque has cambiado encapsulación por consumo de memoria.

## Error común
Responder solo "porque es una buena práctica". En entrevista pesa más decir: "porque cada instancia replicaría el método y eso empeora memoria y dificulta compartir comportamiento, especialmente en dominios con muchos objetos vivos".

## Frase corta de entrevista
> Los datos propios van en la instancia; el comportamiento compartido, en el prototipo. Así evitas crear copias innecesarias de la misma función en cada objeto.
