La memoización es una técnica para guardar el resultado de una función costosa y reutilizarlo cuando se vuelve a llamar con las mismas entradas. Cambias tiempo de CPU por memoria.

## Ejemplo de código
```js
function memoize(fn) {
  const cache = new Map();

  return function (...args) {
    const key = JSON.stringify(args);

    if (!cache.has(key)) {
      cache.set(key, fn(...args));
    }

    return cache.get(key);
  };
}

const sumarLento = memoize((a, b) => a + b);
console.log(sumarLento(1, 2));
console.log(sumarLento(1, 2));
```

## Cuándo ayuda
- cálculos puros y repetidos;
- selectores derivados;
- transformaciones costosas que dependen solo de entradas concretas.

## Matices importantes
- Funciona mejor con funciones puras: mismas entradas, mismo resultado.
- El sistema de claves importa. `JSON.stringify(args)` es una demo útil, pero no siempre es robusta.
- Si el caché crece sin límite, la optimización puede convertirse en fuga de memoria.

## Para interiorizarlo
No memorices "memoizar es cachear" y ya. Entiende el coste total: ganas velocidad, pero pagas complejidad y memoria. Solo compensa cuando el cálculo repetido lo justifica.
