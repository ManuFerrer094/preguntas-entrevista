JavaScript permite declarar variables con `var`, `let` y `const`. Aunque las tres existen, no significan lo mismo y conviene usarlas con criterio.

## Idea clave
- `var` tiene scope de función y además se eleva con inicialización a `undefined`. Hoy se evita salvo en código legacy.
- `let` tiene scope de bloque y permite reasignación.
- `const` también tiene scope de bloque, pero no permite reasignar la variable.

## Ejemplo de código
```js
if (true) {
  var conVar = 'visible fuera del bloque';
  let conLet = 'solo dentro del bloque';
  const conConst = { activa: true };

  conConst.activa = false; // esto sí se puede
}

console.log(conVar);
// console.log(conLet); // ReferenceError
// console.log(conConst); // ReferenceError
```

## Matices importantes
- `const` no vuelve inmutable al objeto; solo impide reasignar la referencia.
- `let` y `const` también se elevan, pero quedan en la temporal dead zone hasta su declaración.
- En código moderno, la regla práctica suele ser: usa `const` por defecto y `let` cuando sepas que necesitas reasignar.

## Para interiorizarlo
No pienses en estas palabras como sinónimos. Piensa en ellas como una forma de expresar intención sobre el ciclo de vida y la mutabilidad de una variable.
