---
title: ¿Cuál es la diferencia entre == y === en JavaScript?
difficulty: easy
tags: [javascript, equality, coercion, comparison]
---

En JavaScript, `==` es el operador de igualdad abstracta (loose equality) que realiza **coerción de tipos** antes de comparar, mientras que `===` es el operador de igualdad estricta (strict equality) que compara **sin coerción de tipos**. En la práctica, casi siempre se debe usar `===`.

## Igualdad estricta (===)

Compara valor **y** tipo. Solo son iguales si tienen el mismo tipo y el mismo valor:

```javascript
// Mismos tipo y valor
5 === 5           // true
'hello' === 'hello' // true
true === true     // true

// Distinto tipo
5 === '5'         // false
0 === false       // false
null === undefined // false

// Objetos: comparan por referencia, no por valor
const a = { x: 1 };
const b = { x: 1 };
a === b  // false (referencias diferentes)
a === a  // true (misma referencia)
```

## Igualdad abstracta (==)

Aplica reglas de coerción de tipos antes de comparar:

```javascript
// Número vs string: convierte string a número
5 == '5'    // true  (convierte '5' → 5)
5 == '5.0'  // true
0 == ''     // true  ('' → 0)
0 == '0'    // true  ('0' → 0)
0 == ' '    // true  (' ' → 0)

// Boolean: convierte boolean a número
true == 1   // true  (true → 1)
false == 0  // true  (false → 0)
true == '1' // true  (true → 1, '1' → 1)
false == ''  // true  (false → 0, '' → 0)
false == '0' // true

// null y undefined son iguales entre sí (y solo entre sí)
null == undefined  // true
null == 0          // false ← sorpresa!
null == false      // false ← sorpresa!
undefined == false // false

// Objetos vs primitivos: llama toString() o valueOf()
[] == 0         // true  ([] → '' → 0)
[] == false     // true
[''] == false   // true
[0] == false    // true
```

## Reglas de coerción (simplificadas)

```
Tipo de x    Tipo de y    Resultado
boolean      cualquiera   convierte boolean a número, recompara
string       number       convierte string a número
object       string/number llama valueOf() o toString()
null         undefined    true (siempre)
null/undefined cualquier otro false (siempre)
```

## Por qué usar === siempre

```javascript
// ❌ Comparaciones confusas con ==
'' == false    // true ← ¿esperabas esto?
0 == false     // true
' ' == 0       // true
[] == false    // true
'' == 0        // true ← ¡ahora '' == false == 0 pero '' !== 0 con ===!

// ✅ Con === el comportamiento es predecible
'' === false   // false
0 === false    // false
' ' === 0      // false
```

## Casos especiales importantes

```javascript
// NaN no es igual a nada, ni a sí mismo
NaN === NaN  // false
NaN == NaN   // false

// Usar Number.isNaN() para verificar
Number.isNaN(NaN)      // true
Number.isNaN(undefined) // false (diferente a isNaN global)

// Object.is() — igualdad aún más estricta
Object.is(NaN, NaN)   // true  ← diferente a ===
Object.is(0, -0)      // false ← diferente a ===
0 === -0              // true
```

## Comparación de objetos por valor

```javascript
// Para comparar objetos "por valor", hay que hacerlo manualmente
const a = { x: 1, y: 2 };
const b = { x: 1, y: 2 };

// Solución simple (sin objetos anidados ni funciones)
JSON.stringify(a) === JSON.stringify(b); // true (pero inestable con orden de propiedades)

// Mejor: usar lodash isEqual o implementar deep equal
_.isEqual(a, b); // true (lodash)
```

**Consejo de entrevista**: La regla de oro es "siempre usa `===`". Las únicas excepciones donde `==` puede ser útil son: verificar null o undefined en un solo check (`value == null` equivale a `value === null || value === undefined`), y en legacy code. Conocer la tabla de coerción de `==` demuestra comprensión profunda de JavaScript, aunque en la práctica se evita activamente.
