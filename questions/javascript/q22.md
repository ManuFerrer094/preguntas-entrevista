title: ¿Qué son los closures en JavaScript y para qué sirven en la práctica?

Una closure aparece cuando una función "recuerda" variables de su entorno léxico incluso después de que la función exterior haya terminado de ejecutarse. No es magia: la función interna conserva acceso al scope en el que fue creada.

## Ejemplo de código
```js
function crearContador() {
  let valor = 0;

  return function incrementar() {
    valor += 1;
    return valor;
  };
}

const contador = crearContador();
console.log(contador()); // 1
console.log(contador()); // 2
console.log(contador()); // 3
```

## Qué está pasando
`crearContador` termina, pero la función `incrementar` sigue necesitando `valor`. Por eso ese dato no desaparece: la closure mantiene vivo ese acceso.

## Para qué sirven de verdad
- encapsular estado privado sin exponerlo directamente;
- crear factorías configuradas;
- construir callbacks que recuerdan contexto;
- implementar memoización, throttling, debounce y muchos patrones reales.

## Matices importantes
- Las closures son muy poderosas, pero también pueden retener memoria si capturan más de lo necesario.
- Entender closures aclara por qué ciertos callbacks o listeners siguen "recordando" datos antiguos.

## Para interiorizarlo
No memorices la definición. Piensa en una closure como una función con mochila: además del código, lleva consigo parte del contexto donde nació.
