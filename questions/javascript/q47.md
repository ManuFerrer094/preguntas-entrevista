La programación dirigida por eventos organiza el flujo del programa alrededor de sucesos: clicks, respuestas de red, timers, mensajes, cambios de estado, etc. En lugar de ejecutar una secuencia lineal cerrada, reaccionas a cosas que van ocurriendo.

## Ejemplo de código
```js
button.addEventListener('click', () => {
  console.log('El usuario ha hecho click');
});

socket.addEventListener('message', (event) => {
  console.log('Mensaje recibido', event.data);
});
```

## Dónde aparece
- interfaces de usuario;
- sockets y mensajería;
- eventos del DOM;
- sistemas pub/sub;
- arquitecturas reactivas.

## Qué conviene entender
En un sistema orientado a eventos, las piezas están más desacopladas, pero también es más fácil perder trazabilidad si no defines bien contratos, nombres y responsables de cada evento.

## Para interiorizarlo
Piensa en los eventos como mensajes del sistema: “algo ha ocurrido”. El diseño importante no es solo escucharlos, sino decidir quién los emite, quién los consume y qué garantías existen sobre ellos.
