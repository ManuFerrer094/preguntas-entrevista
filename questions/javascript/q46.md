Una IIFE es una Immediately Invoked Function Expression: una función que se define y se ejecuta en el mismo momento. Fue muy usada antes de los módulos modernos para crear scope aislado y no contaminar el ámbito global.

## Ejemplo de código
```js
(function () {
  const secreto = 'solo vive aquí dentro';
  console.log(secreto);
})();
```

## Por qué se usaban tanto
- crear encapsulación;
- evitar colisiones de nombres en el global;
- construir módulos simples antes de ES Modules.

## Qué papel tienen hoy
Con `let`, `const` y módulos, ya no son tan necesarias para el trabajo diario. Aun así, siguen siendo útiles para entender código legacy y ciertos patrones de encapsulación rápida.

## Para interiorizarlo
Las IIFEs importan menos por su vigencia actual y más porque enseñan una idea muy profunda de JavaScript: puedes crear scope ejecutando funciones, no solo declarando bloques.
