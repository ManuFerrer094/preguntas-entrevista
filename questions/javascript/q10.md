---
title: ¿Qué es el operador spread y rest en JavaScript?
difficulty: easy
tags: [javascript, spread, rest, es6, syntax]
---

El operador `...` (tres puntos) tiene dos usos distintos según el contexto: como **spread** expande un iterable en elementos individuales, y como **rest** agrupa múltiples elementos en un array. Son complementarios: spread "dispersa", rest "recolecta".

## Operador Spread — expandir iterables

```javascript
// Spread en arrays
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];

const combined = [...arr1, ...arr2];        // [1, 2, 3, 4, 5, 6]
const withExtra = [0, ...arr1, 3.5, ...arr2]; // [0, 1, 2, 3, 3.5, 4, 5, 6]

// Clonar array (shallow copy)
const clone = [...arr1]; // Nueva referencia, mismos valores primitivos

// Convertir string a array de caracteres
const chars = [..."hello"]; // ['h', 'e', 'l', 'l', 'o']

// Convertir Set a array
const unique = [...new Set([1, 2, 2, 3, 3])]; // [1, 2, 3]

// Spread en objetos (ES2018)
const defaults = { theme: 'light', language: 'es', notifications: true };
const userPrefs = { theme: 'dark', fontSize: 16 };

// Combinar objetos (las propiedades del derecho sobreescriben)
const config = { ...defaults, ...userPrefs };
// { theme: 'dark', language: 'es', notifications: true, fontSize: 16 }

// Clonar objeto con modificación
const updatedUser = { ...user, name: 'Nuevo Nombre', updatedAt: new Date() };
```

## Spread en llamadas a funciones

```javascript
// En lugar de apply()
const numbers = [5, 1, 9, 3, 7];
const max = Math.max(...numbers);   // Equivale a Math.max(5, 1, 9, 3, 7)
const min = Math.min(...numbers);   // 1

// Con funciones propias
function sum(a, b, c) { return a + b + c; }
const args = [1, 2, 3];
console.log(sum(...args)); // 6

// Pasar array como múltiples argumentos
const dates = [2024, 0, 15]; // [año, mes(0-11), día]
const date = new Date(...dates);
```

## Parámetros Rest — recoger argumentos

```javascript
// Rest en parámetros de función
function logMessages(level, ...messages) {
  // 'messages' es un array con todos los argumentos después del primero
  messages.forEach(msg => console.log(`[${level}] ${msg}`));
}

logMessages('INFO', 'Server started', 'Port: 3000', 'Environment: production');
// [INFO] Server started
// [INFO] Port: 3000
// [INFO] Environment: production

// Función sum variádica
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
console.log(sum(1, 2, 3, 4, 5)); // 15
```

## Rest en desestructuración

```javascript
// Rest en array destructuring
const [first, second, ...remaining] = [1, 2, 3, 4, 5];
console.log(first);     // 1
console.log(remaining); // [3, 4, 5]

// Rest en object destructuring
const { id, name, ...otherProps } = user;
// otherProps tiene todo excepto id y name
```

## Casos de uso prácticos

```javascript
// Inmutabilidad — actualizar estado sin mutar
function updateUser(users, userId, updates) {
  return users.map(user =>
    user.id === userId
      ? { ...user, ...updates } // Crear nuevo objeto con cambios
      : user
  );
}

// Merge profundo simplificado
function mergeConfigs(...configs) {
  return configs.reduce((merged, config) => ({ ...merged, ...config }), {});
}

// Función que acepta cualquier número de arrays
function flatten(...arrays) {
  return arrays.flat();
}
console.log(flatten([1, 2], [3, 4], [5, 6])); // [1, 2, 3, 4, 5, 6]

// HOF que añade logging a cualquier función
function withLogging(fn) {
  return function(...args) {
    console.log(`Llamando ${fn.name} con`, args);
    const result = fn(...args);
    console.log(`Resultado:`, result);
    return result;
  };
}
```

**Consejo de entrevista**: Spread y rest son ubicuos en JavaScript moderno. Un punto importante: el spread en objetos hace una **copia superficial** (shallow copy): los valores primitivos se copian por valor, pero los objetos anidados se copian por referencia. Para clonar objetos profundamente anidados, se necesita `structuredClone()` (moderno) o `JSON.parse(JSON.stringify(obj))` (con limitaciones).
