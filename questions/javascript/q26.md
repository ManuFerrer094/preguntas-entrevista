Los **Web Workers** permiten ejecutar código JavaScript en un **hilo de trabajo** separado del hilo principal, evitando que operaciones costosas bloqueen la interfaz de usuario. Son la única forma de multithreading real en JavaScript del navegador.

## Problema: el hilo principal bloqueado

```javascript
// ❌ Sin Web Workers: cálculo costoso bloquea la UI
function calcularNumerosPrimos(limite) {
  const primos = [];
  for (let n = 2; n <= limite; n++) {
    let esPrimo = true;
    for (let i = 2; i <= Math.sqrt(n); i++) {
      if (n % i === 0) { esPrimo = false; break; }
    }
    if (esPrimo) primos.push(n);
  }
  return primos;
}

// Esto CONGELA la UI durante segundos
const primos = calcularNumerosPrimos(10_000_000);
// El usuario no puede hacer clic, hacer scroll, ni nada
```

## Crear y usar un Web Worker

```javascript
// worker.js — Se ejecuta en el hilo secundario
self.addEventListener('message', function(evento) {
  const { tipo, datos } = evento.data;

  if (tipo === 'calcular-primos') {
    const primos = calcularNumerosPrimos(datos.limite);
    // Enviar resultado de vuelta al hilo principal
    self.postMessage({ tipo: 'primos-calculados', resultado: primos });
  }
});

function calcularNumerosPrimos(limite) {
  const primos = [];
  for (let n = 2; n <= limite; n++) {
    let esPrimo = true;
    for (let i = 2; i <= Math.sqrt(n); i++) {
      if (n % i === 0) { esPrimo = false; break; }
    }
    if (esPrimo) primos.push(n);
  }
  return primos;
}
```

```javascript
// main.js — Hilo principal
const worker = new Worker('./worker.js');

// Recibir mensajes del worker
worker.addEventListener('message', (evento) => {
  const { tipo, resultado } = evento.data;

  if (tipo === 'primos-calculados') {
    console.log(`Encontrados ${resultado.length} primos`);
    mostrarResultados(resultado);
  }
});

// Manejo de errores
worker.addEventListener('error', (error) => {
  console.error('Error en Worker:', error.message);
});

// Enviar trabajo al worker (no bloquea la UI)
worker.postMessage({ tipo: 'calcular-primos', datos: { limite: 10_000_000 } });
// La UI sigue siendo interactiva mientras el worker calcula
```

## Transferir datos eficientemente con Transferable Objects

```javascript
// Por defecto, los datos se COPIAN (serialización JSON) entre hilos
// Para datos grandes (ArrayBuffer, ImageData), usar Transferencia para evitar la copia

// main.js
const buffer = new ArrayBuffer(1024 * 1024 * 100);  // 100MB

// ❌ Sin transferencia: copia los 100MB (lento)
worker.postMessage({ buffer });

// ✅ Con transferencia: mueve la memoria al worker (rápido, el hilo principal pierde acceso)
worker.postMessage({ buffer }, [buffer]);
// buffer ahora es inaccesible en el hilo principal

// worker.js — Procesar y devolver
self.addEventListener('message', ({ data: { buffer } }) => {
  const vista = new Float32Array(buffer);
  // Procesar datos...
  vista.forEach((v, i) => vista[i] = v * 2);

  // Devolver el buffer al hilo principal
  self.postMessage({ buffer }, [buffer]);
});
```

## SharedArrayBuffer — Memoria compartida

```javascript
// SharedArrayBuffer permite COMPARTIR memoria entre hilos (sin copia)
// Requiere cabeceras de seguridad: Cross-Origin-Isolation
const bufferCompartido = new SharedArrayBuffer(1024);
const vistaCompartida = new Int32Array(bufferCompartido);

// worker.js
self.addEventListener('message', ({ data: { bufferCompartido } }) => {
  const vista = new Int32Array(bufferCompartido);
  // Usar Atomics para operaciones thread-safe
  Atomics.add(vista, 0, 1);  // Incremento atómico
  Atomics.notify(vista, 0);   // Notificar al hilo principal
});

// main.js
const worker = new Worker('./worker.js');
worker.postMessage({ bufferCompartido });
// Leer el valor actualizado por el worker:
Atomics.wait(vistaCompartida, 0, 0);  // Esperar a que cambie
console.log(vistaCompartida[0]);  // 1
```

## Inline Worker con Blob (sin archivo separado)

```javascript
// Crear el worker como string y convertirlo a URL
const codigoWorker = `
  self.addEventListener('message', (e) => {
    const { items } = e.data;
    const resultado = items
      .filter(item => item.activo)
      .map(item => ({ ...item, procesado: true }));
    self.postMessage(resultado);
  });
`;

const blob = new Blob([codigoWorker], { type: 'application/javascript' });
const urlWorker = URL.createObjectURL(blob);
const worker = new Worker(urlWorker);

// Limpiar la URL cuando ya no se necesite
worker.addEventListener('message', (e) => {
  console.log('Procesado:', e.data);
  URL.revokeObjectURL(urlWorker);  // Liberar memoria
  worker.terminate();
});
```

## Casos de uso reales

```javascript
// 1. Procesamiento de imágenes
// 2. Parsing de archivos grandes (CSV, JSON de varios MB)
// 3. Algoritmos de compresión/descompresión
// 4. Cálculos matemáticos intensivos (simulaciones, ML)
// 5. Cifrado/descifrado de datos
// 6. Búsqueda en datos grandes (full-text search offline)

// Ejemplo: parsear CSV grande sin bloquear UI
function parsearCSVGrande(archivoTexto) {
  return new Promise((resolve, reject) => {
    const worker = new Worker('./csv-parser.worker.js');
    worker.postMessage({ csv: archivoTexto });
    worker.onmessage = (e) => resolve(e.data);
    worker.onerror = reject;
  });
}
```

**Consejo de entrevista**: Los Web Workers son esenciales cuando una operación supera los ~16ms (el tiempo de un frame a 60fps). Los límites de los Workers: no pueden acceder al DOM, window, ni document —solo tienen acceso a APIs de worker como `fetch`, `IndexedDB`, `setTimeout`, `WebSockets` y la mayoría de APIs modernas. Para comunicación compleja, considera usar `comlink` (librería de Google) que envuelve el Worker en una API más amigable con Promises.
