---
title: ¿Qué son las promesas en JavaScript y por qué ayudan a gestionar asincronía?
difficulty: medium
tags: [promises, async, error-handling]
lastReviewed: 2026-04-06
---

Una promesa representa el resultado futuro de una operación asíncrona. Puede estar pendiente, cumplida o rechazada. Su valor principal no es solo “esperar algo”, sino darte un modelo uniforme para encadenar operaciones y manejar errores.

## Ejemplo de código
```js
function pedirUsuario() {
  return fetch('/api/user')
    .then((response) => {
      if (!response.ok) {
        throw new Error('Respuesta no válida');
      }
      return response.json();
    })
    .then((data) => data.name);
}

pedirUsuario()
  .then((nombre) => console.log(nombre))
  .catch((error) => console.error(error.message));
```

## Qué aportan frente a callbacks puros
- composición más clara;
- manejo de errores centralizado con `.catch()`;
- cadenas legibles de pasos asíncronos;
- APIs estándar (`all`, `allSettled`, `race`, `any`).

## Matices importantes
- Una promesa resuelta no significa “éxito de negocio”; solo significa que terminó en el estado fulfilled.
- Lanzar un error dentro de `.then()` convierte la cadena en rechazada.
- Una promesa empieza a ejecutarse cuando se crea, no cuando haces `.then()`.

## Para interiorizarlo
Mira una promesa como un contrato del futuro: “todavía no tengo el valor, pero sé cómo lo recibiré o cómo sabré que falló”.
