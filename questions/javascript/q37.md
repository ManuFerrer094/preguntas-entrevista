Aquí quieren comprobar si sabes construir componentes de verdad y no solo consumirlos. En especial, si entiendes dónde inicializar, dónde limpiar efectos y cómo reaccionar a cambios de atributos sin meter lógica frágil.

## Qué evalúa el entrevistador
- Si sabes cuándo usar `connectedCallback()` y `disconnectedCallback()`.
- Si conoces `attributeChangedCallback()` y `observedAttributes`.
- Si entiendes que el constructor no es el mejor lugar para depender del DOM exterior.

## Respuesta sólida
- `constructor()`: configura estado interno básico, pero evita leer atributos o tocar DOM externo si no es necesario.
- `connectedCallback()`: se ejecuta cuando el elemento entra en el DOM; es un buen sitio para render inicial, suscripciones o listeners.
- `disconnectedCallback()`: sirve para limpiar listeners, observers, timers o cualquier efecto colgante.
- `attributeChangedCallback(name, oldValue, newValue)`: reacciona a cambios en atributos observados.
- `adoptedCallback()`: menos frecuente, pero útil si el nodo cambia de documento.

## Ejemplo de código

```js
class UserBadge extends HTMLElement {
  static get observedAttributes() {
    return ['status'];
  }

  connectedCallback() {
    this.render();
  }

  disconnectedCallback() {
    this.cleanup?.();
  }

  attributeChangedCallback(name, oldValue, newValue) {
    if (name === 'status' && oldValue !== newValue) {
      this.render();
    }
  }

  render() {
    this.textContent = `Estado: ${this.getAttribute('status') ?? 'offline'}`;
  }
}
```

## Lo que suma en una respuesta senior
- Nombrar la idempotencia: `connectedCallback()` puede ejecutarse más de una vez si el elemento se desconecta y vuelve a conectarse.
- Explicar que el sincronismo entre propiedades y atributos merece una estrategia clara, sobre todo si el componente va a convivir con frameworks.

## Error común
Meter toda la inicialización en el constructor y asumir que el elemento ya está montado, con hijos disponibles o atributos ya listos. En Web Components esa suposición falla con facilidad.

## Frase corta de entrevista
> El ciclo de vida me da puntos claros para inicializar, reaccionar y limpiar: `connectedCallback` para montar, `attributeChangedCallback` para sincronizar y `disconnectedCallback` para evitar fugas.
