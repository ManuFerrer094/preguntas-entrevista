El currying transforma una función que recibe varios argumentos en una secuencia de funciones que reciben un argumento cada una. No es solo una curiosidad funcional: puede mejorar composición y reutilización.

## Ejemplo de código
```js
function multiplicar(a) {
  return function (b) {
    return a * b;
  };
}

const duplicar = multiplicar(2);
console.log(duplicar(5)); // 10
```

## Cuándo ayuda
- cuando quieres preconfigurar funciones;
- cuando trabajas con pipelines o composición funcional;
- cuando quieres crear utilidades especializadas a partir de una función general.

## Diferencia con “función de muchos parámetros”
Una función normal espera todos los argumentos juntos. Una función curryficada te deja aplicar los argumentos por etapas.

## Matices importantes
- No siempre mejora la legibilidad; en código de negocio a veces una función normal es más clara.
- Muchas librerías funcionales aprovechan esta técnica para construir APIs componibles.

## Para interiorizarlo
No memorices la forma. Piensa en la intención: “quiero fijar una parte del comportamiento ahora y completar el resto después”.
