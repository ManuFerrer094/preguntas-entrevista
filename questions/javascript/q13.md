Puedes definir funciones de varias formas. Las más comunes son la declaración de función, la expresión de función y la arrow function.

## Ejemplo de código
```js
function declarada(a, b) {
  return a + b;
}

const expresada = function (a, b) {
  return a + b;
};

const flecha = (a, b) => a + b;
```

## Idea clave
Las tres crean funciones, pero no son idénticas:
- la declaración de función se eleva completa;
- la expresión depende del momento en que se asigna;
- la arrow function tiene reglas propias para `this`, `arguments` y `new`.

## Matices importantes
- No elijas arrow functions solo porque escriben menos; el significado de `this` cambia.
- Las declaraciones de función suelen ser una muy buena opción para utilidades nombradas.
- Las expresiones y arrows son cómodas cuando quieres pasar funciones como valor.

## Para interiorizarlo
La forma de definir una función también comunica intención. Aprende a elegir la sintaxis por semántica, no solo por costumbre.
