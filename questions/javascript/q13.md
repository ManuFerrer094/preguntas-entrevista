---
title: ¿Qué es el this en JavaScript?

`this` en JavaScript es una referencia al contexto de ejecución actual. A diferencia de otros lenguajes, `this` en JavaScript no está determinado por dónde se **define** la función, sino por cómo se **llama**. Esta es una de las características más confusas de JavaScript pero fundamental entender correctamente.

## Las 4 reglas de binding de this

**1. Binding implícito** — el objeto a la izquierda del punto:

```javascript
const person = {
  name: 'Ana',
  greet() {
    console.log(`Hola, soy ${this.name}`); // this = person
  }
};

person.greet(); // "Hola, soy Ana"

// El problema: perder el contexto
const greet = person.greet;
greet(); // "Hola, soy undefined" — this ya no es person, es undefined (strict) o window
```

**2. Binding explícito** — call, apply, bind:

```javascript
function introduce(greeting, punctuation) {
  console.log(`${greeting}, soy ${this.name}${punctuation}`);
}

const user = { name: 'Carlos' };

// call: argumentos separados por coma
introduce.call(user, 'Hola', '!');    // "Hola, soy Carlos!"

// apply: argumentos como array
introduce.apply(user, ['Buenas', '.']); // "Buenas, soy Carlos."

// bind: crea una nueva función con this fijado
const boundIntroduce = introduce.bind(user);
boundIntroduce('Hey', '?'); // "Hey, soy Carlos?"

// bind parcial (partial application)
const formalIntro = introduce.bind(user, 'Buenos días');
formalIntro('!'); // "Buenos días, soy Carlos!"
```

**3. Binding de new** — constructor:

```javascript
function Person(name, age) {
  // Al usar new, 'this' es el nuevo objeto creado
  this.name = name;
  this.age = age;
}

const ana = new Person('Ana', 28);
// ana = { name: 'Ana', age: 28 }
```

**4. Binding por defecto** — función standalone:

```javascript
function showThis() {
  console.log(this);
}

showThis(); // En strict mode: undefined. En no-strict: window/global
```

## Arrow functions — no tienen su propio this

Las arrow functions heredan el `this` del scope donde se **definen** (lexical this), no del contexto donde se llaman:

```javascript
class Timer {
  constructor() {
    this.seconds = 0;
  }

  start() {
    // ❌ Función regular: this dentro del callback no es el Timer
    setInterval(function() {
      this.seconds++; // ❌ this es undefined (o window)
    }, 1000);

    // ✅ Arrow function: hereda this del método start()
    setInterval(() => {
      this.seconds++; // ✅ this es el Timer
    }, 1000);
  }
}

// En React class components (ahora menos común)
class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
    // Necesario si no se usa arrow function
    this.handleClick = this.handleClick.bind(this);
  }

  // Arrow function como método — lexical this
  handleClick = () => {
    this.setState({ count: this.state.count + 1 });
  };
}
```

## Prioridad de las reglas

La prioridad de binding es: **new** > **bind/call/apply** > **implícito** > **por defecto**

```javascript
function test() {
  console.log(this.name);
}

const obj1 = { name: 'obj1', test };
const obj2 = { name: 'obj2' };

obj1.test();                    // 'obj1' — implícito
obj1.test.call(obj2);           // 'obj2' — explícito sobreescribe implícito
const bound = test.bind(obj1);
bound.call(obj2);               // 'obj1' — bind sobreescribe call
```

## this en clases modernas

```javascript
class EventEmitter {
  private listeners: Map<string, Function[]> = new Map();

  // Las clases en JS usan strict mode automáticamente
  // 'this' en métodos es undefined si se llama fuera del contexto de la instancia
  on(event: string, listener: Function) {
    const list = this.listeners.get(event) || [];
    this.listeners.set(event, [...list, listener]);
    return this; // Método encadenable
  }

  emit(event: string, ...args: any[]) {
    this.listeners.get(event)?.forEach(fn => fn(...args));
    return this;
  }
}
```

**Consejo de entrevista**: La pregunta sobre `this` es casi universal en entrevistas JavaScript. Los puntos clave son: (1) el valor de `this` se determina en tiempo de ejecución según cómo se llama la función, (2) las arrow functions no tienen `this` propio, (3) `bind()` crea una nueva función con `this` fijado permanentemente. Demostrar conocimiento de los 4 modos de binding y la diferencia con arrow functions es lo esperado.
