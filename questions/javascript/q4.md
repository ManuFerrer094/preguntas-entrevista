---
title: ¿Qué diferencia hay entre == y === en JavaScript?
difficulty: easy
tags: [equality, operators, types]
lastReviewed: 2026-04-06
---

La diferencia es que `==` compara tras aplicar coerción de tipos si hace falta, mientras que `===` exige que valor y tipo coincidan sin conversiones implícitas.

## Idea clave
`===` es la opción por defecto porque hace el comportamiento más predecible. `==` puede parecer cómodo, pero a menudo oculta conversiones automáticas difíciles de razonar.

## Ejemplo de código
```js
console.log(5 == '5'); // true
console.log(5 === '5'); // false

console.log(false == 0); // true
console.log(false === 0); // false

console.log(null == undefined); // true
console.log(null === undefined); // false
```

## Matices importantes
- `==` no es "malo" por definición, pero sí más propenso a sorpresas.
- Hay un uso intencional relativamente común: `valor == null` para comprobar a la vez `null` y `undefined`.
- Cuando comparas objetos, ambos operadores comparan referencias, no contenido.

## Para interiorizarlo
Si quieres aprender bien JavaScript, acostúmbrate a preguntar: "¿esta comparación está convirtiendo algo sin que yo lo vea?". Si la respuesta no está clara, usa `===`.
