Los patrones de diseño no son recetas mágicas, sino soluciones recurrentes a problemas que aparecen una y otra vez. En JavaScript suelen verse mucho patrones como módulo, factory, observer, pub/sub y singleton, entre otros.

## Ejemplo de factory
```js
function crearUsuario(nombre) {
  return {
    nombre,
    saludar() {
      return `Hola, soy ${nombre}`;
    },
  };
}
```

## Ejemplo de módulo
```js
const contador = (() => {
  let valor = 0;

  return {
    incrementar() {
      valor += 1;
      return valor;
    },
  };
})();
```

## Cómo pensarlo bien
- `factory`: cuando quieres crear objetos sin acoplarte a `new` o a jerarquías rígidas.
- `observer` o `pub/sub`: cuando necesitas comunicar cambios entre partes desacopladas.
- `singleton`: cuando de verdad necesitas una única instancia global, pero con cuidado para no meter dependencias ocultas.

## Para interiorizarlo
No aprendas patrones como catálogo de nombres. Aprende qué problema resuelve cada uno y qué coste introduce. Un patrón bien usado simplifica; mal usado, complica más de lo que ayuda.
