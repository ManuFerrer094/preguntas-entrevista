Cuando la entrevista baja a fugas de memoria, ya no quieren teoría aislada: quieren ver si sabes localizar referencias que se quedan vivas más de la cuenta. La respuesta buena une dos ideas: qué retiene un objeto y qué patrón usar para que esa retención no sea accidental.

## Qué evalúa el entrevistador
- Si entiendes que las referencias circulares por sí solas no son el problema; lo peligroso es que sigan siendo alcanzables desde una raíz.
- Si sabes detectar las fuentes típicas de retención: caches globales, listeners, timers, observadores, mapas de metadatos y closures de larga vida.
- Si conoces estructuras débiles como `WeakMap` y `WeakSet`.

## Respuesta sólida
- Una referencia circular entre dos objetos no impide el GC si ambos quedan fuera del grafo alcanzable.
- La fuga aparece cuando esa relación queda anclada por una referencia fuerte desde fuera, por ejemplo una caché global o un registro de elementos montados.
- Para asociaciones auxiliares entre objetos y metadatos, `WeakMap` o `WeakSet` son útiles porque no mantienen viva la clave por sí mismos.

## Ejemplo práctico
Imagina una librería de componentes que guarda configuración por elemento DOM:

```js
const metadata = new WeakMap();

function register(element, config) {
  metadata.set(element, config);
}
```

Si el elemento desaparece del DOM y no quedan otras referencias fuertes, esa entrada puede desaparecer junto con él. Con un `Map` tradicional, en cambio, la estructura seguiría reteniendo la referencia.

## Qué diría sobre prevención
- Limpiar listeners en desmontaje o desconexión.
- Cancelar `setInterval`, `MutationObserver`, `ResizeObserver` o suscripciones.
- Evitar caches eternas sin política de invalidación.
- Revisar closures que capturan objetos pesados cuando el contexto ya terminó.
- Usar herramientas de heap snapshots para confirmar quién está reteniendo qué.

## Error común
Responder "las referencias circulares siempre producen fugas". Eso era una simplificación histórica. En motores modernos con `mark-and-sweep`, el problema real es la alcanzabilidad, no el simple hecho de que dos objetos se apunten entre sí.

## Frase corta de entrevista
> No me preocupa una referencia circular aislada; me preocupa quién la sigue reteniendo. Si la asociación es auxiliar, prefiero `WeakMap` o `WeakSet` para no impedir la recolección.
