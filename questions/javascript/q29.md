---
title: ¿Qué es Object.freeze() y cuándo usarlo en JavaScript?
difficulty: easy
tags: [javascript, object-freeze, immutability, const, shallow]
---

`Object.freeze()` hace un objeto **inmutable superficialmente**: previene agregar, eliminar o modificar sus propiedades. Es útil para constantes de configuración, estados que no deben mutar y proteger objetos de modificaciones accidentales.

## Uso básico

```javascript
const config = Object.freeze({
  apiUrl: 'https://api.ejemplo.com',
  version: '2.0',
  timeout: 5000
});

// ❌ Intentar modificar en modo estricto lanza TypeError:
config.apiUrl = 'https://otro.com';      // TypeError (strict mode) / silencioso (sloppy)
config.nuevaPropiedad = 'valor';          // TypeError en strict mode
delete config.version;                    // TypeError en strict mode

// Verificar si está congelado:
console.log(Object.isFrozen(config));    // true

// Las propiedades se pueden leer normalmente:
console.log(config.apiUrl);             // 'https://api.ejemplo.com'
```

## freeze() es SUPERFICIAL (shallow)

```javascript
const estado = Object.freeze({
  usuario: {
    nombre: 'Ana',    // Este objeto anidado NO está congelado
    email: 'ana@ejemplo.com'
  },
  configuracion: {
    tema: 'claro'
  }
});

// ❌ Las propiedades directas no se pueden modificar:
// estado.usuario = null;  // TypeError

// ✅ Pero las propiedades anidadas SÍ se pueden modificar:
estado.usuario.nombre = 'Carlos';  // ¡Funciona! El objeto anidado no está congelado
estado.configuracion.tema = 'oscuro';  // ¡Funciona!

console.log(estado.usuario.nombre);   // 'Carlos' — fue modificado
```

## Deep freeze — Congelación profunda

```javascript
// Función para congelar recursivamente
function deepFreeze(obj) {
  // Obtener propiedades del objeto (incluyendo no-enumerables)
  const propNames = Reflect.ownKeys(obj);

  // Congelar primero las propiedades anidadas (depth-first)
  for (const nombre of propNames) {
    const valor = obj[nombre];
    if (valor && typeof valor === 'object') {
      deepFreeze(valor);
    }
  }

  return Object.freeze(obj);
}

const config = deepFreeze({
  api: {
    url: 'https://api.ejemplo.com',
    timeout: 5000,
    headers: {
      'Content-Type': 'application/json'
    }
  },
  caracteristicas: ['feature1', 'feature2']
});

// Ahora NADA se puede modificar:
config.api.url = 'otro';                     // TypeError
config.api.headers['Authorization'] = '...'; // TypeError
config.caracteristicas.push('feature3');      // TypeError
```

## Casos de uso prácticos

```javascript
// 1. Constantes de configuración global
export const CONFIG = Object.freeze({
  API_URL: process.env.NEXT_PUBLIC_API_URL,
  MAX_REINTENTOS: 3,
  TIMEOUT_MS: 10000,
  ENDPOINTS: Object.freeze({
    USUARIOS: '/api/usuarios',
    PRODUCTOS: '/api/productos',
    PEDIDOS: '/api/pedidos'
  })
});

// 2. Enumeraciones / constantes de dominio
export const ESTADO_PEDIDO = Object.freeze({
  PENDIENTE: 'PENDIENTE',
  PROCESANDO: 'PROCESANDO',
  ENVIADO: 'ENVIADO',
  ENTREGADO: 'ENTREGADO',
  CANCELADO: 'CANCELADO'
});

// Iterable (ventaja sobre TypeScript enum):
Object.values(ESTADO_PEDIDO).forEach(estado => console.log(estado));

// 3. Estado inicial de Redux/Zustand
const ESTADO_INICIAL = Object.freeze({
  usuarios: [],
  cargando: false,
  error: null,
  paginacion: Object.freeze({ pagina: 1, limite: 10, total: 0 })
});

// 4. Proteger objetos devueltos por funciones
function obtenerConfiguracion() {
  const config = {
    host: 'localhost',
    puerto: 3000
  };
  return Object.freeze(config);  // El consumidor no puede modificarla
}
```

## Object.freeze() vs const vs Object.seal()

```javascript
// const: previene reasignación de la variable, NO previene modificación del objeto
const obj = { a: 1 };
obj.a = 2;      // ✅ Funciona — const no hace el objeto inmutable
// obj = {};   // ❌ Error — no se puede reasignar la variable

// Object.seal(): previene agregar/eliminar props, pero SÍ permite modificarlas
const sellado = Object.seal({ a: 1, b: 2 });
sellado.a = 99;       // ✅ Puede modificar propiedades existentes
sellado.c = 3;        // ❌ No puede agregar nuevas propiedades
delete sellado.a;     // ❌ No puede eliminar propiedades

// Object.freeze(): previene agregar, eliminar Y modificar propiedades
const congelado = Object.freeze({ a: 1, b: 2 });
congelado.a = 99;     // ❌ No puede modificar
congelado.c = 3;      // ❌ No puede agregar
delete congelado.a;   // ❌ No puede eliminar
```

| Operación              | `const` | `Object.seal()` | `Object.freeze()` |
|------------------------|---------|-----------------|-------------------|
| Reasignar variable     | ❌      | ✅              | ✅                |
| Modificar propiedades  | ✅      | ✅              | ❌                |
| Agregar propiedades    | ✅      | ❌              | ❌                |
| Eliminar propiedades   | ✅      | ❌              | ❌                |
| Profundidad            | N/A     | Superficial     | Superficial        |

**Consejo de entrevista**: La confusión más común es pensar que `const` hace el objeto inmutable —no es así. `const` solo previene la reasignación de la variable. `Object.freeze()` es la forma correcta de hacer un objeto inmutable (superficialmente). Para inmutabilidad profunda en aplicaciones grandes, se usan librerías como `Immer` (que usa Proxies) o `Immutable.js`, que son más eficientes que el `deepFreeze` manual para objetos grandes y actualizaciones frecuentes.
