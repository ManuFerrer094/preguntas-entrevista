Manejar errores bien no consiste en poner `try/catch` por todas partes, sino en decidir dónde capturar, dónde enriquecer contexto, cuándo propagar y cómo evitar estados incoherentes.

## Estrategias frecuentes
- `try/catch` para código síncrono o `await`;
- `.catch()` en cadenas de promesas;
- lanzar errores con `throw` cuando la función no puede cumplir su contrato;
- crear errores de dominio más expresivos;
- registrar contexto útil para diagnóstico.

## Ejemplo de código
```js
async function cargarPerfil() {
  try {
    const response = await fetch('/api/profile');

    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`);
    }

    return await response.json();
  } catch (error) {
    console.error('No se pudo cargar el perfil', error);
    throw error;
  }
}
```

## Qué conviene evitar
- capturar un error y silenciarlo sin contexto;
- devolver valores ambiguos que mezclan datos válidos y fallos;
- usar excepciones para flujo normal de negocio cuando hay alternativas más claras.

## Para interiorizarlo
Una buena estrategia de errores busca dos cosas: que el sistema pueda reaccionar correctamente y que tú puedas entender por qué falló cuando algo salga mal.
