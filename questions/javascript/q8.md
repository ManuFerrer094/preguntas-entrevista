---
title: ¿Qué es el patrón Module en JavaScript?
difficulty: medium

El patrón Module en JavaScript es una forma de organizar y encapsular código, exponiendo solo lo que es necesario (API pública) y ocultando los detalles de implementación (variables y funciones privadas). En JavaScript moderno, esto se logra con los módulos nativos ES6, aunque el patrón original usaba IIFEs (Immediately Invoked Function Expressions).

## ES Modules (ESM) — la forma moderna

```javascript
// math.js — exportaciones nombradas
const PI = 3.14159;
let _instanceCount = 0; // "privado" por convención (no exportado)

export function add(a, b) { return a + b; }
export function multiply(a, b) { return a * b; }
export function getCircleArea(r) { return PI * r ** 2; }
export const GOLDEN_RATIO = 1.618033;

// math.js — exportación por defecto (una por módulo)
export default class Calculator {
  add(a, b) { return a + b; }
}
```

```javascript
// app.js — importaciones
import Calculator from './math.js';              // Default import
import { add, multiply, PI } from './math.js';  // Named imports
import * as MathUtils from './math.js';          // Namespace import
import { add as sum } from './math.js';          // Renamed import

// Dynamic imports (carga diferida)
const { add } = await import('./math.js');

// Tree-shaking: el bundler elimina lo que no se importa
```

## CommonJS (Node.js tradicional)

```javascript
// utils.js
const _ = require('lodash'); // Dependencia interna

let _cache = {}; // Variable "privada" (no exportada)

function fetchUser(id) {
  if (_cache[id]) return _cache[id];
  const user = db.findById(id);
  _cache[id] = user;
  return user;
}

function clearCache() { _cache = {}; }

// Solo exportar la API pública
module.exports = { fetchUser, clearCache };

// app.js
const { fetchUser } = require('./utils');
```

## El patrón Module clásico con IIFE

Antes de los módulos nativos, se usaba el patrón IIFE para crear alcance privado:

```javascript
const Counter = (() => {
  // Estado privado — no accesible desde fuera
  let count = 0;
  const history = [];

  // API pública
  return {
    increment() {
      count++;
      history.push({ action: 'increment', value: count });
    },
    decrement() {
      count--;
      history.push({ action: 'decrement', value: count });
    },
    getCount() { return count; },
    getHistory() { return [...history]; } // Copia para proteger el array original
  };
})();

Counter.increment();
Counter.increment();
console.log(Counter.getCount()); // 2
// console.log(count); // ❌ ReferenceError: count no está definido
```

## Módulo Singleton

```javascript
// database.js — módulo que se instancia una sola vez
let instance = null;

class Database {
  constructor(connectionString) {
    if (instance) return instance; // Singleton pattern
    this.connection = connect(connectionString);
    instance = this;
  }

  query(sql) { return this.connection.execute(sql); }
}

export default Database;

// En cualquier parte de la aplicación:
import Database from './database.js';
const db = new Database('postgres://...'); // Solo se conecta una vez
```

## Revealing Module Pattern

```javascript
// Patrón que define todo internamente y solo revela lo necesario
const userModule = (() => {
  // Privado
  const users = [];
  let nextId = 1;

  function validateUser(user) {
    return user.name && user.email;
  }

  function findById(id) {
    return users.find(u => u.id === id);
  }

  // Público
  function addUser(userData) {
    if (!validateUser(userData)) throw new Error('Usuario inválido');
    const user = { ...userData, id: nextId++ };
    users.push(user);
    return user;
  }

  function getUser(id) {
    return findById(id);
  }

  // Revelar solo la API pública
  return { addUser, getUser };
})();
```

**Consejo de entrevista**: Los ES Modules son el estándar moderno. Sus ventajas sobre CommonJS son: análisis estático (permite tree-shaking), carga asíncrona nativa, scope estricto por defecto, y soporte nativo en navegadores. La importación dinámica (`import()`) es útil para code-splitting y carga condicional de módulos.
