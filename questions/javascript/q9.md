---
title: ¿Qué son WeakMap y WeakSet en JavaScript?
difficulty: hard

`WeakMap` y `WeakSet` son estructuras de datos especiales de JavaScript que mantienen referencias débiles a sus claves (WeakMap) u objetos (WeakSet). "Débil" significa que si no hay otras referencias al objeto, el garbage collector puede liberarlo de memoria aunque esté en el WeakMap/WeakSet. Esto los hace ideales para situaciones donde no queremos que la estructura de datos impida la liberación de memoria.

## WeakMap — mapa con claves débiles

Las claves de un `WeakMap` deben ser **objetos** (no primitivos). Si el objeto clave es recolectado por el garbage collector, la entrada se elimina automáticamente.

```javascript
const cache = new WeakMap();

function processElement(element) {
  // Verificar si ya tenemos el resultado cacheado para este elemento DOM
  if (cache.has(element)) {
    return cache.get(element);
  }

  const result = expensiveComputation(element);
  cache.set(element, result);
  return result;
}

// Cuando el elemento DOM se elimina del documento, el cache también se libera
// No hay memory leak aunque no llamemos explícitamente a cache.delete(element)
```

**Datos privados con WeakMap:**

```javascript
// Patrón clásico para datos "privados" en clases
const _privateData = new WeakMap();

class BankAccount {
  constructor(owner, balance) {
    _privateData.set(this, { balance, transactions: [] });
    this.owner = owner;
  }

  deposit(amount) {
    const data = _privateData.get(this);
    data.balance += amount;
    data.transactions.push({ type: 'deposit', amount, date: new Date() });
  }

  withdraw(amount) {
    const data = _privateData.get(this);
    if (data.balance < amount) throw new Error('Saldo insuficiente');
    data.balance -= amount;
    data.transactions.push({ type: 'withdrawal', amount, date: new Date() });
  }

  getBalance() {
    return _privateData.get(this).balance;
  }
}

const account = new BankAccount('Ana', 1000);
account.deposit(500);
console.log(account.getBalance()); // 1500
// _privateData no es accesible desde fuera del módulo
```

**Nota**: En JavaScript moderno, los campos privados con `#` son preferibles:

```javascript
class BankAccount {
  #balance;
  #transactions = [];

  constructor(balance) {
    this.#balance = balance;
  }

  get balance() { return this.#balance; }
}
```

## WeakSet — conjunto con referencias débiles

Un `WeakSet` solo puede contener objetos. Útil para marcar objetos sin crear referencias fuertes.

```javascript
// Marcar elementos que ya han sido procesados
const processedElements = new WeakSet();

function processOnce(element) {
  if (processedElements.has(element)) {
    console.log('Ya fue procesado');
    return;
  }

  doProcess(element);
  processedElements.add(element);
}

// Detectar referencias circulares al serializar
function safeStringify(obj) {
  const seen = new WeakSet();

  return JSON.stringify(obj, (key, value) => {
    if (typeof value === 'object' && value !== null) {
      if (seen.has(value)) {
        return '[Circular Reference]';
      }
      seen.add(value);
    }
    return value;
  });
}

const circular = { name: 'test' };
circular.self = circular;
console.log(safeStringify(circular)); // {"name":"test","self":"[Circular Reference]"}
```

## Diferencias con Map y Set normales

| | Map/Set | WeakMap/WeakSet |
|---|---|---|
| Tipo de clave/valor | Cualquiera | Solo objetos |
| Referencias | Fuertes | Débiles |
| Garbage collection | Previene GC | Permite GC |
| Iterable | ✅ (forEach, for...of) | ❌ No iterable |
| `.size` | ✅ | ❌ No disponible |
| Uso típico | Colecciones generales | Cache, datos privados |

**Consejo de entrevista**: La pregunta sobre WeakMap/WeakSet suele aparecer en entrevistas para roles avanzados. Los puntos clave son: las referencias débiles permiten que el garbage collector libere memoria (evitando memory leaks), no son iterables (no se puede saber qué hay dentro), y los casos de uso típicos son caching asociado a objetos DOM/instancias y datos verdaderamente privados en clases.
