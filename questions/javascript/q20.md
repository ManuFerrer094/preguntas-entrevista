`this` es una de las partes más malentendidas de JavaScript porque no depende de dónde se define una función normal, sino de cómo se invoca. Su valor se determina en el momento de la llamada.

## Ejemplo de código
```js
const usuario = {
  nombre: 'Ana',
  saludar() {
    return `Hola, soy ${this.nombre}`;
  },
};

console.log(usuario.saludar()); // 'Hola, soy Ana'
```

## Idea clave
En una función normal, `this` depende del call-site:
- como método: suele apuntar al objeto receptor;
- con `new`: apunta a la nueva instancia;
- con `call`, `apply` o `bind`: puedes fijarlo explícitamente.

Las arrow functions no tienen su propio `this`; heredan el del contexto exterior.

## Ejemplo con arrow function
```js
const contador = {
  valor: 0,
  incrementar() {
    setTimeout(() => {
      this.valor += 1;
      console.log(this.valor);
    }, 100);
  },
};

contador.incrementar();
```

## Para interiorizarlo
Cada vez que uses `this`, deja de mirar solo la función y mira cómo se está invocando. Ahí está casi siempre la respuesta.
