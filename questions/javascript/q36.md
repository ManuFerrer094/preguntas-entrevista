Los generators son funciones especiales que pueden pausar y reanudar su ejecución. Se definen con `function*` y devuelven un iterador. Cada vez que llamas a `.next()`, la función continúa desde donde se había quedado.

## Ejemplo de código
```js
function* secuenciaInfinita() {
  let i = 0;
  while (true) {
    yield i++;
  }
}

const gen = secuenciaInfinita();
console.log(gen.next().value); // 0
console.log(gen.next().value); // 1
console.log(gen.next().value); // 2
```

## Qué aportan
- modelar secuencias perezosas;
- controlar flujos complejos paso a paso;
- construir iteradores personalizados;
- procesar grandes volúmenes de datos sin materializarlo todo a la vez.

## Matices importantes
- `yield` pausa la ejecución y entrega un valor.
- El estado interno del generator se conserva entre llamadas.
- No son lo mismo que `async/await`, aunque ambos introducen pausas en la ejecución.

## Para interiorizarlo
Piensa en un generator como una función que no “termina de golpe”, sino que te entrega valores por tramos mientras conserva su contexto entre cada parada.
