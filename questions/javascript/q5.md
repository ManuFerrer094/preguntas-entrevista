En JavaScript puedes escribir comentarios de una línea con `//` y comentarios multilínea con `/* ... */`.

## Ejemplo de código
```js
// Este comentario explica la intención de la línea siguiente.
const iva = 0.21;

/*
  Este bloque puede usarse para explicar
  una decisión que requiere más contexto.
*/
function calcularTotal(base) {
  return base + base * iva;
}
```

## Idea clave
El objetivo de un comentario no es repetir lo obvio, sino explicar algo que el código no deja claro por sí solo: una decisión de negocio, una limitación técnica, un workaround o una advertencia.

## Matices importantes
- Un comentario como `// suma 1` encima de `x + 1` no aporta casi nada.
- Si necesitas demasiados comentarios para entender una función, quizá el problema es el diseño de la función.
- También existen comentarios de documentación como JSDoc para describir contratos, parámetros o tipos esperados.

## Para interiorizarlo
Antes de escribir un comentario, pregúntate: "¿esto explica el porqué o solo repite el qué?". Los buenos comentarios suelen responder al porqué.
