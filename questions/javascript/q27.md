---
title: ¿Para qué sirven async y await en JavaScript?
difficulty: medium

`async` y `await` permiten escribir código asíncrono con una forma más parecida al código secuencial. No eliminan las promesas: se apoyan en ellas.

## Ejemplo de código
```js
async function cargarUsuario() {
  try {
    const response = await fetch('/api/user');

    if (!response.ok) {
      throw new Error('No se pudo cargar el usuario');
    }

    const data = await response.json();
    return data;
  } catch (error) {
    console.error(error.message);
    throw error;
  }
}
```

## Idea clave
- Una función `async` siempre devuelve una promesa.
- `await` pausa esa función concreta hasta que la promesa se resuelve o se rechaza.
- La pausa no bloquea todo JavaScript; solo suspende esa función asíncrona.

## Error común
Usar `await` en serie cuando tareas independientes podrían ejecutarse en paralelo.

```js
const [usuario, cuentas] = await Promise.all([
  fetch('/api/user').then((r) => r.json()),
  fetch('/api/accounts').then((r) => r.json()),
]);
```

## Para interiorizarlo
`async/await` mejora legibilidad, pero no sustituye el pensamiento sobre concurrencia, errores y dependencias entre operaciones.
