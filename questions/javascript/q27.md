El **currying** es una técnica de programación funcional que transforma una función que recibe múltiples argumentos en una secuencia de funciones que cada una recibe un único argumento. Permite la **aplicación parcial** de funciones y crea funciones más especializadas.

## Concepto básico

```javascript
// Función normal: recibe todos los argumentos a la vez
function sumar(a, b, c) {
  return a + b + c;
}
sumar(1, 2, 3);  // 6

// Función currificada: recibe un argumento a la vez
function sumarCurried(a) {
  return function(b) {
    return function(c) {
      return a + b + c;
    };
  };
}

// Uso:
sumarCurried(1)(2)(3);  // 6

// También permite aplicación parcial:
const sumar1 = sumarCurried(1);      // Fija `a = 1`
const sumar1y2 = sumar1(2);          // Fija `b = 2`
console.log(sumar1y2(3));            // 6
console.log(sumar1y2(10));           // 13
```

## Currying con arrow functions (sintaxis moderna)

```javascript
// Más conciso con arrow functions
const sumar = a => b => c => a + b + c;

const multiplicar = x => y => x * y;
const doble = multiplicar(2);   // x = 2 fijado
const triple = multiplicar(3);  // x = 3 fijado

console.log(doble(5));   // 10
console.log(triple(5));  // 15

// Pipelines de transformación
const calcularPrecio = moneda => impuesto => descuento => precio =>
  `${moneda}${(precio * (1 + impuesto) * (1 - descuento)).toFixed(2)}`;

const precioEUR = calcularPrecio('€');
const precioEURconIVA = precioEUR(0.21);
const precioFinal = precioEURconIVA(0.1);  // 10% descuento

console.log(precioFinal(100));   // "€108.90"
console.log(precioFinal(200));   // "€217.80"
```

## Función curry genérica

```javascript
// Función que convierte cualquier función en currificada
function curry(fn) {
  return function currificada(...args) {
    if (args.length >= fn.length) {
      // Tenemos todos los argumentos necesarios
      return fn.apply(this, args);
    }
    // Faltan argumentos: devolver función que espera más
    return function(...masArgs) {
      return currificada.apply(this, args.concat(masArgs));
    };
  };
}

// Uso:
const sumar = curry((a, b, c) => a + b + c);
console.log(sumar(1)(2)(3));    // 6
console.log(sumar(1, 2)(3));    // 6 (también acepta múltiples a la vez)
console.log(sumar(1)(2, 3));    // 6
console.log(sumar(1, 2, 3));    // 6

const filtrar = curry((predicado, array) => array.filter(predicado));
const filtrarPares = filtrar(n => n % 2 === 0);
const filtrarMayoresDe5 = filtrar(n => n > 5);

console.log(filtrarPares([1, 2, 3, 4, 5, 6]));        // [2, 4, 6]
console.log(filtrarMayoresDe5([1, 4, 6, 7, 8, 2]));   // [6, 7, 8]
```

## Casos de uso prácticos

```javascript
// 1. Configuración de loggers
const crearLogger = nivel => contexto => mensaje =>
  `[${nivel.toUpperCase()}][${contexto}] ${new Date().toISOString()}: ${mensaje}`;

const logInfo = crearLogger('info');
const logError = crearLogger('error');

const logInfoApp = logInfo('App');
const logInfoDB = logInfo('Database');

console.log(logInfoApp('Servidor iniciado'));
console.log(logInfoDB('Conexión establecida'));
console.log(logError('Auth')('Token inválido'));

// 2. Validadores componibles
const validarCampo = nombre => validadores => valor => {
  for (const validar of validadores) {
    const error = validar(valor);
    if (error) return { campo: nombre, error };
  }
  return null;
};

const requerido = v => !v?.trim() ? 'Este campo es requerido' : null;
const minLength = min => v => v.length < min ? `Mínimo ${min} caracteres` : null;
const esEmail = v => !v.includes('@') ? 'Email no válido' : null;

const validarEmail = validarCampo('email')([requerido, esEmail]);
const validarNombre = validarCampo('nombre')([requerido, minLength(2)]);

console.log(validarEmail('no-es-email'));   // { campo: 'email', error: 'Email no válido' }
console.log(validarEmail('ok@test.com'));   // null (válido)
console.log(validarNombre(''));             // { campo: 'nombre', error: 'Este campo es requerido' }

// 3. Event handlers configurables
const manejarEvento = tipo => handler => elemento => {
  elemento.addEventListener(tipo, handler);
  return () => elemento.removeEventListener(tipo, handler);  // Retorna cleanup
};

const alHacerClic = manejarEvento('click');
const alPresionarTecla = manejarEvento('keydown');

const removerClickBoton = alHacerClic(
  () => console.log('botón clicado')
)(document.getElementById('btn'));

// Remover el listener cuando no se necesite:
removerClickBoton();
```

## Diferencia entre currying y aplicación parcial

```javascript
// Currying: siempre transforma en funciones de UN argumento
const currificada = a => b => c => a + b + c;

// Aplicación parcial: fija ALGUNOS argumentos, no necesariamente uno a uno
function sumarTres(a, b, c) { return a + b + c; }

const parcial = sumarTres.bind(null, 10);       // Fija a = 10
console.log(parcial(2, 3));                      // 15 (recibe b y c juntos)

// curry con múltiples argumentos en cada paso
const parcialManual = curry(sumarTres);
console.log(parcialManual(10, 2)(3));            // 15
```

**Consejo de entrevista**: El currying es un pilar de la programación funcional. Sus ventajas son: 1) **reutilización** (crear versiones especializadas de funciones), 2) **composición** (funciones de un argumento se componen fácilmente), 3) **lazy evaluation** (la función no se ejecuta hasta tener todos los argumentos). Librerías como Ramda y lodash/fp proveen curry y funciones currificadas por defecto. En entrevistas de empresas que usan programación funcional, este tema aparece frecuentemente junto con `compose` y `pipe`.
