Depurar bien JavaScript es una mezcla de herramientas y método. Las herramientas te enseñan el estado; el método te ayuda a formular hipótesis y descartarlas rápido.

## Herramientas clave
- `console.log`, `console.table`, `console.time` para inspección rápida;
- breakpoints en DevTools;
- inspección de call stack, scope y valores en tiempo real;
- pestañas de Network, Performance y Memory;
- source maps para depurar código transformado.

## Ejemplo de técnica útil
```js
function procesarPedido(pedido) {
  console.log('Entrada', pedido);
  debugger;
  // ... resto de lógica
}
```

## Método que suele funcionar
1. reproducir el fallo;
2. aislar la ruta mínima;
3. observar estado real, no el estado que crees tener;
4. encontrar el primer punto donde la realidad diverge de tu expectativa.

## Para interiorizarlo
Depurar no es lanzar logs al azar. Es reducir incertidumbre paso a paso hasta localizar el punto exacto donde el programa deja de comportarse como esperabas.
