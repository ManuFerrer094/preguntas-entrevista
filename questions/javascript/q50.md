`WeakMap` y `WeakSet` son colecciones débiles. Su particularidad es que no impiden que los objetos que contienen sean recolectados por el garbage collector si ya no existen otras referencias fuertes hacia ellos.

## Ejemplo con WeakMap
```js
const metadata = new WeakMap();

let elemento = { id: 1 };
metadata.set(elemento, { renderizado: true });

console.log(metadata.get(elemento));

elemento = null;
```

## Diferencias respecto a Map y Set
- `WeakMap` solo acepta objetos como clave.
- `WeakSet` solo acepta objetos como valor.
- No puedes iterarlos directamente porque sus elementos pueden desaparecer en cualquier momento al ser recolectados.

## Cuándo son útiles
- asociar metadatos a objetos sin alargar artificialmente su vida;
- caches auxiliares ligadas a instancias o nodos DOM;
- evitar ciertas fugas de memoria en asociaciones temporales.

## Para interiorizarlo
No los veas como “Map y Set raros”. Son herramientas para modelar relaciones que no deberían mantener vivo un objeto por sí mismas. Ahí está su verdadero valor.
