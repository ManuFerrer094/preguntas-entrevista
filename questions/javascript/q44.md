---
title: ¿Qué diferencia hay entre apply, call y bind en JavaScript?
difficulty: hard

`call`, `apply` y `bind` permiten controlar explícitamente el valor de `this` al trabajar con funciones normales.

## Ejemplo de código
```js
function saludar(saludo, despedida) {
  return `${saludo}, soy ${this.nombre}. ${despedida}`;
}

const persona = { nombre: 'Luis' };

console.log(saludar.call(persona, 'Hola', 'Adiós'));
console.log(saludar.apply(persona, ['Hola', 'Adiós']));

const saludarLuis = saludar.bind(persona, 'Hola');
console.log(saludarLuis('Hasta luego'));
```

## Diferencia exacta
- `call`: invoca la función al momento y recibe argumentos sueltos.
- `apply`: invoca la función al momento y recibe los argumentos como array.
- `bind`: no ejecuta; devuelve una nueva función con `this` fijado y, si quieres, con argumentos preaplicados.

## Para qué sigue importando
Aunque hoy usemos mucho `...args` o arrow functions, entender estas APIs aclara `this`, reuso de funciones y técnicas de partial application.

## Para interiorizarlo
La pregunta útil no es “¿qué método me sé?”, sino “¿quiero ejecutar ya o crear una función preparada para más tarde?”. Ahí se separa `bind` de `call/apply`.
