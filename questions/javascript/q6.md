---
title: ¿Qué es la desestructuración en JavaScript?
difficulty: easy
tags: [javascript, destructuring, es6, syntax]
---

La desestructuración (destructuring) es una sintaxis de JavaScript (ES6) que permite extraer valores de arrays u objetos y asignarlos a variables de forma concisa. Hace el código más legible y reduce la cantidad de código necesario para acceder a propiedades anidadas.

## Desestructuración de objetos

```javascript
const user = {
  id: 1,
  name: 'Ana García',
  email: 'ana@example.com',
  address: {
    city: 'Madrid',
    country: 'España'
  },
  roles: ['admin', 'user']
};

// Extracción básica
const { id, name, email } = user;

// Renombrar la variable
const { name: userName, email: userEmail } = user;
console.log(userName); // 'Ana García'

// Valores por defecto (si la propiedad es undefined)
const { name, age = 25, isActive = true } = user;

// Desestructuración anidada
const { address: { city, country } } = user;
console.log(city); // 'Madrid'

// Combinar extracción y resto
const { id, ...rest } = user;
console.log(rest); // { name, email, address, roles }
```

## Desestructuración de arrays

```javascript
const colors = ['red', 'green', 'blue', 'yellow'];

// Extracción por posición
const [first, second] = colors;
console.log(first);  // 'red'
console.log(second); // 'green'

// Saltar elementos
const [, , third] = colors;
console.log(third); // 'blue'

// Rest en arrays
const [primary, ...others] = colors;
console.log(others); // ['green', 'blue', 'yellow']

// Swap de variables
let a = 1, b = 2;
[a, b] = [b, a];
console.log(a, b); // 2 1

// Valores por defecto
const [x = 10, y = 20] = [5];
console.log(x, y); // 5 20
```

## Desestructuración en parámetros de función

```javascript
// Sin desestructuración (verboso)
function displayUser(user) {
  console.log(`${user.name} (${user.email})`);
}

// Con desestructuración (limpio)
function displayUser({ name, email, age = 'No especificado' }) {
  console.log(`${name} (${email}) - Edad: ${age}`);
}

// Muy común en React/Vue con props
function UserCard({ name, avatar, onClick }) {
  return `<button onclick="${onClick}">${name}</button>`;
}

// Con arrays (útil para hooks de React)
function useState(initial) {
  // ...
  return [value, setter]; // Devuelve un array para desestructurar
}
const [count, setCount] = useState(0);
```

## Desestructuración en bucles

```javascript
const users = [
  { id: 1, name: 'Ana', role: 'admin' },
  { id: 2, name: 'Carlos', role: 'user' }
];

// Desestructurar en el for...of
for (const { id, name, role } of users) {
  console.log(`${id}: ${name} (${role})`);
}

// Con Object.entries()
const config = { host: 'localhost', port: 3000, debug: true };
for (const [key, value] of Object.entries(config)) {
  console.log(`${key}: ${value}`);
}
```

## Casos de uso reales

```javascript
// Importaciones de módulos
const { readFile, writeFile, mkdir } = require('fs/promises');
import { useState, useEffect, useCallback } from 'react';

// Respuestas de API
const { data: users, total, page } = await fetchUsers();

// Funciones que devuelven múltiples valores
function getMinMax(numbers) {
  return { min: Math.min(...numbers), max: Math.max(...numbers) };
}
const { min, max } = getMinMax([1, 5, 3, 9, 2]);

// Promise.all
const [user, posts, comments] = await Promise.all([
  fetchUser(id),
  fetchPosts(id),
  fetchComments(id)
]);
```

**Consejo de entrevista**: La desestructuración es ubicua en JavaScript moderno. Puntos importantes: los valores por defecto solo se aplican cuando la propiedad es `undefined` (no cuando es `null`), el operador rest (`...`) en desestructuración siempre debe ser el último elemento, y renombrar variables (`{ prop: newName }`) es diferente a asignar valor por defecto (`{ prop = defaultValue }`).
