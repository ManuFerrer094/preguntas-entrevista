title: ¿Qué son los Union Types y Intersection Types en TypeScript?

Los Union Types e Intersection Types son dos formas de combinar tipos en TypeScript que permiten expresar relaciones complejas entre tipos de forma precisa.

## Union Types (|) — "A o B"

Un Union Type permite que un valor sea de uno de varios tipos. Se define con el operador `|`.

```typescript
// Un valor puede ser string o number
type ID = string | number;
const userId: ID = '123';    // ✅
const postId: ID = 456;      // ✅
// const invalid: ID = true; // ❌ Error

// Union con tipos literales (muy útil)
type Status = 'pending' | 'active' | 'inactive' | 'deleted';
type Direction = 'north' | 'south' | 'east' | 'west';
type HttpMethod = 'GET' | 'POST' | 'PUT' | 'DELETE' | 'PATCH';

function setStatus(id: number, status: Status): void {
  // TypeScript garantiza que status es uno de los valores válidos
}

// Nullable types — patrón común
type Nullable<T> = T | null;
type Optional<T> = T | null | undefined;

const user: Nullable<User> = null; // El usuario puede no estar cargado todavía
```

## Narrowing con Union Types

Al usar union types, TypeScript requiere manejar cada caso posible:

```typescript
function processInput(input: string | number | boolean) {
  if (typeof input === 'string') {
    return input.toUpperCase(); // TypeScript sabe que es string aquí
  } else if (typeof input === 'number') {
    return input.toFixed(2);   // TypeScript sabe que es number aquí
  } else {
    return input ? 'Sí' : 'No'; // TypeScript sabe que es boolean aquí
  }
}

// Con interfaces — usar discriminated union (ver pregunta dedicada)
type Shape =
  | { kind: 'circle'; radius: number }
  | { kind: 'square'; side: number }
  | { kind: 'rectangle'; width: number; height: number };

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case 'circle':    return Math.PI * shape.radius ** 2;
    case 'square':    return shape.side ** 2;
    case 'rectangle': return shape.width * shape.height;
  }
}
```

## Intersection Types (&) — "A y B"

Un Intersection Type combina múltiples tipos, resultando en un tipo que tiene **todas** las propiedades de todos los tipos combinados.

```typescript
type Timestamped = {
  createdAt: Date;
  updatedAt: Date;
};

type WithId = {
  id: string;
};

type User = {
  name: string;
  email: string;
};

// Un DatabaseRecord tiene todas las propiedades de User, WithId y Timestamped
type DatabaseRecord<T> = T & WithId & Timestamped;

type UserRecord = DatabaseRecord<User>;
// Equivale a: { name: string; email: string; id: string; createdAt: Date; updatedAt: Date }

const user: UserRecord = {
  id: '123',
  name: 'Ana',
  email: 'ana@example.com',
  createdAt: new Date(),
  updatedAt: new Date()
};
```

**Mixins con Intersection Types:**

```typescript
type Serializable = {
  serialize(): string;
  deserialize(data: string): void;
};

type Loggable = {
  log(message: string): void;
  getLog(): string[];
};

type FullService = Serializable & Loggable;

// Una clase que implementa todas las interfaces
class DataService implements FullService {
  private logs: string[] = [];

  serialize(): string { return JSON.stringify(this); }
  deserialize(data: string): void { Object.assign(this, JSON.parse(data)); }
  log(message: string): void { this.logs.push(message); }
  getLog(): string[] { return this.logs; }
}
```

## Combinación práctica

```typescript
// API Response types
type ApiSuccess<T> = {
  success: true;
  data: T;
  meta: { total: number; page: number };
};

type ApiError = {
  success: false;
  error: string;
  code: number;
};

type ApiResponse<T> = ApiSuccess<T> | ApiError;

async function fetchUsers(): Promise<ApiResponse<User[]>> {
  // ...
}

const response = await fetchUsers();
if (response.success) {
  console.log(response.data); // TypeScript sabe que es ApiSuccess
} else {
  console.log(response.error); // TypeScript sabe que es ApiError
}
```

**Consejo de entrevista**: Union Types e Intersection Types son fundamentales en TypeScript. Union (`|`) significa "puede ser uno de", Intersection (`&`) significa "debe ser todos". Los union types son especialmente poderosos combinados con discriminated unions para manejar estados y resultados de forma type-safe.
