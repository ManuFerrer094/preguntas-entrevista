Tipar funciones asíncronas en TypeScript implica entender cómo `async/await` interactúa con el sistema de tipos: una función `async` siempre devuelve una `Promise`, y TypeScript puede inferir o recibir explícitamente el tipo del valor resuelto.

## Tipado básico de async/await

```typescript
// TypeScript infiere automáticamente que el retorno es Promise<User>
async function fetchUser(id: string): Promise<User> {
  const response = await fetch(`/api/users/${id}`);

  if (!response.ok) {
    throw new Error(`HTTP error: ${response.status}`);
  }

  // response.json() devuelve Promise<any> — necesitamos ayudar a TypeScript
  const data: User = await response.json();
  return data;
}

// Uso: TypeScript sabe que user es User
const user = await fetchUser('123');
console.log(user.name); // ✅ Type-safe
```

## Tipar response.json() correctamente

```typescript
// Función genérica para fetch type-safe
async function fetchJson<T>(url: string, options?: RequestInit): Promise<T> {
  const response = await fetch(url, options);

  if (!response.ok) {
    throw new Error(`HTTP ${response.status}: ${response.statusText}`);
  }

  return response.json() as Promise<T>;
}

// Uso con tipo explícito
const users = await fetchJson<User[]>('/api/users');
const product = await fetchJson<Product>('/api/products/1');
```

## Manejo de errores con tipos

```typescript
// Patrón Result type para errores explícitos
type Result<T, E = Error> =
  | { success: true; data: T }
  | { success: false; error: E };

async function safeApiCall<T>(
  fn: () => Promise<T>
): Promise<Result<T>> {
  try {
    const data = await fn();
    return { success: true, data };
  } catch (error) {
    return {
      success: false,
      error: error instanceof Error ? error : new Error(String(error))
    };
  }
}

// Uso completamente type-safe
const result = await safeApiCall(() => fetchUser('123'));

if (result.success) {
  console.log(result.data.name); // TypeScript sabe que data es User
} else {
  console.error(result.error.message);
}
```

## Promise.all, Promise.allSettled tipados

```typescript
// Promise.all — falla si alguna rechaza
async function loadDashboard(userId: string) {
  const [user, posts, notifications] = await Promise.all([
    fetchUser(userId),           // Promise<User>
    fetchPosts(userId),          // Promise<Post[]>
    fetchNotifications(userId)   // Promise<Notification[]>
  ]);
  // TypeScript infiere: [User, Post[], Notification[]]
  return { user, posts, notifications };
}

// Promise.allSettled — nunca rechaza
async function loadWithFallbacks(userId: string) {
  const results = await Promise.allSettled([
    fetchUser(userId),
    fetchPosts(userId)
  ]);

  const [userResult, postsResult] = results;
  // PromiseSettledResult<User> y PromiseSettledResult<Post[]>

  const user = userResult.status === 'fulfilled'
    ? userResult.value  // User
    : null;

  const posts = postsResult.status === 'fulfilled'
    ? postsResult.value  // Post[]
    : [];

  return { user, posts };
}
```

## Generics en funciones asíncronas

```typescript
// Service genérico para CRUD
class ApiService<T extends { id: string }> {
  constructor(private baseUrl: string) {}

  async getAll(): Promise<T[]> {
    return fetchJson<T[]>(this.baseUrl);
  }

  async getById(id: string): Promise<T | null> {
    try {
      return await fetchJson<T>(`${this.baseUrl}/${id}`);
    } catch {
      return null;
    }
  }

  async create(data: Omit<T, 'id'>): Promise<T> {
    return fetchJson<T>(this.baseUrl, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    });
  }

  async update(id: string, data: Partial<T>): Promise<T> {
    return fetchJson<T>(`${this.baseUrl}/${id}`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    });
  }
}

const userService = new ApiService<User>('/api/users');
const users: User[] = await userService.getAll();
const user: User = await userService.create({ name: 'Ana', email: 'ana@test.com' });
```

## Tipado de callbacks asíncronos

```typescript
// Tipo para un handler asíncrono
type AsyncHandler<T = void> = () => Promise<T>;
type AsyncPredicate<T> = (value: T) => Promise<boolean>;
type AsyncMapper<T, R> = (value: T) => Promise<R>;

async function processItems<T, R>(
  items: T[],
  mapper: AsyncMapper<T, R>,
  filter?: AsyncPredicate<T>
): Promise<R[]> {
  const filteredItems = filter
    ? await Promise.all(items.map(async item =>
        await filter(item) ? item : null
      )).then(results => results.filter((item): item is T => item !== null))
    : items;

  return Promise.all(filteredItems.map(mapper));
}
```

**Consejo de entrevista**: Siempre anotar el tipo de retorno de funciones públicas asíncronas: `async function fetch(): Promise<User>`. No asumir que `response.json()` tiene el tipo correcto — siempre hacer el cast o usar una función genérica. Para catch blocks, en strict mode el error es `unknown` y debe verificarse con `instanceof Error` antes de acceder a `.message`.
