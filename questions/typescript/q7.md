---
title: ¿Qué es Type Guards en TypeScript?
difficulty: medium

Los Type Guards son técnicas para reducir o "estrechar" (narrow) el tipo de una variable dentro de un bloque de código. TypeScript analiza el flujo del programa y entiende que dentro de una condición, el tipo es más específico que el declarado originalmente. Esto permite escribir código type-safe cuando se trabaja con union types u objetos de tipo desconocido.

## Type Guards integrados

**typeof** — para primitivos:

```typescript
function formatValue(value: string | number | boolean): string {
  if (typeof value === 'string') {
    return value.trim().toLowerCase(); // TypeScript sabe: value es string
  } else if (typeof value === 'number') {
    return value.toFixed(2);           // TypeScript sabe: value es number
  } else {
    return value ? 'Verdadero' : 'Falso'; // TypeScript sabe: value es boolean
  }
}
```

**instanceof** — para clases:

```typescript
class HttpError extends Error {
  constructor(public statusCode: number, message: string) {
    super(message);
  }
}

class ValidationError extends Error {
  constructor(public fields: string[], message: string) {
    super(message);
  }
}

function handleError(error: unknown): Response {
  if (error instanceof HttpError) {
    return { status: error.statusCode, body: error.message }; // error es HttpError
  }
  if (error instanceof ValidationError) {
    return { status: 400, body: error.fields.join(', ') }; // error es ValidationError
  }
  return { status: 500, body: 'Error desconocido' };
}
```

**Comprobación de propiedades con `in`:**

```typescript
type Cat = { meow: () => void; lives: number };
type Dog = { bark: () => void; fetch: () => void };

type Pet = Cat | Dog;

function makeSound(pet: Pet): void {
  if ('meow' in pet) {
    pet.meow(); // TypeScript sabe que es Cat
  } else {
    pet.bark(); // TypeScript sabe que es Dog
  }
}
```

## User-defined Type Guards (predicados de tipo)

```typescript
// La firma de retorno "param is Type" convierte la función en un type guard
function isString(value: unknown): value is string {
  return typeof value === 'string';
}

function isUser(value: unknown): value is User {
  return (
    typeof value === 'object' &&
    value !== null &&
    'id' in value &&
    'email' in value &&
    typeof (value as User).email === 'string'
  );
}

// Uso
function processApiResponse(data: unknown) {
  if (isUser(data)) {
    console.log(data.email); // TypeScript sabe que data es User
  }
}
```

## Assertion Functions

```typescript
// Versión alternativa: lanza un error si la condición no se cumple
function assertIsString(value: unknown): asserts value is string {
  if (typeof value !== 'string') {
    throw new TypeError(`Expected string, got ${typeof value}`);
  }
}

function processName(name: unknown) {
  assertIsString(name);
  // A partir de aquí, TypeScript sabe que name es string
  console.log(name.toUpperCase());
}
```

## Discriminated Unions con type guards

```typescript
type LoadingState = { status: 'loading' };
type SuccessState<T> = { status: 'success'; data: T };
type ErrorState = { status: 'error'; message: string };

type AsyncState<T> = LoadingState | SuccessState<T> | ErrorState;

function renderState<T>(state: AsyncState<T>): string {
  switch (state.status) {
    case 'loading': return 'Cargando...';
    case 'success': return `Datos: ${JSON.stringify(state.data)}`; // TypeScript sabe que tiene .data
    case 'error':   return `Error: ${state.message}`;             // TypeScript sabe que tiene .message
  }
}
```

## Type Guard para arrays

```typescript
function filterNonNull<T>(items: (T | null | undefined)[]): T[] {
  return items.filter((item): item is T => item != null);
}

const values: (string | null)[] = ['hello', null, 'world', null];
const strings: string[] = filterNonNull(values); // ['hello', 'world']
```

**Consejo de entrevista**: Los type guards son esenciales para escribir código TypeScript seguro cuando se manejan valores de tipo `unknown`, respuestas de APIs, o union types complejos. La función user-defined type guard con el predicado `value is Type` es especialmente útil para validar datos externos (respuestas de API, entradas del usuario).
