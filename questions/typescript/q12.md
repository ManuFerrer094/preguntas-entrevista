---
title: ¿Qué es el operador keyof en TypeScript?
difficulty: medium
tags: [typescript, keyof, index-types, advanced]
---

El operador `keyof` en TypeScript produce un union type con los nombres de todas las propiedades de un tipo. Es fundamental para crear funciones y tipos genéricos que operen sobre las propiedades de objetos de forma type-safe.

## Uso básico de keyof

```typescript
interface User {
  id: number;
  name: string;
  email: string;
  role: 'admin' | 'user';
}

type UserKeys = keyof User; // 'id' | 'name' | 'email' | 'role'

// Uso: garantizar que la clave exista en el tipo
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user: User = { id: 1, name: 'Ana', email: 'ana@test.com', role: 'admin' };

const name = getProperty(user, 'name');   // TypeScript infiere: string
const id = getProperty(user, 'id');       // TypeScript infiere: number
// getProperty(user, 'invalid');           // ❌ Error de compilación
```

## Index Access Types (T[K])

Se usa junto con `keyof` para obtener el tipo de una propiedad:

```typescript
type UserName = User['name'];  // string
type UserId = User['id'];      // number
type UserRole = User['role'];  // 'admin' | 'user'

// Obtener el tipo de un elemento de un array
type Fruits = ['apple', 'banana', 'cherry'];
type FirstFruit = Fruits[0];   // 'apple'
type AnyFruit = Fruits[number]; // 'apple' | 'banana' | 'cherry'
```

## Patrones avanzados con keyof

**Función setProperty type-safe:**

```typescript
function setProperty<T, K extends keyof T>(obj: T, key: K, value: T[K]): T {
  return { ...obj, [key]: value };
}

const updated = setProperty(user, 'name', 'Carlos');      // ✅
// setProperty(user, 'name', 42);                          // ❌ 42 no es string
// setProperty(user, 'nonExistent', 'value');              // ❌ clave inválida
```

**Función pick type-safe:**

```typescript
function pick<T, K extends keyof T>(obj: T, keys: K[]): Pick<T, K> {
  const result = {} as Pick<T, K>;
  keys.forEach(key => { result[key] = obj[key]; });
  return result;
}

const publicUser = pick(user, ['id', 'name', 'email']); // { id, name, email }
// No incluye 'role', TypeScript lo sabe
```

**Pluck de arrays:**

```typescript
function pluck<T, K extends keyof T>(items: T[], key: K): T[K][] {
  return items.map(item => item[key]);
}

const users: User[] = [...];
const names = pluck(users, 'name');  // string[]
const ids = pluck(users, 'id');      // number[]
```

**keyof con typeof para objetos de configuración:**

```typescript
const COLORS = {
  primary: '#3178C6',
  secondary: '#F7DF1E',
  danger: '#DD0031',
  success: '#42B883'
} as const;

type ColorName = keyof typeof COLORS;
// 'primary' | 'secondary' | 'danger' | 'success'

type ColorValue = typeof COLORS[ColorName];
// '#3178C6' | '#F7DF1E' | '#DD0031' | '#42B883'

function getColor(name: ColorName): ColorValue {
  return COLORS[name]; // 100% type-safe
}
```

**Mapper de objetos genérico:**

```typescript
function mapValues<T extends object, R>(
  obj: T,
  transform: <K extends keyof T>(key: K, value: T[K]) => R
): Record<keyof T, R> {
  const result = {} as Record<keyof T, R>;
  (Object.keys(obj) as (keyof T)[]).forEach(key => {
    result[key] = transform(key, obj[key]);
  });
  return result;
}

const lengths = mapValues(
  { name: 'Ana', email: 'ana@test.com' },
  (key, value) => value.length
);
// { name: 3, email: 12 }
```

**Consejo de entrevista**: `keyof T` siempre produce un union type de las claves, y `T[K]` (index access) produce el tipo del valor para esa clave. Combinados, son la base para crear funciones genéricas que operan sobre objetos de forma completamente type-safe, eliminando errores en tiempo de ejecución causados por acceder a propiedades que no existen.
