---
title: ¿Qué es el operador typeof en TypeScript y cómo se diferencia del typeof de JavaScript?
difficulty: easy
tags: [typescript, typeof, type-queries, types, runtime]
---

En TypeScript, `typeof` tiene **dos comportamientos** distintos: el comportamiento de **runtime** (heredado de JavaScript, devuelve un string) y el comportamiento de **tipo** (solo en contexto de tipos, crea un tipo TypeScript a partir de una variable).

## typeof en runtime (JavaScript)

```typescript
// Igual que en JavaScript: devuelve un string con el tipo primitivo
typeof "hola"           // "string"
typeof 42               // "number"
typeof true             // "boolean"
typeof undefined        // "undefined"
typeof null             // "object" (quirk histórico de JS)
typeof {}               // "object"
typeof []               // "object"
typeof function(){}     // "function"
typeof Symbol()         // "symbol"
typeof 42n              // "bigint"

// Uso típico en guards de tipo:
function procesar(valor: string | number) {
  if (typeof valor === 'string') {
    // Aquí TypeScript sabe que `valor` es string
    return valor.toUpperCase();
  }
  // Aquí TypeScript sabe que `valor` es number
  return valor.toFixed(2);
}
```

## typeof en contexto de tipo (TypeScript exclusivo)

```typescript
// Capturar el tipo de una variable
const usuario = {
  id: 1,
  nombre: 'Ana García',
  email: 'ana@ejemplo.com',
  rol: 'admin' as const
};

// Crear un tipo TypeScript a partir de la variable
type TipoUsuario = typeof usuario;
/*
TipoUsuario = {
  id: number;
  nombre: string;
  email: string;
  rol: 'admin';
}
*/

// Muy útil con funciones
function crearConfig(opciones: { debug: boolean; nivel: number }) {
  return { ...opciones, version: '1.0.0' };
}

type ConfigResultado = ReturnType<typeof crearConfig>;
// ConfigResultado = { debug: boolean; nivel: number; version: string }
```

## Capturar tipos de objetos complejos

```typescript
// En lugar de definir el tipo manualmente, derivarlo del valor
const COLORES_DISPONIBLES = {
  primario: '#3B82F6',
  secundario: '#8B5CF6',
  peligro: '#EF4444',
  exito: '#10B981',
  advertencia: '#F59E0B'
} as const;

type NombreColor = keyof typeof COLORES_DISPONIBLES;
// type NombreColor = 'primario' | 'secundario' | 'peligro' | 'exito' | 'advertencia'

type ValorColor = typeof COLORES_DISPONIBLES[NombreColor];
// type ValorColor = '#3B82F6' | '#8B5CF6' | '#EF4444' | '#10B981' | '#F59E0B'

function obtenerColor(nombre: NombreColor): ValorColor {
  return COLORES_DISPONIBLES[nombre];
}
```

## typeof con funciones para capturar tipos de parámetros y retorno

```typescript
async function obtenerUsuario(id: number) {
  const respuesta = await fetch(`/api/usuarios/${id}`);
  return respuesta.json() as Promise<{ id: number; nombre: string; activo: boolean }>;
}

// Tipos derivados de la función:
type ParamsObtenerUsuario = Parameters<typeof obtenerUsuario>;
// [id: number]

type RetornoObtenerUsuario = Awaited<ReturnType<typeof obtenerUsuario>>;
// { id: number; nombre: string; activo: boolean }

// Útil para tipar mocks en tests:
const mockObtenerUsuario: typeof obtenerUsuario = async (id) => ({
  id,
  nombre: 'Usuario Mock',
  activo: true
});
```

## typeof con enums y clases

```typescript
enum Direccion {
  Norte = 'NORTE',
  Sur = 'SUR',
  Este = 'ESTE',
  Oeste = 'OESTE'
}

// typeof en un enum captura el tipo del objeto enum (no el tipo de los valores)
type TipoDireccionEnum = typeof Direccion;
/*
TipoDireccionEnum = {
  Norte: string;
  Sur: string;
  Este: string;
  Oeste: string;
}
*/

// Capturar los valores del enum:
type ValorDireccion = (typeof Direccion)[keyof typeof Direccion];
// 'NORTE' | 'SUR' | 'ESTE' | 'OESTE'


// Con clases:
class Servicio {
  static version = '2.0.0';
  static crear(): Servicio { return new Servicio(); }
  procesar(datos: string): number { return datos.length; }
}

// typeof Clase captura el tipo estático (no la instancia)
type TipoEstatico = typeof Servicio;
// { version: string; crear(): Servicio; new(): Servicio }

type Instancia = InstanceType<typeof Servicio>;
// { procesar(datos: string): number }
```

## Diferencia entre typeof y keyof

```typescript
const config = {
  host: 'localhost',
  puerto: 3000,
  debug: false
};

// typeof: captura el tipo completo del objeto
type TipoConfig = typeof config;
// { host: string; puerto: number; debug: boolean }

// keyof: captura las claves del tipo
type ClavesConfig = keyof typeof config;
// 'host' | 'puerto' | 'debug'

// Combinación frecuente para acceso tipado:
function obtenerValor<T, K extends keyof T>(obj: T, clave: K): T[K] {
  return obj[clave];
}

const puerto = obtenerValor(config, 'puerto');  // type: number
const host = obtenerValor(config, 'host');       // type: string
```

**Consejo de entrevista**: La distinción entre `typeof` en runtime y en tipos es fundamental en TypeScript. El uso más poderoso es `typeof` con `keyof` para crear tipos derivados de objetos constantes (`as const`), evitando la duplicación entre el objeto real y su definición de tipo. Otro uso avanzado es `ReturnType<typeof fn>` y `Parameters<typeof fn>` para capturar los tipos de funciones sin duplicar la definición —especialmente útil cuando los tipos de retorno son complejos o inferidos.
