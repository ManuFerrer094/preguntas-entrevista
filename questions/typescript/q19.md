TypeScript incluye tipos de utilidad como `Partial`, `Required`, `Pick` o `Omit`, pero puedes crear los tuyos propios combinando Mapped Types, Conditional Types e `infer`. Esto permite construir transformaciones de tipos reutilizables y expresivas.

## Recrear los tipos de utilidad nativos

```typescript
// Partial<T> — hace todas las propiedades opcionales
type MyPartial<T> = {
  [K in keyof T]?: T[K];
};

// Required<T> — hace todas las propiedades requeridas
type MyRequired<T> = {
  [K in keyof T]-?: T[K]; // -? elimina el modificador opcional
};

// Readonly<T> — hace todas las propiedades de solo lectura
type MyReadonly<T> = {
  readonly [K in keyof T]: T[K];
};

// Record<K, V>
type MyRecord<K extends keyof any, V> = {
  [P in K]: V;
};

// Pick<T, K>
type MyPick<T, K extends keyof T> = {
  [P in K]: T[P];
};

// Omit<T, K>
type MyOmit<T, K extends keyof T> = MyPick<T, Exclude<keyof T, K>>;
```

## Tipos personalizados útiles

```typescript
// DeepPartial — Partial recursivo para objetos anidados
type DeepPartial<T> = T extends object
  ? { [K in keyof T]?: DeepPartial<T[K]> }
  : T;

interface Config {
  server: { host: string; port: number };
  database: { url: string; name: string };
  features: { darkMode: boolean; beta: boolean };
}

const patch: DeepPartial<Config> = {
  server: { port: 4000 },  // No necesita incluir host
  features: { darkMode: true }
};

// Mutable<T> — elimina readonly de todas las propiedades
type Mutable<T> = {
  -readonly [K in keyof T]: T[K];
};

// NonNullable profundo
type DeepNonNullable<T> = T extends null | undefined
  ? never
  : T extends object
  ? { [K in keyof T]: DeepNonNullable<T[K]> }
  : T;
```

## Extraer tipos con infer

```typescript
// Extraer el tipo de retorno de una función
type ReturnType<T extends (...args: any) => any> =
  T extends (...args: any) => infer R ? R : never;

// Extraer el tipo del primer argumento
type FirstArgument<T extends (...args: any) => any> =
  T extends (first: infer F, ...rest: any) => any ? F : never;

// Extraer el tipo de un array
type ArrayElement<T> = T extends (infer E)[] ? E : never;
type NumberElement = ArrayElement<number[]>; // number

// Extraer el tipo resuelto de una Promise
type Awaited<T> = T extends Promise<infer R>
  ? R extends Promise<any> ? Awaited<R> : R
  : T;

type Result = Awaited<Promise<Promise<string>>>; // string
```

## Tipos condicionales avanzados

```typescript
// XOR — exactamente uno de los dos tipos
type XOR<T, U> =
  | (T & { [K in Exclude<keyof U, keyof T>]?: never })
  | (U & { [K in Exclude<keyof T, keyof U>]?: never });

type WithId = { id: string };
type WithSlug = { slug: string };
type Resource = XOR<WithId, WithSlug>;

const r1: Resource = { id: '123' };        // ✅
const r2: Resource = { slug: 'my-post' };  // ✅
const r3: Resource = { id: '1', slug: 'p' }; // ❌ No se puede tener ambos

// IsArray — comprueba si un tipo es array
type IsArray<T> = T extends any[] ? true : false;
type Test1 = IsArray<number[]>;  // true
type Test2 = IsArray<string>;    // false

// Flatten — aplanar un tipo array
type Flatten<T> = T extends (infer U)[] ? U : T;
type Str = Flatten<string[]>;  // string
type Num = Flatten<number>;    // number (ya plano)
```

## Builder de tipos para formularios

```typescript
// Generar tipos de errores a partir del tipo del formulario
type FormErrors<T extends object> = {
  [K in keyof T]?: T[K] extends object
    ? FormErrors<T[K]>  // Error anidado para objetos
    : string;           // Mensaje de error para primitivos
};

interface RegistrationForm {
  email: string;
  password: string;
  address: {
    street: string;
    city: string;
    zip: string;
  };
}

const errors: FormErrors<RegistrationForm> = {
  email: 'Email inválido',
  address: {
    zip: 'Código postal incorrecto'
    // No requiere todos los campos de address
  }
};

// Tipo para métodos de un objeto
type MethodsOf<T> = {
  [K in keyof T]: T[K] extends (...args: any) => any ? K : never;
}[keyof T];

class UserService {
  getUser(id: string) { return null; }
  updateUser(id: string, data: any) { return null; }
  private secret = 'private';
}

type ServiceMethods = MethodsOf<UserService>; // 'getUser' | 'updateUser'
```

**Consejo de entrevista**: Los tipos de utilidad personalizados demuestran dominio avanzado de TypeScript. Las tres herramientas clave son: Mapped Types (`[K in keyof T]`) para transformar propiedades, Conditional Types (`T extends U ? X : Y`) para tipos que dependen de condiciones, e `infer` para extraer tipos de estructuras complejas. Comenzar recreando los tipos nativos (`Partial`, `Required`) es una excelente forma de entender cómo funcionan internamente.
