---
title: ¿Qué son los Index Signatures en TypeScript?
difficulty: medium
tags: [typescript, index-signatures, dynamic-keys, objects, types]
---

Los Index Signatures permiten tipar objetos con claves dinámicas (cuando no se conocen todos los nombres de las propiedades en tiempo de compilación). Especifican el tipo de las claves y el tipo de los valores asociados.

## Sintaxis básica

```typescript
// Objeto con cualquier número de propiedades string → number
interface ScoreMap {
  [playerName: string]: number;
}

const scores: ScoreMap = {
  'Ana': 100,
  'Luis': 85,
  'Marta': 92
};

scores['NuevoJugador'] = 75; // ✅ Válido
scores['Ana'] = 'cien';      // ❌ Error: string no asignable a number
```

## Index Signatures con propiedades fijas

```typescript
interface Config {
  [key: string]: string | number | boolean; // Firma de índice
  host: string;   // Propiedad fija — debe ser compatible con el tipo del índice
  port: number;   // ✅ number es asignable a string | number | boolean
  debug: boolean; // ✅
  // ❌ items: string[] — no compatible con la firma de índice
}

const config: Config = {
  host: 'localhost',
  port: 3000,
  debug: true,
  customOption: 'value',  // Propiedad dinámica válida
  timeout: 5000
};
```

## Claves numéricas

```typescript
interface ArrayLike<T> {
  [index: number]: T;  // Clave numérica
  length: number;
}

const arr: ArrayLike<string> = {
  0: 'primero',
  1: 'segundo',
  length: 2
};

// Nota: en JavaScript, las claves de objeto siempre son strings.
// TypeScript permite tanto string como number en firmas de índice,
// pero en runtime los números se convierten a strings.
```

## Record<K, V> — alternativa más expresiva

```typescript
// Record<K, V> es equivalente a una firma de índice pero más legible
type Scores = Record<string, number>;
// Equivalente a: { [key: string]: number }

type Status = 'active' | 'inactive' | 'pending';
type UserStatus = Record<string, Status>;

// Con union de claves conocidas
type HttpMethods = 'GET' | 'POST' | 'PUT' | 'DELETE';
type MethodHandlers = Record<HttpMethods, (req: Request) => Response>;
// Todas las keys deben estar presentes (a diferencia de la firma de índice)
```

## Caso de uso: traducciones i18n

```typescript
type Language = 'es' | 'en' | 'fr';

interface Translations {
  [key: string]: {
    [lang in Language]: string;
  };
}

const i18n: Translations = {
  greeting: {
    es: 'Hola',
    en: 'Hello',
    fr: 'Bonjour'
  },
  farewell: {
    es: 'Adiós',
    en: 'Goodbye',
    fr: 'Au revoir'
  }
};
```

## Mapped Types vs Index Signatures

```typescript
// Index Signature: cualquier clave del tipo dado
interface DynamicObject {
  [key: string]: number;
}

// Mapped Type: solo las claves del tipo dado (más restrictivo)
type ExactKeys = { [K in 'a' | 'b' | 'c']: number };
// Equivalente a: { a: number; b: number; c: number }

// Mapped Type con modificadores opcionales
type PartialObject = { [K in 'a' | 'b' | 'c']?: number };
// Equivalente a: { a?: number; b?: number; c?: number }
```

## Problema del acceso no seguro y solución con noUncheckedIndexedAccess

```typescript
interface StringMap {
  [key: string]: string;
}

const map: StringMap = { name: 'Ana' };

// Sin noUncheckedIndexedAccess:
const value = map['unknownKey']; // TypeScript infiere string (no string | undefined)
value.toUpperCase(); // No hay error en TypeScript, pero falla en runtime

// Con "noUncheckedIndexedAccess": true en tsconfig.json:
const safeValue = map['unknownKey']; // TypeScript infiere string | undefined
safeValue?.toUpperCase(); // Correcto — optional chaining requerido
if (safeValue !== undefined) {
  safeValue.toUpperCase(); // ✅
}
```

## Index Signatures para cache y memoización

```typescript
class Memoizer<T> {
  private cache: { [key: string]: T } = {};

  get(key: string): T | undefined {
    return this.cache[key];
  }

  set(key: string, value: T): void {
    this.cache[key] = value;
  }

  has(key: string): boolean {
    return key in this.cache;
  }

  clear(): void {
    this.cache = {};
  }
}

const numberCache = new Memoizer<number>();
numberCache.set('fibonacci-10', 55);
const value = numberCache.get('fibonacci-10'); // number | undefined
```

**Consejo de entrevista**: Activa `noUncheckedIndexedAccess` en `tsconfig.json` para que TypeScript infiera `T | undefined` cuando accedes con una firma de índice, haciendo el código más seguro. Usa `Record<K, V>` cuando las claves son un conjunto conocido (union type) y firmas de índice solo cuando realmente son dinámicas. Mencionar esta distinción y la opción del compilador demuestra un uso maduro de TypeScript.
