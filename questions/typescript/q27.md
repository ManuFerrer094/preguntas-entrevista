---
title: ¿Qué es la covarianza y contravarianza en TypeScript?
difficulty: hard
tags: [typescript, variance, covariance, contravariance, type-system]
---

La **varianza** describe cómo se comportan los tipos compuestos (arrays, funciones, genéricos) cuando sus tipos base tienen una relación de subtipo. TypeScript usa un sistema de varianza basado en la estructura, lo que puede llevar a comportamientos sorprendentes.

## Subtipado estructural en TypeScript

```typescript
class Animal {
  nombre: string = '';
}

class Perro extends Animal {
  raza: string = '';
}

// Perro es subtipo de Animal (Perro tiene todo lo que Animal tiene, más)
const perro: Perro = new Perro();
const animal: Animal = perro;  // ✅ Asignación válida: Perro es subtipo de Animal
```

## Covarianza — Arrays y retornos de función

```typescript
// Los arrays son COVARIANTES en TypeScript (aunque esto puede ser problemático)
let perros: Perro[] = [new Perro()];
let animales: Animal[] = perros;  // ✅ TypeScript lo permite (covariante)

// Pero esto puede causar problemas:
animales.push(new Animal());    // TS lo permite... pero perros[1] no sería Perro!
// Esto es un "hole" de seguridad en el sistema de tipos de TypeScript

// Los tipos de retorno de funciones son COVARIANTES ✅
type ObtenerAnimal = () => Animal;
type ObtenerPerro = () => Perro;

const obtenerPerro: ObtenerPerro = () => new Perro();
const obtenerAnimal: ObtenerAnimal = obtenerPerro;  // ✅ Válido: Perro es subtipo de Animal
```

## Contravarianza — Parámetros de función

```typescript
// Los parámetros de función son CONTRAVARIANTES (dirección opuesta)
type ProcesarAnimal = (animal: Animal) => void;
type ProcesarPerro = (perro: Perro) => void;

const procesarAnimal: ProcesarAnimal = (a) => console.log(a.nombre);

// ✅ Contravarianza: una función que acepta Animal puede usarse donde se espera ProcesarPerro
// (porque puede manejar cualquier Animal, incluyendo Perros)
const procesarPerro: ProcesarPerro = procesarAnimal;  // ✅ Válido

// ❌ Una función que solo acepta Perros NO puede usarse donde se espera ProcesarAnimal
const soloPerros: ProcesarAnimal = (perro: Perro) => console.log(perro.raza);  // ❌ Error
// Porque si llamas a soloPerros(new Gato()), fallaría
```

## Bivarianza — Métodos de clase (TypeScript por compatibilidad)

```typescript
// TypeScript (con --strictFunctionTypes) hace parámetros contravariantes
// PERO los métodos de clase son bivariantes (por compatibilidad histórica)

interface Procesador {
  // Método: bivariante (menos seguro pero compatible con más código)
  procesar(animal: Animal): void;
}

interface ProcesadorFn {
  // Propiedad función: contravariante (más seguro)
  procesar: (animal: Animal) => void;
}

// Con strictFunctionTypes, las funciones en propiedades son contravariantes
// pero los métodos de interfaz siguen siendo bivariantes
```

## Invarianza — Genéricos en posición de escritura y lectura

```typescript
// Un contenedor genérico es INVARIANTE cuando puede leer Y escribir
class Contenedor<T> {
  valor: T;
  constructor(valor: T) { this.valor = valor; }
  obtener(): T { return this.valor; }
  establecer(v: T) { this.valor = v; }
}

const contenedorPerro = new Contenedor<Perro>(new Perro());
// const contenedorAnimal: Contenedor<Animal> = contenedorPerro;  // ❌ Error en TS estricto
// Si lo permitiera:
// contenedorAnimal.establecer(new Gato());  // ← Ahora contenedorPerro.valor sería un Gato!

// Solo lectura → covariante (readonly)
interface SoloLectura<T> {
  readonly valor: T;
}
const soloLecturaAnimal: SoloLectura<Animal> = contenedorPerro;  // ✅ Covariante

// Solo escritura → contravariante
interface SoloEscritura<T> {
  establecer(v: T): void;
}
```

## Ejemplo práctico: callbacks y event handlers

```typescript
interface Boton {
  onClick: (e: MouseEvent) => void;
}

interface BotonConTecla extends Boton {
  onKeyDown: (e: KeyboardEvent) => void;
}

// Función que acepta el evento base (MouseEvent) puede manejar eventos más específicos
const manejador: (e: Event) => void = (e) => console.log(e.type);
const boton: Boton = { onClick: manejador };  // ✅ Contravarianza: Event es supertipo de MouseEvent
```

## Varianza con tipos condicionales y mapped types

```typescript
// TypeScript infiere covarianza en posición de retorno
type Mapa<T> = T extends string ? string[] : number[];

type ResultadoString = Mapa<string>;   // string[]
type ResultadoNumber = Mapa<number>;   // number[]

// El operador in de mapped types puede controlar la varianza con + y -
type SoloLecturaGenerico<T> = {
  readonly [K in keyof T]: T[K];  // Todas las propiedades readonly
};

type Mutable<T> = {
  -readonly [K in keyof T]: T[K];  // Elimina readonly (usando -)
};
```

**Consejo de entrevista**: La varianza es un concepto avanzado que aparece en preguntas de nivel senior. Los puntos clave son: 1) los **tipos de retorno** son covariantes (subtipo compatible), 2) los **parámetros de función** son contravariantes (supertipo compatible), 3) los **genéricos de lectura+escritura** son invariantes. En TypeScript, `--strictFunctionTypes` hace que las funciones en propiedades sean contravariantes (más seguro), pero los métodos de interfaz mantienen la bivarianza por compatibilidad con código existente.
