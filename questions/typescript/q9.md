Las Discriminated Unions (también llamadas Tagged Unions o Algebraic Data Types) son un patrón de TypeScript donde cada miembro de un union type tiene una propiedad literal común (el "discriminante") que permite a TypeScript estrechar el tipo de forma exhaustiva. Es uno de los patrones más poderosos para modelar estados y eventos en TypeScript.

## Estructura básica

```typescript
// Cada variante tiene una propiedad 'type' con un valor literal único
type Circle = { type: 'circle'; radius: number };
type Rectangle = { type: 'rectangle'; width: number; height: number };
type Triangle = { type: 'triangle'; base: number; height: number };

type Shape = Circle | Rectangle | Triangle;

function calculateArea(shape: Shape): number {
  switch (shape.type) {
    case 'circle':
      return Math.PI * shape.radius ** 2; // TypeScript sabe que es Circle
    case 'rectangle':
      return shape.width * shape.height;  // TypeScript sabe que es Rectangle
    case 'triangle':
      return (shape.base * shape.height) / 2; // TypeScript sabe que es Triangle
  }
  // TypeScript sabe que todos los casos están cubiertos (exhaustive check)
}
```

## Exhaustive checking — detectar casos olvidados

```typescript
// Función auxiliar para exhaustive checking
function assertNever(value: never): never {
  throw new Error(`Caso no manejado: ${JSON.stringify(value)}`);
}

function describeShape(shape: Shape): string {
  switch (shape.type) {
    case 'circle':    return `Círculo de radio ${shape.radius}`;
    case 'rectangle': return `Rectángulo ${shape.width}x${shape.height}`;
    case 'triangle':  return `Triángulo de base ${shape.base}`;
    default:
      return assertNever(shape); // Si se añade un nuevo tipo y no se maneja aquí, ¡error de compilación!
  }
}
```

## Modelar estados de la aplicación (Redux-like)

```typescript
// Estado de carga de datos
type AsyncState<T, E = string> =
  | { status: 'idle' }
  | { status: 'loading' }
  | { status: 'success'; data: T }
  | { status: 'error'; error: E };

// Componente React con todos los estados cubiertos
function UserComponent({ state }: { state: AsyncState<User> }) {
  switch (state.status) {
    case 'idle':    return <button>Cargar usuario</button>;
    case 'loading': return <Spinner />;
    case 'success': return <UserCard user={state.data} />;
    case 'error':   return <ErrorMessage message={state.error} />;
  }
}
```

## Modelar eventos de dominio

```typescript
type UserEvent =
  | { type: 'USER_CREATED'; payload: { id: string; name: string; email: string } }
  | { type: 'USER_UPDATED'; payload: { id: string; changes: Partial<User> } }
  | { type: 'USER_DELETED'; payload: { id: string } }
  | { type: 'USER_ROLE_CHANGED'; payload: { id: string; oldRole: Role; newRole: Role } };

function processEvent(event: UserEvent): void {
  switch (event.type) {
    case 'USER_CREATED':
      createUserInDB(event.payload);
      sendWelcomeEmail(event.payload.email);
      break;
    case 'USER_UPDATED':
      updateUserInDB(event.payload.id, event.payload.changes);
      break;
    case 'USER_DELETED':
      softDeleteUser(event.payload.id);
      break;
    case 'USER_ROLE_CHANGED':
      updatePermissions(event.payload.id, event.payload.newRole);
      auditLog(`Role changed from ${event.payload.oldRole} to ${event.payload.newRole}`);
      break;
  }
}
```

## Manejo de errores con discriminated unions

```typescript
type Result<T, E = Error> =
  | { success: true; value: T }
  | { success: false; error: E };

async function fetchUser(id: string): Promise<Result<User>> {
  try {
    const user = await db.findUser(id);
    if (!user) return { success: false, error: new NotFoundError(id) };
    return { success: true, value: user };
  } catch (error) {
    return { success: false, error: error as Error };
  }
}

// Uso type-safe sin try/catch en el llamador
const result = await fetchUser('123');
if (result.success) {
  console.log(result.value.name); // TypeScript sabe que tiene .value
} else {
  console.error(result.error.message); // TypeScript sabe que tiene .error
}
```

**Consejo de entrevista**: Las Discriminated Unions son el equivalente TypeScript del pattern matching funcional. Son especialmente valiosas para modelar estados finitos (formulario: idle/loading/success/error), comandos y eventos. El exhaustive checking con `assertNever` es una técnica que demuestra madurez en TypeScript, ya que garantiza que al añadir un nuevo caso al union type, el compilador force actualizar todos los switch/if que lo usan.
