title: ¿Qué es Declaration Merging en TypeScript?

Declaration Merging es la capacidad de TypeScript para combinar múltiples declaraciones con el mismo nombre en una sola definición. Ocurre principalmente con interfaces, namespaces y la combinación de funciones/clases con namespaces.

## Merging de interfaces

El caso más común: dos declaraciones de la misma interfaz se fusionan.

```typescript
// Declaración original de una librería
interface Window {
  history: History;
  location: Location;
}

// Tu declaración (en un .d.ts o en tu código)
interface Window {
  myCustomProperty: string;
  analytics: {
    track(event: string, data?: object): void;
  };
}

// TypeScript fusiona ambas declaraciones:
// window.myCustomProperty es válido
window.myCustomProperty = 'hola';
window.analytics.track('page_view', { url: '/home' });
```

```typescript
// Caso práctico: extender Express Request
// En express.d.ts o global.d.ts:
declare global {
  namespace Express {
    interface Request {
      user?: {
        id: string;
        email: string;
        roles: string[];
      };
    }
  }
}

// Ahora en tu middleware:
app.use((req, res, next) => {
  req.user = { id: '1', email: 'ana@test.com', roles: ['admin'] };
  next();
});

// Y en tus controladores:
app.get('/profile', (req, res) => {
  if (req.user) { // TypeScript sabe que user puede existir
    res.json(req.user.email); // ✅
  }
});
```

## Merging de namespaces

```typescript
// Los namespaces con el mismo nombre se fusionan
namespace Validation {
  export function isEmail(value: string): boolean {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value);
  }
}

namespace Validation {
  export function isPhone(value: string): boolean {
    return /^\+?[\d\s-]{9,}$/.test(value);
  }
}

// Acceso a ambas funciones fusionadas
Validation.isEmail('test@test.com'); // ✅
Validation.isPhone('+34 666 777 888'); // ✅
```

## Merging de función con namespace

```typescript
// Patrón para añadir propiedades estáticas a una función
function createUser(name: string): User {
  return { id: generateId(), name };
}

namespace createUser {
  export const defaultRole = 'user';
  export function withRole(name: string, role: string): User {
    return { id: generateId(), name, role };
  }
}

// Ahora la función tiene propiedades "estáticas"
const user1 = createUser('Ana');
const user2 = createUser.withRole('Luis', 'admin');
console.log(createUser.defaultRole); // 'user'
```

## Merging de clase con namespace (patrón de propiedades estáticas)

```typescript
class Logger {
  constructor(private context: string) {}

  log(message: string) {
    console.log(`[${this.context}] ${message}`);
  }
}

namespace Logger {
  export type Level = 'debug' | 'info' | 'warn' | 'error';

  export function create(context: string): Logger {
    return new Logger(context);
  }

  export const globalLogger = new Logger('Global');
}

// Acceso al tipo y a las propiedades del namespace
const logger: Logger = Logger.create('MyService'); // ✅
Logger.globalLogger.log('Aplicación iniciada'); // ✅
const level: Logger.Level = 'info'; // ✅
```

## Cuándo ocurre el conflicto (interfaces vs types)

```typescript
// Las interfaces se pueden fusionar — los types NO
interface User {
  name: string;
}
interface User {
  email: string;  // ✅ Merge permitido
}

type Product = { name: string };
type Product = { price: number }; // ❌ Error: Duplicate identifier 'Product'

// Por eso, para extensibilidad (librerías, augmentation) se prefieren interfaces
```

## Module Augmentation — extender módulos de terceros

```typescript
// Extender la librería 'axios' para añadir propiedades personalizadas
import 'axios';

declare module 'axios' {
  interface AxiosRequestConfig {
    retries?: number;           // Reintentos automáticos
    retryDelay?: number;        // Delay entre reintentos en ms
    skipAuth?: boolean;         // Saltar autenticación para esta petición
  }
}

// Ahora puedes usar las nuevas propiedades con tipado
axios.get('/api/users', {
  retries: 3,
  retryDelay: 1000,
  skipAuth: false
});
```

**Consejo de entrevista**: Declaration Merging es fundamental cuando trabajas con TypeScript en librerías y cuando extiendes tipos de terceros (Express Request, Axios config, Vue/React globals). El caso más frecuente es el module augmentation para tipar propiedades añadidas dinámicamente (como `req.user` en Express). La diferencia clave entre `interface` (se puede fusionar) y `type` (no se puede fusionar) determina cuándo usar cada uno en una librería.
