---
title: ¿Qué es el patrón Singleton con TypeScript?
difficulty: medium
tags: [typescript, singleton, design-patterns, classes]
---

El patrón **Singleton** garantiza que una clase tenga **una única instancia** durante toda la vida de la aplicación y proporciona un punto de acceso global a ella. Es útil para gestores de configuración, conexiones de base de datos, loggers y caches.

## Implementación clásica con clase

```typescript
class ConfiguracionApp {
  private static instancia: ConfiguracionApp;

  private readonly configuracion: Map<string, string>;

  // Constructor privado: nadie puede usar `new ConfiguracionApp()`
  private constructor() {
    this.configuracion = new Map();
    // Cargar configuración por defecto
    this.configuracion.set('tema', 'claro');
    this.configuracion.set('idioma', 'es');
    this.configuracion.set('apiUrl', 'https://api.ejemplo.com');
  }

  // Método estático para obtener la única instancia
  static obtenerInstancia(): ConfiguracionApp {
    if (!ConfiguracionApp.instancia) {
      ConfiguracionApp.instancia = new ConfiguracionApp();
    }
    return ConfiguracionApp.instancia;
  }

  obtener(clave: string): string | undefined {
    return this.configuracion.get(clave);
  }

  establecer(clave: string, valor: string): void {
    this.configuracion.set(clave, valor);
  }
}

// Uso: siempre la misma instancia
const config1 = ConfiguracionApp.obtenerInstancia();
const config2 = ConfiguracionApp.obtenerInstancia();
console.log(config1 === config2);  // true — son la misma instancia

config1.establecer('tema', 'oscuro');
console.log(config2.obtener('tema'));  // 'oscuro' — comparten el mismo estado
```

## Singleton thread-safe con inicialización lazy

```typescript
class ConexionBaseDatos {
  private static instancia: ConexionBaseDatos | null = null;
  private static creando = false;

  private pool: Pool;

  private constructor(config: DBConfig) {
    this.pool = createPool(config);
  }

  static async obtenerInstancia(config?: DBConfig): Promise<ConexionBaseDatos> {
    if (ConexionBaseDatos.instancia) {
      return ConexionBaseDatos.instancia;
    }

    if (!config) throw new Error('Config requerida para primera inicialización');

    // Prevenir múltiples inicializaciones concurrentes
    if (ConexionBaseDatos.creando) {
      // Esperar a que termine la inicialización en curso
      await new Promise(resolve => setTimeout(resolve, 50));
      return ConexionBaseDatos.obtenerInstancia();
    }

    ConexionBaseDatos.creando = true;
    const instancia = new ConexionBaseDatos(config);
    await instancia.verificarConexion();
    ConexionBaseDatos.instancia = instancia;
    ConexionBaseDatos.creando = false;

    return instancia;
  }

  async query<T>(sql: string, params?: unknown[]): Promise<T[]> {
    return this.pool.execute<T>(sql, params);
  }

  private async verificarConexion(): Promise<void> {
    await this.pool.getConnection();
    console.log('Base de datos conectada');
  }
}
```

## Singleton como módulo ES (patrón moderno)

```typescript
// logger.ts — El módulo ES es un Singleton por naturaleza
// (se ejecuta una vez y se cachea por el runtime)

class Logger {
  private logs: string[] = [];
  private nivel: 'debug' | 'info' | 'error' = 'info';

  setNivel(nivel: 'debug' | 'info' | 'error') {
    this.nivel = nivel;
  }

  debug(mensaje: string) {
    if (this.nivel === 'debug') {
      const entrada = `[DEBUG ${new Date().toISOString()}] ${mensaje}`;
      this.logs.push(entrada);
      console.debug(entrada);
    }
  }

  info(mensaje: string) {
    const entrada = `[INFO ${new Date().toISOString()}] ${mensaje}`;
    this.logs.push(entrada);
    console.info(entrada);
  }

  error(mensaje: string, error?: Error) {
    const entrada = `[ERROR ${new Date().toISOString()}] ${mensaje}${error ? ': ' + error.message : ''}`;
    this.logs.push(entrada);
    console.error(entrada);
  }

  exportarLogs(): string[] {
    return [...this.logs];
  }
}

// Exportar la instancia única — patrón de módulo Singleton
export const logger = new Logger();
```

```typescript
// Uso en cualquier archivo:
import { logger } from './logger';

logger.info('Aplicación iniciada');
logger.error('Error en la API', new Error('404 Not Found'));
```

## Singleton con Symbol para máxima privacidad

```typescript
const INSTANCIA_KEY = Symbol('EventBus.instancia');
const SUSCRIPTORES_KEY = Symbol('EventBus.suscriptores');

type Manejador<T = unknown> = (payload: T) => void;

class EventBus {
  private [SUSCRIPTORES_KEY] = new Map<string, Manejador[]>();
  private static [INSTANCIA_KEY]: EventBus | undefined;

  private constructor() {}

  static getInstance(): EventBus {
    if (!EventBus[INSTANCIA_KEY]) {
      EventBus[INSTANCIA_KEY] = new EventBus();
    }
    return EventBus[INSTANCIA_KEY]!;
  }

  on<T>(evento: string, manejador: Manejador<T>): () => void {
    const manejadores = this[SUSCRIPTORES_KEY].get(evento) ?? [];
    manejadores.push(manejador as Manejador);
    this[SUSCRIPTORES_KEY].set(evento, manejadores);

    // Retorna función de desuscripción
    return () => this.off(evento, manejador as Manejador);
  }

  emit<T>(evento: string, payload: T): void {
    this[SUSCRIPTORES_KEY].get(evento)?.forEach(h => h(payload));
  }

  private off(evento: string, manejador: Manejador): void {
    const manejadores = this[SUSCRIPTORES_KEY].get(evento) ?? [];
    this[SUSCRIPTORES_KEY].set(
      evento,
      manejadores.filter(h => h !== manejador)
    );
  }
}

export const eventBus = EventBus.getInstance();
```

## Cuándo usar Singleton (y cuándo no)

```
✅ Casos válidos:
- Logger de aplicación
- Configuración global
- Conexión a base de datos (pool)
- Cache en memoria
- Event bus

❌ Antipatrones:
- Estado de UI (usar Context, Zustand, Pinia, etc.)
- Estado por usuario/sesión
- Cuando dificulta el testing (difícil hacer mock)
- Cuando necesitas múltiples configuraciones diferentes
```

**Consejo de entrevista**: El patrón moderno en JavaScript/TypeScript es el **módulo ES como Singleton**: exportar una instancia creada al nivel del módulo (`export const logger = new Logger()`). El runtime de Node.js y los navegadores cachean los módulos, por lo que la misma instancia se comparte en toda la aplicación. La clase con constructor privado es más explícita pero más verbosa. Para tests, el Singleton puede ser problemático —una solución es exportar también la clase para poder instanciarla independientemente en los tests.
