Los **Enums** en TypeScript permiten definir un conjunto de constantes con nombre, creando un tipo especial que representa valores relacionados. Existen tres tipos: numéricos, de cadena y heterogéneos.

## Enum numérico (por defecto)

```typescript
enum Direccion {
  Norte,   // 0
  Sur,     // 1
  Este,    // 2
  Oeste    // 3
}

const miDireccion: Direccion = Direccion.Norte;
console.log(miDireccion);          // 0
console.log(Direccion[0]);         // "Norte" (mapping inverso)
console.log(Direccion.Norte);      // 0

// Con valores personalizados
enum CodigoHTTP {
  OK = 200,
  Creado = 201,
  NoEncontrado = 404,
  Error = 500
}

function manejarRespuesta(codigo: CodigoHTTP) {
  if (codigo === CodigoHTTP.OK) {
    console.log('Éxito');
  }
}
```

## Enum de cadena (más recomendado en proyectos modernos)

```typescript
enum EstadoPedido {
  Pendiente = 'PENDIENTE',
  Procesando = 'PROCESANDO',
  Enviado = 'ENVIADO',
  Entregado = 'ENTREGADO',
  Cancelado = 'CANCELADO'
}

interface Pedido {
  id: number;
  estado: EstadoPedido;
}

function actualizarEstado(pedido: Pedido, nuevoEstado: EstadoPedido): Pedido {
  return { ...pedido, estado: nuevoEstado };
}

// El valor es legible en logs y APIs
const pedido = { id: 1, estado: EstadoPedido.Pendiente };
console.log(pedido.estado);  // "PENDIENTE" (no "0")

// Guardias de tipo con enum
function estaEnTransito(estado: EstadoPedido): boolean {
  return estado === EstadoPedido.Procesando || estado === EstadoPedido.Enviado;
}
```

## const enum — Optimización de bundle

```typescript
// const enum: TypeScript los inlinea en tiempo de compilación
// No genera objeto JavaScript → bundle más pequeño
const enum Permiso {
  Leer = 'READ',
  Escribir = 'WRITE',
  Eliminar = 'DELETE',
  Admin = 'ADMIN'
}

// El compilador convierte esto:
if (permiso === Permiso.Leer) { /* ... */ }

// En esto (no hay objeto Permiso en el JS generado):
if (permiso === 'READ') { /* ... */ }

// ⚠️ Limitación: no se puede usar en .d.ts de librerías publicadas
// ni cuando se importan desde archivos distintos con isolatedModules
```

## La alternativa moderna: Union Types con as const

```typescript
// Muchos desarrolladores TypeScript modernos prefieren esto a los enums:
const EstadoPedido = {
  Pendiente: 'PENDIENTE',
  Procesando: 'PROCESANDO',
  Enviado: 'ENVIADO',
  Entregado: 'ENTREGADO',
  Cancelado: 'CANCELADO'
} as const;

type EstadoPedido = typeof EstadoPedido[keyof typeof EstadoPedido];
// type EstadoPedido = "PENDIENTE" | "PROCESANDO" | "ENVIADO" | "ENTREGADO" | "CANCELADO"

// ✅ Sin objeto extra en runtime (con const enum también)
// ✅ Funciona con isolatedModules
// ✅ Los valores son strings literales (mejor autocompletado)
// ✅ Iterable: Object.values(EstadoPedido)
```

## Enum vs Union Types: comparativa

```typescript
// Enum:
enum Color { Rojo = 'RED', Verde = 'GREEN', Azul = 'BLUE' }
function pintarColor(c: Color) { /* ... */ }
pintarColor(Color.Rojo);     // ✅
pintarColor('RED');           // ❌ Error! No se puede asignar string a Color

// Union Type:
type Color = 'RED' | 'GREEN' | 'BLUE';
function pintarColor(c: Color) { /* ... */ }
pintarColor('RED');           // ✅ Strings literales compatibles
```

## Uso con switch exhaustivo

```typescript
enum Figura {
  Circulo = 'CIRCULO',
  Rectangulo = 'RECTANGULO',
  Triangulo = 'TRIANGULO'
}

function calcularArea(figura: Figura, ...medidas: number[]): number {
  switch (figura) {
    case Figura.Circulo:
      return Math.PI * medidas[0] ** 2;
    case Figura.Rectangulo:
      return medidas[0] * medidas[1];
    case Figura.Triangulo:
      return (medidas[0] * medidas[1]) / 2;
    default:
      // TypeScript detecta si falta algún case
      const _exhaustivo: never = figura;
      throw new Error(`Figura no manejada: ${_exhaustivo}`);
  }
}
```

**Consejo de entrevista**: Los enums en TypeScript son más verbosos que en otros lenguajes porque generan código JavaScript real (excepto `const enum`). La tendencia moderna es preferir **Union Types de strings literales** (`type Estado = 'activo' | 'inactivo'`) o el patrón `as const` para objetos de constantes, ya que tienen mejor integración con el sistema de tipos, son más ligeros en bundle y funcionan bien con `isolatedModules`. Sin embargo, los enums siguen siendo útiles cuando necesitas el mapping inverso o un conjunto de constantes claramente encapsulado.
