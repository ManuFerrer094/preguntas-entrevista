---
title: ¿Qué son los Conditional Types en TypeScript?
difficulty: hard
tags: [typescript, conditional-types, advanced, generics]
---

Los Conditional Types permiten expresar tipos que dependen de una condición: `T extends U ? X : Y`. Si el tipo `T` es asignable a `U`, el resultado es `X`; de lo contrario, es `Y`. Son fundamentales para crear utilidades de tipo avanzadas y son la base de muchos utility types de TypeScript.

## Sintaxis básica

```typescript
type IsString<T> = T extends string ? true : false;

type A = IsString<string>;  // true
type B = IsString<number>;  // false
type C = IsString<'hello'>; // true (string literal extends string)
```

## Conditional Types con genéricos

```typescript
// NonNullable<T> — eliminar null y undefined
type NonNullable<T> = T extends null | undefined ? never : T;

type WithNull = string | null | undefined;
type WithoutNull = NonNullable<WithNull>; // string

// ReturnType<T> — obtener el tipo de retorno de una función
type ReturnType<T extends (...args: any) => any> =
  T extends (...args: any) => infer R ? R : never;

function getUser(): { id: number; name: string } {
  return { id: 1, name: 'Ana' };
}
type UserType = ReturnType<typeof getUser>; // { id: number; name: string }

// Parameters<T> — obtener los tipos de los parámetros
type Parameters<T extends (...args: any) => any> =
  T extends (...args: infer P) => any ? P : never;

function createUser(name: string, age: number, email: string): void {}
type CreateUserParams = Parameters<typeof createUser>; // [string, number, string]
```

## La palabra clave `infer`

`infer` permite extraer y "capturar" un tipo de dentro de otro tipo durante la evaluación condicional:

```typescript
// Extraer el tipo del elemento de un array
type ElementType<T> = T extends (infer E)[] ? E : never;

type StringArr = ElementType<string[]>;   // string
type NumberArr = ElementType<number[]>;   // number

// Extraer el tipo de una promesa
type Awaited<T> = T extends Promise<infer R> ? Awaited<R> : T; // Recursivo

type PromiseResult = Awaited<Promise<Promise<string>>>; // string

// Extraer el tipo del primer parámetro
type FirstParam<T extends (...args: any) => any> =
  T extends (first: infer F, ...rest: any[]) => any ? F : never;

function fetchData(url: string, options?: RequestInit) {}
type FirstArg = FirstParam<typeof fetchData>; // string
```

## Distribución sobre union types

Los conditional types se distribuyen automáticamente sobre union types:

```typescript
type ToArray<T> = T extends any ? T[] : never;

type StringOrNumber = ToArray<string | number>;
// Se distribuye: ToArray<string> | ToArray<number>
// = string[] | number[]

// Excluir tipos de un union
type Exclude<T, U> = T extends U ? never : T;
type Extract<T, U> = T extends U ? T : never;

type WithoutString = Exclude<string | number | boolean, string>;
// = number | boolean
```

## Casos de uso prácticos

```typescript
// Deep partial — hacer todos los niveles del objeto opcionales
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object ? DeepPartial<T[K]> : T[K];
};

// Filtrar keys por tipo de valor
type KeysOfType<T, V> = {
  [K in keyof T]: T[K] extends V ? K : never;
}[keyof T];

interface User {
  id: number;
  name: string;
  email: string;
  age: number;
  isActive: boolean;
}

type StringKeys = KeysOfType<User, string>; // 'name' | 'email'
type NumberKeys = KeysOfType<User, number>; // 'id' | 'age'

// Tipo condicional basado en la longitud del array
type IsNonEmpty<T extends any[]> = T extends [] ? false : true;
type A = IsNonEmpty<[]>;          // false
type B = IsNonEmpty<[1, 2, 3]>;   // true
```

**Consejo de entrevista**: Los Conditional Types son TypeScript avanzado y suelen aparecer en entrevistas para roles senior. El punto clave es entender `infer`: permite "extraer" tipos anidados, como el tipo de retorno de una función, el tipo de elementos de un array o el tipo resuelto de una promesa. Saber que los conditional types se distribuyen sobre union types (`T extends U ? X : Y` con `T = A | B` resulta en `(A extends U ? X : Y) | (B extends U ? X : Y)`) es fundamental.
