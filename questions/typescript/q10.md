---
title: ¿Qué es la inferencia de tipos en TypeScript?
difficulty: medium
tags: [typescript, type-inference, type-system, basics]
---

La inferencia de tipos es la capacidad de TypeScript para deducir automáticamente el tipo de una variable, función o expresión sin que el programador lo declare explícitamente. TypeScript es inteligente: analiza el contexto y la inicialización para inferir el tipo más preciso posible.

Esto significa que en muchos casos, añadir anotaciones de tipo manualmente es redundante (y más verboso) cuando TypeScript puede inferirlas correctamente.

## Inferencia en variables

```typescript
// TypeScript infiere el tipo en la inicialización
let name = 'Ana';        // Inferido como: string
let age = 28;            // Inferido como: number
let active = true;       // Inferido como: boolean
let score = 9.5;         // Inferido como: number

// Tipos literales inferidos con const
const status = 'active'; // Inferido como: 'active' (no string)
const MAX_SIZE = 100;    // Inferido como: 100 (no number)

// Arrays e objetos
const items = [1, 2, 3];           // Inferido como: number[]
const user = { id: 1, name: 'Ana' }; // Inferido como: { id: number; name: string }
```

## Inferencia en funciones

```typescript
// Tipo de retorno inferido
function add(a: number, b: number) {
  return a + b; // TypeScript infiere: number
}

// Con genéricos
function identity<T>(value: T) {
  return value; // TypeScript infiere el tipo de retorno como T
}

const str = identity('hello'); // TypeScript infiere: string
const num = identity(42);      // TypeScript infiere: number

// Funciones de array
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(n => n * 2); // Inferido como: number[]
const evens = numbers.filter(n => n % 2 === 0); // Inferido como: number[]
const sum = numbers.reduce((acc, n) => acc + n, 0); // Inferido como: number
```

## Inferencia contextual

TypeScript puede inferir tipos del contexto en el que se usa una expresión:

```typescript
// El tipo del callback se infiere del tipo del array
const words = ['hello', 'world'];
words.forEach(word => {
  // TypeScript sabe que word es string, sin necesidad de anotarlo
  console.log(word.toUpperCase());
});

// Tipos de eventos
document.addEventListener('click', (event) => {
  // TypeScript infiere que event es MouseEvent (del contexto 'click')
  console.log(event.clientX, event.clientY);
});

// Object destructuring
function processUser({ name, age }: { name: string; age: number }) {
  // TypeScript infiere los tipos de name y age del parámetro
  return `${name} tiene ${age} años`;
}
```

## Cuándo anotar tipos explícitamente

La inferencia no siempre es suficiente. Se recomienda anotar tipos cuando:

```typescript
// 1. Parámetros de función sin valor por defecto
function greet(name: string): string { // Parámetros SIEMPRE anotados
  return `Hola, ${name}`;
}

// 2. Variables que se inicializan después
let user: User | null = null; // Sin anotación, TypeScript inferiría null
user = await fetchUser();

// 3. El tipo inferido es demasiado amplio
const config = {
  mode: 'development' as 'development' | 'production',
  // Sin 'as', TypeScript infiere: string (demasiado amplio)
};

// 4. Función que devuelve un tipo complejo (claridad para otros devs)
function buildQuery(params: FilterParams): QueryBuilder {
  // Anotar el retorno mejora la legibilidad y el self-documentation
}

// 5. Variables de array inicialmente vacías
const users: User[] = []; // Sin anotación: never[]
```

## typeof para capturar tipos inferidos

```typescript
const defaultConfig = {
  host: 'localhost',
  port: 3000,
  debug: false,
  allowedOrigins: ['http://localhost:4200']
};

// Capturar el tipo inferido de un objeto existente
type Config = typeof defaultConfig;
// { host: string; port: number; debug: boolean; allowedOrigins: string[] }

function configure(options: Partial<Config>): Config {
  return { ...defaultConfig, ...options };
}
```

**Consejo de entrevista**: La guía de TypeScript recomienda "escribir tipos en los límites de la API" (parámetros de funciones públicas, propiedades de interfaces) y dejar que TypeScript infiera el resto. Anotar todo de forma redundante hace el código más verboso sin añadir seguridad. Una señal de código TypeScript maduro es el uso inteligente de la inferencia para reducir la verbosidad.
