---
title: ¿Qué son los Template Literal Types en TypeScript?
difficulty: medium
tags: [typescript, template-literal-types, advanced, string-types]
---

Los Template Literal Types (introducidos en TypeScript 4.1) permiten crear tipos de cadena de texto usando la sintaxis de template literals de JavaScript. Son extremadamente útiles para crear tipos que representan combinaciones de strings, como rutas de API, nombres de eventos, o keys de objetos.

## Sintaxis básica

```typescript
type Greeting = `Hello, ${string}!`; // Cualquier string con ese formato

const greet1: Greeting = 'Hello, World!';  // ✅
const greet2: Greeting = 'Hello, Ana!';    // ✅
// const greet3: Greeting = 'Hi, Ana!';    // ❌ Error

// Combinar tipos literales
type Color = 'red' | 'green' | 'blue';
type Size = 'sm' | 'md' | 'lg';

type ColorSize = `${Color}-${Size}`;
// 'red-sm' | 'red-md' | 'red-lg' | 'green-sm' | 'green-md' | ...
// ¡TypeScript genera todas las combinaciones!

const classNames: ColorSize = 'blue-lg'; // ✅
```

## Intrinsic String Manipulation Types

TypeScript incluye tipos para transformar strings:

```typescript
type Upper = Uppercase<'hello world'>;    // 'HELLO WORLD'
type Lower = Lowercase<'HELLO WORLD'>;    // 'hello world'
type Cap   = Capitalize<'hello world'>;   // 'Hello world'
type Uncap = Uncapitalize<'Hello World'>; // 'hELLO wORLD'
```

## Casos de uso prácticos

**Rutas de API type-safe:**

```typescript
type ApiVersion = 'v1' | 'v2';
type Resource = 'users' | 'products' | 'orders';
type ApiRoute = `/api/${ApiVersion}/${Resource}`;

function fetchApi(route: ApiRoute): Promise<Response> {
  return fetch(route);
}

fetchApi('/api/v1/users');    // ✅
fetchApi('/api/v2/products'); // ✅
// fetchApi('/api/v3/users'); // ❌ 'v3' no es válido
// fetchApi('/api/v1/items'); // ❌ 'items' no es válido
```

**Event handler types:**

```typescript
type EventName = 'click' | 'focus' | 'blur' | 'change' | 'submit';
type EventHandler = `on${Capitalize<EventName>}`;
// 'onClick' | 'onFocus' | 'onBlur' | 'onChange' | 'onSubmit'

type ButtonProps = {
  [K in EventHandler]?: (event: Event) => void;
};

// CSS class helpers
type CssProperty = 'margin' | 'padding' | 'border';
type CssDirection = 'Top' | 'Right' | 'Bottom' | 'Left';
type CssProperty4 = `${CssProperty}${CssDirection}`;
// 'marginTop' | 'marginRight' | ... | 'paddingTop' | ... | 'borderTop' | ...
```

**Getters y setters automáticos:**

```typescript
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

type Setters<T> = {
  [K in keyof T as `set${Capitalize<string & K>}`]: (value: T[K]) => void;
};

interface Config {
  host: string;
  port: number;
  debug: boolean;
}

type ConfigApi = Getters<Config> & Setters<Config>;
// {
//   getHost: () => string;   setHost: (value: string) => void;
//   getPort: () => number;   setPort: (value: number) => void;
//   getDebug: () => boolean; setDebug: (value: boolean) => void;
// }
```

**Keys de objeto concatenadas:**

```typescript
type FlattenKeys<T, Prefix extends string = ''> = {
  [K in keyof T]: T[K] extends object
    ? FlattenKeys<T[K], `${Prefix}${string & K}.`>
    : `${Prefix}${string & K}`;
}[keyof T];

interface NestedObject {
  user: {
    name: string;
    address: {
      city: string;
      zip: string;
    };
  };
  settings: {
    theme: string;
  };
}

type Keys = FlattenKeys<NestedObject>;
// 'user.name' | 'user.address.city' | 'user.address.zip' | 'settings.theme'
```

**Consejo de entrevista**: Los Template Literal Types son relativamente nuevos (TypeScript 4.1+) y demuestran conocimiento actualizado del lenguaje. Sus usos más prácticos en el día a día son: rutas de API type-safe, nombres de eventos con prefijo (`on${EventName}`), y la combinación con mapped types para generar getters/setters automáticamente. Combinados con `Capitalize`, `Uppercase`, etc., son muy poderosos para trabajar con convenciones de nomenclatura.
