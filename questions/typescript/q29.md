`NoInfer<T>` es una utilidad de tipo incorporada en **TypeScript 5.4** que bloquea la inferencia de tipo en una posición específica de un genérico. Permite controlar con precisión dónde TypeScript infiere el tipo de un genérico, evitando inferencias no deseadas desde ciertos argumentos.

## El problema sin NoInfer

```typescript
// Función que establece un valor por defecto dentro de un array de opciones
function crearSelector<T>(
  opciones: T[],
  porDefecto: T   // ← TypeScript infiere T desde AMBOS parámetros
): { opciones: T[]; porDefecto: T } {
  return { opciones, porDefecto };
}

// El problema: TypeScript ensancha el tipo inferido
const selector = crearSelector(
  ['rojo', 'verde', 'azul'],   // Infiere T = string
  'amarillo'                    // ✅ Acepta cualquier string, incluso 'amarillo'
                                //    que NO está en las opciones
);

// El tipo es correcto pero la lógica es incorrecta:
// 'amarillo' no está en ['rojo', 'verde', 'azul']
// TypeScript no detecta este error
```

## Solución con NoInfer (TypeScript 5.4+)

```typescript
function crearSelector<T>(
  opciones: T[],
  porDefecto: NoInfer<T>   // ← NO infiere T desde este parámetro
): { opciones: T[]; porDefecto: T } {
  return { opciones, porDefecto };
}

// Ahora T se infiere SOLO desde `opciones`
const selector1 = crearSelector(
  ['rojo', 'verde', 'azul'],   // T = 'rojo' | 'verde' | 'azul'
  'amarillo'                    // ❌ Error: 'amarillo' no es asignable a T
);

const selector2 = crearSelector(
  ['rojo', 'verde', 'azul'],
  'rojo'                        // ✅ 'rojo' está en las opciones
);
```

## Antes de NoInfer: soluciones manuales

```typescript
// Solución 1: Usar [T] extends [never] para bloquear inferencia
function crearSelectorAntes<T>(
  opciones: T[],
  porDefecto: [T] extends [T] ? T : never  // Truco para bloquear inferencia
): void { /* ... */ }

// Solución 2: Dividir en dos funciones
function crearSelectorFactory<T>(opciones: T[]) {
  return function withDefault(porDefecto: T) {
    return { opciones, porDefecto };
  };
}
const factory = crearSelectorFactory(['rojo', 'verde', 'azul']);
factory('amarillo');  // ❌ Error (más verboso pero funciona)

// NoInfer es mucho más simple y legible ✅
```

## Casos de uso prácticos

### Estado inicial de formularios

```typescript
function crearFormulario<T extends Record<string, unknown>>(
  esquema: { [K in keyof T]: Validador<T[K]> },
  valorInicial: NoInfer<Partial<T>>   // T se infiere del esquema, no del valor inicial
): FormularioReactivo<T> {
  // implementación
}

const formulario = crearFormulario(
  {
    nombre: validarTexto({ minLength: 2 }),
    email: validarEmail(),
    edad: validarNumero({ min: 0, max: 150 })
  },
  {
    nombre: 'Ana',
    // edad: 'texto'  ← ❌ Error: string no es asignable a number (correcto!)
  }
);
```

### Configuración de animaciones

```typescript
type Animacion = 'entrada' | 'salida' | 'pulso' | 'deslizar';

function configurarAnimacion<T extends string>(
  animaciones: T[],
  porDefecto: NoInfer<T>,
  alHover: NoInfer<T>
): ConfigAnimacion {
  return { animaciones, porDefecto, alHover };
}

const config = configurarAnimacion(
  ['entrada', 'salida', 'pulso'] as const,
  'entrada',     // ✅
  'deslizar'     // ❌ Error: 'deslizar' no está en las opciones
);
```

### Event emitters tipados

```typescript
type Handler<T> = (event: T) => void;

function crearEmitter<EventMap extends Record<string, unknown>>() {
  const handlers = new Map<keyof EventMap, Handler<unknown>[]>();

  return {
    on<K extends keyof EventMap>(
      evento: K,
      handler: Handler<NoInfer<EventMap[K]>>  // No permite re-inferir EventMap
    ) {
      const lista = handlers.get(evento) ?? [];
      lista.push(handler as Handler<unknown>);
      handlers.set(evento, lista);
    },
    emit<K extends keyof EventMap>(evento: K, payload: EventMap[K]) {
      handlers.get(evento)?.forEach(h => h(payload));
    }
  };
}

type MisEventos = {
  'usuario-login': { id: number; nombre: string };
  'carrito-actualizado': { total: number; items: number };
};

const emitter = crearEmitter<MisEventos>();
emitter.on('usuario-login', ({ id, nombre }) => {
  console.log(`Login: ${nombre} (${id})`);
});
```

## Comparativa: con y sin NoInfer

```typescript
// Sin NoInfer: TypeScript infiere desde todos los argumentos
function sin<T>(a: T, b: T): T { return a; }
sin(['a', 'b'], 'c');  // T = string[] | string — inesperado

// Con NoInfer: inferencia controlada
function con<T>(a: T, b: NoInfer<T>): T { return a; }
con(['a', 'b'], 'c');  // ❌ Error: 'c' (string) no es asignable a string[] (T inferido de `a`)
```

**Consejo de entrevista**: `NoInfer<T>` es una adición de TypeScript 5.4 muy solicitada por la comunidad. Antes de ella, el patrón `[T] extends [T] ? T : never` era el truco común para bloquear la inferencia en una posición, pero era poco legible. `NoInfer` resuelve un problema sutil pero frecuente: cuando tienes un genérico inferido desde varios parámetros y quieres que solo algunos participen en la inferencia. Es especialmente útil en librerías de UI que necesitan que los valores por defecto sean compatibles con los arrays de opciones.
