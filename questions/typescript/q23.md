---
title: ¿Qué son los Namespaces en TypeScript?
difficulty: medium
tags: [typescript, namespaces, modules, organization, declaration-files]
---

Los Namespaces agrupan código relacionado bajo un identificador para evitar colisiones de nombres en el espacio global. Son el sistema de organización anterior a los módulos ES6 y todavía se usan principalmente en archivos de declaración (`.d.ts`) y para augmentar módulos de terceros.

## Namespaces básicos

```typescript
namespace Validation {
  export interface StringValidator {
    isAcceptable(s: string): boolean;
  }

  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/; // No exportado → privado

  export class EmailValidator implements StringValidator {
    isAcceptable(s: string): boolean {
      return emailRegex.test(s);
    }
  }

  export class LengthValidator implements StringValidator {
    constructor(private minLength: number, private maxLength: number) {}

    isAcceptable(s: string): boolean {
      return s.length >= this.minLength && s.length <= this.maxLength;
    }
  }
}

// Uso
const emailValidator = new Validation.EmailValidator();
const lengthValidator = new Validation.LengthValidator(3, 50);

emailValidator.isAcceptable('test@test.com');   // true
lengthValidator.isAcceptable('hola');           // true

// El emailRegex no es accesible desde fuera
// Validation.emailRegex  ❌ Error: privado
```

## Namespaces anidados

```typescript
namespace App {
  export namespace Models {
    export interface User {
      id: string;
      name: string;
      email: string;
    }

    export interface Product {
      id: string;
      name: string;
      price: number;
    }
  }

  export namespace Services {
    export class UserService {
      getUser(id: string): App.Models.User {
        return { id, name: 'Ana', email: 'ana@test.com' };
      }
    }
  }

  export namespace Utils {
    export function formatCurrency(amount: number, currency = 'EUR'): string {
      return new Intl.NumberFormat('es-ES', {
        style: 'currency',
        currency
      }).format(amount);
    }
  }
}

// Uso
const userService = new App.Services.UserService();
const user: App.Models.User = userService.getUser('1');
console.log(App.Utils.formatCurrency(99.99)); // '99,99 €'
```

## Alias para namespaces anidados

```typescript
// Alias para evitar escribir el namespace completo
import UserService = App.Services.UserService;
import Models = App.Models;

const service = new UserService();
const user: Models.User = service.getUser('1');
```

## Namespaces en archivos de declaración (el uso más importante hoy en día)

```typescript
// typings/global.d.ts — Extender el objeto global del navegador
declare namespace NodeJS {
  interface ProcessEnv {
    NODE_ENV: 'development' | 'test' | 'production';
    PORT?: string;
    DATABASE_URL: string;
    JWT_SECRET: string;
  }
}

// Ahora process.env tiene tipos correctos
const port = process.env.PORT ?? '3000'; // string | undefined
const env = process.env.NODE_ENV; // 'development' | 'test' | 'production'
```

```typescript
// Declarar librerías sin tipos (variables globales)
declare namespace MyLibrary {
  function makeGreeting(s: string): string;
  let numberOfGreetings: number;
}

// Usar la librería con tipado
MyLibrary.makeGreeting('World');
```

## Modules vs Namespaces

```typescript
// ✅ MÓDULOS ES6 (recomendado para código nuevo)
// user.service.ts
export class UserService {
  getUser(id: string) { /* ... */ }
}

// main.ts
import { UserService } from './user.service';
const service = new UserService();

// ⚠️ NAMESPACES (solo para archivos .d.ts y código legado)
namespace UserModule {
  export class UserService {
    getUser(id: string) { /* ... */ }
  }
}
```

| | Módulos ES6 | Namespaces |
|---|---|---|
| Ámbito | Fichero | Global |
| Recomendado para | Todo código nuevo | `.d.ts`, augmentation |
| Bundling | Soportado (tree-shaking) | Requiere referencia manual |
| Lazy loading | Nativo (`import()`) | No soportado |

## Cuándo usar namespaces hoy en día

```typescript
// ✅ 1. Extender tipos globales (module augmentation)
declare global {
  interface Window {
    analyticsTracker: (event: string) => void;
  }
}

// ✅ 2. Agrupar tipos relacionados en archivos .d.ts
declare namespace API {
  interface User { id: string; name: string; }
  interface Post { id: string; title: string; userId: string; }
  interface Response<T> { data: T; status: number; }
}

// ✅ 3. Archivos de declaración de librerías sin módulos
declare namespace jQuery {
  function ajax(url: string, options?: any): void;
  function get(url: string): void;
}

// ❌ No usar para organizar código de aplicación — usa módulos ES6
```

**Consejo de entrevista**: En proyectos modernos, los **módulos ES6** son la forma correcta de organizar el código; los namespaces son principalmente para declaraciones de tipos globales y archivos `.d.ts`. Mencionar que la principal diferencia es que los módulos tienen su propio ámbito de fichero mientras que los namespaces viven en el espacio global demuestra comprensión de cuándo y por qué usar cada uno.
