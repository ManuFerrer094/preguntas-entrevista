---
title: ¿Qué son los const assertions en TypeScript?
difficulty: medium

---

`as const` (const assertion) le dice a TypeScript que infiera el tipo más específico posible para un valor: convierte strings a string literals, numbers a number literals, y hace arrays y objetos completamente readonly.

## Problema sin as const

```typescript
// Sin as const: TypeScript infiere tipos amplios
const direction = 'north';
// TypeScript infiere: direction: string (¡podría cambiar!)

let mutableDir = 'north';
// TypeScript infiere: mutableDir: string (correcto, puede cambiar)

const config = {
  host: 'localhost',
  port: 3000
};
// TypeScript infiere: { host: string; port: number }
// Podría cambiar: config.host = 'otro' ✅ (permite mutación)

const directions = ['north', 'south', 'east', 'west'];
// TypeScript infiere: string[] (no el tipo literal exacto)
```

## as const en acción

```typescript
// Con as const: TypeScript infiere el tipo más preciso
const direction = 'north' as const;
// TypeScript infiere: direction: 'north' (string literal, no string)

const config = {
  host: 'localhost',
  port: 3000
} as const;
// TypeScript infiere: { readonly host: 'localhost'; readonly port: 3000 }
// No permite mutación: config.host = 'otro' ❌ Error (readonly)

const directions = ['north', 'south', 'east', 'west'] as const;
// TypeScript infiere: readonly ['north', 'south', 'east', 'west']
// No permite: directions.push('up') ❌ Error
```

## Extraer tipos de unión de arrays

```typescript
const ROLES = ['user', 'admin', 'moderator'] as const;
type Role = typeof ROLES[number]; // 'user' | 'admin' | 'moderator'

// Sin as const: typeof ROLES[number] sería string

function hasRole(role: Role): boolean {
  return ROLES.includes(role);
}

hasRole('admin'); // ✅
hasRole('superadmin'); // ❌ Error: 'superadmin' no es asignable a Role
```

## Constantes de configuración type-safe

```typescript
// Tabla de configuración totalmente type-safe
const HTTP_STATUS = {
  OK: 200,
  CREATED: 201,
  BAD_REQUEST: 400,
  UNAUTHORIZED: 401,
  NOT_FOUND: 404,
  INTERNAL_SERVER_ERROR: 500,
} as const;

type StatusCode = typeof HTTP_STATUS[keyof typeof HTTP_STATUS];
// 200 | 201 | 400 | 401 | 404 | 500

function createResponse(status: StatusCode, data: unknown) {
  return { status, data };
}

createResponse(HTTP_STATUS.OK, { users: [] }); // ✅
createResponse(200, { users: [] }); // ✅
createResponse(999, { users: [] }); // ❌ Error: 999 no es un StatusCode válido
```

## as const en objetos de rutas

```typescript
const ROUTES = {
  HOME: '/',
  LOGIN: '/login',
  DASHBOARD: '/dashboard',
  USERS: {
    LIST: '/users',
    DETAIL: '/users/:id',
    EDIT: '/users/:id/edit'
  }
} as const;

type AppRoute = typeof ROUTES.USERS[keyof typeof ROUTES.USERS];
// '/users' | '/users/:id' | '/users/:id/edit'

function navigate(route: AppRoute | '/') {
  window.location.href = route;
}

navigate(ROUTES.USERS.LIST);     // ✅
navigate('/users');              // ✅
navigate('/inexistent-route');   // ❌ Error
```

## as const vs enum

```typescript
// enum: crea un objeto en runtime, valores numéricos por defecto
enum Direction {
  North = 'north',
  South = 'south',
  East = 'east',
  West = 'west'
}

// as const: sin overhead en runtime, misma funcionalidad
const Direction = {
  North: 'north',
  South: 'south',
  East: 'east',
  West: 'west'
} as const;

type Direction = typeof Direction[keyof typeof Direction];
// 'north' | 'south' | 'east' | 'west'

// Ventajas de as const sobre enum:
// ✅ No genera código extra en el bundle
// ✅ Valores usables directamente como strings/numbers
// ✅ Interoperable con JSON.parse y serialización
// ⚠️ enum tiene mejor soporte en algunos ORM/validadores
```

## Tuples con as const

```typescript
// Sin as const: inferido como (string | number)[]
const point = [10, 20];

// Con as const: inferido como readonly [10, 20] (tuple literal)
const point2 = [10, 20] as const;

function drawPoint([x, y]: readonly [number, number]) {
  console.log(`(${x}, ${y})`);
}

drawPoint(point2); // ✅ — el tipo es compatible

// Útil para crear tuplas type-safe
function useState<T>(initial: T): readonly [T, (value: T) => void] {
  let state = initial;
  return [state, (v) => { state = v; }] as const;
}
```

**Consejo de entrevista**: `as const` es la solución idiomática de TypeScript para crear constantes con tipos literales sin el overhead de los `enum`. El patrón más frecuente en entrevistas es el uso de `typeof ARRAY[number]` para extraer una unión de tipos desde un array con `as const`. Recuerda que `as const` hace el objeto **completamente readonly** en tiempo de compilación, pero no previene la mutación en runtime (eso requeriría `Object.freeze()`).
