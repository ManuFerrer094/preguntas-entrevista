title: ¿Qué son los Mapped Types en TypeScript?

Los Mapped Types son una característica avanzada de TypeScript que permite crear nuevos tipos transformando cada propiedad de un tipo existente. Son la base de muchos Utility Types integrados como `Partial`, `Readonly`, `Required` y `Pick`.

La sintaxis básica es: `{ [K in keyof T]: ... }`

## Utility Types integrados (implementados como Mapped Types)

```typescript
// Partial<T> — todas las propiedades opcionales
type Partial<T> = {
  [K in keyof T]?: T[K];
};

// Required<T> — todas las propiedades requeridas
type Required<T> = {
  [K in keyof T]-?: T[K]; // -? elimina la opcionalidad
};

// Readonly<T> — todas las propiedades de solo lectura
type Readonly<T> = {
  readonly [K in keyof T]: T[K];
};

// Record<K, V> — objeto con claves K y valores V
type Record<K extends keyof any, V> = {
  [P in K]: V;
};

// Pick<T, K> — seleccionar propiedades K de T
type Pick<T, K extends keyof T> = {
  [P in K]: T[P];
};

// Omit<T, K> — excluir propiedades K de T
type Omit<T, K extends keyof T> = Pick<T, Exclude<keyof T, K>>;
```

## Ejemplos prácticos

```typescript
interface User {
  id: number;
  name: string;
  email: string;
  password: string;
  role: 'admin' | 'user';
  createdAt: Date;
}

// Para actualizar, todas las propiedades son opcionales (excepto id)
type UpdateUserDTO = Partial<Omit<User, 'id' | 'createdAt'>>;
// { name?: string; email?: string; password?: string; role?: 'admin' | 'user' }

// Para respuestas públicas, excluir el password
type PublicUser = Omit<User, 'password'>;

// Para el formulario de login
type LoginDTO = Pick<User, 'email' | 'password'>;
```

## Mapped Types personalizados

```typescript
// Hacer todas las propiedades nullable
type Nullable<T> = {
  [K in keyof T]: T[K] | null;
};

// Convertir valores a funciones getter
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

interface Config {
  host: string;
  port: number;
}

type ConfigGetters = Getters<Config>;
// { getHost: () => string; getPort: () => number }

// Filtrar propiedades por tipo de valor
type OnlyStrings<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K];
};

interface Mixed {
  name: string;
  age: number;
  email: string;
  active: boolean;
}

type StringFields = OnlyStrings<Mixed>;
// { name: string; email: string }
```

## Mapped Type con modificadores de acceso

```typescript
// Mutable — inverso de Readonly
type Mutable<T> = {
  -readonly [K in keyof T]: T[K]; // -readonly elimina el readonly
};

const frozen: Readonly<User> = { ... };
// frozen.name = 'Ana'; // ❌ Error

const mutable: Mutable<Readonly<User>> = { ...frozen };
mutable.name = 'Ana'; // ✅
```

## Mapped Types en el mundo real

```typescript
// API response mapper
type ApiResponseWrapper<T> = {
  [K in keyof T]: {
    value: T[K];
    error: string | null;
    loading: boolean;
  };
};

// Form state management
type FormState<T> = {
  values: T;
  errors: { [K in keyof T]?: string };
  touched: { [K in keyof T]?: boolean };
  isDirty: boolean;
  isSubmitting: boolean;
};

// Validadores para cada campo
type Validators<T> = {
  [K in keyof T]?: (value: T[K]) => string | undefined;
};
```

**Consejo de entrevista**: Los Mapped Types son una de las características más poderosas de TypeScript para DRY (Don't Repeat Yourself) en las definiciones de tipos. Saber implementar `Partial`, `Pick` y `Omit` desde cero demuestra comprensión profunda del sistema de tipos. Los utility types de TypeScript están todos implementados con Mapped Types y otros tipos avanzados.
