---
title: ¿Qué es infer en TypeScript y cómo usarlo en Conditional Types?
difficulty: hard
tags: [typescript, infer, conditional-types, generics, advanced]
---

`infer` permite **extraer** y **capturar** un tipo dentro de un tipo condicional. En lugar de especificar el tipo explícitamente, TypeScript lo infiere de la estructura del tipo que se está comprobando.

## Sintaxis básica

```typescript
// Sin infer: comprobar si es un array
type IsArray<T> = T extends any[] ? true : false;

// Con infer: extraer el tipo del elemento del array
type ArrayElement<T> = T extends (infer E)[] ? E : never;

type StringElement = ArrayElement<string[]>;  // string
type NumberElement = ArrayElement<number[]>;  // number
type Mixed = ArrayElement<(string | number)[]>; // string | number
type NotArray = ArrayElement<string>; // never
```

## Extraer tipos de funciones

```typescript
// Tipo de retorno de una función
type ReturnType<T extends (...args: any) => any> =
  T extends (...args: any) => infer R ? R : never;

// Tipo de los parámetros (como tuple)
type Parameters<T extends (...args: any) => any> =
  T extends (...args: infer P) => any ? P : never;

// Primer parámetro
type FirstParam<T extends (...args: any) => any> =
  T extends (first: infer F, ...rest: any[]) => any ? F : never;

// Último parámetro
type LastParam<T extends (...args: any) => any> =
  T extends (...args: [...infer _, infer Last]) => any ? Last : never;

function fetchUser(id: string, options?: { cache: boolean }) {
  return { id, name: 'Ana' };
}

type FetchReturn = ReturnType<typeof fetchUser>;  // { id: string; name: string }
type FetchParams = Parameters<typeof fetchUser>;   // [string, { cache: boolean }?]
type FetchFirst = FirstParam<typeof fetchUser>;    // string
```

## Extraer tipos de promesas

```typescript
// Tipo resuelto de una Promise (versión simplificada de Awaited)
type UnwrapPromise<T> = T extends Promise<infer R> ? R : T;

// Versión recursiva para promesas anidadas
type DeepUnwrap<T> = T extends Promise<infer R> ? DeepUnwrap<R> : T;

type A = UnwrapPromise<Promise<string>>;          // string
type B = UnwrapPromise<Promise<User[]>>;           // User[]
type C = DeepUnwrap<Promise<Promise<number>>>;    // number
type D = UnwrapPromise<string>;                   // string (no era Promise)
```

## Extraer tipos de clases y constructores

```typescript
// Tipo de instancia de un constructor
type InstanceType<T extends new (...args: any) => any> =
  T extends new (...args: any) => infer R ? R : never;

// Parámetros del constructor
type ConstructorParameters<T extends new (...args: any) => any> =
  T extends new (...args: infer P) => any ? P : never;

class ApiClient {
  constructor(
    private baseUrl: string,
    private apiKey: string,
    private timeout = 5000
  ) {}
}

type ClientConstructorParams = ConstructorParameters<typeof ApiClient>;
// [string, string, number?]
```

## infer en tipos distribuidos

```typescript
// Extraer el tipo de un observable (RxJS style)
type ObservableValue<T> =
  T extends { subscribe: (observer: (value: infer V) => void) => any }
    ? V
    : never;

// Extraer el tipo de un generador
type GeneratorYield<T> =
  T extends Generator<infer Y, any, any> ? Y : never;

function* numberGenerator(): Generator<number, string, void> {
  yield 1;
  yield 2;
  return 'done';
}

type Yielded = GeneratorYield<ReturnType<typeof numberGenerator>>; // number
```

## Casos prácticos reales

```typescript
// 1. Extraer tipos de Event Handlers
type EventHandler<T extends Event = Event> = (event: T) => void;
type HandlerEventType<T> = T extends EventHandler<infer E> ? E : never;

type MyHandler = EventHandler<MouseEvent>;
type MyEvent = HandlerEventType<MyHandler>; // MouseEvent

// 2. Desempaquetar un tipo Result
type Ok<T> = { success: true; data: T };
type Err<E> = { success: false; error: E };
type Result<T, E> = Ok<T> | Err<E>;

type ExtractSuccess<T> =
  T extends Result<infer D, any> ? D : never;
type ExtractError<T> =
  T extends Result<any, infer E> ? E : never;

type ApiResult = Result<User, string>;
type SuccessData = ExtractSuccess<ApiResult>; // User
type ErrorData = ExtractError<ApiResult>;     // string

// 3. Inferir el tipo de una propiedad concreta
type PropType<T, K extends keyof T> =
  T extends { [P in K]: infer V } ? V : never;

interface Config { host: string; port: number; debug: boolean; }
type PortType = PropType<Config, 'port'>; // number
```

## infer con varianza (TypeScript 4.7+)

```typescript
// infer in/out para controlar varianza en tipos complejos
type Getter<T> = () => T;
type GetterValue<T> = T extends Getter<infer out R> ? R : never;
// 'out' indica que R es covariante (se usa en posición de salida)
```

**Consejo de entrevista**: `infer` es la herramienta que diferencia un usuario intermedio de uno avanzado en TypeScript. La clave es entender que `infer` solo funciona en la cláusula `extends` de un tipo condicional, y que el nombre dado con `infer` (como `infer R`) captura el tipo que TypeScript necesita "adivinar" para que la expresión sea verdadera. Recrear `ReturnType` y `Parameters` en una entrevista demuestra comprensión profunda del sistema de tipos.
