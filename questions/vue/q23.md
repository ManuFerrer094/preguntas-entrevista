---
title: ¿Cuál es la diferencia entre Options API y Composition API en Vue?
difficulty: easy

---

Vue ofrece dos estilos de escritura de componentes: la **Options API** (Vue 2 y Vue 3) y la **Composition API** (introducida en Vue 3). La Composition API es la opción recomendada en Vue 3.

## Options API — Organización por tipo de opción

```vue
<script>
export default {
  name: 'ContadorComponent',
  // Estado
  data() {
    return {
      contador: 0,
      nombre: 'Usuario'
    };
  },
  // Propiedades calculadas
  computed: {
    mensajeBienvenida() {
      return `Hola, ${this.nombre}! Contador: ${this.contador}`;
    }
  },
  // Métodos
  methods: {
    incrementar() {
      this.contador++;
    },
    resetear() {
      this.contador = 0;
    }
  },
  // Watchers
  watch: {
    contador(nuevoValor, valorAnterior) {
      console.log(`Contador cambió de ${valorAnterior} a ${nuevoValor}`);
    }
  },
  // Lifecycle
  mounted() {
    console.log('Componente montado');
  }
}
</script>

<template>
  <div>
    <p>{{ mensajeBienvenida }}</p>
    <button @click="incrementar">+1</button>
    <button @click="resetear">Reset</button>
  </div>
</template>
```

## Composition API con `<script setup>` — Organización por lógica

```vue
<script setup>
import { ref, computed, watch, onMounted } from 'vue';

// Estado
const contador = ref(0);
const nombre = ref('Usuario');

// Computed
const mensajeBienvenida = computed(() =>
  `Hola, ${nombre.value}! Contador: ${contador.value}`
);

// Métodos (funciones normales)
function incrementar() {
  contador.value++;
}

function resetear() {
  contador.value = 0;
}

// Watcher
watch(contador, (nuevoValor, valorAnterior) => {
  console.log(`Contador cambió de ${valorAnterior} a ${nuevoValor}`);
});

// Lifecycle
onMounted(() => {
  console.log('Componente montado');
});
</script>

<template>
  <div>
    <p>{{ mensajeBienvenida }}</p>
    <button @click="incrementar">+1</button>
    <button @click="resetear">Reset</button>
  </div>
</template>
```

## Ventaja de la Composition API: Reutilización de lógica

```vue
<!-- Con Options API, la lógica se dispersa entre data, methods, computed, watch -->
<!-- Con Composition API, la lógica relacionada se agrupa en Composables: -->

<!-- useContador.js (Composable) -->
<script>
import { ref, computed } from 'vue';

export function useContador(valorInicial = 0) {
  const contador = ref(valorInicial);
  const esPar = computed(() => contador.value % 2 === 0);

  function incrementar() { contador.value++; }
  function decrementar() { contador.value--; }
  function resetear() { contador.value = valorInicial; }

  return { contador, esPar, incrementar, decrementar, resetear };
}
</script>

<!-- Uso en componente -->
<script setup>
import { useContador } from './useContador';

const { contador, esPar, incrementar, decrementar } = useContador(10);
</script>
```

## Comparativa de características

| Característica           | Options API              | Composition API              |
|--------------------------|--------------------------|------------------------------|
| Organización             | Por tipo (data/methods)  | Por funcionalidad             |
| Reutilización de lógica  | Mixins (conflictos)      | Composables ✅                |
| TypeScript               | Soporte parcial          | ✅ Soporte completo           |
| Legibilidad (simple)     | ✅ Muy intuitiva         | Requiere más contexto        |
| Legibilidad (complejo)   | Código disperso          | ✅ Lógica agrupada            |
| Bundle size              | Similar                  | Mejor tree-shaking           |
| Acceso a `this`          | Necesario                | ❌ No necesario              |

## Cuándo usar cada una

```
Options API → Ideal para:
✅ Proyectos Vue 2 migrados a Vue 3
✅ Desarrolladores nuevos en Vue
✅ Componentes simples con poca lógica
✅ Equipos que prefieren la familiaridad de la API

Composition API → Ideal para:
✅ Proyectos nuevos en Vue 3
✅ Lógica compleja que se puede extraer en Composables
✅ Proyectos con TypeScript
✅ Equipos con experiencia en React Hooks
```

**Consejo de entrevista**: Ambas APIs son válidas en Vue 3 y pueden coexistir en el mismo proyecto. La Composition API brilla cuando el componente crece y la lógica relacionada está dispersa entre `data`, `methods`, `computed` y `watch`. Los Composables son la solución definitiva a los problemas de los Mixins (colisiones de nombres, fuente de verdad oculta). En proyectos nuevos, la recomendación oficial de Vue 3 es usar Composition API con `<script setup>`.
