---
title: ¿Qué son los slots en Vue?
difficulty: medium
tags: [vue, slots, component-composition, content-projection]
---

Los slots son un mecanismo de Vue para la proyección de contenido: permiten que un componente padre inserte HTML o componentes dentro de las "ranuras" definidas por el componente hijo. Son la forma en que Vue implementa la composición de componentes, similar al `<ng-content>` de Angular o a `children` en React.

## Slot básico (default slot)

```vue
<!-- BaseCard.vue -->
<template>
  <div class="card">
    <div class="card-body">
      <slot /> <!-- El contenido se proyecta aquí -->
    </div>
  </div>
</template>

<!-- Uso -->
<BaseCard>
  <h2>Título del card</h2>
  <p>Descripción del contenido.</p>
</BaseCard>
```

## Contenido por defecto

```vue
<!-- Button.vue -->
<template>
  <button class="btn" :class="variant">
    <slot>Click aquí</slot> <!-- "Click aquí" se muestra si no se proyecta contenido -->
  </button>
</template>
```

## Named slots (slots con nombre)

```vue
<!-- PageLayout.vue -->
<template>
  <div class="layout">
    <header>
      <slot name="header" />
    </header>

    <main>
      <slot /> <!-- slot por defecto -->
    </main>

    <footer>
      <slot name="footer">
        <p>© 2024 Mi Empresa</p> <!-- Contenido por defecto -->
      </slot>
    </footer>
  </div>
</template>

<!-- Uso con v-slot (o su atajo #) -->
<PageLayout>
  <template #header>
    <h1>Mi Aplicación</h1>
    <nav>...</nav>
  </template>

  <!-- Contenido para el slot por defecto -->
  <div class="content">Contenido principal</div>

  <template #footer>
    <p>Pie de página personalizado</p>
  </template>
</PageLayout>
```

## Scoped slots — el slot expone datos al padre

Los scoped slots permiten que el componente hijo pase datos al contenido que proyecta el padre. Son poderosos para componentes de lista reutilizables:

```vue
<!-- DataTable.vue -->
<template>
  <table>
    <tbody>
      <tr v-for="item in items" :key="item.id">
        <!-- El slot expone 'item' al componente padre -->
        <slot :item="item" :index="items.indexOf(item)" />
      </tr>
    </tbody>
  </table>
</template>

<script setup>
defineProps({ items: Array });
</script>

<!-- Uso: el padre decide cómo renderizar cada fila -->
<DataTable :items="users">
  <template #default="{ item, index }">
    <td>{{ index + 1 }}</td>
    <td>{{ item.name }}</td>
    <td>{{ item.email }}</td>
    <td>
      <button @click="deleteUser(item.id)">Eliminar</button>
    </td>
  </template>
</DataTable>
```

**Componente de lista genérico con scoped slots:**

```vue
<!-- VirtualList.vue: componente reutilizable que no sabe qué renderizará -->
<template>
  <div class="virtual-list">
    <div
      v-for="item in visibleItems"
      :key="item.id"
      class="list-item"
    >
      <slot name="item" :item="item" :isSelected="selectedId === item.id" />
    </div>

    <slot name="empty" v-if="items.length === 0">
      <p class="empty">No hay elementos</p>
    </slot>
  </div>
</template>

<!-- Uso flexible -->
<VirtualList :items="products">
  <template #item="{ item, isSelected }">
    <ProductCard :product="item" :highlighted="isSelected" />
  </template>

  <template #empty>
    <EmptyState message="No hay productos disponibles" />
  </template>
</VirtualList>
```

**Consejo de entrevista**: Los scoped slots son una de las características más poderosas y menos comprendidas de Vue. Permiten crear componentes "renderless" (sin markup propio) que solo proveen lógica y exponen datos al padre. Un ejemplo clásico es un componente que gestiona el fetching de datos y expone el resultado mediante scoped slot, permitiendo al padre decidir completamente el renderizado.
