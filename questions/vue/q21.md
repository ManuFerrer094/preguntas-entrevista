---
title: ¿Qué es defineProps y defineEmits en Vue 3 con script setup?
difficulty: easy
tags: [vue, defineProps, defineEmits, script-setup, composition-api, typescript]
---

`defineProps` y `defineEmits` son macros del compilador de Vue 3 disponibles exclusivamente en `<script setup>`. Reemplazan a `props` y `emits` de la Options API, ofreciendo tipado completo con TypeScript y una sintaxis más concisa.

## defineProps básico

```vue
<!-- ChildComponent.vue -->
<script setup>
// defineProps sin TypeScript — validación en runtime
const props = defineProps({
  title: {
    type: String,
    required: true
  },
  count: {
    type: Number,
    default: 0
  },
  items: {
    type: Array,
    default: () => []
  },
  variant: {
    type: String,
    validator: (value) => ['primary', 'secondary', 'danger'].includes(value),
    default: 'primary'
  }
});

// Las props son accesibles directamente (con desestructuración pierdes reactividad)
console.log(props.title);
console.log(props.count);
</script>

<template>
  <!-- En el template, acceso directo sin "props." -->
  <h1>{{ title }}</h1>
  <span>{{ count }}</span>
</template>
```

## defineProps con TypeScript (recomendado)

```vue
<script setup lang="ts">
interface Props {
  title: string;
  count?: number;   // ? significa opcional
  items?: string[];
  variant?: 'primary' | 'secondary' | 'danger';
  onClick?: (item: string) => void;
}

// Sintaxis de tipo genérico — más expresiva con TypeScript
const props = defineProps<Props>();

// withDefaults — proporcionar valores por defecto con tipado
const propsWithDefaults = withDefaults(defineProps<Props>(), {
  count: 0,
  items: () => [],  // Arrays/objetos: usar función para evitar compartir referencia
  variant: 'primary'
});
</script>
```

## defineEmits básico

```vue
<script setup>
// defineEmits sin TypeScript
const emit = defineEmits(['update:modelValue', 'submit', 'cancel']);

function handleSubmit(data) {
  emit('submit', data);
}

function handleCancel() {
  emit('cancel');
}
</script>
```

## defineEmits con TypeScript (recomendado)

```vue
<script setup lang="ts">
// Tipado estricto de eventos
const emit = defineEmits<{
  'update:modelValue': [value: string];  // [tipos de argumentos...]
  'submit': [data: FormData];
  'cancel': [];  // Sin argumentos
  'change': [oldValue: string, newValue: string];  // Múltiples argumentos
}>();

// TypeScript verifica que los argumentos sean correctos
emit('submit', formData);         // ✅
emit('update:modelValue', 'hola'); // ✅
emit('submit');                    // ❌ Error: falta el argumento FormData
emit('unknown-event');             // ❌ Error: evento no declarado
</script>
```

## v-model con defineProps y defineEmits

```vue
<!-- CustomInput.vue — implementar v-model personalizado -->
<script setup lang="ts">
const props = defineProps<{
  modelValue: string;  // Nombre estándar para v-model
  placeholder?: string;
}>();

const emit = defineEmits<{
  'update:modelValue': [value: string];
}>();
</script>

<template>
  <input
    :value="modelValue"
    :placeholder="placeholder"
    @input="emit('update:modelValue', ($event.target as HTMLInputElement).value)"
    class="custom-input"
  />
</template>
```

```vue
<!-- ParentComponent.vue — usar v-model con CustomInput -->
<template>
  <CustomInput v-model="searchQuery" placeholder="Buscar..." />
  <!-- Equivalente a:
  <CustomInput
    :modelValue="searchQuery"
    @update:modelValue="searchQuery = $event"
  /> -->
</template>
```

## Múltiples v-model

```vue
<!-- FormField.vue — múltiples v-model en un componente -->
<script setup lang="ts">
defineProps<{
  modelValue: string;        // v-model (por defecto)
  label?: string;            // v-model:label
  error?: string;            // v-model:error
}>();

defineEmits<{
  'update:modelValue': [value: string];
  'update:error': [error: string];
}>();
</script>

<template>
  <label v-if="label">{{ label }}</label>
  <input
    :value="modelValue"
    @input="$emit('update:modelValue', $event.target.value)"
    @blur="validateField(modelValue)"
  />
  <span v-if="error" class="error">{{ error }}</span>
</template>
```

```vue
<!-- Uso de múltiples v-model -->
<FormField
  v-model="form.email"
  v-model:label="'Correo electrónico'"
  v-model:error="formErrors.email"
/>
```

## defineModel (Vue 3.4+, simplifica v-model)

```vue
<!-- Vue 3.4+: defineModel simplifica la implementación de v-model -->
<script setup lang="ts">
// Equivalente a defineProps({ modelValue }) + defineEmits(['update:modelValue'])
const model = defineModel<string>({ required: true });

// Con nombre personalizado
const title = defineModel<string>('title');
const count = defineModel<number>('count', { default: 0 });
</script>

<template>
  <input v-model="model" />
  <!-- model.value se actualiza y emite automáticamente el evento -->
</template>
```

**Consejo de entrevista**: `defineProps` y `defineEmits` son macros del compilador — no son funciones importadas y solo funcionan en `<script setup>`. Con TypeScript, la sintaxis de tipo genérico es preferible a la validación en runtime porque el compilador puede verificarla estáticamente. Para `v-model`, el patrón `modelValue` + `update:modelValue` es el estándar. En Vue 3.4+, `defineModel()` simplifica esta implementación.
