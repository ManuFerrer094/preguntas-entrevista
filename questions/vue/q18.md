---
title: ¿Qué es keep-alive en Vue?
difficulty: medium
tags: [vue, keep-alive, caching, performance, lifecycle]
---

`<KeepAlive>` es un componente integrado de Vue que mantiene en memoria instancias de componentes cuando se desmontan, evitando recrearlos al volver a montarlos. Preserva el estado y mejora el rendimiento en tabs, wizards y listas con filtros.

## Problema sin keep-alive

```vue
<script setup>
import { ref } from 'vue';

const activeTab = ref('search');
</script>

<template>
  <nav>
    <button @click="activeTab = 'search'">Búsqueda</button>
    <button @click="activeTab = 'favorites'">Favoritos</button>
  </nav>

  <!-- ❌ Sin keep-alive: el componente se destruye y recrea al cambiar de tab -->
  <!-- Se pierden: el texto del buscador, la posición del scroll, los resultados -->
  <SearchTab v-if="activeTab === 'search'" />
  <FavoritesTab v-if="activeTab === 'favorites'" />
</template>
```

## Uso básico de KeepAlive

```vue
<template>
  <nav>
    <button @click="activeTab = 'search'">Búsqueda</button>
    <button @click="activeTab = 'favorites'">Favoritos</button>
  </nav>

  <!-- ✅ Con keep-alive: el componente se "duerme" en lugar de destruirse -->
  <KeepAlive>
    <SearchTab v-if="activeTab === 'search'" />
    <FavoritesTab v-if="activeTab === 'favorites'" />
  </KeepAlive>
</template>
```

## Incluir y excluir componentes específicos

```vue
<template>
  <KeepAlive
    include="SearchTab,DataTable"
    :max="5"
  >
    <!-- Solo SearchTab y DataTable se cachean (por nombre del componente) -->
    <component :is="currentComponent" />
  </KeepAlive>

  <!-- Excluir componentes específicos del caché -->
  <KeepAlive :exclude="['VideoPlayer', 'LiveChat']">
    <component :is="currentComponent" />
  </KeepAlive>

  <!-- Limitar el número de instancias cacheadas (LRU cache) -->
  <KeepAlive :max="10">
    <component :is="currentComponent" />
  </KeepAlive>
</template>
```

## Hooks onActivated y onDeactivated

```vue
<!-- SearchTab.vue — con hooks específicos de KeepAlive -->
<script setup>
import { ref, onActivated, onDeactivated, onMounted, onUnmounted } from 'vue';

const query = ref('');
const results = ref([]);
let scrollPosition = 0;

// Ciclo de vida normal: solo se ejecuta al crear/destruir por primera vez
onMounted(() => {
  console.log('SearchTab montado por primera vez');
  loadInitialData();
});

onUnmounted(() => {
  console.log('SearchTab desmontado completamente (si no hay KeepAlive)');
});

// Hooks específicos de KeepAlive
onActivated(() => {
  // Se ejecuta cada vez que el componente "se activa" (se vuelve visible)
  console.log('SearchTab activado');
  window.scrollTo(0, scrollPosition); // Restaurar posición de scroll
  refreshDataIfStale();               // Actualizar datos si han pasado más de 5 min
});

onDeactivated(() => {
  // Se ejecuta cuando el componente "se desactiva" (queda en caché pero invisible)
  console.log('SearchTab desactivado');
  scrollPosition = window.scrollY;    // Guardar posición de scroll
});

function refreshDataIfStale() {
  const FIVE_MINUTES = 5 * 60 * 1000;
  if (Date.now() - lastFetchTime.value > FIVE_MINUTES) {
    fetchResults(query.value);
  }
}
</script>
```

## KeepAlive con Router View

```vue
<!-- App.vue o el componente de layout -->
<script setup>
import { ref } from 'vue';
</script>

<template>
  <RouterView v-slot="{ Component, route }">
    <!-- Cachear solo ciertas rutas usando meta -->
    <KeepAlive :include="cachedRoutes">
      <component
        :is="Component"
        :key="route.fullPath"
      />
    </KeepAlive>
  </RouterView>
</template>
```

```javascript
// router/index.js — marcar qué rutas se deben cachear
const routes = [
  {
    path: '/search',
    component: () => import('../views/SearchView.vue'),
    meta: { keepAlive: true }  // Esta ruta se cachea
  },
  {
    path: '/profile',
    component: () => import('../views/ProfileView.vue'),
    meta: { keepAlive: false } // Esta no
  }
];
```

## Ciclo de vida completo con KeepAlive

```
Primera visita:
  beforeCreate → created → beforeMount → mounted → onActivated

Al navegar a otra tab/ruta:
  onDeactivated (NO onUnmounted — el componente vive en caché)

Al volver:
  onActivated (NO onMounted — el componente ya estaba creado)

Al destruir el caché (max excedido o exclusión):
  onDeactivated → onUnmounted
```

**Consejo de entrevista**: `KeepAlive` es ideal para tabs, wizards multi-paso, y cualquier UI donde el usuario espera que su estado se preserve al navegar. El hook `onActivated` es el equivalente a `mounted` para componentes cacheados: úsalo para refrescar datos potencialmente obsoletos. El atributo `max` implementa una caché LRU (Least Recently Used): cuando se supera el límite, el componente menos recientemente usado se desmonta.
