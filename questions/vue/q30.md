---
title: ¿Qué es shallowRef vs ref en Vue 3?
difficulty: medium
tags: [vue, shallowRef, ref, reactivity, performance, vue3]
---

`shallowRef` crea una referencia reactiva **superficial**: solo el `.value` en sí mismo es reactivo, pero las propiedades internas del objeto **no son reactivas**. Esto contrasta con `ref`, que aplica reactividad profunda a objetos y arrays.

## Diferencia fundamental

```javascript
import { ref, shallowRef, watch } from 'vue';

// ref — Reactividad PROFUNDA (deep reactive)
const usuarioRef = ref({
  nombre: 'Ana',
  direccion: { ciudad: 'Madrid', pais: 'España' }
});

// Estas mutaciones SÍ disparan reactividad:
usuarioRef.value.nombre = 'Carlos';                    // ✅ Reactivo
usuarioRef.value.direccion.ciudad = 'Barcelona';       // ✅ Reactivo (deep)

// shallowRef — Reactividad SUPERFICIAL (shallow reactive)
const usuarioShallow = shallowRef({
  nombre: 'Ana',
  direccion: { ciudad: 'Madrid', pais: 'España' }
});

// Estas mutaciones NO disparan reactividad:
usuarioShallow.value.nombre = 'Carlos';                // ❌ No reactivo (mutación interna)
usuarioShallow.value.direccion.ciudad = 'Barcelona';   // ❌ No reactivo

// Solo cambiar el .value completo es reactivo:
usuarioShallow.value = { ...usuarioShallow.value, nombre: 'Carlos' };  // ✅ Reactivo
```

## Cuándo usar shallowRef

```javascript
// ✅ Para objetos grandes que vienen de la API y no necesitan reactividad profunda
const tablaDatos = shallowRef([]);

async function cargarDatos() {
  const respuesta = await fetch('/api/tabla');
  const datos = await respuesta.json();
  // Reemplazamos el array completo (reactivo)
  tablaDatos.value = datos;  // ✅ Dispara actualización de la vista
}

// ❌ Si mutas elementos individuales, la vista NO se actualiza:
tablaDatos.value[0].nombre = 'Nuevo nombre';  // No reactivo

// ✅ Para actualizar un elemento, crea nuevo array:
tablaDatos.value = tablaDatos.value.map(item =>
  item.id === idActualizar ? { ...item, nombre: 'Nuevo nombre' } : item
);
```

## Performance: shallowRef vs ref para objetos grandes

```javascript
import { ref, shallowRef, reactive, shallowReactive } from 'vue';

// ref convierte recursivamente todo el objeto con Proxy
// Para objetos grandes (1000+ propiedades), esto tiene overhead:
const estadoRef = ref({
  usuarios: Array(1000).fill(null).map((_, i) => ({
    id: i, nombre: `Usuario ${i}`, datos: { /* muchos campos */ }
  }))
});

// shallowRef: solo el wrapper es reactivo, el objeto interno es plano
// Mucho menos overhead para objetos grandes:
const estadoShallow = shallowRef({
  usuarios: Array(1000).fill(null).map((_, i) => ({
    id: i, nombre: `Usuario ${i}`, datos: { /* muchos campos */ }
  }))
});
// Solo se re-renderiza cuando se reemplaza .value completo
```

## shallowReactive — Lo mismo para reactive

```javascript
import { reactive, shallowReactive } from 'vue';

// reactive — Profundo (todo es reactivo)
const estado = reactive({
  config: { tema: 'oscuro', idioma: 'es' },
  datos: { lista: [], total: 0 }
});
estado.config.tema = 'claro';       // ✅ Reactivo
estado.datos.lista.push(item);      // ✅ Reactivo

// shallowReactive — Superficial (solo propiedades del primer nivel)
const estadoShallow = shallowReactive({
  config: { tema: 'oscuro', idioma: 'es' },
  datos: { lista: [], total: 0 }
});
estadoShallow.config = { tema: 'claro', idioma: 'es' };  // ✅ Reactivo (reemplaza el objeto)
estadoShallow.config.tema = 'claro';                      // ❌ No reactivo (mutación interna)
```

## triggerRef — Forzar actualización de shallowRef

```javascript
import { shallowRef, triggerRef } from 'vue';

const lista = shallowRef([
  { id: 1, nombre: 'Item 1' }
]);

// Si necesitas mutar internamente y forzar la actualización:
lista.value.push({ id: 2, nombre: 'Item 2' });  // Mutación no reactiva
triggerRef(lista);  // ✅ Fuerza la actualización de la vista manualmente

// Úsalo solo si realmente necesitas mutar el objeto interno
// Normalmente es mejor reemplazar el valor completo
```

## Tabla comparativa

| API               | Profundidad | Uso óptimo                          | Overhead |
|-------------------|-------------|-------------------------------------|----------|
| `ref()`           | Profunda    | Objetos pequeños/medianos           | Alto     |
| `shallowRef()`    | Superficial | Objetos grandes, datos de API       | Bajo     |
| `reactive()`      | Profunda    | Estado de componente complejo       | Alto     |
| `shallowReactive()` | Superficial | Config, estado de solo reemplazo  | Bajo     |
| `readonly()`      | Profunda    | Props inmutables                    | Medio    |

## Regla práctica

```javascript
// Usa shallowRef cuando:
// 1. El objeto viene de una API y no necesita reactividad de sus propiedades internas
// 2. Siempre reemplazas el objeto completo (no mutes sus propiedades)
// 3. El objeto es grande y quieres evitar el overhead de proxies recursivos

// Usa ref cuando:
// 1. Necesitas reactividad en propiedades anidadas
// 2. El template accede a propiedades profundas del objeto
// 3. Usas v-model sobre propiedades del objeto
```

**Consejo de entrevista**: `shallowRef` es una optimización de rendimiento que se usa cuando sabes que siempre vas a reemplazar el valor completo (patrón inmutable). Es el análogo al patrón de Redux: en lugar de mutar el estado, creas uno nuevo. Combinar `shallowRef` con la inmutabilidad (spread operator, `map`, `filter`) da el mejor rendimiento al evitar el overhead de los Proxies recursivos de Vue.
