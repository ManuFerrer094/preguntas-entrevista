**Vue I18n** es la librería oficial para internacionalización en Vue. Permite gestionar múltiples idiomas, formateo de fechas/números, pluralización y mensajes con interpolación de variables.

## Instalación y configuración

```bash
npm install vue-i18n@9
```

```javascript
// src/i18n/index.js
import { createI18n } from 'vue-i18n';

const mensajes = {
  es: {
    nav: {
      inicio: 'Inicio',
      sobre: 'Sobre nosotros',
      contacto: 'Contacto',
    },
    bienvenida: {
      titulo: 'Hola, {nombre}!',
      subtitulo: 'Bienvenido a nuestra aplicación',
    },
    carrito: {
      items: '{n} artículo | {n} artículos',  // Pluralización
      vacio: 'Tu carrito está vacío',
      total: 'Total: {precio}',
    },
    fechas: {
      registro: 'Registrado el {fecha}',
    }
  },
  en: {
    nav: {
      inicio: 'Home',
      sobre: 'About',
      contacto: 'Contact',
    },
    bienvenida: {
      titulo: 'Hello, {nombre}!',
      subtitulo: 'Welcome to our application',
    },
    carrito: {
      items: '{n} item | {n} items',
      vacio: 'Your cart is empty',
      total: 'Total: {precio}',
    },
    fechas: {
      registro: 'Registered on {fecha}',
    }
  }
};

export const i18n = createI18n({
  legacy: false,           // Usar Composition API
  locale: 'es',           // Idioma por defecto
  fallbackLocale: 'en',   // Fallback si no hay traducción
  messages: mensajes,
});
```

```javascript
// main.js
import { createApp } from 'vue';
import App from './App.vue';
import { i18n } from './i18n';

createApp(App).use(i18n).mount('#app');
```

## Uso en componentes con Composition API

```vue
<script setup>
import { useI18n } from 'vue-i18n';

const { t, locale, d, n } = useI18n();

const nombreUsuario = 'Ana';
const numArticulos = 3;
const precio = 99.99;
const fechaRegistro = new Date('2024-01-15');
</script>

<template>
  <!-- Traducción simple -->
  <h1>{{ t('nav.inicio') }}</h1>

  <!-- Traducción con variable -->
  <p>{{ t('bienvenida.titulo', { nombre: nombreUsuario }) }}</p>

  <!-- Pluralización -->
  <span>{{ t('carrito.items', numArticulos, { n: numArticulos }) }}</span>
  <!-- n=1: "1 artículo" | n=3: "3 artículos" -->

  <!-- Formato de número como moneda -->
  <p>{{ t('carrito.total', { precio: n(precio, 'currency') }) }}</p>

  <!-- Formato de fecha -->
  <p>{{ t('fechas.registro', { fecha: d(fechaRegistro, 'short') }) }}</p>
</template>
```

## Cambio de idioma dinámico

```vue
<script setup>
import { useI18n } from 'vue-i18n';

const { locale } = useI18n();

const idiomasDisponibles = [
  { codigo: 'es', nombre: 'Español', bandera: '🇪🇸' },
  { codigo: 'en', nombre: 'English', bandera: '🇬🇧' },
  { codigo: 'fr', nombre: 'Français', bandera: '🇫🇷' },
];

function cambiarIdioma(nuevoCodigo) {
  locale.value = nuevoCodigo;
  localStorage.setItem('idioma-preferido', nuevoCodigo);
  document.documentElement.setAttribute('lang', nuevoCodigo);
}
</script>

<template>
  <div class="selector-idioma">
    <button
      v-for="idioma in idiomasDisponibles"
      :key="idioma.codigo"
      :class="{ activo: locale === idioma.codigo }"
      @click="cambiarIdioma(idioma.codigo)"
    >
      {{ idioma.bandera }} {{ idioma.nombre }}
    </button>
  </div>
</template>
```

## Carga lazy de traducciones (para apps grandes)

```javascript
// i18n/index.js — Con lazy loading
export const i18n = createI18n({
  legacy: false,
  locale: 'es',
  fallbackLocale: 'en',
  messages: { es: {} }  // Solo idioma por defecto inicialmente
});

// Función para cargar un idioma bajo demanda
const idiomasCargados = new Set(['es']);

export async function cargarIdioma(locale) {
  if (idiomasCargados.has(locale)) return;

  // Cargar el JSON del idioma dinámicamente
  const mensajes = await import(`./locales/${locale}.json`);
  i18n.global.setLocaleMessage(locale, mensajes.default);
  idiomasCargados.add(locale);
}

// Uso:
async function cambiarIdioma(codigo) {
  await cargarIdioma(codigo);
  i18n.global.locale.value = codigo;
}
```

## Formateo de fechas y números

```javascript
// Configuración de formatos
const i18n = createI18n({
  legacy: false,
  locale: 'es',
  datetimeFormats: {
    es: {
      short: { year: 'numeric', month: 'short', day: 'numeric' },
      long: { year: 'numeric', month: 'long', day: 'numeric', weekday: 'long' },
    },
    en: {
      short: { year: 'numeric', month: 'short', day: 'numeric' },
    }
  },
  numberFormats: {
    es: {
      currency: { style: 'currency', currency: 'EUR', minimumFractionDigits: 2 },
      decimal: { style: 'decimal', minimumFractionDigits: 2 },
      percent: { style: 'percent', minimumFractionDigits: 1 },
    },
    en: {
      currency: { style: 'currency', currency: 'USD' },
    }
  }
});
```

```vue
<script setup>
const { d, n } = useI18n();
</script>

<template>
  <!-- Fechas formateadas según el locale -->
  <p>{{ d(new Date(), 'short') }}</p>   <!-- es: "15 ene 2024" | en: "Jan 15, 2024" -->
  <p>{{ d(new Date(), 'long') }}</p>    <!-- "lunes, 15 de enero de 2024" -->

  <!-- Números y moneda -->
  <p>{{ n(1234.56, 'currency') }}</p>   <!-- es: "1.234,56 €" | en: "$1,234.56" -->
  <p>{{ n(0.75, 'percent') }}</p>       <!-- es: "75,0 %" -->
</template>
```

**Consejo de entrevista**: Al hablar de i18n en Vue, menciona: 1) la diferencia entre `legacy: false` (Composition API) y `legacy: true` (Options API), 2) la carga lazy de idiomas para reducir el bundle inicial, 3) el `fallbackLocale` para manejar traducciones incompletas, 4) la integración con `document.lang` para accesibilidad. Vue I18n también soporta ICU Message Format para pluralización más avanzada, útil para casos como "1 persona, 2 personas".
