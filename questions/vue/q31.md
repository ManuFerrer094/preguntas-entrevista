**Vue Test Utils** es la librería oficial para testear componentes Vue, y **Vitest** es el runner de tests recomendado para proyectos Vue 3 (integrado con Vite). Juntos permiten montar componentes en un entorno DOM simulado y verificar su comportamiento.

## Configuración

```bash
npm install --save-dev vitest @vue/test-utils jsdom @vitejs/plugin-vue
```

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';

export default defineConfig({
  plugins: [vue()],
  test: {
    globals: true,           // describe, it, expect sin imports
    environment: 'jsdom',    // DOM simulado
    setupFiles: ['./tests/setup.ts']  // Configuración global
  }
});
```

## Test básico de un componente

```vue
<!-- Contador.vue -->
<script setup>
import { ref } from 'vue';
const props = defineProps({ inicial: { type: Number, default: 0 } });
const contador = ref(props.inicial);
</script>
<template>
  <div>
    <p data-testid="valor">Contador: {{ contador }}</p>
    <button @click="contador++">Incrementar</button>
    <button @click="contador = props.inicial" :disabled="contador === props.inicial">
      Resetear
    </button>
  </div>
</template>
```

```javascript
// Contador.test.js
import { describe, it, expect } from 'vitest';
import { mount } from '@vue/test-utils';
import Contador from './Contador.vue';

describe('Contador', () => {
  it('renderiza el valor inicial', () => {
    const wrapper = mount(Contador, {
      props: { inicial: 5 }
    });

    expect(wrapper.find('[data-testid="valor"]').text()).toBe('Contador: 5');
  });

  it('incrementa el contador al hacer clic', async () => {
    const wrapper = mount(Contador);

    await wrapper.find('button').trigger('click');

    expect(wrapper.find('[data-testid="valor"]').text()).toBe('Contador: 1');
  });

  it('el botón resetear está deshabilitado al valor inicial', () => {
    const wrapper = mount(Contador, { props: { inicial: 0 } });

    const btnReset = wrapper.findAll('button')[1];
    expect(btnReset.attributes('disabled')).toBeDefined();
  });
});
```

## Testear emits y props

```javascript
// ListaItem.test.js
import { mount } from '@vue/test-utils';
import ListaItem from './ListaItem.vue';

describe('ListaItem', () => {
  it('emite el evento "eliminar" con el id correcto', async () => {
    const wrapper = mount(ListaItem, {
      props: { item: { id: 42, nombre: 'Test Item' } }
    });

    await wrapper.find('.btn-eliminar').trigger('click');

    // Verificar el evento emitido
    expect(wrapper.emitted('eliminar')).toBeTruthy();
    expect(wrapper.emitted('eliminar')[0]).toEqual([42]);
  });

  it('muestra el nombre del item', () => {
    const wrapper = mount(ListaItem, {
      props: { item: { id: 1, nombre: 'Mi Producto' } }
    });

    expect(wrapper.text()).toContain('Mi Producto');
  });
});
```

## Testear componentes con slots

```javascript
import { mount } from '@vue/test-utils';
import Tarjeta from './Tarjeta.vue';

it('renderiza contenido del slot por defecto', () => {
  const wrapper = mount(Tarjeta, {
    slots: {
      default: '<p>Contenido del slot</p>',
      header: '<h2>Título</h2>',
      footer: '<small>Pie</small>'
    }
  });

  expect(wrapper.find('p').text()).toBe('Contenido del slot');
  expect(wrapper.find('h2').text()).toBe('Título');
});
```

## Testear con stubs y mocks

```javascript
// FormularioPago.test.js
import { mount } from '@vue/test-utils';
import FormularioPago from './FormularioPago.vue';

describe('FormularioPago', () => {
  it('llama al servicio de pago al enviar', async () => {
    const mockProcesarPago = vi.fn().mockResolvedValue({ ok: true, transaccionId: 'abc123' });

    const wrapper = mount(FormularioPago, {
      global: {
        // Mock de componentes hijos
        stubs: { 'TarjetaCredito': true, 'ModalConfirmacion': true },
        // Mock de provides (para inject)
        provide: { pagoService: { procesar: mockProcesarPago } }
      }
    });

    await wrapper.find('input[name="monto"]').setValue('50');
    await wrapper.find('form').trigger('submit');
    await nextTick();  // Esperar la actualización del DOM

    expect(mockProcesarPago).toHaveBeenCalledWith(50);
  });
});
```

## Testear con Pinia

```javascript
import { setActivePinia, createPinia } from 'pinia';
import { mount } from '@vue/test-utils';
import { useCarritoStore } from '../stores/carrito';
import ProductoCard from './ProductoCard.vue';

describe('ProductoCard con Pinia', () => {
  beforeEach(() => {
    setActivePinia(createPinia());  // Crear nueva Pinia para cada test
  });

  it('añade el producto al carrito', async () => {
    const carritoStore = useCarritoStore();
    const wrapper = mount(ProductoCard, {
      props: { producto: { id: 1, nombre: 'Laptop', precio: 999 } },
      global: { plugins: [createPinia()] }
    });

    await wrapper.find('.btn-comprar').trigger('click');

    expect(carritoStore.items).toHaveLength(1);
    expect(carritoStore.items[0].id).toBe(1);
  });
});
```

## Testear operaciones asíncronas

```javascript
it('muestra la lista de usuarios después de la carga', async () => {
  // Mock de fetch
  global.fetch = vi.fn().mockResolvedValue({
    ok: true,
    json: () => Promise.resolve([
      { id: 1, nombre: 'Ana' },
      { id: 2, nombre: 'Carlos' }
    ])
  });

  const wrapper = mount(ListaUsuarios);

  // Verificar estado de carga inicial
  expect(wrapper.text()).toContain('Cargando...');

  // Esperar a que se resuelvan las promesas y se actualice el DOM
  await flushPromises();

  expect(wrapper.findAll('.usuario')).toHaveLength(2);
  expect(wrapper.text()).toContain('Ana');
  expect(wrapper.text()).toContain('Carlos');
});
```

**Consejo de entrevista**: La filosofía de Vue Test Utils es similar a React Testing Library: testear comportamiento, no implementación. Usa `wrapper.trigger()` para simular eventos, `wrapper.emitted()` para verificar eventos emitidos, y `flushPromises()` para resolver operaciones asíncronas antes de hacer assertions. Evita `wrapper.vm.método()` (accede a la implementación interna) y prefiere interacciones simuladas del usuario.
