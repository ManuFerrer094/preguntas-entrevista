---
title: ¿Qué es el ciclo de vida de un componente Vue?
difficulty: medium
tags: [vue, lifecycle, composition-api, hooks]
---

El ciclo de vida de un componente Vue describe las etapas por las que pasa desde que se crea hasta que se destruye. Vue proporciona hooks (funciones) que se ejecutan en cada etapa, permitiendo ejecutar código en momentos específicos del ciclo de vida.

## Diagrama del ciclo de vida

```
Creación          → beforeCreate → created
Montaje DOM       → beforeMount → mounted
Actualización     → beforeUpdate → updated
Desmontaje        → beforeUnmount → unmounted
```

## Composition API (Vue 3 — forma recomendada)

```vue
<script setup>
import {
  onBeforeMount,
  onMounted,
  onBeforeUpdate,
  onUpdated,
  onBeforeUnmount,
  onUnmounted,
  ref
} from 'vue';

const data = ref(null);
let subscription;
let intervalId;

// setup() se ejecuta antes de que el componente se monte
// Equivale a beforeCreate + created de Options API
console.log('setup: componente inicializando');

onBeforeMount(() => {
  console.log('onBeforeMount: antes de insertar en el DOM');
  // El DOM aún no existe; no acceder a elementos
});

onMounted(async () => {
  console.log('onMounted: componente insertado en el DOM');

  // ✅ Aquí es seguro acceder al DOM y hacer peticiones iniciales
  data.value = await fetchData();

  // Iniciar subscriptions, timers, etc.
  subscription = eventBus.subscribe('update', handleUpdate);
  intervalId = setInterval(pollData, 5000);
});

onBeforeUpdate(() => {
  console.log('onBeforeUpdate: antes de actualizar el DOM reactivo');
});

onUpdated(() => {
  console.log('onUpdated: DOM actualizado');
  // Cuidado: puede causar bucles infinitos si se modifica estado aquí
});

onBeforeUnmount(() => {
  console.log('onBeforeUnmount: componente a punto de desmontarse');
  // ✅ Limpiar timers, subscriptions, event listeners
  clearInterval(intervalId);
  subscription?.unsubscribe();
});

onUnmounted(() => {
  console.log('onUnmounted: componente desmontado');
});
</script>
```

## Options API (Vue 2/3)

```vue
<script>
export default {
  data() {
    return { users: [] };
  },

  beforeCreate() {
    // Antes de inicializar datos y eventos
    // 'this.users' aún no existe
  },

  created() {
    // Datos e instancia listos, pero sin DOM
    // Ideal para peticiones HTTP iniciales
    this.fetchUsers();
  },

  beforeMount() {
    // Antes de insertar en el DOM
  },

  mounted() {
    // DOM disponible
    // Ideal para integraciones con librerías externas (charts, maps)
    this.initializeChart();
  },

  beforeUpdate() {
    // Antes de que el DOM se actualice por cambios reactivos
  },

  updated() {
    // DOM actualizado
  },

  beforeUnmount() {
    // Antes de destruir: limpiar recursos
    this.chart?.destroy();
  },

  unmounted() {
    // Componente destruido
  }
};
</script>
```

## Caso de uso práctico: integración con librería externa

```vue
<script setup>
import { onMounted, onUnmounted, ref } from 'vue';
import Chart from 'chart.js/auto';

const canvasRef = ref(null);
let chart = null;

onMounted(() => {
  chart = new Chart(canvasRef.value, {
    type: 'line',
    data: { labels: ['Ene', 'Feb', 'Mar'], datasets: [{ data: [10, 20, 15] }] }
  });
});

onUnmounted(() => {
  chart?.destroy(); // Evitar memory leaks
});
</script>

<template>
  <canvas ref="canvasRef"></canvas>
</template>
```

**Consejo de entrevista**: El hook más usado en entrevistas es `onMounted`: sirve para hacer peticiones iniciales de datos, inicializar librerías externas y acceder al DOM. Enfatizar que en `setup()` o `created()` el DOM no está disponible. También es importante mencionar la limpieza de recursos en `onUnmounted` para evitar memory leaks con timers, event listeners o suscripciones.
