---
title: ¿Qué son los Teleports en Vue 3?
difficulty: medium
tags: [vue, teleport, portals, dom]
---

`Teleport` es un componente integrado de Vue 3 que permite renderizar el contenido de un componente en un nodo del DOM diferente al del árbol de componentes. Es especialmente útil para modales, tooltips, notificaciones y cualquier elemento que necesite escapar del contexto visual de su componente padre (por ejemplo, para evitar problemas con `overflow: hidden`, `z-index` o `position: relative`).

Sin Teleport, un modal definido dentro de un componente profundamente anidado se renderiza dentro de esa jerarquía del DOM, lo que puede causar problemas de posicionamiento o z-index. Con Teleport, el modal puede renderizarse directamente en `<body>`.

**Uso básico:**

```vue
<script setup>
import { ref } from 'vue';

const isOpen = ref(false);
</script>

<template>
  <div class="some-deep-component">
    <button @click="isOpen = true">Abrir Modal</button>

    <!-- El contenido se renderizará en <body>, no aquí -->
    <Teleport to="body">
      <div v-if="isOpen" class="modal-overlay" @click.self="isOpen = false">
        <div class="modal">
          <h2>Modal Teleportado</h2>
          <p>Este modal está en el body del DOM, pero aquí en el template.</p>
          <button @click="isOpen = false">Cerrar</button>
        </div>
      </div>
    </Teleport>
  </div>
</template>
```

**Teleport condicional con :disabled:**

```vue
<template>
  <!-- Solo teleportar en modo móvil -->
  <Teleport to="#mobile-drawer" :disabled="!isMobile">
    <NavigationMenu />
  </Teleport>
</template>
```

**Múltiples Teleports al mismo destino:**

```vue
<!-- Se añaden en orden de aparición en el DOM -->
<Teleport to="#notifications">
  <div class="notification">Primera notificación</div>
</Teleport>

<Teleport to="#notifications">
  <div class="notification">Segunda notificación</div>
</Teleport>
```

**Componente de Modal reutilizable con Teleport:**

```vue
<!-- BaseModal.vue -->
<script setup>
defineProps({
  modelValue: Boolean,
  title: String
});

const emit = defineEmits(['update:modelValue']);
const close = () => emit('update:modelValue', false);
</script>

<template>
  <Teleport to="body">
    <Transition name="modal">
      <div v-if="modelValue" class="modal-overlay" @click.self="close">
        <div class="modal-container" role="dialog" aria-modal="true">
          <header class="modal-header">
            <h2>{{ title }}</h2>
            <button @click="close" aria-label="Cerrar">✕</button>
          </header>
          <div class="modal-body">
            <slot />
          </div>
          <footer class="modal-footer">
            <slot name="footer">
              <button @click="close">Cerrar</button>
            </slot>
          </footer>
        </div>
      </div>
    </Transition>
  </Teleport>
</template>

<style>
.modal-enter-active, .modal-leave-active { transition: opacity 0.3s; }
.modal-enter-from, .modal-leave-to { opacity: 0; }
.modal-overlay {
  position: fixed; inset: 0;
  background: rgba(0,0,0,0.5);
  display: flex; align-items: center; justify-content: center;
  z-index: 1000;
}
</style>
```

**Uso del BaseModal:**

```vue
<script setup>
import { ref } from 'vue';
import BaseModal from '@/components/BaseModal.vue';

const showModal = ref(false);
</script>

<template>
  <button @click="showModal = true">Abrir</button>

  <BaseModal v-model="showModal" title="Confirmar acción">
    <p>¿Estás seguro de que deseas continuar?</p>
    <template #footer>
      <button @click="showModal = false">Cancelar</button>
      <button @click="confirm">Confirmar</button>
    </template>
  </BaseModal>
</template>
```

**Consejo de entrevista**: Teleport es la solución oficial de Vue para el patrón "portal" (también presente en React con `createPortal`). Es importante mencionar que aunque el nodo se renderiza en otro lugar del DOM, el componente sigue siendo parte del árbol de componentes de Vue: mantiene acceso al estado del padre, los eventos fluyen normalmente y se desmonta cuando el padre lo hace.
