---
title: ¿Qué es Suspense en Vue 3?
difficulty: medium
tags: [vue, suspense, async-components, loading, error-handling]
---

`<Suspense>` es un componente integrado de Vue 3 que maneja automáticamente el estado de carga de componentes asíncronos y componentes con `setup()` asíncrono. Muestra un contenido de fallback mientras el árbol de componentes resuelve sus promesas.

## Componente con setup asíncrono

```vue
<!-- UserProfile.vue — setup puede ser async en Vue 3 -->
<script setup>
// Cuando setup() es async, el componente "suspende" hasta que se resuelve
const response = await fetch(`/api/users/${props.userId}`);
const user = await response.json();
// Este componente requiere estar envuelto en <Suspense>
</script>

<template>
  <div class="profile">
    <h2>{{ user.name }}</h2>
    <p>{{ user.email }}</p>
  </div>
</template>
```

## Uso básico de Suspense

```vue
<!-- ParentComponent.vue -->
<script setup>
import UserProfile from './UserProfile.vue';
</script>

<template>
  <Suspense>
    <!-- Slot por defecto: contenido que puede suspender -->
    <template #default>
      <UserProfile :userId="userId" />
    </template>

    <!-- Slot fallback: se muestra mientras carga -->
    <template #fallback>
      <div class="loading">
        <Spinner />
        <p>Cargando perfil...</p>
      </div>
    </template>
  </Suspense>
</template>
```

## Suspense con manejo de errores

```vue
<script setup>
import { ref, onErrorCaptured } from 'vue';

const error = ref(null);

// onErrorCaptured captura errores de componentes hijos, incluyendo los de Suspense
onErrorCaptured((err) => {
  error.value = err;
  return false; // Evita que el error se propague más arriba
});
</script>

<template>
  <div v-if="error" class="error-state">
    <h3>Error al cargar</h3>
    <p>{{ error.message }}</p>
    <button @click="error = null; $forceUpdate()">Reintentar</button>
  </div>

  <Suspense v-else>
    <template #default>
      <AsyncComponent />
    </template>
    <template #fallback>
      <LoadingSkeleton />
    </template>
  </Suspense>
</template>
```

## Suspense anidado para UI granular

```vue
<template>
  <div class="product-page">
    <!-- Suspense externo: estructura principal de la página -->
    <Suspense>
      <template #default>
        <ProductHeader :productId="id" />

        <!-- Suspense interno: sección de reviews independiente -->
        <Suspense>
          <template #default>
            <ProductReviews :productId="id" />
          </template>
          <template #fallback>
            <ReviewsSkeleton />
          </template>
        </Suspense>

      </template>
      <template #fallback>
        <ProductPageSkeleton />
      </template>
    </Suspense>
  </div>
</template>
```

## Eventos de Suspense

```vue
<script setup>
function onPending() {
  console.log('Suspense: comenzando a cargar');
}

function onResolve() {
  console.log('Suspense: todos los async completados');
}

function onFallback() {
  console.log('Suspense: mostrando fallback');
}
</script>

<template>
  <Suspense
    @pending="onPending"
    @resolve="onResolve"
    @fallback="onFallback"
  >
    <template #default>
      <AsyncDashboard />
    </template>
    <template #fallback>
      <DashboardSkeleton />
    </template>
  </Suspense>
</template>
```

## Suspense con defineAsyncComponent

```vue
<script setup>
import { defineAsyncComponent } from 'vue';

// Ambos enfoques funcionan con Suspense
const AsyncChart = defineAsyncComponent(() => import('./Chart.vue'));
// O directamente en el template: <component :is="() => import('./Chart.vue')" />
</script>

<template>
  <Suspense>
    <template #default>
      <AsyncChart :data="chartData" />
    </template>
    <template #fallback>
      <ChartSkeleton />
    </template>
  </Suspense>
</template>
```

## Combinación con vue-router

```vue
<!-- RouterView con Suspense para transiciones de página -->
<template>
  <RouterView v-slot="{ Component }">
    <Transition name="page" mode="out-in">
      <Suspense>
        <template #default>
          <component :is="Component" />
        </template>
        <template #fallback>
          <PageLoader />
        </template>
      </Suspense>
    </Transition>
  </RouterView>
</template>
```

**Consejo de entrevista**: `<Suspense>` es experimental en Vue 3 pero estable en la práctica. Su característica más poderosa es permitir `await` directamente en `<script setup>`, haciendo el código asíncrono mucho más limpio. Combínalo siempre con `onErrorCaptured` o un componente de gestión de errores para manejar los fallos de las peticiones asíncronas. A diferencia de mostrar un spinner manualmente, `Suspense` sincroniza automáticamente múltiples componentes async del mismo árbol.
