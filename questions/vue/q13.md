Nuxt.js es el framework meta para Vue.js, al igual que Next.js lo es para React. Proporciona Server Side Rendering (SSR), Static Site Generation (SSG), file-based routing automático, layouts, middleware, y una arquitectura full-stack con API routes, todo configurado automáticamente.

Nuxt 3 está construido sobre Vue 3, Vite y Nitro (su servidor universal), lo que lo hace extremadamente rápido y flexible.

## Estructura de un proyecto Nuxt 3

```
my-nuxt-app/
├── pages/           → Rutas automáticas basadas en archivos
│   ├── index.vue    → /
│   ├── about.vue    → /about
│   └── users/
│       ├── index.vue    → /users
│       └── [id].vue     → /users/:id (ruta dinámica)
├── components/      → Auto-importados
├── layouts/         → Layouts reutilizables
├── composables/     → Auto-importados
├── server/
│   └── api/         → API routes del servidor
│       └── users.get.ts → GET /api/users
├── middleware/      → Middleware de navegación
├── plugins/         → Plugins de Vue
└── nuxt.config.ts
```

## File-based routing

```vue
<!-- pages/users/[id].vue -->
<script setup>
// useRoute() está auto-importado en Nuxt
const route = useRoute();
const { id } = route.params;

// useFetch es un composable de Nuxt con SSR y caché automático
const { data: user, pending, error } = await useFetch(`/api/users/${id}`);
</script>

<template>
  <div>
    <div v-if="pending">Cargando...</div>
    <div v-else-if="error">Error: {{ error.message }}</div>
    <div v-else>
      <h1>{{ user.name }}</h1>
      <p>{{ user.email }}</p>
    </div>
  </div>
</template>
```

## Server API Routes

```typescript
// server/api/users/[id].get.ts
export default defineEventHandler(async (event) => {
  const id = getRouterParam(event, 'id');

  // Acceso directo a base de datos (solo en servidor)
  const user = await db.user.findUnique({ where: { id: Number(id) } });

  if (!user) {
    throw createError({ statusCode: 404, message: 'Usuario no encontrado' });
  }

  return user;
});
```

## Modos de renderizado

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  // SSR (por defecto): renderizado en servidor en cada request
  ssr: true,

  // Configurar por ruta
  routeRules: {
    '/': { prerender: true },           // SSG: pre-renderizar en build
    '/dashboard/**': { ssr: false },    // SPA: solo cliente
    '/api/**': { cors: true },
    '/blog/**': { isr: 3600 },          // ISR: revalidar cada hora
  }
});
```

## Composables de Nuxt

```vue
<script setup>
// Data fetching con caché y SSR automático
const { data, refresh } = await useFetch('/api/products');

// Para peticiones manuales (no SSR)
const { data: user } = await useAsyncData('user', () =>
  $fetch(`/api/users/${userId.value}`),
  { watch: [userId] }
);

// Estado global compartido entre cliente y servidor
const counter = useState('counter', () => 0);

// SEO
useSeoMeta({
  title: 'Mi Página',
  description: 'Descripción para SEO',
  ogImage: '/og-image.jpg'
});
</script>
```

## Layouts

```vue
<!-- layouts/default.vue -->
<template>
  <div>
    <AppHeader />
    <main>
      <slot /> <!-- El contenido de la página va aquí -->
    </main>
    <AppFooter />
  </div>
</template>

<!-- En una página, cambiar el layout -->
<!-- pages/admin/dashboard.vue -->
<script setup>
definePageMeta({ layout: 'admin' });
</script>
```

**Consejo de entrevista**: Nuxt es la elección natural cuando se necesita SSR o SEO con Vue. Sus características más destacadas son: auto-imports (no hay que importar componentes ni composables manualmente), la integración SSR/SSG transparente con `useFetch`, y las API routes del servidor que permiten hacer aplicaciones full-stack sin necesitar un backend separado para operaciones simples.
