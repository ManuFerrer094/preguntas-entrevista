---
title: ¿Cómo funciona v-model en Vue 3 y cómo crear componentes con v-model personalizado?
difficulty: medium
tags: [vue, v-model, two-way-binding, components, vue3]
---

`v-model` en Vue 3 es un azúcar sintáctico para el enlace bidireccional de datos. En Vue 3, fue rediseñado respecto a Vue 2 para ser más flexible y permitir múltiples v-models en un solo componente.

## v-model en elementos nativos

```vue
<script setup>
import { ref } from 'vue';

const nombre = ref('');
const activo = ref(false);
const color = ref('rojo');
</script>

<template>
  <!-- input text: bind :value + @input -->
  <input v-model="nombre" />
  <!-- equivale a: <input :value="nombre" @input="nombre = $event.target.value" /> -->

  <!-- checkbox: bind :checked + @change -->
  <input type="checkbox" v-model="activo" />

  <!-- select: bind :value + @change -->
  <select v-model="color">
    <option value="rojo">Rojo</option>
    <option value="azul">Azul</option>
  </select>
</template>
```

## v-model en componentes personalizados (Vue 3)

```vue
<!-- CustomInput.vue — Componente hijo -->
<script setup>
// defineModel() es la forma moderna (Vue 3.4+)
const modelo = defineModel();           // nombre por defecto: 'modelValue'
</script>

<template>
  <input
    :value="modelo"
    @input="modelo = $event.target.value"
  />
</template>

<!-- Uso en el padre: -->
<!-- <CustomInput v-model="texto" /> -->
```

### Con defineModel() (Vue 3.4+ — forma recomendada)

```vue
<!-- CustomInput.vue -->
<script setup>
const modelo = defineModel({
  default: '',
  // Validación opcional del tipo
});
</script>

<template>
  <div class="custom-input">
    <input v-model="modelo" />
  </div>
</template>
```

```vue
<!-- Forma clásica (compatible con Vue < 3.4) -->
<script setup>
const props = defineProps({
  modelValue: String
});

const emit = defineEmits(['update:modelValue']);

function handleInput(event) {
  emit('update:modelValue', event.target.value);
}
</script>

<template>
  <input :value="modelValue" @input="handleInput" />
</template>
```

## Múltiples v-model en Vue 3

```vue
<!-- ModalFormulario.vue -->
<script setup>
const titulo = defineModel('titulo', { default: '' });
const visible = defineModel('visible', { default: false });
const contenido = defineModel('contenido', { default: '' });
</script>

<template>
  <div v-if="visible" class="modal">
    <input v-model="titulo" placeholder="Título del modal" />
    <textarea v-model="contenido"></textarea>
    <button @click="visible = false">Cerrar</button>
  </div>
</template>

<!-- Uso en el padre: múltiples v-model -->
<script setup>
import { ref } from 'vue';
const tituloModal = ref('Mi Modal');
const modalVisible = ref(false);
const textoModal = ref('');
</script>

<template>
  <button @click="modalVisible = true">Abrir Modal</button>
  <ModalFormulario
    v-model:titulo="tituloModal"
    v-model:visible="modalVisible"
    v-model:contenido="textoModal"
  />
</template>
```

## Modificadores personalizados de v-model

```vue
<!-- InputFormateado.vue — Con modificador personalizado 'capitalize' -->
<script setup>
const [modelo, modificadores] = defineModel({
  set(valor) {
    if (modificadores.capitalize) {
      return valor.charAt(0).toUpperCase() + valor.slice(1);
    }
    return valor;
  }
});
</script>

<template>
  <input v-model="modelo" />
</template>

<!-- Uso: -->
<!-- <InputFormateado v-model.capitalize="texto" /> -->
<!-- Al escribir "hola" → se almacena "Hola" -->
```

## Modificadores nativos de v-model

```vue
<script setup>
const precio = ref(0);
const descripcion = ref('');
const busqueda = ref('');
</script>

<template>
  <!-- .number: convierte el valor a número automáticamente -->
  <input v-model.number="precio" type="number" />
  <!-- precio es Number, no String -->

  <!-- .trim: elimina espacios al inicio y al final -->
  <input v-model.trim="descripcion" />
  <!-- "  hola  " → "hola" -->

  <!-- .lazy: actualiza solo en el evento 'change' (no 'input') -->
  <!-- Útil para validaciones que no queremos disparar en cada tecla -->
  <input v-model.lazy="busqueda" />
</template>
```

## v-model con validación en el componente hijo

```vue
<!-- ValidatedInput.vue -->
<script setup>
import { computed } from 'vue';

const props = defineProps({
  modelValue: String,
  minLength: { type: Number, default: 0 },
  maxLength: { type: Number, default: Infinity }
});

const emit = defineEmits(['update:modelValue']);

const esValido = computed(() =>
  props.modelValue.length >= props.minLength &&
  props.modelValue.length <= props.maxLength
);

function handleInput(e) {
  emit('update:modelValue', e.target.value);
}
</script>

<template>
  <div :class="{ invalido: !esValido }">
    <input :value="modelValue" @input="handleInput" />
    <span v-if="!esValido" class="error">
      Entre {{ minLength }} y {{ maxLength }} caracteres
    </span>
  </div>
</template>
```

**Consejo de entrevista**: En Vue 2, `v-model` era azúcar para `:value` + `@input`. En Vue 3 con `defineModel()` (Vue 3.4+), es incluso más simple: el prop y el emit se generan automáticamente. La posibilidad de tener **múltiples v-model** en un componente es una mejora clave de Vue 3 respecto a Vue 2. Los modificadores personalizados permiten transformaciones automáticas del valor.
