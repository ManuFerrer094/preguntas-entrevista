La reactividad en Vue 3 es el sistema que detecta automáticamente cuándo cambian los datos y actualiza el DOM en consecuencia. Vue 3 reescribió completamente el sistema de reactividad usando **Proxies de JavaScript** (ES6), reemplazando los `Object.defineProperty` de Vue 2, lo que elimina varias limitaciones importantes.

## Cómo funciona internamente

Vue 3 envuelve los objetos reactivos en un `Proxy` que intercepta las operaciones de lectura (get) y escritura (set). Cuando un componente accede a una propiedad reactiva durante su renderizado, Vue registra esa dependencia. Cuando la propiedad cambia, Vue notifica a todos los suscriptores para que se actualicen.

```javascript
// Implementación simplificada del sistema de reactividad
function reactive(target) {
  return new Proxy(target, {
    get(target, key, receiver) {
      track(target, key); // Registrar dependencia
      return Reflect.get(target, key, receiver);
    },
    set(target, key, value, receiver) {
      const result = Reflect.set(target, key, value, receiver);
      trigger(target, key); // Notificar suscriptores
      return result;
    }
  });
}
```

## Primitivas de reactividad

**`ref`** — para valores primitivos (y objetos):

```vue
<script setup>
import { ref } from 'vue';

const count = ref(0);       // Envuelve el valor en { value: 0 }
const name = ref('Vue');

// En JS: acceso con .value
count.value++;
console.log(count.value); // 1

// En el template: .value se desenvuelve automáticamente
</script>

<template>
  <p>{{ count }}</p>  <!-- No necesita .value -->
  <button @click="count++">+1</button>
</template>
```

**`reactive`** — para objetos y arrays:

```vue
<script setup>
import { reactive } from 'vue';

const state = reactive({
  user: { name: 'Ana', age: 28 },
  items: [],
  loading: false
});

// No necesita .value, acceso directo
state.user.name = 'Carlos'; // ✅ Vue detecta este cambio
state.items.push({ id: 1 }); // ✅ También detecta métodos de array

// ❌ No reasignar el objeto completo (pierde reactividad)
// state = reactive({ user: {...} }); // MAL
</script>
```

**`computed`** — valores derivados con caché:

```vue
<script setup>
import { ref, computed } from 'vue';

const firstName = ref('Ana');
const lastName = ref('García');

const fullName = computed(() => `${firstName.value} ${lastName.value}`);
// Solo recalcula cuando firstName o lastName cambian

// Computed writable
const fullNameWritable = computed({
  get: () => `${firstName.value} ${lastName.value}`,
  set: (value) => {
    [firstName.value, lastName.value] = value.split(' ');
  }
});
</script>
```

## Ventajas de Proxy sobre Vue 2

Vue 2 usaba `Object.defineProperty` y tenía limitaciones:

```javascript
// ❌ Vue 2: NO detectaba estas operaciones
this.myObject.newProperty = 'value'; // Añadir nueva propiedad
delete this.myObject.someProperty;   // Eliminar propiedad
this.myArray[0] = 'value';           // Modificar por índice

// ✅ Soluciones de Vue 2
this.$set(this.myObject, 'newProperty', 'value');
this.$delete(this.myObject, 'someProperty');

// ✅ Vue 3 con Proxy: todo esto funciona automáticamente
state.newProperty = 'value';         // Detectado
delete state.someProperty;           // Detectado
state.array[0] = 'value';            // Detectado
```

## Reactividad con `toRefs` y `toRef`

```javascript
const state = reactive({ count: 0, name: 'Vue' });

// ❌ Desestructurar pierde la reactividad
const { count } = state; // count es un número normal, no reactivo

// ✅ toRefs convierte cada propiedad en un ref
const { count, name } = toRefs(state); // Ambas son refs reactivas
count.value++; // Actualiza state.count
```

**Consejo de entrevista**: Una pregunta clásica es la diferencia entre `ref` y `reactive`. `ref` puede contener cualquier tipo de valor y siempre requiere `.value`. `reactive` solo funciona con objetos/arrays pero tiene acceso directo sin `.value`. En la práctica, muchos equipos prefieren usar `ref` para todo por consistencia.
