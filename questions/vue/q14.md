---
title: ¿Cuál es la diferencia entre ref y reactive en Vue 3?
difficulty: medium
tags: [vue, ref, reactive, composition-api, reactivity]
---

`ref` y `reactive` son las dos primitivas principales para crear estado reactivo en Vue 3. Ambas crean datos reactivos, pero difieren en su uso, la forma de acceder a los valores y los tipos de datos que manejan mejor.

## ref

`ref` puede contener **cualquier tipo de valor**: primitivos (string, number, boolean), objetos, arrays, null, etc. Siempre envuelve el valor en un objeto con una propiedad `.value`.

```vue
<script setup>
import { ref } from 'vue';

// Primitivos
const count = ref(0);
const name = ref('Vue');
const isActive = ref(false);

// Objetos y arrays
const user = ref({ name: 'Ana', age: 28 });
const items = ref([1, 2, 3]);

// Acceso y modificación — SIEMPRE con .value en JS
count.value++;
name.value = 'React';
user.value.age = 29;         // Modificar propiedad
user.value = { name: 'Carlos', age: 30 }; // Reasignar (funciona)
items.value.push(4);
</script>

<template>
  <!-- En el template: .value se desenvuelve automáticamente -->
  <p>{{ count }}</p>       <!-- No {{ count.value }} -->
  <p>{{ user.name }}</p>   <!-- No {{ user.value.name }} -->
</template>
```

## reactive

`reactive` solo funciona con **objetos y arrays** (tipos de referencia). No necesita `.value` para acceder a las propiedades.

```vue
<script setup>
import { reactive } from 'vue';

const state = reactive({
  count: 0,
  user: { name: 'Ana', age: 28 },
  items: ['a', 'b', 'c']
});

// Acceso directo sin .value
state.count++;
state.user.age = 29;
state.items.push('d');

// ❌ NO reasignar el objeto completo: pierde reactividad
// state = reactive({ count: 10 }); // MAL - pierde la referencia reactiva

// ❌ NO desestructurar: pierde reactividad
// const { count } = state; // count es un número normal, no reactivo

// ✅ Para desestructurar, usar toRefs
import { toRefs } from 'vue';
const { count, user } = toRefs(state); // Ahora son refs reactivas
</script>
```

## Diferencias clave

| Característica | ref | reactive |
|---|---|---|
| Tipos soportados | Cualquiera | Solo objetos/arrays |
| Acceso en JS | `.value` requerido | Directo |
| Acceso en template | Auto-unwrapped | Directo |
| Reasignación | ✅ `ref.value = newObj` | ❌ Pierde reactividad |
| Desestructuración | ✅ Directo | ❌ Necesita `toRefs` |
| Consistencia | Alta | Variable |

## ¿Cuándo usar cada uno?

```vue
<script setup>
import { ref, reactive } from 'vue';

// ✅ ref: para primitivos y valores que se reasignan
const isLoading = ref(false);
const selectedId = ref(null);
const count = ref(0);

// ✅ ref: también para objetos que pueden ser reemplazados completamente
const currentUser = ref(null); // Puede ser null y luego un objeto

// ✅ reactive: para estado de formulario o estado relacionado
const form = reactive({
  email: '',
  password: '',
  rememberMe: false
});

// ✅ reactive: para estado complejo de un componente
const tableState = reactive({
  page: 1,
  pageSize: 10,
  sortBy: 'name',
  sortOrder: 'asc',
  filters: {}
});
</script>
```

## Recomendación de la comunidad

La mayoría de la comunidad Vue 3 recomienda **usar `ref` para todo** por razones de consistencia: siempre sabes que necesitas `.value`, no hay sorpresas con desestructuración, y puedes reasignar el valor completo fácilmente. El equipo de Vue acepta ambos enfoques, pero la guía de estilo tiende hacia `ref`.

```vue
<script setup>
// Patrón coherente: todo con ref
const isOpen = ref(false);
const users = ref([]);
const config = ref({ theme: 'dark', lang: 'es' });
const selectedUser = ref(null);
</script>
```

**Consejo de entrevista**: Esta es una pregunta muy frecuente en entrevistas de Vue. La respuesta clave es: `ref` requiere `.value` en JS pero no en templates (auto-unwrapping), mientras que `reactive` no requiere `.value` pero no funciona con primitivos y pierde reactividad si se desestructura o reasigna.
