Las directivas personalizadas en Vue 3 permiten encapsular manipulaciones de bajo nivel del DOM que no encajan bien en componentes. Son ideales para: focus automático, tooltips, resize observers, lazy loading de imágenes, y permisos de acceso.

## Estructura de una directiva

```javascript
// Una directiva es un objeto con hooks del ciclo de vida del elemento DOM
const myDirective = {
  // Cuando la directiva se vincula al elemento, antes de que el componente se monte
  created(el, binding, vnode) {},

  // Justo antes de que el componente padre se monte
  beforeMount(el, binding, vnode) {},

  // Después de que el componente padre y todos sus hijos se han montado
  mounted(el, binding, vnode) {},

  // Antes de actualizar el componente padre
  beforeUpdate(el, binding, vnode, prevVnode) {},

  // Después de actualizar el componente padre y sus hijos
  updated(el, binding, vnode, prevVnode) {},

  // Antes de desmontar el componente padre
  beforeUnmount(el, binding, vnode) {},

  // Cuando el componente padre se desmonta
  unmounted(el, binding, vnode) {}
};

// binding contiene:
// binding.value    — el valor pasado a la directiva: v-my="value"
// binding.oldValue — valor anterior (solo en updated)
// binding.arg      — el argumento: v-my:arg
// binding.modifiers — modificadores: v-my.mod1.mod2 → { mod1: true, mod2: true }
// binding.instance — instancia del componente
```

## Directiva v-focus

```javascript
// directivas/focus.js
export const vFocus = {
  mounted(el) {
    el.focus();
  }
};

// Registrar globalmente
app.directive('focus', vFocus);

// O registrar localmente en un componente
// <script setup>
// import { vFocus } from './directivas/focus';
// </script>
```

```vue
<template>
  <!-- Uso: el input se enfoca automáticamente al montar -->
  <input v-focus placeholder="Buscador" />
</template>
```

## Directiva v-tooltip

```javascript
// directivas/tooltip.js
export const vTooltip = {
  mounted(el, binding) {
    const tooltip = document.createElement('div');
    tooltip.className = 'tooltip-popup';
    tooltip.textContent = binding.value;
    tooltip.style.cssText = `
      position: fixed;
      background: #333;
      color: white;
      padding: 4px 8px;
      border-radius: 4px;
      font-size: 12px;
      pointer-events: none;
      display: none;
      z-index: 9999;
    `;

    document.body.appendChild(tooltip);
    el._tooltip = tooltip;

    el._showTooltip = (e) => {
      tooltip.style.display = 'block';
      tooltip.style.top = `${e.clientY - tooltip.offsetHeight - 8}px`;
      tooltip.style.left = `${e.clientX - tooltip.offsetWidth / 2}px`;
    };
    el._hideTooltip = () => {
      tooltip.style.display = 'none';
    };

    el.addEventListener('mouseenter', el._showTooltip);
    el.addEventListener('mousemove', el._showTooltip);
    el.addEventListener('mouseleave', el._hideTooltip);
  },

  updated(el, binding) {
    if (el._tooltip) el._tooltip.textContent = binding.value;
  },

  unmounted(el) {
    el.removeEventListener('mouseenter', el._showTooltip);
    el.removeEventListener('mousemove', el._showTooltip);
    el.removeEventListener('mouseleave', el._hideTooltip);
    el._tooltip?.remove();
  }
};
```

```vue
<template>
  <button v-tooltip="'Eliminar este elemento'">🗑️</button>
  <button v-tooltip="tooltipText">ℹ️</button>
</template>
```

## Directiva v-permission (control de acceso)

```javascript
// directivas/permission.js
export const vPermission = {
  mounted(el, binding) {
    const { value: requiredPermission, modifiers } = binding;
    const userPermissions = getCurrentUserPermissions(); // De tu store/composable

    const hasPermission = Array.isArray(requiredPermission)
      ? modifiers.any
        ? requiredPermission.some(p => userPermissions.includes(p))
        : requiredPermission.every(p => userPermissions.includes(p))
      : userPermissions.includes(requiredPermission);

    if (!hasPermission) {
      // Ocultar o eliminar el elemento
      if (modifiers.hide) {
        el.style.display = 'none';
      } else {
        el.parentNode?.removeChild(el);
      }
    }
  }
};
```

```vue
<template>
  <!-- Visible solo para admins -->
  <button v-permission="'admin:delete'">Eliminar usuario</button>

  <!-- Visible si tiene alguno de los permisos -->
  <button v-permission.any="['admin:write', 'moderator:write']">
    Editar contenido
  </button>

  <!-- Ocultar con CSS (preserva el espacio) en lugar de eliminar -->
  <button v-permission.hide="'superadmin'">
    Acción peligrosa
  </button>
</template>
```

## Directiva v-lazy-img (lazy loading de imágenes)

```javascript
// directivas/lazy-img.js
export const vLazyImg = {
  mounted(el, binding) {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          el.src = binding.value;
          el.onload = () => el.classList.add('loaded');
          observer.unobserve(el);
        }
      },
      { threshold: 0.1, rootMargin: '50px' }
    );

    el.src = binding.arg || '/placeholder.png'; // Imagen de placeholder
    observer.observe(el);
    el._lazyObserver = observer;
  },

  unmounted(el) {
    el._lazyObserver?.disconnect();
  }
};
```

```vue
<template>
  <!-- src por defecto, carga la imagen real cuando es visible -->
  <img v-lazy-img:placeholder.jpg="product.imageUrl" alt="Producto" />
</template>
```

**Consejo de entrevista**: Las directivas son para manipulación directa del DOM que no encaja en componentes. Siempre limpia los recursos en el hook `unmounted` (event listeners, observers, timers) para evitar memory leaks. Para casos más complejos con estado, considera si un componente con `<slot>` sería más apropiado que una directiva. Las directivas más comunes en entrevistas: `v-focus`, `v-tooltip`, y control de permisos.
