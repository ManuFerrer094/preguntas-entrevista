---
title: ¿Qué es Pinia?
difficulty: medium
tags: [vue, pinia, state-management, store]
---

Pinia es la librería oficial de gestión de estado para Vue 3. Es el sucesor de Vuex y fue diseñada específicamente para aprovechar la Composition API y TypeScript. Ofrece una API más sencilla, mejor soporte para TypeScript, DevTools integradas y módulos automáticos sin necesidad de namespacing.

A diferencia de Vuex, en Pinia no hay mutations: el estado se modifica directamente en las acciones, lo que reduce significativamente el boilerplate.

**Instalación y configuración:**

```javascript
// main.ts
import { createApp } from 'vue';
import { createPinia } from 'pinia';
import App from './App.vue';

const app = createApp(App);
app.use(createPinia());
app.mount('#app');
```

**Definición de un store:**

```typescript
// stores/cart.ts
import { defineStore } from 'pinia';
import { ref, computed } from 'vue';

// Setup Store (estilo Composition API) — preferido
export const useCartStore = defineStore('cart', () => {
  // State
  const items = ref<CartItem[]>([]);
  const loading = ref(false);

  // Getters (equivalente a computed)
  const totalItems = computed(() =>
    items.value.reduce((sum, item) => sum + item.quantity, 0)
  );
  const totalPrice = computed(() =>
    items.value.reduce((sum, item) => sum + item.price * item.quantity, 0)
  );

  // Actions
  function addItem(product: Product) {
    const existing = items.value.find(i => i.id === product.id);
    if (existing) {
      existing.quantity++;
    } else {
      items.value.push({ ...product, quantity: 1 });
    }
  }

  function removeItem(id: number) {
    const index = items.value.findIndex(i => i.id === id);
    if (index !== -1) items.value.splice(index, 1);
  }

  async function syncWithServer() {
    loading.value = true;
    try {
      await fetch('/api/cart', {
        method: 'POST',
        body: JSON.stringify(items.value)
      });
    } finally {
      loading.value = false;
    }
  }

  function $reset() {
    items.value = [];
  }

  return { items, loading, totalItems, totalPrice, addItem, removeItem, syncWithServer, $reset };
});
```

**Uso en componentes:**

```vue
<script setup>
import { useCartStore } from '@/stores/cart';
import { storeToRefs } from 'pinia';

const cartStore = useCartStore();

// storeToRefs mantiene la reactividad al desestructurar
const { items, totalItems, totalPrice, loading } = storeToRefs(cartStore);

// Las acciones se desestructuran directamente (no necesitan storeToRefs)
const { addItem, removeItem } = cartStore;
</script>

<template>
  <div>
    <p>{{ totalItems }} artículos - ${{ totalPrice }}</p>
    <ul>
      <li v-for="item in items" :key="item.id">
        {{ item.name }} x{{ item.quantity }}
        <button @click="removeItem(item.id)">✕</button>
      </li>
    </ul>
  </div>
</template>
```

**Plugin de persistencia:**

```javascript
import { createPinia } from 'pinia';
import piniaPluginPersistedstate from 'pinia-plugin-persistedstate';

const pinia = createPinia();
pinia.use(piniaPluginPersistedstate);

// En el store:
export const useSettingsStore = defineStore('settings', () => {
  const theme = ref('light');
  return { theme };
}, {
  persist: true // Persiste automáticamente en localStorage
});
```

**Consejo de entrevista**: Una diferencia clave con Vuex es que no hay mutations: las acciones pueden modificar el estado directamente. Otra diferencia importante es que `storeToRefs` es necesario para mantener la reactividad al desestructurar propiedades del store. Las acciones pueden ser asíncronas sin configuración adicional, a diferencia de Vuex donde se necesitaban actions separadas de mutations.
