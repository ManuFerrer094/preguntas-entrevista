---
title: ¿Cómo implementar lazy loading de componentes en Vue 3?
difficulty: medium
tags: [vue, lazy-loading, code-splitting, performance, dynamic-import]
---

El lazy loading divide el bundle en chunks que se cargan bajo demanda, reduciendo el tiempo de carga inicial. En Vue 3 se implementa con `defineAsyncComponent` para componentes individuales y con importaciones dinámicas en Vue Router para rutas completas.

## defineAsyncComponent — carga diferida de componentes

```vue
<script setup>
import { defineAsyncComponent } from 'vue';

// Carga básica: el componente se importa cuando se necesita
const HeavyChart = defineAsyncComponent(() =>
  import('./components/HeavyChart.vue')
);

// Con opciones avanzadas: loading, error y timeout
const DataTable = defineAsyncComponent({
  loader: () => import('./components/DataTable.vue'),
  loadingComponent: LoadingSpinner, // Mostrar mientras carga
  errorComponent: ErrorMessage,     // Mostrar si hay error
  delay: 200,                       // Retrasar el loading (evita flash)
  timeout: 10000,                   // Error si tarda más de 10s
  onError(error, retry, fail, attempts) {
    if (attempts <= 3) {
      retry(); // Reintentar hasta 3 veces
    } else {
      fail(); // Mostrar errorComponent
    }
  }
});
</script>

<template>
  <!-- Suspense muestra el fallback mientras carga el componente async -->
  <Suspense>
    <template #default>
      <HeavyChart :data="chartData" />
    </template>
    <template #fallback>
      <div>Cargando gráfico...</div>
    </template>
  </Suspense>

  <!-- Componentes async también funcionan sin Suspense (tienen su propio loading) -->
  <DataTable :rows="tableData" />
</template>
```

## Lazy loading por rutas (más impactante)

```javascript
// router/index.js
import { createRouter, createWebHistory } from 'vue-router';

const router = createRouter({
  history: createWebHistory(),
  routes: [
    {
      path: '/',
      component: () => import('../views/HomeView.vue')  // Chunk separado
    },
    {
      path: '/dashboard',
      component: () => import('../views/DashboardView.vue')
    },
    {
      path: '/admin',
      // Agrupar en el mismo chunk con webpackChunkName
      component: () => import(/* webpackChunkName: "admin" */ '../views/AdminView.vue')
    },
    {
      path: '/admin/users',
      component: () => import(/* webpackChunkName: "admin" */ '../views/AdminUsersView.vue')
    }
  ]
});
```

## Preloading para mejorar la percepción de velocidad

```vue
<script setup>
import { defineAsyncComponent } from 'vue';

const ModalComponent = defineAsyncComponent(() =>
  import('./ModalComponent.vue')
);

// Precargar el modal cuando el usuario pasa el ratón por el botón
function preloadModal() {
  import('./ModalComponent.vue'); // Inicia la descarga sin renderizar
}
</script>

<template>
  <button
    @mouseenter="preloadModal"
    @click="showModal = true"
  >
    Abrir modal
  </button>

  <ModalComponent v-if="showModal" />
</template>
```

## Lazy loading condicional

```vue
<script setup>
import { defineAsyncComponent, shallowRef } from 'vue';

const activeTab = ref('overview');

// Mapa de componentes lazy — solo se cargan cuando se activan
const tabComponents = {
  overview: defineAsyncComponent(() => import('./tabs/OverviewTab.vue')),
  analytics: defineAsyncComponent(() => import('./tabs/AnalyticsTab.vue')),
  settings: defineAsyncComponent(() => import('./tabs/SettingsTab.vue'))
};

// El tipo correcto para componentes dinámicos
const currentTabComponent = shallowRef(tabComponents.overview);

function switchTab(tab) {
  activeTab.value = tab;
  currentTabComponent.value = tabComponents[tab];
}
</script>

<template>
  <nav>
    <button
      v-for="tab in ['overview', 'analytics', 'settings']"
      :key="tab"
      :class="{ active: activeTab === tab }"
      @click="switchTab(tab)"
    >
      {{ tab }}
    </button>
  </nav>

  <Suspense>
    <component :is="currentTabComponent" />
    <template #fallback>
      <TabSkeleton />
    </template>
  </Suspense>
</template>
```

**Consejo de entrevista**: El lazy loading por rutas en Vue Router es el más impactante para reducir el bundle inicial. `defineAsyncComponent` es la herramienta de Vue 3 para componentes individuales. El componente `<Suspense>` de Vue 3 funciona de manera similar a React Suspense y permite mostrar un fallback mientras el componente async carga. Para grandes aplicaciones, apunta a que el bundle inicial sea menor de 150KB gzipped.
