---
title: ¿Qué son los watchers en Vue?
difficulty: medium
tags: [vue, watchers, reactivity, composition-api]
---

Los watchers en Vue permiten observar cambios en datos reactivos y ejecutar código como efecto secundario cuando esos datos cambian. Son útiles para operaciones que no pueden expresarse como datos computados: llamadas a APIs, manipulación del DOM, escritura en localStorage, etc.

Vue ofrece varias formas de definir watchers: `watch` y `watchEffect` en la Composition API, y `watch` en la Options API.

## watch — observa una fuente específica

```vue
<script setup>
import { ref, watch } from 'vue';

const searchQuery = ref('');
const results = ref([]);
const loading = ref(false);

// Observa un ref específico
watch(searchQuery, async (newValue, oldValue) => {
  if (newValue === oldValue || newValue.length < 2) return;

  loading.value = true;
  try {
    const response = await fetch(`/api/search?q=${newValue}`);
    results.value = await response.json();
  } finally {
    loading.value = false;
  }
}, {
  debounce: 300 // No nativo, usar con VueUse
});

// Observar un objeto reactivo
const user = ref({ name: '', age: 0 });

watch(
  () => user.value.age, // getter para propiedades anidadas
  (newAge) => {
    if (newAge < 0) user.value.age = 0;
  }
);

// Observar múltiples fuentes
const firstName = ref('');
const lastName = ref('');

watch([firstName, lastName], ([newFirst, newLast], [oldFirst, oldLast]) => {
  console.log(`Nombre completo cambió a: ${newFirst} ${newLast}`);
});
</script>
```

## watchEffect — ejecuta y rastrea automáticamente

`watchEffect` ejecuta el callback inmediatamente y rastrea automáticamente todos los datos reactivos que se acceden dentro de él:

```vue
<script setup>
import { ref, watchEffect, onUnmounted } from 'vue';

const userId = ref(1);
const userData = ref(null);

// Se ejecuta inmediatamente y cada vez que userId cambia
const stop = watchEffect(async (onCleanup) => {
  let cancelled = false;

  // onCleanup evita condiciones de carrera
  onCleanup(() => { cancelled = true; });

  const data = await fetch(`/api/users/${userId.value}`).then(r => r.json());

  if (!cancelled) {
    userData.value = data;
  }
});

// Detener el watcher manualmente
onUnmounted(() => stop());
</script>
```

## watch en Options API

```vue
<script>
export default {
  data() {
    return {
      count: 0,
      user: { name: '', address: { city: '' } }
    };
  },
  watch: {
    // Watcher simple
    count(newVal, oldVal) {
      console.log(`Count: ${oldVal} → ${newVal}`);
    },

    // Watcher profundo para objetos
    user: {
      handler(newVal) {
        this.saveToServer(newVal);
      },
      deep: true,        // Observa cambios en propiedades anidadas
      immediate: true    // Ejecuta inmediatamente al montar
    },

    // Atajo para propiedad anidada
    'user.address.city'(newCity) {
      this.updateMap(newCity);
    }
  }
}
</script>
```

## watch vs computed

| | watch | computed |
|---|---|---|
| Propósito | Efectos secundarios | Valores derivados |
| Devuelve | Nada | Valor calculado |
| Ejecución | Al cambiar la fuente | Solo cuando se accede |
| Uso típico | API calls, DOM | Transformar datos |

**Consejo de entrevista**: Un error común es usar `watch` cuando `computed` sería suficiente. La regla general es: si necesitas un valor derivado, usa `computed`; si necesitas ejecutar código en respuesta a un cambio (side effect), usa `watch`. El opción `immediate: true` es útil para ejecutar la lógica tanto al montar como al actualizar.
