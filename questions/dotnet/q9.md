---
title: ¿Qué es la inyección de dependencias (DI) en .NET y cuáles son los tiempos de vida de los servicios?
difficulty: medium
tags: [dotnet, dependency-injection, di, servicios, lifetime, aspnet-core]
---

La **Inyección de Dependencias (DI)** es un patrón de diseño en el que las dependencias de una clase se suministran desde el exterior en lugar de crearlas internamente. ASP.NET Core incluye un contenedor DI nativo.

**Registro de servicios en `Program.cs`:**

```csharp
builder.Services.AddTransient<IEmailService, SmtpEmailService>();
builder.Services.AddScoped<ICarritoService, CarritoService>();
builder.Services.AddSingleton<IConfiguracion, ConfiguracionApp>();
```

**Tiempos de vida (lifetime):**

| Lifetime | Creación | Destrucción | Uso típico |
|---|---|---|---|
| **Transient** | Cada vez que se solicita | Al salir del scope | Servicios ligeros sin estado |
| **Scoped** | Una vez por petición HTTP | Al finalizar la petición | DbContext, servicios de negocio |
| **Singleton** | Una sola vez en la app | Al apagar la app | Caché, configuración, logging |

**Inyección en controladores y clases:**

```csharp
public class PedidosController : Controller
{
    private readonly IPedidoService _pedidos;
    private readonly ILogger<PedidosController> _logger;

    public PedidosController(IPedidoService pedidos,
        ILogger<PedidosController> logger)
    {
        _pedidos = pedidos;
        _logger = logger;
    }
}
```

**Captive Dependency (error común):**

Inyectar un servicio `Scoped` dentro de un `Singleton` crea una dependencia cautiva: el scoped se convierte en singleton de facto, lo que puede causar problemas de estado compartido. ASP.NET Core detecta y lanza excepción en modo desarrollo.

**Acceso al contenedor fuera de constructores:**

```csharp
// Evitar en lo posible (Service Locator anti-pattern)
var service = app.Services.GetRequiredService<IEmailService>();
```

Prefiere siempre la inyección por constructor.
