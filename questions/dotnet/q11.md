---
title: ¿Qué es la configuración en ASP.NET Core y cómo se accede a appsettings.json?
difficulty: easy
tags: [dotnet, configuracion, appsettings, options-pattern, aspnet-core]
---

ASP.NET Core usa un sistema de configuración flexible y jerárquico que puede combinar múltiples fuentes: `appsettings.json`, variables de entorno, argumentos de línea de comandos y secretos de usuario.

**Estructura de `appsettings.json`:**

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information"
    }
  },
  "ConnectionStrings": {
    "Default": "Server=.;Database=MiApp;Trusted_Connection=True;"
  },
  "Email": {
    "SmtpHost": "smtp.ejemplo.com",
    "SmtpPort": 587,
    "RemitentePorDefecto": "no-reply@ejemplo.com"
  }
}
```

**Acceso directo con `IConfiguration`:**

```csharp
public class MiServicio
{
    private readonly IConfiguration _config;

    public MiServicio(IConfiguration config) => _config = config;

    public void Enviar()
    {
        var host = _config["Email:SmtpHost"];
        var puerto = _config.GetValue<int>("Email:SmtpPort");
        var connStr = _config.GetConnectionString("Default");
    }
}
```

**Options Pattern (recomendado):**

```csharp
// Clase de configuración fuertemente tipada
public class EmailOptions
{
    public const string SectionName = "Email";
    public string SmtpHost { get; set; }
    public int SmtpPort { get; set; }
    public string RemitentePorDefecto { get; set; }
}

// Registro en Program.cs
builder.Services.Configure<EmailOptions>(
    builder.Configuration.GetSection(EmailOptions.SectionName));

// Inyección en servicio
public class EmailService
{
    private readonly EmailOptions _opts;

    public EmailService(IOptions<EmailOptions> opts)
        => _opts = opts.Value;
}
```

**Jerarquía de sobreescritura:** `appsettings.json` → `appsettings.{Entorno}.json` → Variables de entorno → Argumentos CLI. Cada fuente puede sobreescribir la anterior.
