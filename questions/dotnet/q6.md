---
title: ¿Para qué sirve el bloque using en el contexto de acceso a datos?
difficulty: easy
tags: [dotnet, using, idisposable, memoria, base-de-datos]
---

El bloque **`using`** en C# se utiliza para garantizar que los objetos que implementan la interfaz `IDisposable` sean liberados correctamente al salir del bloque, incluso si ocurre una excepción.

En el contexto de acceso a datos es especialmente importante porque:

- **Libera conexiones a la base de datos:** Las conexiones son recursos costosos. El bloque `using` garantiza que la conexión se cierre y devuelva al pool de conexiones tan pronto como ya no sea necesaria.
- **Destruye el contexto de Entity Framework:** El `DbContext` mantiene un caché de entidades en memoria. Usarlo con `using` evita fugas de memoria y comportamientos inesperados.
- **Evita fugas de recursos:** Aunque exista una excepción, el método `Dispose()` siempre se invoca.

```csharp
// Con Entity Framework 6
using (var context = new MiDbContext())
{
    var clientes = context.Clientes.Where(c => c.Activo).ToList();
    // Al salir del bloque, context.Dispose() se llama automáticamente
}

// Con ADO.NET
using (var connection = new SqlConnection(connectionString))
using (var command = new SqlCommand("SELECT * FROM Productos", connection))
{
    connection.Open();
    var reader = command.ExecuteReader();
    // conexión y comando se liberan automáticamente
}
```

Desde C# 8.0 existe la sintaxis `using` de declaración (sin llaves), que libera el recurso al final del ámbito del método.
