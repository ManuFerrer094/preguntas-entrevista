---
title: ¿Qué es un ORM y por qué usar Entity Framework?
difficulty: easy
tags: [dotnet, entity-framework, orm, base-de-datos]
---

Un **ORM (Object-Relational Mapper)** es una herramienta que actúa como puente entre una base de datos relacional y el código orientado a objetos. Convierte las tablas de la base de datos en clases y las filas en objetos, permitiendo manipular los datos usando la programación orientada a objetos en lugar de escribir consultas SQL manuales.

**Entity Framework (EF)** es el ORM oficial de Microsoft para .NET. Sus ventajas son:

- **Productividad:** Se trabaja directamente con objetos .NET en lugar de SQL, reduciendo código repetitivo.
- **Mantenibilidad:** Los cambios en el modelo de datos se reflejan en el código C# sin necesidad de reescribir consultas SQL.
- **LINQ integration:** Permite escribir consultas tipadas y con IntelliSense usando LINQ.
- **Migraciones:** Gestiona los cambios en el esquema de la base de datos mediante migraciones controladas.

```csharp
// Sin ORM (ADO.NET puro)
SqlCommand cmd = new SqlCommand("SELECT * FROM Clientes WHERE Id = @id", conn);
cmd.Parameters.AddWithValue("@id", 1);

// Con Entity Framework
var cliente = context.Clientes.Find(1);
```

En .NET 4.8 se usa principalmente **Entity Framework 6 (EF6)**, la versión madura del ORM clásico, con soporte completo para bases de datos relacionales como SQL Server.
