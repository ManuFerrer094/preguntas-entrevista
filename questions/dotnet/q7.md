---
title: ¿Cómo se implementa programación asíncrona con async/await en .NET?
difficulty: medium
tags: [dotnet, async, await, task, hilos]
---

La programación asíncrona en .NET permite ejecutar operaciones largas (llamadas a bases de datos, peticiones HTTP, lectura de archivos) sin bloquear el hilo actual. Se basa en las palabras clave **`async`** y **`await`** junto con la clase **`Task`**, introducidas desde .NET 4.0.

**Conceptos clave:**

- **`async`:** Marca un método como asíncrono. El método puede contener una o más expresiones `await`.
- **`await`:** Suspende la ejecución del método hasta que la tarea esperada complete, sin bloquear el hilo.
- **`Task` / `Task<T>`:** Representa una operación asíncrona. `Task<T>` retorna un valor de tipo `T`.

```csharp
// Método asíncrono que llama a la base de datos
public async Task<List<Cliente>> ObtenerClientesAsync()
{
    using (var context = new MiDbContext())
    {
        return await context.Clientes.ToListAsync();
    }
}

// Uso con Task.Run para operaciones CPU-intensivas
public async Task ProcesarDatosAsync()
{
    var resultado = await Task.Run(() => OperacionPesada());
    Console.WriteLine(resultado);
}
```

**Diferencia entre `Task.Run` y métodos async nativos:**
- Los métodos de I/O asíncronos (`ToListAsync`, `ReadAsync`, etc.) no crean un nuevo hilo; usan callbacks del sistema operativo.
- `Task.Run` ejecuta código en un hilo del ThreadPool, ideal para operaciones que consumen CPU.
