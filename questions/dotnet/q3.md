---
title: ¿Cómo se maneja la compatibilidad de versiones en .NET 4.8?
difficulty: medium
tags: [dotnet, compatibilidad, versiones, migracion]
---

El .NET Framework está diseñado con **compatibilidad hacia atrás (backward compatibility)** como principio fundamental:

- **Ejecución sin cambios:** Las aplicaciones compiladas para versiones anteriores del .NET Framework (por ejemplo, 3.5 o 4.0) generalmente se ejecutan sin modificaciones en .NET 4.8, ya que Microsoft garantiza la compatibilidad binaria entre versiones de la misma familia (4.x).

- **Ejecución en paralelo (Side-by-Side Execution):** Es posible tener múltiples versiones del .NET Framework instaladas simultáneamente en el mismo equipo. Cada aplicación puede especificar qué versión del runtime desea utilizar mediante el archivo de configuración (`app.config`) con la sección `<supportedRuntime>`.

- **Target Framework:** En Visual Studio, cada proyecto especifica el framework de destino. Una aplicación que apunta a .NET 4.8 utiliza todas las APIs disponibles en esa versión.

- **Cambios de comportamiento:** Aunque la compatibilidad binaria está garantizada, algunos comportamientos pueden cambiar entre versiones secundarias. Microsoft documenta estos cambios en las notas de compatibilidad de cada versión.

```xml
<!-- app.config - especificar versión de runtime -->
<configuration>
  <startup>
    <supportedRuntime version="v4.0" sku=".NETFramework,Version=v4.8"/>
  </startup>
</configuration>
```
