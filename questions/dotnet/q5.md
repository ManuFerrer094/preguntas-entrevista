---
title: ¿Cuál es la diferencia entre LINQ y expresiones Lambda en .NET?
difficulty: medium
tags: [dotnet, linq, lambda, consultas]
---

**LINQ (Language Integrated Query)** y las **expresiones Lambda** son dos formas relacionadas pero distintas de trabajar con colecciones de datos en .NET:

**LINQ (sintaxis de consulta):**
Es un metalenguaje integrado en C# que tiene una sintaxis similar a SQL. Es más legible para desarrolladores familiarizados con SQL.

```csharp
// Sintaxis LINQ de consulta
var resultado = from c in clientes
                where c.Edad > 18
                orderby c.Nombre
                select c;
```

**Expresiones Lambda:**
Son una forma más compacta y funcional de escribir lógica usando métodos de extensión. Resultan más concisas para operaciones simples y son preferidas en código moderno.

```csharp
// Sintaxis Lambda (método de extensión)
var resultado = clientes
    .Where(c => c.Edad > 18)
    .OrderBy(c => c.Nombre)
    .ToList();
```

**Diferencias clave:**
- Ambas producen el mismo resultado; LINQ es azúcar sintáctica que el compilador traduce a llamadas Lambda.
- Las expresiones Lambda son más concisas para operaciones cortas y encadenadas.
- La sintaxis LINQ es más legible para consultas complejas con múltiples joins y agrupaciones.
- Con Entity Framework, ambas generan la misma consulta SQL en la base de datos.
