---
title: ¿Qué diferencia hay entre .NET Framework y .NET moderno (.NET Core / .NET 8)?
difficulty: easy
tags: [dotnet, dotnet-framework, dotnet-core, dotnet8, multiplataforma]
---

**.NET Framework** es la plataforma original de Microsoft, lanzada en 2002, diseñada exclusivamente para **Windows**. Su versión más reciente es la 4.8 y está en modo de mantenimiento: no recibirá nuevas características.

**.NET moderno** (llamado .NET Core hasta la versión 3.1, y simplemente **.NET** a partir de la versión 5) es la reimplementación multiplataforma, de código abierto y con soporte activo. La versión actual con soporte a largo plazo (LTS) es **.NET 8**.

**Comparativa principal:**

| Característica | .NET Framework | .NET moderno (.NET 8) |
|---|---|---|
| Plataformas | Solo Windows | Windows, Linux, macOS |
| Código abierto | No | Sí |
| Rendimiento | Bueno | Muy superior (mejoras continuas) |
| Nuevas características | No (mantenimiento) | Sí (ciclo anual) |
| Soporte a largo plazo | Incluido en Windows | LTS cada 2 años |
| Tipo de apps | WinForms, WPF, ASP.NET clásico | WinForms*, WPF*, ASP.NET Core, MAUI, Blazor |
| Despliegue | GAC / instalador | Self-contained, contenedores Docker |

> \* WinForms y WPF también están disponibles en .NET moderno, pero solo en Windows.

**¿Cuándo usar cada uno?**

- Usa **.NET Framework** si mantienes una aplicación legada que depende de componentes exclusivos (COM, WCF completo, WebForms) y no es viable migrar.
- Usa **.NET moderno** para cualquier proyecto nuevo: ofrece mejor rendimiento, soporte activo y la posibilidad de ejecutar en múltiples plataformas.

```csharp
// Comprobar la versión en tiempo de ejecución
Console.WriteLine(Environment.Version);            // e.g. 8.0.x
Console.WriteLine(RuntimeInformation.FrameworkDescription); // .NET 8.0.x
```
