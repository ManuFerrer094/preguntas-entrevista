---
title: ¿Cómo funciona el Razor Class Library (RCL) y cuándo conviene usarla?
difficulty: hard
tags: [razor, rcl, razor-class-library, componentes-compartidos, aspnet-core]
---

Una **Razor Class Library (RCL)** es un proyecto de tipo biblioteca que puede contener vistas Razor, Pages, View Components, Tag Helpers, archivos estáticos y código C#, todo empaquetable como un NuGet reutilizable entre múltiples aplicaciones ASP.NET Core.

**Casos de uso:**

- Componentes de UI corporativos compartidos entre proyectos (cabeceras, footers, formularios de login).
- Paquetes de funcionalidad UI de terceros (ej.: Identity UI de ASP.NET Core usa una RCL).
- Microfrontends o módulos de aplicación grandes.

**Crear una RCL:**

```bash
dotnet new razorclasslib -n MiComponentesUI
```

**Estructura típica:**

```
MiComponentesUI/
  wwwroot/        ← archivos estáticos (CSS, JS, imágenes)
  Pages/          ← Razor Pages
  Views/          ← Vistas MVC
  TagHelpers/
  ViewComponents/
```

**Consumir la RCL en la app principal:**

```xml
<!-- .csproj de la app -->
<ProjectReference Include="../MiComponentesUI/MiComponentesUI.csproj" />
```

Los archivos estáticos de la RCL se sirven automáticamente bajo la ruta `/_content/{NombreEnsamblado}/`.

**Sobreescribir vistas de una RCL:**

Si la app principal tiene una vista en la misma ruta que la RCL, la vista de la app toma precedencia (override). Esto permite personalizar componentes sin modificar la librería.

**RCL con Blazor:**

Las RCL también pueden contener componentes Blazor (`.razor`), siendo el mecanismo estándar para compartir componentes Blazor entre proyectos.
