---
title: ¿Cuál es la diferencia entre @{ } y @expression en Razor?
difficulty: easy
tags: [razor, sintaxis, expresiones, bloques]
---

En Razor existen dos formas principales de insertar código C#:

**1. Expresiones implícitas (`@expression`):**

Se usan para evaluar y renderizar el resultado directamente en el HTML. No llevan llaves y el parser de Razor detecta automáticamente dónde termina la expresión.

```razor
<p>Hola, @Model.UserName</p>
<p>Fecha: @DateTime.Now.ToShortDateString()</p>
```

**2. Bloques de código (`@{ }`):**

Permiten ejecutar código C# sin renderizar salida directamente. Se usan para declarar variables, bucles, condiciones u otras sentencias.

```razor
@{
    var titulo = "Bienvenido";
    var esAdmin = User.IsInRole("Admin");
}
<h1>@titulo</h1>
@if (esAdmin)
{
    <p>Eres administrador.</p>
}
```

**Regla clave:** dentro de un bloque `@{ }`, el texto plano NO se renderiza automáticamente; hay que usar `@:` o la etiqueta `<text>` para emitir HTML/texto:

```razor
@{
    var items = new[] { "A", "B", "C" };
    foreach (var i in items)
    {
        @: Elemento: @i <br/>
    }
}
```

**Expresiones complejas** (que contienen espacios u operadores) se encierran en paréntesis:

```razor
<p>@(Model.Precio * 1.21) €</p>
```
