---
title: ¿Cómo funciona el model binding en Razor Pages y ASP.NET MVC con [BindProperty] y [FromForm]?
difficulty: hard
tags: [razor, model-binding, bindproperty, fromform, aspnet-core]
---

El **model binding** es el proceso por el cual ASP.NET Core mapea automáticamente los datos de la petición HTTP (query string, body de formulario, route values, JSON) a parámetros de métodos o propiedades de modelos.

**En Razor Pages — `[BindProperty]`:**

```csharp
public class ContactoModel : PageModel
{
    [BindProperty]
    public ContactoInput Input { get; set; }

    // Solo bindea en POST/PUT/PATCH (no en GET por defecto)
    public IActionResult OnPost()
    {
        if (!ModelState.IsValid) return Page();
        // Usar Input...
        return RedirectToPage("Gracias");
    }
}

public class ContactoInput
{
    [Required] public string Nombre { get; set; }
    [EmailAddress] public string Email { get; set; }
}
```

Para bindear también en GET: `[BindProperty(SupportsGet = true)]`

**En MVC — parámetros de action:**

```csharp
[HttpPost]
public IActionResult Guardar([FromForm] ProductoViewModel model)
{
    // FromForm: datos de formulario (Content-Type: multipart o urlencoded)
    // FromBody: JSON/XML
    // FromQuery: query string
    // FromRoute: segmentos de ruta
    // FromHeader: cabeceras HTTP
}
```

**Convenciones de binding por nombre:**

El binder busca valores en este orden: `Form fields` → `Route values` → `Query string`. El nombre del campo HTML debe coincidir con el nombre de la propiedad (case-insensitive):

```razor
<input asp-for="Input.Nombre" /> @* genera name="Input.Nombre" *@
```

**Tipos complejos anidados:** el binder usa la notación de punto (`Input.Direccion.Calle`) para poblar objetos anidados automáticamente.
