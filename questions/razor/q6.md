---
title: ¿Cómo previene Razor los ataques XSS y cuándo debes usar Html.Raw?
difficulty: medium
tags: [razor, xss, seguridad, html-raw, encoding]
---

Por defecto, **Razor codifica automáticamente en HTML** cualquier valor que se renderiza con `@expression`. Esto previene ataques de Cross-Site Scripting (XSS) convirtiendo caracteres peligrosos como `<`, `>`, `"` y `&` en sus entidades HTML.

**Ejemplo de protección automática:**

```csharp
// Si Model.Comentario = "<script>alert('xss')</script>"
```

```razor
<p>@Model.Comentario</p>
```

Renderiza como texto escapado:

```html
<p>&lt;script&gt;alert(&#x27;xss&#x27;)&lt;/script&gt;</p>
```

**`Html.Raw()` — uso controlado:**

`Html.Raw()` desactiva la codificación y emite el HTML tal cual. **Solo debe usarse cuando la fuente es de confianza**, por ejemplo, HTML generado por el propio servidor o sanitizado previamente:

```razor
@* Peligroso si Descripcion viene del usuario sin sanear *@
@Html.Raw(Model.DescripcionHtml)
```

**Buenas prácticas de seguridad:**

- Nunca uses `Html.Raw()` con datos directamente del usuario.
- Si necesitas renderizar HTML del usuario, sanitízalo primero con una librería como **HtmlSanitizer** (NuGet).
- Usa `@` (encoding por defecto) siempre que sea posible.
- Para JSON incrustado en `<script>`, usa `Json.Serialize()` que también escapa correctamente:

```razor
<script>
    var data = @Json.Serialize(Model.Datos);
</script>
```

> El encoding automático de Razor es una de las principales razones por las que es más seguro que la concatenación manual de strings HTML.
