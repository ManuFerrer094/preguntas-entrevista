title: ¿Qué son las Razor Pages y en qué se diferencian del patrón MVC?

**Razor Pages** es un modelo de programación introducido en ASP.NET Core 2.0 que organiza la lógica de UI en torno a páginas individuales en lugar de seguir el flujo Controller → Action → View del patrón MVC clásico.

**Estructura de una Razor Page:**

Cada página consta de dos archivos:

- `Index.cshtml` — La vista Razor con directiva `@page`.
- `Index.cshtml.cs` — El **PageModel**, clase C# con los manejadores de petición.

```csharp
// Index.cshtml.cs
public class IndexModel : PageModel
{
    [BindProperty]
    public string Nombre { get; set; }

    public void OnGet()
    {
        // Se ejecuta en GET
    }

    public IActionResult OnPost()
    {
        if (!ModelState.IsValid) return Page();
        return RedirectToPage("Confirmacion");
    }
}
```

```razor
@* Index.cshtml *@
@page
@model IndexModel

<form method="post">
    <input asp-for="Nombre" />
    <button type="submit">Enviar</button>
</form>
```

**Comparativa MVC vs Razor Pages:**

| Aspecto | MVC | Razor Pages |
|---|---|---|
| Organización | Por función (Controladores/Vistas) | Por página/funcionalidad |
| Código repetitivo | Mayor (routing explícito) | Menor |
| Ideal para | APIs + UI complejas | Formularios CRUD sencillos |
| Convención de ruta | Atributo o convención | Ruta basada en ruta de archivo |

Razor Pages aplica el principio de **responsabilidad única** al nivel de página y es la opción recomendada para aplicaciones orientadas a formularios.
