---
title: ¿Qué es el Anti-Forgery Token en Razor y cómo protege contra ataques CSRF?
difficulty: hard
tags: [razor, csrf, antiforgery, seguridad, aspnet-core]
---

El **Cross-Site Request Forgery (CSRF)** es un ataque en el que un sitio malicioso provoca que el navegador de un usuario autenticado envíe peticiones no deseadas a tu aplicación. ASP.NET Core lo mitiga mediante **Anti-Forgery Tokens**.

**Mecanismo:**

1. Al renderizar el formulario, el servidor genera un token único vinculado a la sesión/usuario.
2. El token se incluye como campo oculto en el formulario y también en una cookie.
3. Al enviar el formulario (POST), el servidor verifica que ambos tokens coincidan.
4. Si un sitio externo forja la petición, no tendrá acceso a la cookie del dominio legítimo → ataque bloqueado.

**En Razor, el Tag Helper `<form>` con `asp-action` inyecta el token automáticamente:**

```razor
<form asp-action="Guardar" method="post">
    @* El token se añade automáticamente como campo oculto *@
    <input asp-for="Nombre" />
    <button type="submit">Guardar</button>
</form>
```

**Si usas `Html.BeginForm()` o un `<form>` sin Tag Helpers:**

```razor
<form method="post">
    @Html.AntiForgeryToken()
    ...
</form>
```

**Validación en el servidor:**

```csharp
// En el controlador — verifica automáticamente con el atributo:
[ValidateAntiForgeryToken]
[HttpPost]
public IActionResult Guardar(MiModelo model) { ... }

// Aplicar globalmente a todos los formularios POST:
builder.Services.AddControllersWithViews(options =>
{
    options.Filters.Add(new AutoValidateAntiforgeryTokenAttribute());
});
```

**Razor Pages** valida el token automáticamente en todos los handlers POST/PUT/PATCH/DELETE sin necesidad de atributo adicional.

**Para peticiones AJAX (SPA + API):** se puede leer el token de la cookie `XSRF-TOKEN` y enviarlo como cabecera `X-XSRF-TOKEN`.
