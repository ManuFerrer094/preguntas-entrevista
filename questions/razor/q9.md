---
title: ¿Qué es un View Component en Razor y cuándo debes usarlo en lugar de un Partial View?
difficulty: hard

---

Un **View Component** es similar a un Partial View pero con su propia lógica de negocio encapsulada, soporte para inyección de dependencias y un ciclo de invocación asíncrono. Se introdujo en ASP.NET Core como reemplazo de los Child Actions de MVC 5.

**Cuándo usar View Component vs Partial View:**

| Situación | Solución recomendada |
|---|---|
| Solo renderizar datos que ya tiene la vista | Partial View |
| Necesita consultar la BD o usar servicios | View Component |
| Reusable con lógica independiente (carrito, notificaciones) | View Component |
| Renderizado condicional complejo | View Component |

**Implementación:**

```csharp
// ViewComponents/CarritoViewComponent.cs
public class CarritoViewComponent : ViewComponent
{
    private readonly ICarritoService _carritoService;

    public CarritoViewComponent(ICarritoService carritoService)
    {
        _carritoService = carritoService;
    }

    public async Task<IViewComponentResult> InvokeAsync()
    {
        var items = await _carritoService.ObtenerItemsAsync(HttpContext.User);
        return View(items);
    }
}
```

**Vista del componente** (`Views/Shared/Components/Carrito/Default.cshtml`):

```razor
@model IEnumerable<CarritoItem>
<div class="carrito-badge">@Model.Count() artículos</div>
```

**Invocación desde una vista o layout:**

```razor
@* Forma de Tag Helper (preferida) *@
<vc:carrito></vc:carrito>

@* Forma programática *@
@await Component.InvokeAsync("Carrito")
```

Para usar la sintaxis Tag Helper hay que registrar el ensamblado en `_ViewImports.cshtml`:

```razor
@addTagHelper *, MiApp
```
