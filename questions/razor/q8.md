---
title: ¿Cómo se realiza la validación de formularios en Razor con Data Annotations y ValidationSummary?
difficulty: medium
tags: [razor, validacion, data-annotations, model-state, tag-helpers]
---

ASP.NET Core integra un sistema de validación basado en **Data Annotations** en el modelo, **ModelState** en el servidor y helpers de Razor para mostrar errores en el cliente.

**1. Anotar el modelo:**

```csharp
public class RegistroViewModel
{
    [Required(ErrorMessage = "El nombre es obligatorio")]
    [StringLength(50)]
    public string Nombre { get; set; }

    [Required]
    [EmailAddress(ErrorMessage = "Email inválido")]
    public string Email { get; set; }

    [Range(18, 99, ErrorMessage = "Debe tener entre 18 y 99 años")]
    public int Edad { get; set; }
}
```

**2. Vista Razor con validación:**

```razor
@model RegistroViewModel

<form asp-action="Registrar" method="post">
    <div asp-validation-summary="ModelOnly" class="text-danger"></div>

    <div>
        <label asp-for="Nombre"></label>
        <input asp-for="Nombre" class="form-control" />
        <span asp-validation-for="Nombre" class="text-danger"></span>
    </div>

    <div>
        <label asp-for="Email"></label>
        <input asp-for="Email" class="form-control" />
        <span asp-validation-for="Email" class="text-danger"></span>
    </div>

    <button type="submit">Registrar</button>
</form>

@section Scripts {
    @{await Html.RenderPartialAsync("_ValidationScriptsPartial");}
}
```

**3. Controlador:**

```csharp
[HttpPost]
public IActionResult Registrar(RegistroViewModel model)
{
    if (!ModelState.IsValid)
        return View(model);

    // Guardar...
    return RedirectToAction("Exito");
}
```

**Validación del lado del cliente:** `jquery.validate` y `jquery.validate.unobtrusive` activan la validación en el navegador sin roundtrip al servidor, mejorando la UX. Se incluyen mediante `_ValidationScriptsPartial`.
