---
title: ¿Qué son los Filters en ASP.NET MVC/Razor Pages y cómo se usan para lógica transversal?
difficulty: hard
tags: [razor, filters, action-filters, authorization, aspnet-core]
---

Los **Filters** en ASP.NET Core son clases que se ejecutan en puntos específicos del pipeline de procesamiento de una petición, antes o después de que se ejecuten las actions/handlers. Implementan el patrón AOP (Aspect-Oriented Programming).

**Tipos de filtros (por orden de ejecución):**

1. **Authorization Filters** — primeros en ejecutarse; comprueban si el usuario tiene permiso.
2. **Resource Filters** — envuelven el resto del pipeline; útiles para caché.
3. **Action Filters** — se ejecutan antes/después de la action del controlador.
4. **Exception Filters** — capturan excepciones no controladas.
5. **Result Filters** — se ejecutan antes/después de serializar el resultado.

**Crear un Action Filter personalizado:**

```csharp
public class LogActionFilter : IActionFilter
{
    private readonly ILogger<LogActionFilter> _logger;

    public LogActionFilter(ILogger<LogActionFilter> logger)
    {
        _logger = logger;
    }

    public void OnActionExecuting(ActionExecutingContext context)
    {
        _logger.LogInformation("Ejecutando: {Action}", context.ActionDescriptor.DisplayName);
    }

    public void OnActionExecuted(ActionExecutedContext context)
    {
        _logger.LogInformation("Completado: {Action}", context.ActionDescriptor.DisplayName);
    }
}
```

**Registro y aplicación:**

```csharp
// Global (para todos los controladores)
builder.Services.AddControllersWithViews(options =>
{
    options.Filters.Add<LogActionFilter>();
});

// Por controlador o action (atributo)
[ServiceFilter(typeof(LogActionFilter))]
public IActionResult Index() => View();
```

**En Razor Pages** se usan `IPageFilter` o `IAsyncPageFilter` en lugar de `IActionFilter`:

```csharp
public class AuditPageFilter : IAsyncPageFilter
{
    public async Task OnPageHandlerExecutionAsync(
        PageHandlerExecutingContext context, PageHandlerExecutionDelegate next)
    {
        // Antes del handler
        var result = await next();
        // Después del handler
    }
}
```
