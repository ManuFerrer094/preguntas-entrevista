---
title: ¿Qué es el Tag Helper en Razor y en qué se diferencia de los HTML Helpers?
difficulty: medium
tags: [razor, taghelpers, htmlhelpers, aspnet-core]
---

**HTML Helpers** son métodos estáticos de C# que generan HTML como strings y se invocan con la sintaxis `@Html.*`:

```razor
@Html.LabelFor(m => m.Email)
@Html.TextBoxFor(m => m.Email, new { @class = "form-control" })
@Html.ValidationMessageFor(m => m.Email)
```

**Tag Helpers** (ASP.NET Core) son clases C# que se adjuntan a elementos HTML estándar, haciendo el código más parecido al HTML nativo y mejorando la legibilidad e IntelliSense:

```razor
<label asp-for="Email"></label>
<input asp-for="Email" class="form-control" />
<span asp-validation-for="Email"></span>
```

**Diferencias clave:**

| Aspecto | HTML Helpers | Tag Helpers |
|---|---|---|
| Sintaxis | Métodos C# inline | Atributos HTML (`asp-*`) |
| Legibilidad | Menor | Mayor (HTML puro) |
| IntelliSense | Parcial | Completo para atributos |
| Disponibilidad | MVC 4 / MVC 5 / Core | Solo ASP.NET Core |
| Extensibilidad | Métodos de extensión | Clases que heredan `TagHelper` |

Para activar los Tag Helpers en una vista Razor se añade:

```razor
@addTagHelper *, Microsoft.AspNetCore.Mvc.TagHelpers
```

Habitualmente esto se coloca en `_ViewImports.cshtml` para que esté disponible en todas las vistas.

**Tag Helper personalizado:**

```csharp
[HtmlTargetElement("email")]
public class EmailTagHelper : TagHelper
{
    public string Address { get; set; }
    public override void Process(TagHelperContext context, TagHelperOutput output)
    {
        output.TagName = "a";
        output.Attributes.SetAttribute("href", $"mailto:{Address}");
        output.Content.SetContent(Address);
    }
}
```
