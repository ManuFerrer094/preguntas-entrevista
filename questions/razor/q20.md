---
title: ¿Cuál es la diferencia entre Razor sintaxis en MVC Views, Razor Pages y Blazor?
difficulty: hard
tags: [razor, blazor, mvc, razor-pages, componentes]
---

Razor es un motor de plantillas compartido en varios modelos de programación de ASP.NET Core, pero su uso y características varían significativamente según el contexto.

**Comparativa:**

| Característica | MVC Views (.cshtml) | Razor Pages (.cshtml) | Blazor (.razor) |
|---|---|---|---|
| Directiva inicial | — | `@page` | `@page` (opcional) |
| Clase base | `RazorPage<TModel>` | `PageModel` | `ComponentBase` |
| Ejecución | Servidor (SSR) | Servidor (SSR) | Server o WebAssembly |
| Interactividad | HTML estático | HTML estático | DOM reactivo |
| Binding bidireccional | No nativo | `[BindProperty]` | `@bind` |
| Eventos de usuario | No (requiere JS) | No (requiere JS) | `@onclick`, `@oninput`… |
| Ciclo de vida | — | `OnGet`, `OnPost`… | `OnInitializedAsync`, `OnParametersSet`… |

**Razor en Blazor — características exclusivas:**

```razor
@* Componente Blazor *@
@page "/contador"

<h1>Contador: @contador</h1>
<button @onclick="Incrementar">+1</button>

@code {
    private int contador = 0;

    private void Incrementar() => contador++;
}
```

**Directivas exclusivas de Blazor:**

- `@code { }` — define el bloque C# del componente.
- `@bind` — two-way data binding.
- `@onclick`, `@onchange`… — event handlers.
- `@ref` — referencia al elemento o componente.
- `@inject` — inyección de dependencias.
- `[Parameter]` — propiedades recibidas del componente padre.
- `[CascadingParameter]` — parámetros en cascada desde un ancestro.

**Razor en MVC/Pages:** sintaxis puramente de plantilla sin interactividad; la UI reactiva se logra con JavaScript (por ej., AJAX, Alpine.js, HTMX).
