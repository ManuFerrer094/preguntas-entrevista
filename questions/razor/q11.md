---
title: ¿Cómo se implementan y gestionan las Areas en ASP.NET MVC con Razor?
difficulty: hard
tags: [razor, areas, mvc, routing, aspnet-core]
---

Las **Areas** permiten dividir una aplicación ASP.NET MVC grande en módulos funcionales con sus propios controladores, vistas y modelos, evitando conflictos de nombres y mejorando la organización del proyecto.

**Estructura de directorios:**

```
Areas/
  Admin/
    Controllers/
      DashboardController.cs
    Views/
      Dashboard/
        Index.cshtml
      Shared/
        _Layout.cshtml
      _ViewImports.cshtml
      _ViewStart.cshtml
  Tienda/
    Controllers/
      ProductosController.cs
    Views/
      ...
```

**Registrar el área en el controlador:**

```csharp
[Area("Admin")]
public class DashboardController : Controller
{
    public IActionResult Index() => View();
}
```

**Configurar las rutas de área en `Program.cs`:**

```csharp
app.MapControllerRoute(
    name: "areas",
    pattern: "{area:exists}/{controller=Home}/{action=Index}/{id?}");

app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");
```

**Generar URLs hacia un área:**

```razor
@* Desde dentro del área *@
<a asp-controller="Dashboard" asp-action="Index">Admin</a>

@* Desde fuera del área — especificar area explícitamente *@
<a asp-area="Admin" asp-controller="Dashboard" asp-action="Index">Panel Admin</a>

@* Sin area (ruta raíz) *@
<a asp-area="" asp-controller="Home" asp-action="Index">Inicio</a>
```

**Resolución de vistas:** Razor busca las vistas en el área antes que en la carpeta `Views` global, lo que permite override de vistas compartidas a nivel de área.
