---
title: ¿Cómo se usa TempData en Razor para pasar mensajes entre redirecciones?
difficulty: medium
tags: [razor, tempdata, viewdata, viewbag, redirecciones]
---

**TempData** es un diccionario especial en ASP.NET Core / MVC que persiste datos durante **una sola petición adicional** (típicamente un redirect). Se usa para pasar mensajes de éxito, error o advertencia tras operaciones POST → Redirect → GET.

**Cuándo usar cada mecanismo:**

| Mecanismo | Duración | Uso típico |
|---|---|---|
| `ViewData` / `ViewBag` | Solo la petición actual | Datos de la vista actual |
| `TempData` | Petición actual + 1 (tras redirect) | Mensajes post-acción |
| `Session` | Toda la sesión del usuario | Estado persistente |

**Ejemplo en controlador:**

```csharp
[HttpPost]
public IActionResult Guardar(ProductoViewModel model)
{
    if (!ModelState.IsValid) return View(model);

    _db.Productos.Add(model);
    _db.SaveChanges();

    TempData["Exito"] = "Producto guardado correctamente.";
    return RedirectToAction("Index");
}

public IActionResult Index()
{
    // TempData["Exito"] sigue disponible aquí tras el redirect
    return View();
}
```

**Consumo en la vista Razor:**

```razor
@if (TempData["Exito"] is string msg)
{
    <div class="alert alert-success">@msg</div>
}
```

**`TempData.Keep()` y `TempData.Peek()`:**

```csharp
// Peek: leer sin marcar para eliminación
var msg = TempData.Peek("Exito") as string;

// Keep: marcar para que persista una petición más
TempData.Keep("Exito");
```

**Proveedor de TempData:** por defecto usa cookies. Para aplicaciones con múltiples servidores, configura el proveedor basado en sesión (`AddSessionStateTempDataProvider`).
