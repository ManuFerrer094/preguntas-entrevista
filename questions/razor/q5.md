---
title: ¿Cómo funciona el Partial View en Razor y cuándo debes usar _ViewStart y _ViewImports?
difficulty: medium
tags: [razor, partial-views, layout, viewstart, viewimports]
---

**Partial Views** son vistas Razor reutilizables que se renderizan dentro de otra vista. Son útiles para fragmentos HTML repetibles como menús, tarjetas de productos o formularios compartidos.

**Renderizar un partial:**

```razor
@* Forma moderna (ASP.NET Core) — evita duplicación de ViewData *@
<partial name="_ProductCard" model="@product" />

@* Con HTML Helper (también válido) *@
@await Html.PartialAsync("_ProductCard", product)
```

**Archivo `_ViewStart.cshtml`:**

Se ejecuta **antes** de cualquier vista en su directorio o subdirectorios. Típicamente define el layout por defecto:

```razor
@{
    Layout = "_Layout";
}
```

Si una vista no quiere layout: `Layout = null;`

**Archivo `_ViewImports.cshtml`:**

Centraliza directivas que deben estar disponibles en todas las vistas del directorio:

```razor
@using MiApp.Models
@using MiApp.ViewModels
@addTagHelper *, Microsoft.AspNetCore.Mvc.TagHelpers
@addTagHelper *, MiApp
```

**Jerarquía de búsqueda:** Razor busca `_ViewStart.cshtml` y `_ViewImports.cshtml` desde el directorio de la vista hacia arriba hasta la raíz (`/Views`). Esto permite configuraciones específicas por área o controlador.

**View Components** (alternativa avanzada a Partial Views):

Para parciales que necesitan lógica propia (inyección de dependencias, consultas a base de datos) se usa `ViewComponent` en lugar de un simple partial.
