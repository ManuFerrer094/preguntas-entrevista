---
title: ¿Cómo se maneja la internacionalización (i18n) y localización en vistas Razor?
difficulty: hard
tags: [razor, i18n, localizacion, resx, aspnet-core]
---

ASP.NET Core proporciona un sistema de localización basado en archivos `.resx` y la interfaz `IStringLocalizer` que se integra directamente con las vistas Razor.

**Configuración en `Program.cs`:**

```csharp
builder.Services.AddLocalization(options => options.ResourcesPath = "Resources");
builder.Services.AddControllersWithViews()
    .AddViewLocalization(LanguageViewLocationExpanderFormat.Suffix);

var supportedCultures = new[] { "es", "en", "fr" };
app.UseRequestLocalization(options =>
{
    options.SetDefaultCulture("es")
           .AddSupportedCultures(supportedCultures)
           .AddSupportedUICultures(supportedCultures);
});
```

**Estructura de archivos de recursos:**

```
Resources/
  Views/Home/Index.es.resx
  Views/Home/Index.en.resx
  Controllers/HomeController.es.resx
  Controllers/HomeController.en.resx
```

**Uso en vistas Razor con `IViewLocalizer`:**

```razor
@inject Microsoft.AspNetCore.Mvc.Localization.IViewLocalizer Localizer

<h1>@Localizer["Bienvenido"]</h1>
<p>@Localizer["TotalProductos", Model.Cantidad]</p>
```

**Uso en controladores con `IStringLocalizer`:**

```csharp
public class HomeController : Controller
{
    private readonly IStringLocalizer<HomeController> _localizer;

    public HomeController(IStringLocalizer<HomeController> localizer)
    {
        _localizer = localizer;
    }

    public IActionResult Index()
    {
        ViewData["Mensaje"] = _localizer["Bienvenido"];
        return View();
    }
}
```

**Selector de idioma:** la cultura se puede determinar por query string (`?culture=en`), cookie o cabecera `Accept-Language`. Se configura con `RequestCultureProviders`.
