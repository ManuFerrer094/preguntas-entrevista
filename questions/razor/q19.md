---
title: ¿Cómo se optimiza el rendimiento de renderizado en Razor con Output Caching y Response Caching?
difficulty: hard
tags: [razor, output-caching, response-caching, rendimiento, aspnet-core]
---

Cachear respuestas de vistas Razor reduce drásticamente la carga del servidor para contenido que no cambia frecuentemente. ASP.NET Core ofrece dos enfoques complementarios.

**1. Response Caching (HTTP level):**

Añade cabeceras HTTP `Cache-Control` para que navegadores y proxies intermedios cacheen la respuesta:

```csharp
// Program.cs
builder.Services.AddResponseCaching();
app.UseResponseCaching();
```

```csharp
[ResponseCache(Duration = 60, VaryByQueryKeys = new[] { "page" })]
public IActionResult Catalogo(int page = 1) => View();
```

**Limitación:** no cachea en el servidor; depende del cliente/proxy.

**2. Output Caching (ASP.NET Core 7+, server-side):**

```csharp
// Program.cs
builder.Services.AddOutputCache(options =>
{
    options.AddPolicy("Catalogo", policy =>
        policy.Expire(TimeSpan.FromMinutes(5))
              .SetVaryByQuery("page"));
});
app.UseOutputCache();
```

```csharp
[OutputCache(PolicyName = "Catalogo")]
public IActionResult Catalogo(int page = 1) => View();
```

**Invalidación de caché:**

```csharp
// Inyectar IOutputCacheStore y llamar:
await _cacheStore.EvictByTagAsync("catalogo-tag", cancellationToken);
```

**Cache de fragmentos en Razor (Cache Tag Helper):**

Para cachear solo parte de una vista sin cachear toda la respuesta:

```razor
<cache expires-after="@TimeSpan.FromMinutes(10)" vary-by-user="false">
    @* Contenido costoso de generar *@
    @foreach (var p in Model.ProductosDestacados)
    {
        <partial name="_ProductCard" model="p" />
    }
</cache>
```

El `<cache>` Tag Helper usa `IMemoryCache` internamente. Para entornos distribuidos (múltiples servidores), usa `<distributed-cache>` que se apoya en `IDistributedCache` (Redis, SQL Server).
