---
title: ¿Qué es el Layout en Razor y cómo se implementa una jerarquía de layouts?
difficulty: medium
tags: [razor, layout, secciones, renderBody, aspnet-core]
---

Un **Layout** en Razor es una plantilla maestra que define la estructura HTML común (cabecera, menú, pie de página) que comparten múltiples vistas. Funciona de forma similar a un template padre en otros frameworks.

**Definición del layout (`_Layout.cshtml`):**

```razor
<!DOCTYPE html>
<html>
<head>
    <title>@ViewData["Title"] - Mi App</title>
    @await RenderSectionAsync("Styles", required: false)
</head>
<body>
    <nav><!-- menú --></nav>

    <main>
        @RenderBody()
    </main>

    <footer>© 2025</footer>

    @RenderSection("Scripts", required: false)
</body>
</html>
```

- `@RenderBody()` — obligatorio; inserta el contenido de la vista hija.
- `@RenderSection("nombre", required: false)` — sección opcional que la vista hija puede rellenar.

**Vista hija usando el layout:**

```razor
@{
    Layout = "_Layout";
    ViewData["Title"] = "Inicio";
}

<h1>Bienvenido</h1>

@section Scripts {
    <script src="~/js/home.js"></script>
}
```

**Jerarquía de layouts (nested layouts):**

Un layout puede a su vez heredar de otro layout. Esto permite layouts de área o sección que extienden el layout principal:

```razor
@{
    Layout = "_Layout"; // El layout de área hereda del layout global
}
@RenderBody()
```

**`_ViewStart.cshtml`** evita repetir la asignación `Layout = "_Layout"` en cada vista; cualquier vista que no especifique layout usará el definido en `_ViewStart.cshtml`.
