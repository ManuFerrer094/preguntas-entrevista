---
title: ¿Qué es el hook useState y cómo funciona internamente?
difficulty: easy
tags: [react, hooks, useState, state]
---

`useState` es el hook fundamental de React para añadir **estado local** a componentes funcionales. Retorna un par: el valor actual del estado y una función para actualizarlo, que provoca un re-render del componente.

## Uso básico

```jsx
import { useState } from 'react';

function Contador() {
  const [contador, setContador] = useState(0);  // valor inicial: 0

  return (
    <div>
      <p>Contador: {contador}</p>
      <button onClick={() => setContador(contador + 1)}>Incrementar</button>
      <button onClick={() => setContador(0)}>Resetear</button>
    </div>
  );
}
```

## Actualización funcional (basada en estado anterior)

```jsx
// ❌ Puede tener condiciones de carrera en batching:
setContador(contador + 1);

// ✅ Actualización funcional: siempre usa el valor más reciente
setContador(prev => prev + 1);

// Ejemplo donde importa la diferencia:
function incrementarTres() {
  // Las 3 llamadas ven el mismo `contador` del render actual
  setContador(contador + 1);  // → contador + 1
  setContador(contador + 1);  // → contador + 1 (¡no +2!)
  setContador(contador + 1);  // → contador + 1 (¡no +3!)

  // Con actualización funcional:
  setContador(prev => prev + 1);  // → contador + 1
  setContador(prev => prev + 1);  // → contador + 2 ✅
  setContador(prev => prev + 1);  // → contador + 3 ✅
}
```

## Estado con objetos y arrays

```jsx
function FormularioPerfil() {
  const [usuario, setUsuario] = useState({
    nombre: '',
    email: '',
    edad: 0
  });

  // ❌ Mutación directa (no provoca re-render):
  // usuario.nombre = 'Ana';

  // ✅ Spreado para crear nuevo objeto:
  const actualizarNombre = (nombre) => {
    setUsuario(prev => ({ ...prev, nombre }));
  };

  // Actualizar array: siempre crear un nuevo array
  const [tareas, setTareas] = useState([]);

  const agregarTarea = (texto) => {
    setTareas(prev => [...prev, { id: Date.now(), texto, completada: false }]);
  };

  const eliminarTarea = (id) => {
    setTareas(prev => prev.filter(t => t.id !== id));
  };

  const toggleTarea = (id) => {
    setTareas(prev =>
      prev.map(t => t.id === id ? { ...t, completada: !t.completada } : t)
    );
  };
}
```

## Inicialización perezosa (lazy initialization)

```jsx
// ❌ Se ejecuta en cada render (aunque no se use después del primero):
const [estado, setEstado] = useState(calcularEstadoInicial());

// ✅ Solo se ejecuta en el primer render (lazy initialization):
const [estado, setEstado] = useState(() => calcularEstadoInicial());

// Caso real: leer de localStorage
function useLocalStorage(key, valorPorDefecto) {
  const [valor, setValor] = useState(() => {
    try {
      const item = localStorage.getItem(key);
      return item ? JSON.parse(item) : valorPorDefecto;
    } catch {
      return valorPorDefecto;
    }
  });

  const guardar = (nuevoValor) => {
    setValor(nuevoValor);
    localStorage.setItem(key, JSON.stringify(nuevoValor));
  };

  return [valor, guardar];
}
```

## Batching automático (React 18+)

```jsx
// En React 18, múltiples setStates en cualquier contexto se agrupan en un solo re-render
async function handleClick() {
  const datos = await fetchDatos();

  // React 18: solo UN re-render (batch automático)
  setNombre(datos.nombre);
  setEmail(datos.email);
  setLoading(false);

}
```

## Cómo funciona internamente

```
Primer render:
  useState(0) → React crea una celda en la "fiber" del componente
              → Devuelve [0, setter]

Llamar a setter(1):
  → React encola una actualización de estado
  → Planifica un nuevo render
  → En el nuevo render, useState() devuelve [1, setter]

React identifica el estado por ORDEN de llamada de hooks:
  1ª llamada useState → celda 0
  2ª llamada useState → celda 1
  3ª llamada useState → celda 2
  → Por eso los hooks NO pueden estar dentro de if/for/funciones
```

**Consejo de entrevista**: La regla más importante de `useState`: el estado es inmutable —siempre crea nuevas referencias para objetos y arrays. React usa `Object.is()` para comparar: si la referencia no cambia, no re-renderiza. Por eso mutaciones directas no funcionan. La inicialización perezosa con función es una optimización importante para valores costosos de calcular.
