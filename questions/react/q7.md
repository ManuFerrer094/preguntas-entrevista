---
title: ¿Qué es useMemo y useCallback?
difficulty: medium
tags: [react, hooks, performance, memoization]
---

`useMemo` y `useCallback` son hooks de React para optimizar el rendimiento mediante memoización, evitando cálculos y re-creaciones innecesarias en cada renderizado.

## useMemo

Memoiza el **resultado de un cálculo**. Solo recalcula cuando alguna de sus dependencias cambia. Ideal para operaciones computacionalmente costosas.

```jsx
import { useMemo, useState } from 'react';

function ProductList({ products, filter }) {
  // Sin useMemo: se filtra en CADA renderizado del componente
  // Con useMemo: solo se filtra cuando 'products' o 'filter' cambian
  const filteredProducts = useMemo(() => {
    console.log('Filtrando productos...');
    return products.filter(p =>
      p.name.toLowerCase().includes(filter.toLowerCase())
    );
  }, [products, filter]);

  return (
    <ul>
      {filteredProducts.map(p => <li key={p.id}>{p.name}</li>)}
    </ul>
  );
}
```

## useCallback

Memoiza la **definición de una función**. Devuelve la misma referencia de función entre renderizados, siempre que las dependencias no cambien. Esto es importante porque en JavaScript cada renderizado crea nuevas funciones, lo que rompe la igualdad por referencia.

```jsx
import { useCallback, useState, memo } from 'react';

// Componente hijo optimizado con React.memo
const Button = memo(({ onClick, label }) => {
  console.log(`Renderizando botón: ${label}`);
  return <button onClick={onClick}>{label}</button>;
});

function Counter() {
  const [count, setCount] = useState(0);
  const [other, setOther] = useState(0);

  // Sin useCallback: 'increment' sería una nueva función en cada render
  // Button se re-renderizaría aunque 'count' no haya cambiado
  const increment = useCallback(() => {
    setCount(c => c + 1);
  }, []); // Sin dependencias porque usa el updater funcional

  return (
    <div>
      <p>Count: {count}</p>
      <p>Other: {other}</p>
      <Button onClick={increment} label="Incrementar" />
      <button onClick={() => setOther(o => o + 1)}>Cambiar otro</button>
    </div>
  );
}
```

## Cuándo NO usarlos

Un error común es usarlos en todo, lo que añade complejidad sin beneficio real. Solo tiene sentido cuando:

- `useMemo`: el cálculo es genuinamente costoso (filtrar miles de elementos, cálculos matemáticos pesados).
- `useCallback`: la función se pasa a un componente hijo optimizado con `React.memo` o es una dependencia de otro hook.

```jsx
// ❌ Sobreoptimización innecesaria
const double = useMemo(() => count * 2, [count]); // trivial, no vale la pena

// ✅ Tiene sentido
const sortedData = useMemo(() =>
  [...hugeDataset].sort((a, b) => a.value - b.value),
  [hugeDataset]
);
```

**Consejo de entrevista**: React Compiler (antes conocido como React Forget), disponible desde React 19, puede memoizar automáticamente los componentes, reduciendo la necesidad de `useMemo` y `useCallback` manuales. Mencionar esto demuestra conocimiento actualizado del ecosistema.
