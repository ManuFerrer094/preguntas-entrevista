Server Side Rendering (SSR) es la técnica de generar el HTML de una aplicación React en el servidor antes de enviarlo al navegador. Esto mejora el tiempo de primera carga perceptible (FCP) y el SEO, ya que los bots de búsqueda reciben HTML completo en lugar de una página vacía que se llena con JavaScript.

Next.js es el framework más popular de React para SSR, y con la introducción de los **React Server Components** (RSC) en Next.js 13/14 con el App Router, el modelo ha evolucionado significativamente.

## Patrones de renderizado en Next.js

**1. Server Components (por defecto en App Router):**

```jsx
// app/users/page.tsx - Server Component por defecto
// Se ejecuta en el servidor, NO se envía JavaScript al cliente

async function UsersPage() {
  // Fetch directo sin useEffect ni useState
  const users = await fetch('https://api.example.com/users', {
    next: { revalidate: 3600 } // revalidar cada hora (ISR)
  }).then(r => r.json());

  return (
    <div>
      <h1>Usuarios</h1>
      {users.map(user => (
        <UserCard key={user.id} user={user} />
      ))}
    </div>
  );
}
```

**2. Client Components (cuando se necesita interactividad):**

```jsx
'use client'; // Directiva que marca el componente como cliente

import { useState } from 'react';

export function SearchBar({ onSearch }) {
  const [query, setQuery] = useState('');

  return (
    <input
      value={query}
      onChange={e => {
        setQuery(e.target.value);
        onSearch(e.target.value);
      }}
    />
  );
}
```

**3. Streaming con Suspense:**

```jsx
import { Suspense } from 'react';

export default function Dashboard() {
  return (
    <div>
      <h1>Dashboard</h1>
      {/* El header se muestra inmediatamente */}
      <Suspense fallback={<SkeletonStats />}>
        <Stats /> {/* Carga en streaming cuando esté listo */}
      </Suspense>
      <Suspense fallback={<SkeletonFeed />}>
        <ActivityFeed />
      </Suspense>
    </div>
  );
}
```

## Static Site Generation (SSG)

```jsx
// Página estática: se genera en build time
export default async function BlogPost({ params }) {
  const post = await getPost(params.slug);
  return <article>{post.content}</article>;
}

// generateStaticParams para pre-renderizar rutas específicas
export async function generateStaticParams() {
  const posts = await getAllPosts();
  return posts.map(post => ({ slug: post.slug }));
}
```

## Metadata y SEO

```jsx
// app/products/[id]/page.tsx
export async function generateMetadata({ params }) {
  const product = await getProduct(params.id);
  return {
    title: product.name,
    description: product.description,
    openGraph: {
      images: [product.image]
    }
  };
}
```

## Comparación de estrategias

| Estrategia | Cuándo usar |
|---|---|
| Static (SSG) | Contenido que no cambia (blog, documentación) |
| ISR (Incremental Static Regeneration) | Contenido que cambia con poca frecuencia |
| SSR dinámico | Contenido personalizado por usuario |
| Client-side | Dashboards, contenido muy interactivo |

**Consejo de entrevista**: La distinción entre Server Components y Client Components es fundamental en Next.js moderno. Los Server Components reducen el JavaScript enviado al cliente y permiten acceso directo a bases de datos y sistemas de archivos. Un error común es usar `'use client'` en exceso; la regla es "server first" y marcar como cliente solo lo que necesita interactividad.
