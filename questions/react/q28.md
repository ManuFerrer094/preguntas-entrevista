Los **React Server Components (RSC)** son componentes que se renderizan **exclusivamente en el servidor** y envían HTML serializado al cliente. No tienen JavaScript en el bundle del cliente, no tienen estado, no pueden usar hooks, pero pueden acceder directamente a bases de datos, sistema de archivos y APIs sin exponer credenciales.

## Server Component vs Client Component

```jsx
// ---- SERVER COMPONENT (por defecto en Next.js App Router) ----
// app/usuarios/page.tsx — Se ejecuta SOLO en el servidor

import { db } from '@/lib/database';  // ✅ Acceso directo a la BD

// async/await directo sin useEffect ni useState
export default async function PaginaUsuarios() {
  // Consulta a la BD sin exponer credenciales al cliente
  const usuarios = await db.query('SELECT * FROM usuarios LIMIT 50');

  return (
    <div>
      <h1>Usuarios ({usuarios.length})</h1>
      {usuarios.map(u => (
        <div key={u.id}>{u.nombre} — {u.email}</div>
      ))}
    </div>
  );
}
// Este componente NO añade nada al bundle JS del cliente
```

```jsx
// ---- CLIENT COMPONENT — requiere 'use client' ----
'use client';  // ← Directiva que lo convierte en Client Component

import { useState } from 'react';

export function BuscadorUsuarios({ usuarios }) {
  const [filtro, setFiltro] = useState('');

  const filtrados = usuarios.filter(u =>
    u.nombre.toLowerCase().includes(filtro.toLowerCase())
  );

  return (
    <div>
      <input
        value={filtro}
        onChange={e => setFiltro(e.target.value)}
        placeholder="Buscar..."
      />
      {filtrados.map(u => <div key={u.id}>{u.nombre}</div>)}
    </div>
  );
}
```

## Composición: pasar Server Components a Client Components

```jsx
// ✅ Pasar Server Components como children de Client Components
// app/layout.tsx (Server Component)
import { Sidebar } from './Sidebar';        // Client Component
import { ContenidoUsuarios } from './ContenidoUsuarios';  // Server Component

export default function Layout() {
  return (
    <Sidebar>
      {/* Server Component pasado como children al Client Component */}
      <ContenidoUsuarios />
    </Sidebar>
  );
}

// Sidebar.tsx (Client Component)
'use client';
import { useState } from 'react';

export function Sidebar({ children }) {
  const [abierto, setAbierto] = useState(true);

  return (
    <div className={abierto ? 'abierto' : 'cerrado'}>
      <button onClick={() => setAbierto(!abierto)}>Toggle</button>
      {children}  {/* Los Server Components se renderizan en el servidor */}
    </div>
  );
}
```

## Acceso a datos directamente en el servidor

```jsx
// app/producto/[id]/page.tsx
import { notFound } from 'next/navigation';

// Función que accede a la BD (solo existe en el servidor)
async function obtenerProducto(id: string) {
  const producto = await db.productos.findUnique({
    where: { id },
    include: { categoria: true, imagenes: true }
  });

  if (!producto) notFound();
  return producto;
}

// Server Component con datos de la BD
export default async function PaginaProducto({ params }) {
  const producto = await obtenerProducto(params.id);

  return (
    <article>
      <h1>{producto.nombre}</h1>
      <p>{producto.descripcion}</p>
      <span>{producto.precio}€</span>
      {/* Importar Client Component para interactividad */}
      <BotonAnadirCarrito productoId={producto.id} />
    </article>
  );
}
```

## Cuándo usar Server vs Client Components

```
¿Necesitas...?
├── Acceder a BD/FS/APIs sin exponer credenciales → Server Component
├── Reducir el bundle JS del cliente → Server Component
├── SEO óptimo sin JavaScript → Server Component
├── useState/useReducer → Client Component
├── useEffect/lifecycle → Client Component
├── Event listeners (onClick, onChange) → Client Component
├── APIs del navegador (localStorage, window) → Client Component
└── Contexto de React (useContext) → Client Component
```

## Diferencias clave

| Característica        | Server Component     | Client Component        |
|-----------------------|----------------------|-------------------------|
| `'use client'`        | No (default)         | ✅ Requerido            |
| JavaScript en bundle  | ❌ Cero              | ✅ Incluido             |
| `useState`/`useEffect`| ❌ No permitido      | ✅ Permitido            |
| Acceso directo a BD   | ✅ Sí               | ❌ Solo via API         |
| async/await en render | ✅ Nativo            | Solo con useEffect      |
| Event handlers        | ❌ No                | ✅ onClick, onChange... |
| Refresh del servidor  | Con `revalidatePath` | Con fetch/setState      |

**Consejo de entrevista**: Los RSC son el cambio más importante en React desde los hooks. La diferencia mental clave: los Server Components son para **datos y estructura** (sin interactividad), los Client Components son para **interactividad**. El patrón ideal es hacer todo lo posible como Server Component y usar Client Components solo en los "leaves" (hojas) del árbol de componentes donde necesitas interactividad.
