---
title: ¿Qué es React.forwardRef y cuándo usarlo?
difficulty: medium
tags: [react, forwardRef, refs, dom, components]
---

`React.forwardRef` permite a un componente **reenviar** la `ref` que recibe hacia un elemento DOM interno o hacia un componente hijo. Por defecto, las refs no se pasan a través de los componentes —`forwardRef` soluciona esto.

## El problema: las refs no se pasan automáticamente

```jsx
// ❌ Sin forwardRef: la ref apunta al componente, no al input interno
function InputBonito({ placeholder }) {
  return <input className="bonito" placeholder={placeholder} />;
}

function App() {
  const inputRef = useRef(null);

  // inputRef.current será null — la ref no llega al <input>
  return (
    <>
      <InputBonito ref={inputRef} placeholder="Escribe aquí" />
      <button onClick={() => inputRef.current?.focus()}>Focus</button>
    </>
  );
}
```

## Solución con forwardRef

```jsx
import { forwardRef, useRef } from 'react';

// El segundo argumento `ref` recibe la ref del componente padre
const InputBonito = forwardRef(function InputBonito({ placeholder, ...props }, ref) {
  return (
    <input
      ref={ref}          // ← La ref se reenvía al elemento DOM interno
      className="bonito"
      placeholder={placeholder}
      {...props}
    />
  );
});

function App() {
  const inputRef = useRef(null);

  return (
    <>
      <InputBonito
        ref={inputRef}   // ✅ Ahora sí llega al <input> interno
        placeholder="Escribe aquí"
      />
      <button onClick={() => inputRef.current?.focus()}>
        Enfocar input
      </button>
    </>
  );
}
```

## Componente de campo de formulario con forwardRef

```jsx
const CampoTexto = forwardRef(function CampoTexto(
  { label, error, ...inputProps },
  ref
) {
  return (
    <div className="campo">
      <label>{label}</label>
      <input
        ref={ref}
        className={error ? 'input-error' : 'input'}
        {...inputProps}
      />
      {error && <span className="error-msg">{error}</span>}
    </div>
  );
});

// Uso con react-hook-form (usa refs internamente)
import { useForm } from 'react-hook-form';

function Formulario() {
  const { register, handleSubmit, formState: { errors } } = useForm();

  return (
    <form onSubmit={handleSubmit(console.log)}>
      {/* register() retorna un objeto con ref, onChange, onBlur, name */}
      <CampoTexto
        label="Email"
        {...register('email', { required: 'El email es requerido' })}
        error={errors.email?.message}
      />
    </form>
  );
}
```

## forwardRef con TypeScript

```tsx
import { forwardRef, InputHTMLAttributes } from 'react';

interface InputProps extends InputHTMLAttributes<HTMLInputElement> {
  label: string;
  error?: string;
}

// Tipado: forwardRef<TipoElemento, TipoProps>
const Input = forwardRef<HTMLInputElement, InputProps>(
  ({ label, error, ...props }, ref) => (
    <div>
      <label>{label}</label>
      <input ref={ref} {...props} />
      {error && <p>{error}</p>}
    </div>
  )
);

Input.displayName = 'Input';  // Importante para debugging en DevTools
```

## Combinar forwardRef con useImperativeHandle

```jsx
// Exponer solo una API controlada (no todo el elemento DOM)
const ModalAnimado = forwardRef(function ModalAnimado({ children }, ref) {
  const [visible, setVisible] = useState(false);

  useImperativeHandle(ref, () => ({
    abrir: () => setVisible(true),
    cerrar: () => setVisible(false),
  }));

  return visible ? (
    <div className="modal">
      {children}
      <button onClick={() => setVisible(false)}>Cerrar</button>
    </div>
  ) : null;
});

// El padre controla el modal imperativamentement
function App() {
  const modalRef = useRef(null);

  return (
    <>
      <button onClick={() => modalRef.current.abrir()}>Abrir Modal</button>
      <ModalAnimado ref={modalRef}>
        <h2>Contenido del Modal</h2>
      </ModalAnimado>
    </>
  );
}
```

## React 19: ref como prop (sin forwardRef)

```jsx
// En React 19, las refs se pueden pasar como prop directamente
// forwardRef ya no es necesario

function InputBonito({ placeholder, ref }) {  // ref como prop normal
  return <input ref={ref} className="bonito" placeholder={placeholder} />;
}

// Uso:
<InputBonito ref={miRef} placeholder="Texto" />
```

## Cuándo usar forwardRef

```
✅ Usar forwardRef cuando:
- Necesitas focus() programático en el elemento DOM interno
- La librería de formularios (react-hook-form, Formik) necesita refs
- El padre necesita acceder al DOM para scroll, animaciones o mediciones
- Construyes un Design System con componentes reutilizables

❌ No usar forwardRef cuando:
- Puedes pasar datos via props y callbacks
- No necesitas acceso imperativo al DOM
```

**Consejo de entrevista**: `forwardRef` es fundamental cuando construyes componentes de Design System (inputs, selects, textareas) porque las librerías de formularios como react-hook-form usan refs internamente. En React 19, `forwardRef` se vuelve obsoleto porque las refs se pueden pasar como prop normal. Mencionar esta evolución demuestra conocimiento del roadmap de React.
