---
title: ¿Qué es el hook useTransition en React?
difficulty: medium
tags: [react, hooks, useTransition, concurrent, performance]
---

`useTransition` es un hook de React 18 que permite marcar ciertas actualizaciones de estado como **no urgentes** (transiciones), permitiendo que React priorice actualizaciones más importantes (como respuesta a un input del usuario) sobre actualizaciones costosas de UI.

## Problema que resuelve

```jsx
// ❌ Sin useTransition: el input se congela mientras filtra 10.000 elementos
function Buscador({ items }) {
  const [query, setQuery] = useState('');
  const [filtrados, setFiltrados] = useState(items);

  function handleChange(e) {
    const valor = e.target.value;
    setQuery(valor);             // Actualización urgente (UI del input)
    setFiltrados(             // Actualización costosa (filtrar 10.000 items)
      items.filter(item => item.nombre.includes(valor))
    );
    // React ejecuta ambas a la vez → el input se congela
  }

  return (
    <>
      <input value={query} onChange={handleChange} />
      <ListaLarga items={filtrados} />
    </>
  );
}
```

## Solución con useTransition

```jsx
import { useState, useTransition } from 'react';

function Buscador({ items }) {
  const [query, setQuery] = useState('');
  const [filtrados, setFiltrados] = useState(items);
  const [isPending, startTransition] = useTransition();

  function handleChange(e) {
    const valor = e.target.value;

    // Actualización urgente: el input se actualiza inmediatamente
    setQuery(valor);

    // Actualización no urgente: puede ser interrumpida si el usuario sigue escribiendo
    startTransition(() => {
      setFiltrados(items.filter(item => item.nombre.includes(valor)));
    });
  }

  return (
    <>
      <input value={query} onChange={handleChange} />

      {/* Mostrar feedback visual durante la transición */}
      {isPending && <span>Actualizando resultados...</span>}

      <div style={{ opacity: isPending ? 0.5 : 1 }}>
        <ListaLarga items={filtrados} />
      </div>
    </>
  );
}
```

## Navegación con useTransition (Next.js)

```jsx
'use client';
import { useTransition } from 'react';
import { useRouter } from 'next/navigation';

function MenuNavegacion({ enlaces }) {
  const [isPending, startTransition] = useTransition();
  const router = useRouter();

  function navegar(href) {
    startTransition(() => {
      // La navegación se marca como transición no urgente
      // El spinner solo aparece si tarda más de lo esperado
      router.push(href);
    });
  }

  return (
    <nav>
      {enlaces.map(enlace => (
        <button
          key={enlace.href}
          onClick={() => navegar(enlace.href)}
          disabled={isPending}
          style={{ opacity: isPending ? 0.7 : 1 }}
        >
          {enlace.texto}
          {isPending && ' (cargando...)'}
        </button>
      ))}
    </nav>
  );
}
```

## Tabs con carga pesada

```jsx
function TabsContenido() {
  const [tabActivo, setTabActivo] = useState('inicio');
  const [isPending, startTransition] = useTransition();

  function cambiarTab(tab) {
    startTransition(() => {
      setTabActivo(tab);
      // Si renderizar el tab tarda, React puede interrumpirlo
      // y dar prioridad a clicks más recientes del usuario
    });
  }

  return (
    <div>
      <div className="tabs">
        {['inicio', 'perfil', 'configuracion', 'analytics'].map(tab => (
          <button
            key={tab}
            onClick={() => cambiarTab(tab)}
            className={tabActivo === tab ? 'activo' : ''}
          >
            {tab}
          </button>
        ))}
      </div>

      {/* Overlay de carga sobre el contenido anterior */}
      <div style={{ position: 'relative' }}>
        {isPending && (
          <div className="loading-overlay">Cargando...</div>
        )}
        <ContenidoTab tab={tabActivo} />
      </div>
    </div>
  );
}
```

## useTransition vs useDeferredValue

```jsx
// useTransition: controlas cuándo la actualización es una transición
// → Para cuando tú controlas la actualización de estado
const [isPending, startTransition] = useTransition();
startTransition(() => setState(nuevoValor));

// useDeferredValue: pospones la propagación de un valor
// → Para cuando recibes el valor de props y no controlas el setter
function Componente({ consulta }) {
  const consultaDiferida = useDeferredValue(consulta);
  // `consulta` es inmediato (para el input)
  // `consultaDiferida` se actualiza en transición (para la lista filtrada)
  return <ListaFiltrada filtro={consultaDiferida} />;
}
```

## Comparativa

| Característica              | `useTransition`           | `useDeferredValue`        |
|-----------------------------|---------------------------|---------------------------|
| Control                     | Envuelves el setState     | Wrappeas el valor         |
| `isPending`                 | ✅ Disponible             | Solo con comparación manual |
| Caso de uso                 | Controlas el setter       | Recibes el valor via props |
| Interrupción                | ✅ React puede interrumpir| ✅ React puede interrumpir |

**Consejo de entrevista**: `useTransition` es una de las características clave del modo Concurrente de React 18. La regla de uso: si una actualización de estado provoca re-renders costosos que bloquean la UI, márcala como transición. El flag `isPending` permite mostrar feedback visual sin comprometer la responsividad. Es especialmente valioso en búsquedas en tiempo real, filtros y navegación entre páginas con contenido pesado.
