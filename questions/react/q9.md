---
title: ¿Qué es el hook useReducer?
difficulty: medium
tags: [react, hooks, useReducer, state-management]
---

`useReducer` es un hook de React que ofrece una alternativa a `useState` para gestionar estado complejo. Está inspirado en el patrón Redux: el estado se actualiza mediante **acciones** que son procesadas por una función **reducer** pura. Es especialmente útil cuando el siguiente estado depende del anterior o cuando hay múltiples sub-valores relacionados.

La firma es: `const [state, dispatch] = useReducer(reducer, initialState)`

```jsx
import { useReducer } from 'react';

// Tipos de acción
const TODO_ACTIONS = {
  ADD: 'ADD',
  TOGGLE: 'TOGGLE',
  DELETE: 'DELETE',
  CLEAR_COMPLETED: 'CLEAR_COMPLETED'
};

// Reducer: función pura que calcula el nuevo estado
function todoReducer(state, action) {
  switch (action.type) {
    case TODO_ACTIONS.ADD:
      return {
        ...state,
        todos: [...state.todos, { id: Date.now(), text: action.payload, done: false }]
      };

    case TODO_ACTIONS.TOGGLE:
      return {
        ...state,
        todos: state.todos.map(todo =>
          todo.id === action.payload ? { ...todo, done: !todo.done } : todo
        )
      };

    case TODO_ACTIONS.DELETE:
      return {
        ...state,
        todos: state.todos.filter(todo => todo.id !== action.payload)
      };

    case TODO_ACTIONS.CLEAR_COMPLETED:
      return {
        ...state,
        todos: state.todos.filter(todo => !todo.done)
      };

    default:
      return state;
  }
}

const initialState = { todos: [] };

function TodoApp() {
  const [state, dispatch] = useReducer(todoReducer, initialState);
  const [input, setInput] = React.useState('');

  const addTodo = () => {
    if (input.trim()) {
      dispatch({ type: TODO_ACTIONS.ADD, payload: input });
      setInput('');
    }
  };

  return (
    <div>
      <input value={input} onChange={e => setInput(e.target.value)} />
      <button onClick={addTodo}>Añadir</button>

      <ul>
        {state.todos.map(todo => (
          <li key={todo.id}>
            <span
              style={{ textDecoration: todo.done ? 'line-through' : 'none' }}
              onClick={() => dispatch({ type: TODO_ACTIONS.TOGGLE, payload: todo.id })}
            >
              {todo.text}
            </span>
            <button onClick={() => dispatch({ type: TODO_ACTIONS.DELETE, payload: todo.id })}>
              ✕
            </button>
          </li>
        ))}
      </ul>

      <button onClick={() => dispatch({ type: TODO_ACTIONS.CLEAR_COMPLETED })}>
        Limpiar completados
      </button>
    </div>
  );
}
```

## useReducer vs useState

```jsx
// useState: para estado simple e independiente
const [count, setCount] = useState(0);
const [name, setName] = useState('');

// useReducer: para estado complejo o relacionado
const [formState, dispatch] = useReducer(formReducer, {
  values: {},
  errors: {},
  touched: {},
  isSubmitting: false
});
```

## Inicialización diferida (lazy initialization)

Para calcular el estado inicial de forma costosa:

```jsx
function init(initialCount) {
  return { count: initialCount, history: [] };
}

const [state, dispatch] = useReducer(reducer, initialCount, init);
// El tercer argumento es una función que recibe el segundo argumento
```

**Consejo de entrevista**: `useReducer` es preferible a múltiples llamadas a `useState` cuando las actualizaciones de estado son complejas o interdependientes. También facilita el testing del reducer de forma aislada (es una función pura), y el dispatch es estable entre renders, lo que evita problemas con `useCallback`.
