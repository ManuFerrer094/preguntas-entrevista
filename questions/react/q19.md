---
title: ¿Qué son los portales en React?
difficulty: medium
tags: [react, portals, dom, modal, tooltip]
---

Los portales de React permiten renderizar un componente fuera de su posición en el árbol del DOM, pero manteniéndolo en el árbol de componentes de React. Son la solución estándar para modales, tooltips y menús desplegables que necesitan escapar del contexto de su contenedor padre.

## El problema que resuelven

```jsx
// Problema: un modal dentro de un div con overflow:hidden queda recortado
function App() {
  return (
    <div style={{ overflow: 'hidden', height: 300 }}>
      {/* ❌ El modal queda recortado por el overflow:hidden del padre */}
      <Modal>
        <p>Este modal queda cortado</p>
      </Modal>
    </div>
  );
}
```

## Uso básico de createPortal

```jsx
import { createPortal } from 'react-dom';

function Modal({ isOpen, onClose, children }) {
  if (!isOpen) return null;

  // Renderiza el modal en document.body, no en su posición del árbol DOM
  return createPortal(
    <div
      className="modal-overlay"
      onClick={onClose}
      role="dialog"
      aria-modal="true"
    >
      <div className="modal-content" onClick={e => e.stopPropagation()}>
        <button
          className="modal-close"
          onClick={onClose}
          aria-label="Cerrar modal"
        >
          ×
        </button>
        {children}
      </div>
    </div>,
    document.body // Segundo argumento: nodo DOM destino
  );
}

// Uso
function App() {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <div style={{ overflow: 'hidden', height: 300 }}>
      <button onClick={() => setIsOpen(true)}>Abrir modal</button>

      {/* El modal se renderiza en document.body aunque esté aquí en el JSX */}
      <Modal isOpen={isOpen} onClose={() => setIsOpen(false)}>
        <h2>Modal con portal</h2>
        <p>Este modal no queda recortado por el overflow del padre.</p>
      </Modal>
    </div>
  );
}
```

## Los eventos burbujean por el árbol de React, no el DOM

```jsx
// Esta es una característica importante: los eventos siguen el árbol React
function Parent() {
  function handleClick() {
    // Este handler SÍ se ejecuta cuando se hace clic en el portal
    // aunque el portal esté en document.body en el DOM real
    console.log('Click capturado en Parent (árbol React)');
  }

  return (
    <div onClick={handleClick}>
      <p>Componente padre</p>
      {createPortal(
        <button>Click en el portal</button>,
        document.body
      )}
    </div>
  );
}
```

## Tooltip con portal

```jsx
function Tooltip({ children, text }) {
  const [visible, setVisible] = useState(false);
  const [position, setPosition] = useState({ top: 0, left: 0 });
  const triggerRef = useRef(null);

  function handleMouseEnter() {
    const rect = triggerRef.current.getBoundingClientRect();
    setPosition({
      top: rect.bottom + window.scrollY + 8,
      left: rect.left + window.scrollX + rect.width / 2
    });
    setVisible(true);
  }

  return (
    <>
      <span
        ref={triggerRef}
        onMouseEnter={handleMouseEnter}
        onMouseLeave={() => setVisible(false)}
      >
        {children}
      </span>

      {visible && createPortal(
        <div
          className="tooltip"
          style={{
            position: 'absolute',
            top: position.top,
            left: position.left,
            transform: 'translateX(-50%)',
            zIndex: 9999
          }}
        >
          {text}
        </div>,
        document.body
      )}
    </>
  );
}

// Uso
<Tooltip text="Información adicional">
  <button>Hover sobre mí</button>
</Tooltip>
```

## Contenedor específico para portales

```jsx
// Crear un contenedor dedicado para los portales
// En el HTML: <div id="portal-root"></div>

function usePortalContainer(id = 'portal-root') {
  const [container, setContainer] = useState(null);

  useEffect(() => {
    let el = document.getElementById(id);
    let created = false;

    if (!el) {
      el = document.createElement('div');
      el.id = id;
      document.body.appendChild(el);
      created = true;
    }

    setContainer(el);

    return () => {
      if (created && el.parentNode) {
        el.parentNode.removeChild(el);
      }
    };
  }, [id]);

  return container;
}

function Portal({ children, containerId }) {
  const container = usePortalContainer(containerId);
  if (!container) return null;
  return createPortal(children, container);
}
```

**Consejo de entrevista**: Los portales son ideales para modales, tooltips, dropdowns y notificaciones que necesitan estar por encima de todo el contenido (z-index, overflow). La clave es que aunque el componente se renderiza en otro nodo del DOM, sigue siendo hijo en el árbol de React: contextos, eventos y `ref` funcionan normalmente. Siempre gestiona el foco (focus trap) y la accesibilidad (aria attributes) en modales creados con portales.
