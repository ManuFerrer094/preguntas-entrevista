---
title: ¿Cuál es la diferencia entre useEffect y useLayoutEffect?
difficulty: medium
tags: [react, hooks, useEffect, useLayoutEffect, dom, performance]
---

`useLayoutEffect` se ejecuta de forma síncrona después de que React actualiza el DOM pero **antes** de que el navegador pinte en pantalla. `useEffect` se ejecuta de forma asíncrona **después** de que el navegador ha pintado. Esto determina cuándo usar cada uno.

## Orden de ejecución

```jsx
function Component() {
  const [count, setCount] = useState(0);

  useLayoutEffect(() => {
    // 2. Se ejecuta DESPUÉS de que React actualiza el DOM
    //    pero ANTES de que el navegador pinte
    console.log('useLayoutEffect:', document.title);
  });

  useEffect(() => {
    // 3. Se ejecuta DESPUÉS de que el navegador ha pintado
    console.log('useEffect');
  });

  console.log('1. Render');
  // Orden: "1. Render" → "2. useLayoutEffect" → (navegador pinta) → "3. useEffect"

  return <div>{count}</div>;
}
```

## Cuándo usar useLayoutEffect

```jsx
// ✅ Medir el DOM para posicionar elementos (evita parpadeo visual)
function Tooltip({ targetRef, children }) {
  const tooltipRef = useRef(null);
  const [position, setPosition] = useState({ top: 0, left: 0 });

  useLayoutEffect(() => {
    // Si usamos useEffect, el tooltip aparecería primero en posición 0,0
    // y luego saltaría a la posición correcta (parpadeo visible)
    // useLayoutEffect calcula la posición ANTES del primer pintado
    if (!tooltipRef.current || !targetRef.current) return;

    const targetRect = targetRef.current.getBoundingClientRect();
    const tooltipRect = tooltipRef.current.getBoundingClientRect();

    setPosition({
      top: targetRect.top - tooltipRect.height - 8,
      left: targetRect.left + (targetRect.width - tooltipRect.width) / 2
    });
  }, [targetRef]);

  return (
    <div
      ref={tooltipRef}
      style={{
        position: 'fixed',
        top: position.top,
        left: position.left
      }}
    >
      {children}
    </div>
  );
}
```

```jsx
// ✅ Sincronizar animaciones con el DOM
function FadeIn({ children }) {
  const ref = useRef(null);

  useLayoutEffect(() => {
    const element = ref.current;
    if (!element) return;

    // Aplicar el estado inicial ANTES del pintado (sin parpadeo)
    element.style.opacity = '0';
    element.style.transform = 'translateY(20px)';

    // Forzar reflow y luego animar
    requestAnimationFrame(() => {
      element.style.transition = 'opacity 0.3s, transform 0.3s';
      element.style.opacity = '1';
      element.style.transform = 'translateY(0)';
    });
  }, []);

  return <div ref={ref}>{children}</div>;
}
```

## Comparación práctica

```jsx
function FlashExample() {
  const [show, setShow] = useState(false);
  const boxRef = useRef(null);

  // ❌ Con useEffect: el box aparece arriba y luego salta al centro (parpadeo)
  useEffect(() => {
    if (show && boxRef.current) {
      const box = boxRef.current;
      const viewportHeight = window.innerHeight;
      const boxHeight = box.offsetHeight;
      box.style.top = `${(viewportHeight - boxHeight) / 2}px`;
    }
  });

  // ✅ Con useLayoutEffect: el box aparece directamente centrado (sin parpadeo)
  useLayoutEffect(() => {
    if (show && boxRef.current) {
      const box = boxRef.current;
      const viewportHeight = window.innerHeight;
      const boxHeight = box.offsetHeight;
      box.style.top = `${(viewportHeight - boxHeight) / 2}px`;
    }
  });

  return (
    <>
      <button onClick={() => setShow(true)}>Mostrar</button>
      {show && <div ref={boxRef} style={{ position: 'fixed' }}>Centrado</div>}
    </>
  );
}
```

## Regla de uso

| | `useEffect` | `useLayoutEffect` |
|---|---|---|
| Cuándo se ejecuta | Después del pintado | Antes del pintado |
| Bloquea el navegador | No | Sí (síncrono) |
| Uso recomendado | Peticiones API, suscripciones, logs | Medición del DOM, posicionamiento, animaciones |
| SSR | Funciona | ⚠️ Warning en SSR (el DOM no existe en servidor) |

```jsx
// ✅ Mayoría de efectos → useEffect
useEffect(() => {
  fetchData();
  const sub = subscribe();
  return () => sub.unsubscribe();
}, []);

// ✅ Medición/posicionamiento del DOM → useLayoutEffect
useLayoutEffect(() => {
  const { width } = ref.current.getBoundingClientRect();
  setElementWidth(width);
}, []);
```

**Consejo de entrevista**: La regla práctica es empezar siempre con `useEffect`. Solo cambia a `useLayoutEffect` si ves un parpadeo visual (flash of incorrect content) al modificar el DOM basándote en sus medidas. `useLayoutEffect` bloquea el pintado del navegador, así que debe ser rápido. En SSR (Next.js), `useLayoutEffect` genera un warning porque el DOM no existe en el servidor; usa `useEffect` o guarda la lógica para después del montaje.
