`StrictMode` es un componente de React que activa comprobaciones y advertencias adicionales durante el desarrollo. **No afecta a la build de producción**. Ayuda a detectar problemas potenciales antes de que lleguen a producción.

## Activación

```jsx
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';

const root = createRoot(document.getElementById('root'));

// Wrappear la app completa o partes específicas
root.render(
  <StrictMode>
    <App />
  </StrictMode>
);

// O solo para partes de la app
function App() {
  return (
    <div>
      <Header />
      <StrictMode>
        <LegacyComponent /> {/* Solo este componente tiene StrictMode */}
      </StrictMode>
      <Footer />
    </div>
  );
}
```

## Doble invocación de funciones (la más sorprendente)

```jsx
// En Strict Mode, React llama dos veces a:
// - El cuerpo de los componentes (función del componente)
// - Las funciones pasadas a useState, useReducer
// - Los setup de useEffect (pero no el cleanup)

// Esto detecta efectos secundarios en lugares donde no deberían existir
function Counter() {
  // ⚠️ En StrictMode, este log aparece DOS VECES en desarrollo
  // Si el segundo render causa problemas → hay efectos secundarios no puros
  console.log('Renderizando Counter');

  const [count, setCount] = useState(() => {
    console.log('Inicializando estado'); // También aparece dos veces
    return 0;
  });

  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

// ❌ Esto revela un bug: el render tiene un efecto secundario
let renderCount = 0;
function BuggyComponent() {
  renderCount++; // Incrementa también en el render "extra" de StrictMode
  return <p>Renders: {renderCount}</p>; // Mostrará 2 en lugar de 1
}
```

## Doble invocación de useEffect en React 18+

```jsx
// React 18 + StrictMode: el efecto se monta, desmonta y vuelve a montar
// Esto simula que el usuario navega fuera y vuelve a la ruta
// Detecta efectos que no tienen cleanup correcto

function ChatRoom({ roomId }) {
  useEffect(() => {
    const connection = createConnection(roomId);
    connection.connect();
    console.log('Conectado');

    return () => {
      connection.disconnect();
      console.log('Desconectado');
    };
    // En StrictMode: Conectado → Desconectado → Conectado
    // Si esto causa errores → el efecto no es "resiliente al remontado"
  }, [roomId]);
}

// ❌ Efecto que no sobrevive al doble montado (bug real)
function BadEffect() {
  useEffect(() => {
    const id = generateUniqueId();
    registerUser(id); // Se registra DOS veces en StrictMode → bug

    // Sin cleanup → StrictMode lo detecta
  }, []);
}

// ✅ Efecto correcto con cleanup
function GoodEffect() {
  useEffect(() => {
    const id = generateUniqueId();
    registerUser(id);

    return () => unregisterUser(id); // Cleanup → sobrevive al doble montado
  }, []);
}
```

## StrictMode y peticiones de red

```jsx
// Un patrón frecuente que se "rompe" con StrictMode (y que debería arreglarse):
function UserList() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    // En StrictMode de React 18: este fetch se ejecuta DOS VECES
    // La solución correcta es usar AbortController para cancelar la primera
    const controller = new AbortController();

    fetch('/api/users', { signal: controller.signal })
      .then(r => r.json())
      .then(setUsers)
      .catch(err => {
        if (err.name !== 'AbortError') console.error(err);
      });

    return () => controller.abort(); // ✅ Cleanup correcto
  }, []);

  return <ul>{users.map(u => <li key={u.id}>{u.name}</li>)}</ul>;
}
```

**Consejo de entrevista**: StrictMode es una herramienta de desarrollo, no afecta producción. El doble render/efecto en React 18+ es intencional y revela bugs reales: si tu código no funciona correctamente cuando un componente se monta dos veces, tienes un bug. La solución no es desactivar StrictMode sino escribir efectos con cleanup correcto.
