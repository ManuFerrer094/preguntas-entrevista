---
title: ¿Qué es Zustand?
difficulty: medium
tags: [react, zustand, state-management, store]
---

Zustand es una librería de gestión de estado para React, minimalista y sin boilerplate. A diferencia de Redux, no requiere actions, reducers ni providers. Su API es simple: defines un store con estado y funciones que lo modifican, y accedes a él desde cualquier componente.

Zustand destaca por su reducido tamaño (~1KB), rendimiento (los componentes solo se re-renderizan cuando la parte del estado que consumen cambia) y facilidad de uso.

**Creación de un store básico:**

```javascript
import { create } from 'zustand';

const useCartStore = create((set, get) => ({
  // Estado
  items: [],
  isOpen: false,

  // Acciones
  addItem: (product) => set((state) => {
    const existing = state.items.find(i => i.id === product.id);
    if (existing) {
      return {
        items: state.items.map(i =>
          i.id === product.id ? { ...i, quantity: i.quantity + 1 } : i
        )
      };
    }
    return { items: [...state.items, { ...product, quantity: 1 }] };
  }),

  removeItem: (id) => set((state) => ({
    items: state.items.filter(i => i.id !== id)
  })),

  clearCart: () => set({ items: [] }),

  // Computed values usando get()
  getTotalPrice: () => {
    return get().items.reduce((sum, item) => sum + item.price * item.quantity, 0);
  },

  toggleCart: () => set((state) => ({ isOpen: !state.isOpen }))
}));
```

**Uso en componentes:**

```jsx
// Selecciona solo la parte del estado que necesitas
function CartIcon() {
  const itemCount = useCartStore(state => state.items.length);
  const toggleCart = useCartStore(state => state.toggleCart);

  return (
    <button onClick={toggleCart}>
      🛒 {itemCount}
    </button>
  );
}

function ProductCard({ product }) {
  const addItem = useCartStore(state => state.addItem);

  return (
    <div>
      <h3>{product.name}</h3>
      <p>${product.price}</p>
      <button onClick={() => addItem(product)}>Añadir al carrito</button>
    </div>
  );
}
```

**Zustand con TypeScript:**

```typescript
import { create } from 'zustand';

interface BearState {
  bears: number;
  increase: () => void;
  reset: () => void;
}

const useBearStore = create<BearState>()((set) => ({
  bears: 0,
  increase: () => set((state) => ({ bears: state.bears + 1 })),
  reset: () => set({ bears: 0 })
}));
```

**Persistencia con middleware:**

```javascript
import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';

const useSettingsStore = create(
  persist(
    (set) => ({
      theme: 'light',
      language: 'es',
      setTheme: (theme) => set({ theme }),
      setLanguage: (language) => set({ language })
    }),
    {
      name: 'app-settings', // clave en localStorage
      storage: createJSONStorage(() => localStorage)
    }
  )
);
```

**Immer middleware para mutaciones inmutables:**

```javascript
import { immer } from 'zustand/middleware/immer';

const useStore = create(immer((set) => ({
  users: [],
  addUser: (user) => set((state) => {
    state.users.push(user); // Immer hace esto inmutable internamente
  })
})));
```

**Consejo de entrevista**: Zustand es una excelente opción para aplicaciones React que necesitan estado global sin la complejidad de Redux. Comparado con Context API, evita los re-renderizados en cascada porque los componentes solo se suscriben a la parte del estado que necesitan. Redux Toolkit sigue siendo preferible para equipos grandes con necesidades de debugging avanzadas.
