`Suspense` es un componente de React que permite mostrar un estado de carga (fallback) mientras sus componentes hijos están esperando algo: ya sea la carga de código (lazy loading) o datos (data fetching con frameworks que soporten Suspense).

## Lazy loading de componentes

El uso más común y estable de Suspense es mostrar un fallback mientras se carga un componente dinámicamente.

```jsx
import { lazy, Suspense } from 'react';

// React.lazy carga el componente solo cuando se necesita
const Dashboard = lazy(() => import('./Dashboard'));
const Settings = lazy(() => import('./Settings'));
const Analytics = lazy(() => import('./Analytics'));

function App() {
  const [page, setPage] = useState('dashboard');

  return (
    <Suspense fallback={<PageSkeleton />}>
      {page === 'dashboard' && <Dashboard />}
      {page === 'settings' && <Settings />}
      {page === 'analytics' && <Analytics />}
    </Suspense>
  );
}

// Con React Router
import { lazy, Suspense } from 'react';
import { Routes, Route } from 'react-router-dom';

const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));

function AppRoutes() {
  return (
    <Suspense fallback={<div>Cargando página...</div>}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </Suspense>
  );
}
```

## Suspense anidado para UX granular

```jsx
function ProductPage({ productId }) {
  return (
    <div>
      <h1>Producto</h1>

      {/* Suspense externo: skeleton de toda la página */}
      <Suspense fallback={<PageSkeleton />}>
        <ProductDetails productId={productId} />

        {/* Suspense interno: solo el panel de reviews tiene su propio fallback */}
        <Suspense fallback={<ReviewsSkeleton />}>
          <ProductReviews productId={productId} />
        </Suspense>
      </Suspense>
    </div>
  );
}
```

## Suspense con data fetching (Next.js, Remix, React Query)

```jsx
// Con React Query (soporte experimental de Suspense)
import { useSuspenseQuery } from '@tanstack/react-query';

// El componente "suspende" mientras los datos cargan
function UserProfile({ userId }) {
  const { data: user } = useSuspenseQuery({
    queryKey: ['user', userId],
    queryFn: () => fetch(`/api/users/${userId}`).then(r => r.json())
  });

  // Sin necesidad de manejar loading/error aquí
  // Suspense muestra el fallback, ErrorBoundary maneja errores
  return <div>{user.name}</div>;
}

// Envolver con Suspense y ErrorBoundary en el componente padre
function App() {
  return (
    <ErrorBoundary fallback={<ErrorMessage />}>
      <Suspense fallback={<UserSkeleton />}>
        <UserProfile userId={1} />
      </Suspense>
    </ErrorBoundary>
  );
}
```

## SuspenseList para coordinar múltiples Suspense

```jsx
import { SuspenseList, Suspense } from 'react';

// SuspenseList coordina el orden en que aparecen los componentes
function FeedPage() {
  return (
    <SuspenseList revealOrder="forwards" tail="collapsed">
      <Suspense fallback={<PostSkeleton />}>
        <Post id={1} />
      </Suspense>
      <Suspense fallback={<PostSkeleton />}>
        <Post id={2} />
      </Suspense>
      <Suspense fallback={<PostSkeleton />}>
        <Post id={3} />
      </Suspense>
    </SuspenseList>
  );
  // revealOrder="forwards": aparecen en orden, aunque el 3 cargue antes
  // tail="collapsed": solo muestra 1 skeleton a la vez
}
```

## useTransition con Suspense

```jsx
import { useTransition, Suspense } from 'react';

function Navigation() {
  const [page, setPage] = useState('home');
  const [isPending, startTransition] = useTransition();

  function navigate(newPage) {
    // startTransition marca la navegación como no urgente
    // React mantiene la UI actual mientras carga la nueva página
    startTransition(() => setPage(newPage));
  }

  return (
    <>
      <nav>
        <button onClick={() => navigate('home')}>Inicio</button>
        <button onClick={() => navigate('about')}>Sobre nosotros</button>
        {isPending && <Spinner size="small" />}
      </nav>

      <Suspense fallback={<PageSkeleton />}>
        {page === 'home' && <HomePage />}
        {page === 'about' && <AboutPage />}
      </Suspense>
    </>
  );
}
```

**Consejo de entrevista**: `Suspense` con `lazy()` para code splitting es la característica más estable y usada. El soporte de Suspense para data fetching requiere frameworks que implementen el protocolo de promesas de React (Next.js App Router, React Query con `useSuspenseQuery`). Siempre combina `Suspense` con `ErrorBoundary` para manejar los errores que ocurran durante la carga.
