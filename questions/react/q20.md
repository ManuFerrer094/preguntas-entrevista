---
title: ¿Qué es el hook useRef y para qué se usa?
difficulty: medium
tags: [react, hooks, useRef, dom, mutable-ref]
---

`useRef` devuelve un objeto mutable `{ current: initialValue }` que persiste entre renderizados sin causar re-renders cuando cambia. Se usa principalmente para: acceder directamente a nodos del DOM, almacenar valores mutables que no deben disparar renderizados, y guardar la versión anterior de un valor.

## Acceso al DOM (caso de uso más común)

```jsx
function AutoFocusInput() {
  const inputRef = useRef(null);

  useEffect(() => {
    // Enfocar el input al montar el componente
    inputRef.current?.focus();
  }, []);

  return <input ref={inputRef} placeholder="Soy enfocado automáticamente" />;
}

// Scroll a un elemento
function ChatWindow({ messages }) {
  const bottomRef = useRef(null);

  useEffect(() => {
    bottomRef.current?.scrollIntoView({ behavior: 'smooth' });
  }, [messages]);

  return (
    <div className="chat-window">
      {messages.map(msg => <Message key={msg.id} {...msg} />)}
      <div ref={bottomRef} />
    </div>
  );
}
```

## Almacenar valores mutables sin re-renders

```jsx
// ✅ useRef para timers, contadores y valores que no afectan la UI
function SearchInput({ onSearch }) {
  const [value, setValue] = useState('');
  const debounceTimerRef = useRef(null);

  function handleChange(e) {
    const newValue = e.target.value;
    setValue(newValue);

    // Cancelar el timer anterior
    if (debounceTimerRef.current) {
      clearTimeout(debounceTimerRef.current);
    }

    // Crear un nuevo timer (no dispara re-render al asignar current)
    debounceTimerRef.current = setTimeout(() => {
      onSearch(newValue);
    }, 300);
  }

  useEffect(() => {
    // Limpiar el timer al desmontar
    return () => clearTimeout(debounceTimerRef.current);
  }, []);

  return <input value={value} onChange={handleChange} />;
}
```

## Almacenar la versión anterior de un valor

```jsx
function usePreviousValue(value) {
  const prevRef = useRef(undefined);

  useEffect(() => {
    prevRef.current = value; // Se actualiza DESPUÉS del render
  });

  return prevRef.current; // Devuelve el valor del render anterior
}

function Counter() {
  const [count, setCount] = useState(0);
  const prevCount = usePreviousValue(count);

  return (
    <div>
      <p>Actual: {count}</p>
      <p>Anterior: {prevCount ?? 'N/A'}</p>
      <button onClick={() => setCount(c => c + 1)}>+1</button>
    </div>
  );
}
```

## useRef vs useState

```jsx
// La diferencia fundamental:

// useState: cambiar el valor DISPARA un re-render
const [count, setCount] = useState(0);
setCount(1); // → React re-renderiza el componente

// useRef: cambiar current NO dispara re-render
const countRef = useRef(0);
countRef.current = 1; // → El componente NO se re-renderiza

// ¿Cuándo usar useRef?
// ✅ Para valores que necesitas leer/escribir pero que NO afectan la UI
// ✅ Para referencias al DOM
// ✅ Para prevenir re-renders innecesarios (ej: timers, IDs, flags)
// ❌ NO para valores que se deben mostrar en la UI → usa useState
```

## Ref callback para elementos dinámicos

```jsx
function DynamicList({ items }) {
  const itemRefs = useRef(new Map());

  function scrollToItem(id) {
    itemRefs.current.get(id)?.scrollIntoView({ behavior: 'smooth' });
  }

  return (
    <>
      <button onClick={() => scrollToItem(items[2].id)}>
        Ir al tercer elemento
      </button>
      <ul>
        {items.map(item => (
          <li
            key={item.id}
            ref={node => {
              // Ref callback: se llama con el nodo DOM (o null al desmontar)
              if (node) {
                itemRefs.current.set(item.id, node);
              } else {
                itemRefs.current.delete(item.id);
              }
            }}
          >
            {item.name}
          </li>
        ))}
      </ul>
    </>
  );
}
```

## forwardRef: pasar refs a componentes hijos

```jsx
// Los componentes funcionales no aceptan ref por defecto
// forwardRef permite pasar la ref al elemento DOM interno
const CustomInput = forwardRef(function CustomInput(props, ref) {
  return (
    <input
      ref={ref}
      className="custom-input"
      {...props}
    />
  );
});

// Ahora el padre puede acceder al DOM del CustomInput
function Form() {
  const inputRef = useRef(null);

  return (
    <>
      <CustomInput ref={inputRef} placeholder="Input personalizado" />
      <button onClick={() => inputRef.current?.focus()}>
        Enfocar input
      </button>
    </>
  );
}
```

**Consejo de entrevista**: La regla de oro: usa `useRef` cuando necesitas un valor que persista entre renders pero cuyo cambio no debe causar re-renders (timers, referencias al DOM, valores previos). Si el cambio del valor debe actualizar la UI, usa `useState`. Mencionar `forwardRef` para pasar refs a componentes hijos también es un punto importante en entrevistas.
