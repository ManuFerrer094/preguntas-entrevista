---
title: ¿Qué son los Error Boundaries en React?
difficulty: medium
tags: [react, error-boundaries, error-handling, resilience]
---

Los Error Boundaries son componentes de React que capturan errores de JavaScript en el árbol de componentes hijo, los registran y muestran una interfaz de usuario alternativa en lugar de hacer que toda la aplicación se rompa. Son la solución de React para manejar errores de renderizado de forma elegante.

Sin Error Boundaries, un error en cualquier componente provoca que toda la aplicación deje de funcionar. Con ellos, puedes aislar fallos en partes específicas de la UI.

**Error Boundary como clase (única forma nativa actualmente):**

```jsx
import { Component } from 'react';

class ErrorBoundary extends Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, error: null, errorInfo: null };
  }

  static getDerivedStateFromError(error) {
    // Actualiza el estado para mostrar la UI de error
    return { hasError: true, error };
  }

  componentDidCatch(error, errorInfo) {
    // Aquí puedes registrar el error en un servicio de monitoreo
    console.error('Error capturado:', error, errorInfo);
    // Ejemplo: logErrorToService(error, errorInfo);
    this.setState({ errorInfo });
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback || (
        <div style={{ padding: '20px', border: '1px solid red', borderRadius: '8px' }}>
          <h2>Algo salió mal</h2>
          <p>{this.state.error?.message}</p>
          <button onClick={() => this.setState({ hasError: false, error: null })}>
            Intentar de nuevo
          </button>
        </div>
      );
    }

    return this.props.children;
  }
}
```

**Uso estratégico en la aplicación:**

```jsx
function App() {
  return (
    <ErrorBoundary fallback={<FullPageError />}>
      <Header />
      <main>
        {/* Cada sección tiene su propio boundary */}
        <ErrorBoundary fallback={<p>Error en el sidebar</p>}>
          <Sidebar />
        </ErrorBoundary>

        <ErrorBoundary fallback={<ContentError />}>
          <MainContent />
        </ErrorBoundary>
      </main>

      <ErrorBoundary fallback={<p>Error en los comentarios</p>}>
        <Comments />
      </ErrorBoundary>
    </ErrorBoundary>
  );
}
```

**Librería react-error-boundary (más conveniente):**

```jsx
import { ErrorBoundary, useErrorBoundary } from 'react-error-boundary';

function ErrorFallback({ error, resetErrorBoundary }) {
  return (
    <div role="alert">
      <p>Error: {error.message}</p>
      <button onClick={resetErrorBoundary}>Reintentar</button>
    </div>
  );
}

function UserCard({ userId }) {
  // Lanzar errores manualmente desde componentes funcionales
  const { showBoundary } = useErrorBoundary();

  const { data, error } = useFetch(`/api/users/${userId}`);

  if (error) showBoundary(error); // Delegar al Error Boundary más cercano

  return <div>{data?.name}</div>;
}

// Uso
<ErrorBoundary
  FallbackComponent={ErrorFallback}
  onReset={() => queryClient.invalidateQueries()}
  onError={(error) => logToSentry(error)}
>
  <UserCard userId="123" />
</ErrorBoundary>
```

## Limitaciones importantes

Los Error Boundaries **NO** capturan:
- Errores en manejadores de eventos (usar try/catch normal)
- Código asíncrono (setTimeout, Promises) — usar try/catch o `useErrorBoundary`
- Errores en el propio Error Boundary
- Errores en Server-Side Rendering

```jsx
// ❌ El Error Boundary NO captura esto
<button onClick={() => {
  throw new Error('Este error no lo captura el boundary');
}}>
  Click

// ✅ Manejo correcto en event handlers
<button onClick={() => {
  try {
    riskyOperation();
  } catch (error) {
    setError(error); // o usar showBoundary de react-error-boundary
  }
}}>
```

**Consejo de entrevista**: Los Error Boundaries son una característica de resiliencia crítica en aplicaciones de producción. Mencionar la integración con servicios de monitoreo como Sentry dentro del `componentDidCatch` y el uso de `react-error-boundary` para simplificar la implementación son puntos que demuestran experiencia real.
