---
title: ¿Qué es React.memo?
difficulty: medium
tags: [react, performance, memoization, components]
---

`React.memo` es un Higher Order Component (HOC) que envuelve un componente funcional para evitar re-renderizados innecesarios. Cuando un componente padre se re-renderiza, normalmente todos sus hijos también lo hacen, aunque sus props no hayan cambiado. `React.memo` resuelve este problema realizando una comparación superficial (shallow comparison) de las props.

```jsx
import { memo, useState } from 'react';

// Sin React.memo: se re-renderiza SIEMPRE que el padre lo haga
const ExpensiveChild = ({ title, items }) => {
  console.log('Renderizando ExpensiveChild');
  return (
    <div>
      <h2>{title}</h2>
      <ul>{items.map(item => <li key={item.id}>{item.name}</li>)}</ul>
    </div>
  );
};

// Con React.memo: solo se re-renderiza si 'title' o 'items' cambian
const MemoizedChild = memo(({ title, items }) => {
  console.log('Renderizando MemoizedChild');
  return (
    <div>
      <h2>{title}</h2>
      <ul>{items.map(item => <li key={item.id}>{item.name}</li>)}</ul>
    </div>
  );
});

function Parent() {
  const [count, setCount] = useState(0);
  const [items] = useState([{ id: 1, name: 'Item 1' }]);

  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>Count: {count}</button>
      {/* Este se re-renderiza en cada click */}
      <ExpensiveChild title="Sin memo" items={items} />
      {/* Este NO se re-renderiza en cada click (items no cambia) */}
      <MemoizedChild title="Con memo" items={items} />
    </div>
  );
}
```

## Comparación personalizada

Por defecto hace una comparación superficial, pero se puede proporcionar una función de comparación personalizada:

```jsx
const UserCard = memo(
  ({ user }) => <div>{user.name} - {user.email}</div>,
  (prevProps, nextProps) => {
    // Devuelve true si las props son "iguales" (NO debe re-renderizar)
    return prevProps.user.id === nextProps.user.id &&
           prevProps.user.name === nextProps.user.name;
  }
);
```

## Problema con funciones como props

La comparación superficial falla con funciones porque cada renderizado crea nuevas referencias:

```jsx
function Parent() {
  const [count, setCount] = useState(0);

  // ❌ Nueva función en cada render → memo no ayuda
  const handleClick = () => console.log('click');

  // ✅ Misma referencia → memo funciona correctamente
  const handleClick = useCallback(() => console.log('click'), []);

  return <MemoizedChild onClick={handleClick} />;
}
```

## Cuándo usar React.memo

- Componentes que se renderizan frecuentemente con las mismas props.
- Componentes con renderizado costoso (listas largas, cálculos).
- Componentes hijos en listas grandes.

**No** tiene sentido en: componentes que siempre reciben props nuevas, componentes muy simples donde el costo de la comparación supera el ahorro, o componentes que reciben children (que siempre crea nuevas referencias).

**Consejo de entrevista**: `React.memo` debe usarse junto con `useCallback` para las funciones pasadas como props. De lo contrario, la memoización no tendrá efecto. Con React Compiler (React 19+), muchos de estos casos se optimizan automáticamente.
