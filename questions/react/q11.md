React Router es la librería de enrutamiento más popular para React. Permite crear aplicaciones de página única (SPA) con múltiples "páginas" o vistas, gestionando la navegación sin recargar la página. La versión 6 (y sus iteraciones) introdujo cambios significativos respecto a versiones anteriores.

**Configuración básica con React Router v6:**

```jsx
import { createBrowserRouter, RouterProvider } from 'react-router-dom';
import { lazy, Suspense } from 'react';

// Lazy loading de rutas
const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));
const UserDetail = lazy(() => import('./pages/UserDetail'));

const router = createBrowserRouter([
  {
    path: '/',
    element: <RootLayout />,
    errorElement: <ErrorPage />,
    children: [
      { index: true, element: <Home /> },
      { path: 'about', element: <About /> },
      {
        path: 'users',
        element: <UsersLayout />,
        children: [
          { index: true, element: <UserList /> },
          { path: ':id', element: <UserDetail /> }
        ]
      }
    ]
  }
]);

function App() {
  return (
    <Suspense fallback={<LoadingSpinner />}>
      <RouterProvider router={router} />
    </Suspense>
  );
}
```

**Layout con Outlet para rutas anidadas:**

```jsx
import { Outlet, Link, NavLink } from 'react-router-dom';

function RootLayout() {
  return (
    <div>
      <nav>
        <NavLink to="/" end className={({ isActive }) => isActive ? 'active' : ''}>
          Inicio
        </NavLink>
        <NavLink to="/users">Usuarios</NavLink>
      </nav>
      <main>
        <Outlet /> {/* Aquí se renderizan las rutas hijas */}
      </main>
    </div>
  );
}
```

**Hooks de React Router:**

```jsx
import { useNavigate, useParams, useSearchParams, useLocation } from 'react-router-dom';

function UserDetail() {
  const { id } = useParams(); // /users/:id
  const navigate = useNavigate();
  const [searchParams, setSearchParams] = useSearchParams();
  const location = useLocation();

  const tab = searchParams.get('tab') || 'profile';

  const goBack = () => navigate(-1); // Equivalente a window.history.back()
  const goToEdit = () => navigate(`/users/${id}/edit`, { state: { from: location } });

  return (
    <div>
      <p>Usuario ID: {id}</p>
      <button onClick={goBack}>Volver</button>
      <button onClick={goToEdit}>Editar</button>
      <button onClick={() => setSearchParams({ tab: 'settings' })}>
        Ver configuración
      </button>
    </div>
  );
}
```

**Data loading con loaders (React Router 6.4+):**

```jsx
// Carga de datos integrada en el router
const router = createBrowserRouter([
  {
    path: 'users/:id',
    element: <UserDetail />,
    loader: async ({ params }) => {
      const user = await fetch(`/api/users/${params.id}`).then(r => r.json());
      if (!user) throw new Response('Not Found', { status: 404 });
      return user;
    }
  }
]);

// En el componente
import { useLoaderData } from 'react-router-dom';

function UserDetail() {
  const user = useLoaderData();
  return <div>{user.name}</div>;
}
```

**Rutas protegidas:**

```jsx
function ProtectedRoute({ children }) {
  const { isAuthenticated } = useAuth();
  const location = useLocation();

  if (!isAuthenticated) {
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  return children;
}
```

**Consejo de entrevista**: React Router v6.4+ introdujo los "Data APIs" (loaders, actions) que acercan React Router a frameworks como Remix, permitiendo co-localizar la lógica de carga de datos con la definición de rutas. Esto es un patrón cada vez más común en aplicaciones modernas.
