Optimizar el rendimiento en React implica **evitar re-renders innecesarios**, reducir el trabajo computacional y mejorar los tiempos de carga. React es rápido por defecto, pero aplicaciones complejas necesitan optimizaciones estratégicas.

## 1. React.memo — Evitar re-renders de componentes puros

```jsx
// Sin memo: re-renderiza cada vez que el padre re-renderiza
function TarjetaProducto({ nombre, precio }) {
  console.log('re-render:', nombre);
  return <div>{nombre}: {precio}€</div>;
}

// Con memo: solo re-renderiza si sus props cambian
const TarjetaProducto = React.memo(function TarjetaProducto({ nombre, precio }) {
  return <div>{nombre}: {precio}€</div>;
});

// Con función de comparación personalizada
const TarjetaProducto = React.memo(
  function TarjetaProducto({ producto }) {
    return <div>{producto.nombre}</div>;
  },
  (prevProps, nextProps) => prevProps.producto.id === nextProps.producto.id
  // true = son iguales → NO re-renderizar
);
```

## 2. useCallback — Estabilizar referencias de funciones

```jsx
function ListaProductos({ productos, categoriaId }) {
  // ❌ Nueva función en cada render → React.memo del hijo no sirve
  const handleClick = (id) => agregarAlCarrito(id, categoriaId);

  // ✅ Misma referencia mientras categoriaId no cambie
  const handleClick = useCallback(
    (id) => agregarAlCarrito(id, categoriaId),
    [categoriaId]
  );

  return productos.map(p => (
    <TarjetaProducto key={p.id} producto={p} onComprar={handleClick} />
  ));
}
```

## 3. useMemo — Memorizar cálculos costosos

```jsx
function Dashboard({ pedidos, filtros }) {
  // ❌ Recalcula en cada render aunque pedidos y filtros no cambien
  const estadisticas = calcularEstadisticas(pedidos, filtros);

  // ✅ Solo recalcula si pedidos o filtros cambian
  const estadisticas = useMemo(
    () => calcularEstadisticas(pedidos, filtros),
    [pedidos, filtros]
  );

  // ✅ También para evitar crear nuevos objetos en props
  const estilos = useMemo(() => ({
    color: tema === 'oscuro' ? 'white' : 'black',
    fontSize: tamano * 1.5
  }), [tema, tamano]);

  return <GraficoEstadisticas datos={estadisticas} style={estilos} />;
}
```

## 4. Code Splitting y lazy loading

```jsx
import { lazy, Suspense } from 'react';

// Cargar componentes pesados solo cuando se necesitan
const EditorRiqueza = lazy(() => import('./EditorRiqueza'));
const GraficoAvanzado = lazy(() => import('./GraficoAvanzado'));
const MapaInteractivo = lazy(() => import('./MapaInteractivo'));

function App() {
  const [mostrarEditor, setMostrarEditor] = useState(false);

  return (
    <div>
      <button onClick={() => setMostrarEditor(true)}>Abrir Editor</button>

      {mostrarEditor && (
        <Suspense fallback={<div>Cargando editor...</div>}>
          <EditorRiqueza />
        </Suspense>
      )}
    </div>
  );
}
```

## 5. Virtualización de listas largas

```jsx
// Para listas de miles de elementos: react-window o TanStack Virtual
import { FixedSizeList } from 'react-window';

function ListaVirtualizada({ items }) {
  const Row = ({ index, style }) => (
    <div style={style} className="fila">
      {items[index].nombre}
    </div>
  );

  return (
    // Solo renderiza los elementos visibles en pantalla
    <FixedSizeList
      height={500}        // Altura del contenedor
      itemCount={items.length}
      itemSize={50}       // Altura de cada fila en px
      width="100%"
    >
      {Row}
    </FixedSizeList>
  );
}
// Sin virtualización: 10.000 filas en el DOM
// Con virtualización: ~15 filas visibles en el DOM
```

## 6. Optimizar el Context API

```jsx
// ❌ Un solo contexto con muchos valores: todo re-renderiza si algo cambia
const AppContext = createContext({ usuario, tema, idioma, carrito, ... });

// ✅ Separar contextos por dominio de cambio
const UsuarioContext = createContext(null);    // Cambia raramente
const TemaContext = createContext('claro');     // Cambia con tema
const CarritoContext = createContext([]);       // Cambia frecuentemente

// Los componentes solo se re-renderizan cuando su contexto específico cambia
```

## 7. useTransition para actualizaciones no urgentes

```jsx
const [isPending, startTransition] = useTransition();

// Las actualizaciones costosas no bloquean la UI
startTransition(() => {
  setFiltroComplejo(nuevoFiltro);  // Costoso de calcular
});
```

## Regla del 80/20 para optimizaciones

1. **Medir primero** con Profiler antes de optimizar
2. **React.memo** para componentes puros con props frecuentes
3. **useCallback** solo cuando la función se pasa a un componente memoizado
4. **useMemo** solo para cálculos verdaderamente costosos (>1ms)
5. **Lazy loading** siempre para bundles grandes
6. **Virtualización** para listas > 100 elementos

**Consejo de entrevista**: El error más común es sobre-optimizar con `useMemo` y `useCallback` en todas partes. Estas optimizaciones tienen su propio costo (comparación de dependencias). El flujo correcto es: medir con el Profiler → identificar el cuello de botella → aplicar la optimización específica. Mencionar que React 19 y el React Compiler automatizan muchas de estas memoizaciones demuestra conocimiento avanzado.
