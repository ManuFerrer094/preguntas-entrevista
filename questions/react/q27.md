---
title: ¿Qué son las keys en React y por qué son importantes?

Las **keys** son atributos especiales que React usa para identificar qué elementos de una lista han cambiado, se han añadido o se han eliminado. Son fundamentales para el algoritmo de reconciliación y el rendimiento.

## Por qué existen las keys

```jsx
// Sin keys (o con índices), React tiene que adivinar qué cambió:
const lista = ['Ana', 'Carlos', 'María'];

// Si eliminamos 'Carlos' y añadimos 'Pedro' al principio:
// Nueva: ['Pedro', 'Ana', 'María']
// React sin keys: modificó 3 nodos (caro)
// React con keys: elimina 'Carlos', mueve 'Ana' y 'María', añade 'Pedro' (eficiente)
```

## Uso correcto: ID único y estable

```jsx
// ✅ Usar ID único del dato (no el índice)
function ListaProductos({ productos }) {
  return (
    <ul>
      {productos.map(producto => (
        <li key={producto.id}>         {/* ID único y estable */}
          {producto.nombre} — {producto.precio}€
        </li>
      ))}
    </ul>
  );
}

// ✅ Con componentes también:
function ListaUsuarios({ usuarios }) {
  return (
    <div>
      {usuarios.map(usuario => (
        <PerfilCard
          key={usuario.id}            {/* key en el componente raíz del map */}
          nombre={usuario.nombre}
          avatar={usuario.avatar}
        />
      ))}
    </div>
  );
}
```

## Por qué el índice como key es problemático

```jsx
// ❌ Usar el índice como key (problemático con listas que se reordenan/filtran)
{items.map((item, index) => (
  <Item key={index} texto={item.texto} />
))}

// Problema concreto: formulario con inputs
function ListaFormulario() {
  const [items, setItems] = useState([
    { id: 1, texto: 'Ana' },
    { id: 2, texto: 'Carlos' },
    { id: 3, texto: 'María' }
  ]);

  const eliminarPrimero = () => setItems(items.slice(1));

  return (
    <div>
      {items.map((item, index) => (
        // ❌ Con key={index}: al eliminar el primero, el input de 'Carlos'
        // hereda el estado del input anterior (el de 'Ana')
        <input key={index} defaultValue={item.texto} />
      ))}
      <button onClick={eliminarPrimero}>Eliminar primero</button>
    </div>
  );
}

// ✅ Con key={item.id}: React sabe exactamente qué input destruir
{items.map(item => (
  <input key={item.id} defaultValue={item.texto} />
))}
```

## Generar keys cuando no hay ID

```jsx
// Si los datos no tienen ID, generar uno al crear el dato (no durante el render):

// ✅ Generar el ID al crear el elemento:
const [comentarios, setComentarios] = useState([]);

const agregarComentario = (texto) => {
  setComentarios(prev => [
    ...prev,
    { id: crypto.randomUUID(), texto, fecha: new Date() }
  ]);
};

// ❌ Generar el ID durante el render (diferente en cada render):
{comentarios.map(c => (
  <div key={Math.random()}>  {/* ❌ Nueva key en cada render = re-mount en cada render */}
    {c.texto}
  </div>
))}
```

## Keys para forzar re-mount (truco avanzado)

```jsx
// Cambiar la key de un componente lo desmonta y remonta completamente
// Útil para resetear el estado interno de un componente

function VisualizadorDatos({ filtro }) {
  const [key, setKey] = useState(0);

  const resetear = () => setKey(k => k + 1);

  return (
    <div>
      <button onClick={resetear}>Resetear vista</button>
      <GraficoComplejo
        key={key}        {/* Cambiar la key fuerza el re-mount completo */}
        filtro={filtro}
      />
    </div>
  );
}

// También útil para resetear un componente cuando cambia una prop:
function EditorTexto({ documentoId }) {
  return (
    <Editor
      key={documentoId}   {/* Se re-monta completamente al cambiar de documento */}
      id={documentoId}
    />
  );
}
```

## Cuándo usar cada tipo de key

| Situación                        | Key recomendada          |
|----------------------------------|--------------------------|
| Lista de datos con ID de base de datos | `item.id`         |
| Lista estática que nunca cambia  | índice (aceptable)       |
| Lista que se reordena/filtra     | ID estable obligatorio   |
| Lista de formularios con inputs  | ID estable obligatorio   |
| Sin ID disponible                | `crypto.randomUUID()` al crear |

**Consejo de entrevista**: El error más común en React de producción es usar el índice como key en listas que se reordenan, filtran o tienen inputs con estado. Explica el problema del "estado huérfano": cuando el índice cambia, React reutiliza el componente DOM del índice anterior, lo que lleva a bugs sutiles donde el estado interno (inputs, checkboxes, animaciones) no corresponde al dato correcto.
