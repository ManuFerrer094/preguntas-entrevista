---
title: ¿Qué es el hook useId en React?
difficulty: easy
tags: [react, hooks, useId, accessibility, ssr]
---

`useId` es un hook de React 18 que genera **IDs únicos y estables** para atributos de accesibilidad como `htmlFor`, `aria-describedby` o `aria-labelledby`. Está diseñado específicamente para evitar problemas de hidratación en aplicaciones con SSR (Server Side Rendering).

## El problema que resuelve

```jsx
// ❌ Sin useId: IDs generados manualmente son problemáticos en SSR
let contador = 0;

function CampoEmail({ label }) {
  const id = `email-${++contador}`;  // El contador es diferente en servidor y cliente

  // Error: "Hydration failed because attribute `id` did not match"
  return (
    <div>
      <label htmlFor={id}>{label}</label>
      <input id={id} type="email" />
    </div>
  );
}
```

## Solución con useId

```jsx
import { useId } from 'react';

function CampoEmail({ label }) {
  const id = useId();  // Mismo valor en servidor y cliente

  return (
    <div>
      <label htmlFor={id}>{label}</label>
      <input id={id} type="email" />
    </div>
  );
}

// Cada instancia del componente obtiene un ID único
// Ejemplo de IDs generados: ":r0:", ":r1:", ":r2:", ...
<CampoEmail label="Email principal" />    {/* id=":r0:" */}
<CampoEmail label="Email secundario" />   {/* id=":r1:" */}
```

## Múltiples IDs desde un mismo useId

```jsx
function FormularioDirección() {
  // Un solo useId puede usarse como prefijo para múltiples atributos relacionados
  const idBase = useId();

  const idCalle = `${idBase}-calle`;
  const idCiudad = `${idBase}-ciudad`;
  const idCP = `${idBase}-cp`;

  return (
    <fieldset>
      <legend>Dirección</legend>

      <div>
        <label htmlFor={idCalle}>Calle</label>
        <input id={idCalle} type="text" />
      </div>

      <div>
        <label htmlFor={idCiudad}>Ciudad</label>
        <input id={idCiudad} type="text" />
      </div>

      <div>
        <label htmlFor={idCP}>Código Postal</label>
        <input id={idCP} type="text" />
      </div>
    </fieldset>
  );
}
```

## Uso con aria-describedby

```jsx
function CampoConAyuda({ label, ayuda, type = 'text' }) {
  const idInput = useId();
  const idAyuda = useId();

  return (
    <div>
      <label htmlFor={idInput}>{label}</label>
      <input
        id={idInput}
        type={type}
        aria-describedby={idAyuda}  // Asocia el input con el texto de ayuda
      />
      <small id={idAyuda} className="ayuda">
        {ayuda}
      </small>
    </div>
  );
}

// Uso:
<CampoConAyuda
  label="Contraseña"
  ayuda="Debe tener al menos 8 caracteres, una mayúscula y un número."
  type="password"
/>
```

## Componente de radio group accesible

```jsx
function RadioGroup({ nombre, opciones, valor, onChange }) {
  const idBase = useId();

  return (
    <fieldset>
      <legend>{nombre}</legend>
      {opciones.map((opcion) => {
        const idOpcion = `${idBase}-${opcion.valor}`;
        return (
          <div key={opcion.valor}>
            <input
              type="radio"
              id={idOpcion}
              name={idBase}  // Agrupa los radios
              value={opcion.valor}
              checked={valor === opcion.valor}
              onChange={() => onChange(opcion.valor)}
            />
            <label htmlFor={idOpcion}>{opcion.etiqueta}</label>
          </div>
        );
      })}
    </fieldset>
  );
}

// Uso:
<RadioGroup
  nombre="Tamaño"
  opciones={[
    { valor: 'S', etiqueta: 'Pequeño' },
    { valor: 'M', etiqueta: 'Mediano' },
    { valor: 'L', etiqueta: 'Grande' }
  ]}
  valor={talla}
  onChange={setTalla}
/>
```

## Reglas importantes

```jsx
// ✅ Usar useId para atributos de accesibilidad DOM
const id = useId();

// ❌ NO usar como key de lista (no está diseñado para eso)
items.map(item => <li key={useId()}>...</li>);  // Error: hooks en loops

// ❌ NO generar IDs para datos de base de datos o lógica de negocio
const idProducto = useId();  // ❌ Usa crypto.randomUUID() o uuid para eso

// ❌ Los IDs de useId tienen formato especial (:r0:) y no son amigables como claves
```

## Cuándo usar useId vs otras alternativas

| Necesidad                         | Solución                      |
|-----------------------------------|-------------------------------|
| IDs para accesibilidad (htmlFor, aria-*) | `useId` ✅         |
| Keys de listas                    | ID de la BD o `crypto.randomUUID()` |
| IDs persistentes en base de datos | `crypto.randomUUID()` o UUID  |
| IDs en tests                      | `data-testid` hardcodeado     |

**Consejo de entrevista**: `useId` resuelve un problema real y frecuentemente ignorado: la accesibilidad correcta con `htmlFor`/`id` en componentes reutilizables. Antes de React 18, los desarrolladores usaban contadores globales o `Math.random()`, que causaban errores de hidratación en SSR. Mencionar que el prefijo `:r` en los IDs es intencional para no colisionar con IDs del desarrollador demuestra conocimiento profundo.
