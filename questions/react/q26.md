Los **componentes de clase** (pre-hooks) y los **componentes funcionales** (con hooks, React 16.8+) son las dos formas de crear componentes en React. Los funcionales son hoy el estándar recomendado.

## Comparativa de sintaxis

```jsx
// ---- COMPONENTE DE CLASE ----
import { Component } from 'react';

class Saludo extends Component {
  constructor(props) {
    super(props);
    this.state = { veces: 0 };
    this.incrementar = this.incrementar.bind(this);  // Binding necesario
  }

  incrementar() {
    this.setState(prev => ({ veces: prev.veces + 1 }));
  }

  componentDidMount() {
    document.title = `Saludo a ${this.props.nombre}`;
  }

  componentDidUpdate(prevProps, prevState) {
    if (prevProps.nombre !== this.props.nombre) {
      document.title = `Saludo a ${this.props.nombre}`;
    }
  }

  componentWillUnmount() {
    document.title = 'App';
  }

  render() {
    return (
      <div>
        <h1>Hola, {this.props.nombre}!</h1>
        <p>Has saludado {this.state.veces} veces</p>
        <button onClick={this.incrementar}>Saludar</button>
      </div>
    );
  }
}
```

```jsx
// ---- COMPONENTE FUNCIONAL ----
import { useState, useEffect } from 'react';

function Saludo({ nombre }) {
  const [veces, setVeces] = useState(0);

  useEffect(() => {
    document.title = `Saludo a ${nombre}`;
    return () => { document.title = 'App'; };  // Cleanup
  }, [nombre]);  // Solo se re-ejecuta si `nombre` cambia

  return (
    <div>
      <h1>Hola, {nombre}!</h1>
      <p>Has saludado {veces} veces</p>
      <button onClick={() => setVeces(v => v + 1)}>Saludar</button>
    </div>
  );
}
```

## Equivalencia de lifecycle methods y hooks

| Clase (lifecycle)            | Funcional (hooks)                           |
|------------------------------|---------------------------------------------|
| `constructor`                | `useState(valorInicial)`                    |
| `componentDidMount`          | `useEffect(() => {...}, [])`                |
| `componentDidUpdate`         | `useEffect(() => {...}, [dep])`             |
| `componentWillUnmount`       | `useEffect(() => { return () => {...} }, [])` |
| `shouldComponentUpdate`      | `React.memo` + `useMemo`/`useCallback`      |
| `getDerivedStateFromProps`   | `useState` + `useEffect`                    |
| `getSnapshotBeforeUpdate`    | `useLayoutEffect` con ref                   |
| `componentDidCatch`          | Error Boundary (solo clases, aún hoy)       |

## Error Boundaries — única ventaja restante de las clases

```jsx
// Los Error Boundaries SOLO pueden implementarse como componentes de clase (en 2024)
class ErrorBoundary extends Component {
  state = { hayError: false, error: null };

  static getDerivedStateFromError(error) {
    return { hayError: true, error };
  }

  componentDidCatch(error, info) {
    console.error('Error capturado:', error, info.componentStack);
  }

  render() {
    if (this.state.hayError) {
      return <h2>Algo salió mal: {this.state.error?.message}</h2>;
    }
    return this.props.children;
  }
}

// Uso:
<ErrorBoundary>
  <ComponenteQuePodriaFallar />
</ErrorBoundary>
```

## Reutilización de lógica: HOC vs Custom Hook

```jsx
// ---- CLASE: Higher-Order Component (HOC) ----
function withAuth(WrappedComponent) {
  return class extends Component {
    state = { usuario: null };

    componentDidMount() {
      this.setState({ usuario: getCurrentUser() });
    }

    render() {
      return <WrappedComponent usuario={this.state.usuario} {...this.props} />;
    }
  };
}

const PaginaProtegida = withAuth(Pagina);


// ---- FUNCIONAL: Custom Hook (más limpio y reutilizable) ----
function useAuth() {
  const [usuario, setUsuario] = useState(null);

  useEffect(() => {
    setUsuario(getCurrentUser());
  }, []);

  return { usuario };
}

function Pagina() {
  const { usuario } = useAuth();  // ✅ Directamente, sin wrapper
  return <div>Hola, {usuario?.nombre}</div>;
}
```

## Por qué preferir funcionales hoy

1. **Menos código**: no hay `this`, binding, ni `super(props)`
2. **Custom Hooks**: la reutilización de lógica es mucho más limpia
3. **Mejor composición**: los hooks se pueden combinar fácilmente
4. **Closures**: sin problema con `this` context
5. **Rendimiento**: React puede optimizar mejor los funcionales
6. **Concurrente**: las futuras características de React (Concurrent Mode) están diseñadas para funcionales

**Consejo de entrevista**: En una entrevista moderna, debes saber por qué los funcionales son preferidos y conocer la equivalencia con los lifecycle methods. El único caso donde aún necesitas clases es para implementar `componentDidCatch` (Error Boundaries), aunque React promete una solución con hooks en el futuro. Mencionar esto demuestra conocimiento actualizado del ecosistema.
