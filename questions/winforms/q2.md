---
title: ¿Cuál es la diferencia entre los eventos Validating y Validated en WinForms?
difficulty: medium
tags: [winforms, validacion, eventos, controles]
---

WinForms proporciona dos eventos de validación que se disparan cuando un control pierde el foco:

**`Validating`:**
- Se dispara **antes** de que el control pierda el foco, permitiendo verificar si los datos ingresados son válidos.
- Si los datos no son correctos, se puede cancelar el cambio de foco estableciendo `e.Cancel = true`, lo que mantiene el foco en el control hasta que se corrija el valor.
- Es el lugar adecuado para mostrar mensajes de error con el componente `ErrorProvider`.

**`Validated`:**
- Se dispara **después** de `Validating`, solo si la validación fue exitosa (es decir, si `e.Cancel` no fue establecido a `true`).
- Se usa para ejecutar lógica que depende de que el valor sea válido, como actualizar otros controles o limpiar mensajes de error.

```csharp
private void txtEdad_Validating(object sender, CancelEventArgs e)
{
    if (!int.TryParse(txtEdad.Text, out int edad) || edad < 0 || edad > 120)
    {
        errorProvider1.SetError(txtEdad, "Ingrese una edad válida (0-120).");
        e.Cancel = true; // mantiene el foco en el control
    }
}

private void txtEdad_Validated(object sender, EventArgs e)
{
    errorProvider1.SetError(txtEdad, ""); // limpia el error
}
```

> **Nota:** Para que los eventos `Validating` y `Validated` se disparen, la propiedad `CausesValidation` del control debe ser `true` (valor por defecto).
