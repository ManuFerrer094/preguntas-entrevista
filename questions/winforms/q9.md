---
title: ¿Cómo se implementa el patrón MVP (Model-View-Presenter) en WinForms?
difficulty: hard
tags: [winforms, mvp, patron-diseno, arquitectura, testabilidad]
---

El patrón **MVP (Model-View-Presenter)** es el más adecuado para WinForms porque desacopla la lógica de presentación del formulario (View), facilitando las pruebas unitarias sin necesidad de instanciar controles de Windows.

**Estructura:**

- **Model:** clases de dominio/datos.
- **View:** el formulario (`Form`), implementa una interfaz.
- **Presenter:** clase C# pura que orquesta la lógica y se comunica con View y Model a través de interfaces.

**Interfaz de la vista:**

```csharp
public interface ILoginView
{
    string Usuario { get; }
    string Password { get; }
    void MostrarError(string mensaje);
    event EventHandler LoginClick;
}
```

**Formulario (View):**

```csharp
public partial class LoginForm : Form, ILoginView
{
    public string Usuario => txtUsuario.Text;
    public string Password => txtPassword.Text;
    public event EventHandler LoginClick;

    private void btnLogin_Click(object sender, EventArgs e)
        => LoginClick?.Invoke(this, EventArgs.Empty);

    public void MostrarError(string mensaje)
        => MessageBox.Show(mensaje, "Error", MessageBoxButtons.OK, MessageBoxIcon.Error);
}
```

**Presenter:**

```csharp
public class LoginPresenter
{
    private readonly ILoginView _view;
    private readonly IAuthService _authService;

    public LoginPresenter(ILoginView view, IAuthService authService)
    {
        _view = view;
        _authService = authService;
        _view.LoginClick += OnLoginClick;
    }

    private async void OnLoginClick(object sender, EventArgs e)
    {
        if (await _authService.AutenticarAsync(_view.Usuario, _view.Password))
            Application.Run(new MainForm());
        else
            _view.MostrarError("Usuario o contraseña incorrectos.");
    }
}
```

El Presenter es completamente testeable con mocks de `ILoginView` e `IAuthService`.
