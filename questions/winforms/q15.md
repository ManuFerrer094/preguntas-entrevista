---
title: ¿Cómo actualizas la UI desde un hilo secundario en WinForms con Invoke y BeginInvoke?
difficulty: medium
tags: [winforms, invoke, begininvoke, hilos, hilo-ui, thread-safety]
---

En WinForms, **los controles solo pueden ser modificados desde el hilo de la UI** (hilo principal). Intentar actualizar un control desde un hilo secundario lanza una `InvalidOperationException`. Para solucionar esto se usan `Invoke` y `BeginInvoke`.

**Diferencia entre Invoke y BeginInvoke:**

| Método | Comportamiento |
|---|---|
| `Control.Invoke` | **Síncrono** — el hilo secundario espera a que la UI ejecute el delegado. |
| `Control.BeginInvoke` | **Asíncrono** — el hilo secundario continúa sin esperar. |

**Patrón clásico con `InvokeRequired`:**

```csharp
private void ActualizarEtiqueta(string texto)
{
    if (label1.InvokeRequired)
    {
        // Estamos en un hilo secundario: marshaling al hilo de la UI
        label1.Invoke(new Action(() => label1.Text = texto));
    }
    else
    {
        // Ya estamos en el hilo de la UI
        label1.Text = texto;
    }
}
```

**Usando BeginInvoke (no bloquea el hilo secundario):**

```csharp
private void ActualizarProgreso(int porcentaje)
{
    progressBar1.BeginInvoke(new Action(() =>
    {
        progressBar1.Value = porcentaje;
        lblEstado.Text = $"Progreso: {porcentaje}%";
    }));
}
```

**Ejemplo completo con Thread manual:**

```csharp
private void btnIniciar_Click(object sender, EventArgs e)
{
    var hilo = new Thread(() =>
    {
        for (int i = 0; i <= 100; i += 10)
        {
            Thread.Sleep(300); // Simula trabajo pesado

            // Actualiza la UI de forma segura
            this.Invoke(new Action(() =>
            {
                progressBar1.Value = i;
                lblEstado.Text = $"Procesando: {i}%";
            }));
        }

        this.Invoke(new Action(() =>
        {
            lblEstado.Text = "¡Completado!";
            btnIniciar.Enabled = true;
        }));
    });

    btnIniciar.Enabled = false;
    hilo.IsBackground = true;
    hilo.Start();
}
```

**Alternativa moderna recomendada — `async`/`await`:**

Con `async`/`await`, el contexto de sincronización de WinForms devuelve automáticamente la ejecución al hilo de la UI tras el `await`, eliminando la necesidad de `Invoke`:

```csharp
private async void btnIniciar_Click(object sender, EventArgs e)
{
    btnIniciar.Enabled = false;
    for (int i = 0; i <= 100; i += 10)
    {
        await Task.Delay(300); // No bloquea la UI
        progressBar1.Value = i;
        lblEstado.Text = $"Procesando: {i}%";
    }
    lblEstado.Text = "¡Completado!";
    btnIniciar.Enabled = true;
}
```

> **Regla general:** usa `async`/`await` para código nuevo y reserva `Invoke`/`BeginInvoke` para cuando trabajes con hilos manuales o `BackgroundWorker` heredado.
