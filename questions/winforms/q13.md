---
title: ¿Cómo se implementa la comunicación entre formularios en WinForms?
difficulty: medium
tags: [winforms, comunicacion-formularios, eventos, propiedades, instancia]
---

Existen varios patrones para intercambiar datos entre formularios en WinForms, cada uno adecuado a un caso de uso diferente.

**1. Pasar datos al abrir un formulario hijo (constructor):**

```csharp
// Formulario padre
var detalle = new DetalleForm(clienteSeleccionado);
detalle.ShowDialog();
```

```csharp
// Formulario hijo
public partial class DetalleForm : Form
{
    public DetalleForm(Cliente cliente)
    {
        InitializeComponent();
        lblNombre.Text = cliente.Nombre;
    }
}
```

**2. Recuperar datos del formulario hijo después de ShowDialog:**

```csharp
// Formulario hijo expone propiedades públicas
public string NombreEditado { get; private set; }

private void btnAceptar_Click(object sender, EventArgs e)
{
    NombreEditado = txtNombre.Text;
    DialogResult = DialogResult.OK;
    Close();
}
```

```csharp
// Padre lo consume
using var dlg = new EditarForm();
if (dlg.ShowDialog() == DialogResult.OK)
    RefrescarLista(dlg.NombreEditado);
```

**3. Comunicación hijo → padre mediante eventos:**

```csharp
// Hijo declara un evento
public event EventHandler<string> DatoActualizado;

private void NotificarCambio(string valor)
    => DatoActualizado?.Invoke(this, valor);
```

```csharp
// Padre se suscribe
var hijo = new HijoForm();
hijo.DatoActualizado += (s, valor) => ActualizarUI(valor);
hijo.Show();
```

**4. Acceso a la instancia del formulario padre:**

```csharp
public partial class HijoForm : Form
{
    private readonly MainForm _padre;

    public HijoForm(MainForm padre)
    {
        InitializeComponent();
        _padre = padre;
    }

    private void btnActualizar_Click(object sender, EventArgs e)
        => _padre.RefrescarDatos();
}
```

**Recomendación:** para aplicaciones medianas o grandes, considera un bus de eventos o el patrón MVP/MVVM para evitar dependencias directas entre formularios.
