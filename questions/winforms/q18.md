---
title: ¿Cómo harías el deployment de una aplicación WinForms?

Existen varias estrategias para desplegar una aplicación WinForms, cada una adaptada a un escenario diferente.

**1. ClickOnce (integrado en Visual Studio)**

La opción más sencilla para distribución a través de red o web. Permite actualizaciones automáticas sin necesidad de privilegios de administrador.

- Visual Studio → Proyecto → **Publicar** → Elegir destino (carpeta, FTP, Azure…).
- Genera un manifiesto de publicación y un instalador `.application`.
- El usuario instala con un clic y las actualizaciones se aplican automáticamente.

```csharp
// Verificar y aplicar actualizaciones en código
if (ApplicationDeployment.IsNetworkDeployed)
{
    var deploy = ApplicationDeployment.CurrentDeployment;
    if (deploy.CheckForUpdate())
        deploy.Update();
}
```

**Cuándo usarlo:** intranets corporativas, usuarios sin privilegios de administrador, necesidad de auto-actualización sencilla.

**2. Instalador MSI con WiX Toolset**

Genera un paquete `.msi` estándar de Windows con control total: accesos directos, entradas de registro, requisitos previos, desinstalación limpia.

```xml
<!-- Fragmento básico de un archivo .wxs -->
<Product Id="*" Name="MiApp" Version="1.0.0" Manufacturer="MiEmpresa" Language="3082">
  <!-- Language 3082 = Español (España, LCID) -->
  <Package InstallerVersion="500" Compressed="yes"/>
  <Feature Id="ProductFeature">
    <ComponentGroupRef Id="ProductComponents"/>
  </Feature>
</Product>
```

**Cuándo usarlo:** distribución pública, aplicaciones empresariales que requieren entradas de registro, componentes adicionales (redistributables) o desinstalación controlada.

**3. Inno Setup**

Alternativa gratuita y muy popular para crear instaladores `.exe` personalizados con scripts.

```pascal
[Setup]
AppName=MiAplicacion
AppVersion=1.0
DefaultDirName={autopf}\MiAplicacion
DefaultGroupName=MiAplicacion
OutputBaseFilename=Setup_MiAplicacion
```

**4. Self-Contained / Single-File (.NET moderno)**

En proyectos que usan .NET 5+, puedes publicar la aplicación como un único ejecutable que incluye el runtime:

```bash
dotnet publish -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true
```

Esto genera un único `.exe` sin necesidad de que el usuario tenga instalado el .NET Runtime.

**Comparativa:**

| Método | Instalador | Auto-actualización | Requiere admin | Complejidad |
|---|---|---|---|---|
| ClickOnce | Sí | ✅ Sí | No (usuario) | Baja |
| MSI / WiX | Sí | Manual | Sí | Media-alta |
| Inno Setup | Sí | Manual | Sí | Media |
| Self-Contained | No (xcopy) | Manual | No | Baja |

> **Recomendación:** para aplicaciones empresariales nuevas en .NET moderno, combina **Self-Contained + Single-File** para la distribución sencilla, o **WiX** cuando se necesite un instalador profesional con prerequisitos.
