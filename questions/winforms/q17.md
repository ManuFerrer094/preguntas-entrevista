---
title: ¿Qué diferencia hay entre WinForms y WPF?
difficulty: easy
tags: [winforms, wpf, diferencias, xaml, ui, escritorio]
---

**WinForms** y **WPF** son las dos tecnologías principales de Microsoft para crear aplicaciones de escritorio Windows, pero tienen enfoques muy diferentes.

**Comparativa:**

| Característica | WinForms | WPF |
|---|---|---|
| Aparición | 2002 (.NET 1.0) | 2006 (.NET 3.0) |
| Diseño de UI | Diseñador visual drag-and-drop | XAML (lenguaje de marcado declarativo) |
| Motor de renderizado | GDI+ (píxeles) | DirectX (vectorial, escalable) |
| Data Binding | Básico (BindingSource) | Avanzado (MVVM, INotifyPropertyChanged) |
| Animaciones y estilos | Limitados | Nativos y potentes (Storyboards, Triggers) |
| Curva de aprendizaje | Baja | Media-alta (XAML + patrones) |
| Patrón recomendado | MVP | MVVM |
| Personalización visual | Difícil (Owner Drawing) | Sencilla (ControlTemplate, DataTemplate) |
| Resolución adaptativa | No (fija en píxeles) | Sí (unidades independientes del dispositivo) |

**Ejemplo de botón en WinForms (diseñador):**
```csharp
// Código generado por el diseñador (Form1.Designer.cs)
this.btnGuardar = new System.Windows.Forms.Button();
this.btnGuardar.Text = "Guardar";
this.btnGuardar.Click += new System.EventHandler(this.btnGuardar_Click);
```

**Ejemplo equivalente en WPF (XAML):**
```xml
<Button Content="Guardar"
        Command="{Binding GuardarCommand}"
        Width="100" Height="30"
        Background="SteelBlue" Foreground="White"/>
```

**¿Cuándo elegir cada uno?**

- **WinForms:** proyectos legados, aplicaciones de gestión sencillas, equipos con poca experiencia en XAML o plazos de entrega cortos. Tooling maduro y gran cantidad de controles de terceros.
- **WPF:** aplicaciones con requisitos visuales avanzados (animaciones, temas, escalado de pantalla), cuando se requiere el patrón MVVM o cuando la aplicación debe adaptarse a distintas resoluciones y monitores 4K.

> Ambas tecnologías están disponibles en .NET moderno (.NET 6+), pero solo en Windows.
