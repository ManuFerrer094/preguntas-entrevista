---
title: ¿Cómo se usan los BackgroundWorker y Task en WinForms para evitar bloquear la UI?
difficulty: medium
tags: [winforms, backgroundworker, async, task, hilo-ui]
---

En WinForms, **toda operación larga ejecutada en el hilo de la UI** congela la ventana. Existen dos enfoques principales para ejecutar trabajo en segundo plano.

**1. `BackgroundWorker` (WinForms clásico):**

```csharp
private void btnProcesar_Click(object sender, EventArgs e)
{
    progressBar1.Value = 0;
    btnProcesar.Enabled = false;

    var worker = new BackgroundWorker { WorkerReportsProgress = true };
    worker.DoWork += (s, args) =>
    {
        for (int i = 0; i <= 100; i++)
        {
            Thread.Sleep(50); // Simulación
            (s as BackgroundWorker).ReportProgress(i);
        }
    };
    worker.ProgressChanged += (s, args) =>
        progressBar1.Value = args.ProgressPercentage;

    worker.RunWorkerCompleted += (s, args) =>
    {
        MessageBox.Show("¡Completado!");
        btnProcesar.Enabled = true;
    };
    worker.RunWorkerAsync();
}
```

**2. `async/await` con `Progress<T>` (moderno, recomendado):**

```csharp
private async void btnProcesar_Click(object sender, EventArgs e)
{
    btnProcesar.Enabled = false;
    progressBar1.Value = 0;

    var progreso = new Progress<int>(pct => progressBar1.Value = pct);

    await Task.Run(() =>
    {
        for (int i = 0; i <= 100; i++)
        {
            Thread.Sleep(50);
            (progreso as IProgress<int>).Report(i);
        }
    });

    MessageBox.Show("¡Completado!");
    btnProcesar.Enabled = true;
}
```

**Regla crítica:** nunca actualices controles de UI desde un hilo diferente al hilo principal. `async/await` gestiona esto automáticamente gracias al `SynchronizationContext` de WinForms. Con hilos manuales, usa `Control.Invoke()`:

```csharp
if (label1.InvokeRequired)
    label1.Invoke(new Action(() => label1.Text = "Actualizado"));
else
    label1.Text = "Actualizado";
```
