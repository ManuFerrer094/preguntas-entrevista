---
title: ¿Cómo se gestionan los recursos y se evitan fugas de memoria con IDisposable en WinForms?
difficulty: hard
tags: [winforms, idisposable, dispose, memoria, gc, handles]
---

WinForms trabaja con recursos no administrados de Windows (handles GDI+, HWND, handles de fichero) que el Garbage Collector no libera automáticamente. Es fundamental implementar el patrón `IDisposable` correctamente.

**Recursos que DEBEN liberarse explícitamente:**

- Objetos GDI+: `Bitmap`, `Graphics`, `Brush`, `Pen`, `Font`, `Icon`.
- Handles de ventana (`HWND`) — liberados por `Form.Dispose()`.
- Streams, conexiones a BD, timers.

**Patrón Dispose estándar en WinForms:**

```csharp
public partial class MiForm : Form
{
    private Bitmap _imagen;
    private System.Timers.Timer _timer;

    public MiForm()
    {
        InitializeComponent();
        _imagen = new Bitmap("logo.png");
        _timer = new System.Timers.Timer(1000);
        _timer.Elapsed += OnTick;
        _timer.Start();
    }

    // Visual Studio genera este método parcial en Designer.cs
    protected override void Dispose(bool disposing)
    {
        if (disposing)
        {
            components?.Dispose();   // Controles del diseñador
            _imagen?.Dispose();
            _timer?.Stop();
            _timer?.Dispose();
        }
        base.Dispose(disposing);
    }
}
```

**Uso de `using` para recursos temporales:**

```csharp
private void Form1_Paint(object sender, PaintEventArgs e)
{
    using var brush = new SolidBrush(Color.Blue);
    using var font  = new Font("Arial", 12);
    e.Graphics.DrawString("Hola", font, brush, 10, 10);
}
```

**Errores comunes:**

- Asignar un nuevo `Bitmap` a un `PictureBox.Image` sin disponer el anterior: `pictureBox1.Image?.Dispose(); pictureBox1.Image = nueva;`
- Suscribirse a eventos de objetos de larga vida sin desuscribirse → impide que el formulario sea recolectado por el GC (memory leak).

```csharp
// Desuscribirse en Dispose:
_timer.Elapsed -= OnTick;
```
