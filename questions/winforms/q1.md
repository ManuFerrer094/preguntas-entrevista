---
title: ¿Cuál es el orden de los eventos al iniciar y cerrar un formulario en WinForms?
difficulty: medium
tags: [winforms, eventos, ciclo-de-vida, formulario]
---

Conocer el ciclo de vida de un formulario WinForms es esencial para ubicar correctamente el código de inicialización y limpieza.

**Al iniciar un formulario**, los eventos se disparan en este orden:

1. `HandleCreated` — Se crea el identificador de ventana (HWND).
2. `BindingContextChanged` — Cambia el contexto de enlace de datos.
3. `Load` — El formulario se carga en memoria. **Aquí se suele inicializar datos.**
4. `VisibleChanged` — El formulario pasa a ser visible.
5. `Activated` — El formulario obtiene el foco de la aplicación.
6. `Shown` — El formulario se muestra por primera vez (solo ocurre una vez).

**Al cerrar un formulario**, los eventos se disparan en este orden:

1. `Closing` (obsoleto) → reemplazado por `FormClosing`.
2. `FormClosing` — Permite cancelar el cierre estableciendo `e.Cancel = true`.
3. `Closed` (obsoleto) → reemplazado por `FormClosed`.
4. `FormClosed` — El formulario ya se ha cerrado. Aquí se liberan recursos.
5. `Deactivate` — El formulario pierde el foco.

```csharp
private void Form1_Load(object sender, EventArgs e)
{
    // Inicializar datos, cargar combos, etc.
}

private void Form1_FormClosing(object sender, FormClosingEventArgs e)
{
    // Confirmar cierre
    if (MessageBox.Show("¿Desea salir?", "Confirmar", MessageBoxButtons.YesNo) == DialogResult.No)
        e.Cancel = true;
}
```
