---
title: ¿Cómo se implementa un método asíncrono en un evento de botón en WinForms?
difficulty: medium
tags: [winforms, async, await, task, boton]
---

Para implementar un manejador de eventos de botón asíncrono en WinForms se deben seguir estos pasos:

1. **Marcar el manejador como `async void`:** Los manejadores de eventos deben retornar `void` (no `Task`), por lo que se usa `async void` como excepción a la regla general.

2. **Crear una `Task` para la operación pesada:** Usar `Task.Run` para mover la carga al ThreadPool, o usar métodos asíncronos nativos (como `ToListAsync` de Entity Framework).

3. **Usar `await`** para esperar el resultado sin bloquear la UI.

```csharp
private async void btnProcesar_Click(object sender, EventArgs e)
{
    // Deshabilitar el botón para evitar doble clic
    btnProcesar.Enabled = false;
    lblEstado.Text = "Procesando...";

    try
    {
        // Operación pesada en hilo secundario
        var resultado = await Task.Run(() => RealizarCalculoComplejo());

        // Este código corre en el hilo de la UI (seguro para actualizar controles)
        lblResultado.Text = resultado.ToString();
        lblEstado.Text = "Listo";
    }
    catch (Exception ex)
    {
        MessageBox.Show($"Error: {ex.Message}");
    }
    finally
    {
        btnProcesar.Enabled = true;
    }
}

private int RealizarCalculoComplejo()
{
    Thread.Sleep(3000); // Simula operación larga
    return 42;
}
```

> **Nota:** `async void` solo debe usarse en manejadores de eventos. Para otros métodos asíncronos, siempre retorne `Task` o `Task<T>` para permitir el manejo correcto de excepciones.
